# ADR-001: Arquitetura de Microserviços V12 - DIYAPP Evolution

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra + Especialista Backend

## CONTEXTO:
O DIYAPP evoluiu de uma aplicação monolítica para uma arquitetura de microserviços distribuídos. Com o crescimento para 12 serviços principais e a necessidade de escalabilidade horizontal, precisamos definir padrões arquiteturais claros para:
1. Comunicação entre serviços (síncrona e assíncrona)
2. Estrutura de repositório que balanceie isolamento e facilidade de desenvolvimento
3. Contratos de API consistentes e versionados
4. Sistema de eventos resiliente para comunicação assíncrona
5. Observabilidade unificada em ambiente distribuído

**Forças em jogo:**
- Velocidade de desenvolvimento vs. Consistência arquitetural
- Autonomia das squads vs. Padronização corporativa
- Latência de comunicação vs. Acoplamento fraco
- Complexidade operacional vs. Escalabilidade

## DECISÃO:
Adotaremos uma arquitetura de microserviços híbrida com:
1. **Comunicação síncrona:** gRPC para comunicação interna entre serviços
2. **Comunicação assíncrona:** Apache Kafka como message broker principal
3. **API Gateway:** GraphQL para APIs externas (clientes web/mobile)
4. **Estrutura de repositório:** Monorepo com workspaces (Lerna/Yarn/Nx)
5. **Contratos:** Protocol Buffers (.proto) para gRPC, GraphQL Schema para APIs externas
6. **Observabilidade:** OpenTelemetry padrão em todos os serviços

## OPÇÕES CONSIDERADAS:

### Opção A: REST + RabbitMQ + Polyrepo
- **Prós:** 
  - Familiaridade da equipe com REST
  - RabbitMQ mais simples de operar inicialmente
  - Repositórios isolados dão autonomia total
- **Contras:**
  - Overhead de JSON parsing em todas as comunicações
  - Versionamento de APIs mais complexo
  - Dificuldade de changesets entre serviços
  - RabbitMQ menos escalável horizontalmente

### Opção B: gRPC + Kafka + Monorepo (ESCOLHIDA)
- **Prós:**
  - gRPC com Protocol Buffers oferece performance 7-10x melhor que REST
  - Contratos fortemente tipados e auto-gerados
  - Kafka oferece melhor escalabilidade e retenção de mensagens
  - Monorepo facilita refatorações cross-service
  - Melhor DX com code sharing e dependências consistentes
- **Contras:**
  - Curva de aprendizado para gRPC
  - Kafka mais complexo de operar
  - Build times maiores no monorepo
  - Acoplamento de deploy se não gerenciado corretamente

### Opção C: GraphQL para tudo + EventBridge + Monorepo
- **Prós:**
  - Single technology stack
  - Flexibilidade máxima para clientes
  - Serverless-friendly com EventBridge
- **Contras:**
  - N+1 query problem em escala
  - Caching mais complexo
  - Over-fetching em comunicações internas
  - Vendor lock-in com AWS

## CONSEQUÊNCIAS:

### Positivas:
1. **Performance:** Comunicação interna 7-10x mais rápida com gRPC
2. **Contratos seguros:** Protocol Buffers garantem compatibilidade entre versões
3. **Escalabilidade:** Kafka permite processamento de milhões de eventos/segundo
4. **Developer Experience:** Monorepo com workspaces facilita desenvolvimento local
5. **Observabilidade:** OpenTelemetry padrão dá visibilidade completa do sistema

### Negativas (trade-offs aceitos):
1. **Complexidade operacional:** Kafka cluster requer expertise dedicada
2. **Curva de aprendizado:** Equipe precisará treinar em gRPC e Protocol Buffers
3. **Build times:** Monorepo pode ter builds mais lentos que polyrepo
4. **Vendor lock-in parcial:** Kafka é open-source mas complexo de migrar

### Riscos e Mitigações:
1. **Risco:** Single point of failure no Kafka cluster
   **Mitigação:** Cluster multi-zone com replicação e monitoring 24/7
2. **Risco:** Monorepo se tornar muito grande e lento
   **Mitigação:** Adotar Nx com affected commands e cache de builds
3. **Risco:** Breaking changes em contratos gRPC
   **Mitigação:** Schema Registry + backward/forward compatibility rules

## REVISÃO: 2024-07-15 (6 meses para reavaliar escalabilidade e DX)

# Contratos de API - Exemplos

## 1. Protocol Buffers (gRPC) - auth.proto
```proto
syntax = "proto3";

package diyapp.auth.v1;

import "google/protobuf/timestamp.proto";

service AuthService {
  rpc Login(LoginRequest) returns (LoginResponse);
  rpc ValidateToken(ValidateTokenRequest) returns (ValidateTokenResponse);
  rpc CreateUser(CreateUserRequest) returns (CreateUserResponse);
}

message LoginRequest {
  string email = 1;
  string password = 2;
  string client_id = 3;
}

message LoginResponse {
  string access_token = 1;
  string refresh_token = 2;
  int32 expires_in = 3;
  User user = 4;
}

message User {
  string id = 1;
  string email = 2;
  string name = 3;
  repeated string roles = 4;
  google.protobuf.Timestamp created_at = 5;
}

message ValidateTokenRequest {
  string token = 1;
}

message ValidateTokenResponse {
  bool valid = 1;
  User user = 2;
  repeated string scopes = 3;
}
```

## 2. GraphQL Schema - gateway
```graphql
type Query {
  me: User
  user(id: ID!): User
  projects(status: ProjectStatus): [Project!]!
}

type Mutation {
  login(email: String!, password: String!): AuthPayload
  createProject(input: CreateProjectInput!): Project
  updateProject(id: ID!, input: UpdateProjectInput!): Project
}

type Subscription {
  projectUpdated(id: ID!): ProjectUpdate
}

type AuthPayload {
  token: String!
  user: User!
}

type User {
  id: ID!
  email: String!
  name: String!
  projects: [Project!]!
  createdAt: DateTime!
}

type Project {
  id: ID!
  name: String!
  description: String
  status: ProjectStatus!
  owner: User!
  collaborators: [User!]!
  createdAt: DateTime!
  updatedAt: DateTime!
}

enum ProjectStatus {
  DRAFT
  ACTIVE
  COMPLETED
  ARCHIVED
}

input CreateProjectInput {
  name: String!
  description: String
  collaboratorIds: [ID!]
}

scalar DateTime
```

# Engineering Standards - Extrato para V12

```markdown
# DIYAPP V12 - Engineering Standards

## 1. Estrutura de Código
- Cada serviço segue estrutura: `src/{grpc, handlers, models, kafka, utils}`
- Testes colocalizados: `src/handlers/user.test.ts` junto de `user.ts`
- Configurações em `src/config/` usando `convict`

## 2. Comunicação entre Serviços
- **Interna:** gRPC com Protocol Buffers
- **Externa:** GraphQL via Gateway
- **Assíncrona:** Kafka com Avro schemas
- **Cache:** Redis para dados frequentemente acessados

## 3. Versionamento
- APIs gRPC: version no package `diyapp.service.v1`
- APIs GraphQL: version no campo `apiVersion` do header
- Eventos Kafka: version no schema namespace `diyapp.events.v1`

## 4. Observabilidade
- Traces: OpenTelemetry com Jaeger
- Metrics: Prometheus metrics em `/metrics`
- Logs: Estrutura JSON com correlation_id
- Health checks: `/health` e `/ready`

## 5. Desenvolvimento Local
```bash
# Iniciar infra
docker-compose up -d kafka postgres redis jaeger

# Instalar dependências
npm run bootstrap

# Desenvolver serviço específico
npm run dev:auth

# Rodar testes
npm run test:auth

# Gerar código gRPC
npm run proto:gen
```

// Mask sensitive data in logs

const maskSensitiveData = (obj) => {
  if (!obj || typeof obj !== 'object') return obj;

  const masked = { ...obj };
  sensitiveFields.forEach((field) => {
    if (masked[field]) {

## 4. Circuit Breaker Utilitário (src/utils/circuitBreaker.js)

```javascript
const CircuitBreaker = require('circuit-breaker-js');
const logger = require('./logger');
const config = require('../config');

class ResilientCircuitBreaker {
  constructor(serviceName, options = {}) {
    this.serviceName = serviceName;
    this.options = {
      timeout: options.timeout || config.circuitBreaker.timeout,
      errorThresholdPercentage: options.errorThresholdPercentage || config.circuitBreaker.errorThresholdPercentage,
      resetTimeout: options.resetTimeout || config.circuitBreaker.resetTimeout,
      ...options,
    };

    this.breaker = new CircuitBreaker({
      timeoutDuration: this.options.timeout,
      errorThreshold: this.options.errorThresholdPercentage,
      resetTimeout: this.options.resetTimeout,
      onCircuitOpen: (metrics) => {
        logger.warn(`Circuit breaker OPEN for ${serviceName}`, {
          service: serviceName,
          metrics,
          state: 'open',
        });
      },
      onCircuitClose: () => {
        logger.info(`Circuit breaker CLOSED for ${serviceName}`, {
          service: serviceName,
          state: 'closed',
        });
      },
    });

    this.metrics = {
      totalRequests: 0,
      successfulRequests: 0,
      failedRequests: 0,
      circuitState: 'closed',
    };
  }

  async execute(fn, fallbackFn = null, context = null) {
    this.metrics.totalRequests += 1;

    return new Promise((resolve, reject) => {
      this.breaker.run(
        (success, fail) => {
          const timeoutId = setTimeout(() => {
            this.metrics.failedRequests += 1;
            fail(new Error(`Timeout after ${this.options.timeout}ms`));
          }, this.options.timeout);

          Promise.resolve()
            .then(() => fn())
            .then((result) => {
              clearTimeout(timeoutId);
              this.metrics.successfulRequests += 1;
              success(result);
            })
            .catch((error) => {
              clearTimeout(timeoutId);
              this.metrics.failedRequests += 1;
              fail(error);
            });
        },
        (result) => {
          // Success callback
          resolve(result);
        },
        (error) => {
          // Failure callback
          logger.error(`Circuit breaker failure for ${this.serviceName}`, {
            service: this.serviceName,
            error: error.message,
            stack: error.stack,
          });

          // Try fallback if available
          if (fallbackFn) {
            try {
              const fallbackResult = fallbackFn();
              logger.info(`Using fallback for ${this.serviceName}`);
              resolve(fallbackResult);
            } catch (fallbackError) {
              reject(fallbackError);
            }
          } else {
            reject(new Error(`Service ${this.serviceName} unavailable: ${error.message}`));
          }
        }
      );
    });
  }

  getMetrics() {
    return {
      ...this.metrics,
      circuitState: this.breaker.isOpen() ? 'open' : 'closed',
      errorRate: this.metrics.totalRequests > 0
        ? (this.metrics.failedRequests / this.metrics.totalRequests) * 100
        : 0,
    };
  }

  reset() {
    this.breaker.reset();
    this.metrics = {
      totalRequests: 0,
      successfulRequests: 0,
      failedRequests: 0,
      circuitState: 'closed',
    };
  }
}

// Factory for creating circuit breakers
const circuitBreakers = new Map();

const getCircuitBreaker = (serviceName, options = {}) => {
  if (!circuitBreakers.has(serviceName)) {
    circuitBreakers.set(serviceName, new ResilientCircuitBreaker(serviceName, options));
  }
  return circuitBreakers.get(serviceName);
};

module.exports = {
  ResilientCircuitBreaker,
  getCircuitBreaker,
};

```javascript
const logger = require('../utils/logger');
const { v4: uuidv4 } = require('uuid');

const loggingMiddleware = (req, res, next) => {
  const startTime = Date.now();
  const correlationId = req.headers['x-correlation-id'] || uuidv4();
  
  // Add correlation ID to request
  req.correlationId = correlationId;
  
  // Create child logger with correlation ID
  req.logger = logger.child(correlationId);
  
  // Log request
  req.logger.info('Incoming request', {
    method: req.method,
    url: req.originalUrl,
    ip: req.ip,
    userAgent: req.get('user-agent'),
    contentType: req.get('content-type'),
  });

  // Capture response
  const originalSend = res.send;
  res.send = function(body) {
    const duration = Date.now() - startTime;
    
    // Log response
    req.logger.info('Request completed', {
      method: req.method,
      url: req.originalUrl,
      statusCode: res.statusCode,
      durationMs: duration,
      contentLength: res.get('Content-Length') || (body ? body.length : 0),
    });

    // Add correlation ID to response headers
    res.setHeader('X-Correlation-ID', correlationId);
    
    // Add request duration header
    res.setHeader('X-Response-Time', `${duration}ms`);

    return originalSend.call(this, body);
  };

  // Log errors
  res.on('finish', () => {
    if (res.statusCode >= 400) {
      const duration = Date.now() - startTime;
      req.logger.warn('Request error', {
        method: req.method,
        url: req.originalUrl,
        statusCode: res.statusCode,
        durationMs: duration,
      });
    }
  });

  next();
};

module.exports = loggingMiddleware;
```

#!/usr/bin/env python3
"""
Processadores customizados para NiFi - DIYAPP V12
"""

import json
import avro.schema
from avro.io import DatumWriter, BinaryEncoder
import io
import uuid
from datetime import datetime
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class DIYAppEventProcessor:
    """Processador de eventos DIYAPP"""
    
    def __init__(self, schema_path):
        with open(schema_path, 'r') as f:
            self.schema = avro.schema.parse(f.read())
    
    def process_event(self, event_data):
        """Processa e valida um evento"""
        try:
            # Adiciona metadados padrão
            processed_event = {
                'event_id': str(uuid.uuid4()),
                'timestamp': int(datetime.now().timestamp() * 1000),
                **event_data
            }
            
            # Valida contra schema Avro
            self.validate_avro(processed_event)
            
            # Serializa para Avro
            avro_bytes = self.serialize_to_avro(processed_event)
            
            return {
                'success': True,
                'data': avro_bytes,
                'event_id': processed_event['event_id']
            }
            
        except Exception as e:
            logger.error(f"Erro ao processar evento: {str(e)}")
            return {
                'success': False,
                'error': str(e),
                'raw_data': event_data
            }
    
    def validate_avro(self, data):
        """Valida dados contra schema Avro"""
        # Implementação simplificada - em produção usar avro.validate
        required_fields = [field.name for field in self.schema.fields 
                          if not isinstance(field.type, avro.schema.UnionSchema) 
                          or 'null' not in [t.type for t in field.type.schemas]]
        
        for field in required_fields:
            if field not in data:
                raise ValueError(f"Campo obrigatório faltando: {field}")
    
    def serialize_to_avro(self, data):
        """Serializa dados para formato Avro binário"""
        writer = DatumWriter(self.schema)
        bytes_writer = io.BytesIO()
        encoder = BinaryEncoder(bytes_writer)
        writer.write(data, encoder)
        return bytes_writer.getvalue()

            }
            
            # Validação básica

            if 'metric_type' not in processed_metric:

## 3. Script de Build (build.sh)

```bash
#!/bin/bash
set -e

echo "🔨 Building DIYAPP V12 Documentation..."

# Instalar dependências se necessário
if [ ! -d "node_modules" ]; then
    echo "📦 Installing dependencies..."
    npm ci
fi

# Validar OpenAPI schemas
echo "🔍 Validating OpenAPI schemas..."
npx swagger-cli validate reference/schemas/openapi/diyapp-api-v1.yaml
npx swagger-cli validate reference/schemas/openapi/webhooks-v1.yaml

# Validar AsyncAPI schema
if [ -f "reference/schemas/asyncapi/events-v1.yaml" ]; then
    echo "🔍 Validating AsyncAPI schema..."
    npx @asyncapi/cli validate reference/schemas/asyncapi/events-v1.yaml
fi

# Lint de documentação
echo "📝 Linting documentation..."
npx vale --config=.vale.ini modules/**/*.adoc || echo "Vale warnings found"

# Gerar documentação
echo "📚 Generating documentation..."
npx antora --fetch antora-playbook.yml

# Gerar bundle OpenAPI para Redoc
echo "🌐 Generating OpenAPI bundles..."
mkdir -p site/diyapp/12.0/api
npx redoc-cli bundle reference/schemas/openapi/diyapp-api-v1.yaml \
    --output site/diyapp/12.0/api/rest-api.html \
    --title "DIYAPP V12 REST API"
npx redoc-cli bundle reference/schemas/openapi/webhooks-v1.yaml \
    --output site/diyapp/12.0/api/webhooks.html \
    --title "DIYAPP V12 Webhooks API"

echo "✅ Documentation built successfully!"
echo "📁 Output: site/diyapp/12.0/index.html"
```
