## 2. ADR-001: Arquitetura de Microsserviços

**docs/adrs/ADR-001-microservices-architecture.md**
```markdown
# ADR-001: Arquitetura de Microsserviços Baseada em Domínios

## Status
Aceita

## Data
2024-01-15

## Contexto
O DIYAPP V11 cresceu para uma aplicação monolítica com acoplamento forte entre funcionalidades. 
Necessitamos de:
1. Escalabilidade independente por componente
2. Deploy contínuo sem downtime
3. Isolamento de falhas
4. Equipes autônomas por domínio

## Decisão
Adotar arquitetura de microsserviços com os seguintes princípios:
- Cada serviço gerencia seu próprio banco de dados
- Comunicação via API Gateway com pattern API Composition
- Event-driven architecture para consistência eventual
- Service Mesh (Istio) para observabilidade e resiliência

## Opções Consideradas
1. **Monolito modularizado**: Menor complexidade operacional, mas limita escalabilidade
2. **Microsserviços leves**: Escalabilidade, mas maior overhead operacional
3. **Serverless Functions**: Escalabilidade automática, mas vendor lock-in

**Escolha: Opção 2** - Balanceia autonomia com controle operacional

## Consequências
### Positivas
- Escalabilidade granular
- Deploy independente
- Stack tecnológica por serviço
- Isolamento de falhas

### Negativas
- Complexidade distribuída
- Overhead de rede
- Gerenciamento de transações distribuídas
- Monitoramento mais complexo

### Riscos
1. Latência de rede - Mitigação: Cache Redis, connection pooling
2. Consistência de dados - Mitigação: Saga pattern, compensação
3. Debug distribuído - Mitigação: Distributed tracing (Jaeger)

## Revisão
2024-07-15 - Avaliar custo operacional vs benefícios
```

## 4. Engineering Standards

**docs/engineering-standards.md**
```markdown
# DIYAPP V12 - Engineering Standards

## 1. Padrões de Código

### JavaScript/Node.js
- Airbnb JavaScript Style Guide
- ESLint com regras específicas
- Prettier para formatação
- Nomenclatura: camelCase (variáveis), PascalCase (classes), UPPER_CASE (constantes)

### TypeScript
- Strict mode obrigatório
- Interfaces sobre types quando possível
- Evitar `any` - usar `unknown` com type guards

### React
- Functional components com hooks
- Custom hooks para lógica reutilizável
- Componentes pequenos e focados (Single Responsibility)

## 2. Convenções de Git

### Branching Strategy
- `main` - produção (protected)
- `develop` - integração (protected)
- `feature/*` - novas funcionalidades
- `fix/*` - correções de bugs
- `release/*` - preparação de release

### Conventional Commits
```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

Types: feat, fix, docs, style, refactor, test, chore, perf, ci, build

## 3. Padrões de API

### REST
- Versionamento: `/api/v1/resource`
- Plural nouns: `/users`, `/users/{id}`
- Nested resources: `/users/{id}/tasks`
- Filter, sort, paginate: `?page=1&limit=20&sort=-createdAt`

### GraphQL (opcional)
- Schema-first design
- Resolvers organizados por domínio
- DataLoader para N+1 queries

## 4. Banco de Dados

### PostgreSQL
- Migrations versionadas
- Índices para queries frequentes
- Constraints no banco (NOT NULL, UNIQUE, FOREIGN KEY)
- Soft delete com `deleted_at`

### Redis
- Cache com TTL
- Session storage
- Rate limiting
- Pub/Sub para eventos

## 5. Segurança

### Autenticação/Autorização
- JWT com refresh tokens
- RBAC (Role-Based Access Control)
- Rate limiting por IP/user
- Input validation em todas as camadas

### Sensitive Data
- Never log passwords, tokens, PII
- Environment variables para secrets
- Encryption at rest e in transit

## 6. Testes

### Cobertura Mínima
- Unit: 80%
- Integration: 70%
- E2E: critical paths only

### Estrutura
```
__tests__/
  ├── unit/
  ├── integration/
  └── e2e/
```

## 7. Docker

### Imagens
- Multi-stage builds
- Non-root user
- Health checks
- Resource limits

## 8. Observabilidade

### Logs
- Structured logging (JSON)
- Correlation IDs
- Log levels: error, warn, info, debug

### Métricas
- Prometheus metrics
- Business metrics
- Health checks

### Tracing
- Distributed tracing
- Span per request
- Service mesh integration
```

## 6. CI/CD Pipeline

**.github/workflows/ci.yml**
```yaml
name: CI Pipeline

on:
  push:
    branches: [develop, feature/*, fix/*]
  pull_request:
    branches: [main, develop]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        service: [api-gateway, user-service, task-service, web-app]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Lint
      run: |
        cd services/${{ matrix.service }}
        npm run lint
    
    - name: Unit tests
      run: |
        cd services/${{ matrix.service }}
        npm test -- --coverage
        npm run test:coverage
    
    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        file: ./services/${{ matrix.service }}/coverage/lcov.info
        flags: ${{ matrix.service }}
        name: ${{ matrix.service }}-coverage

  security-scan:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Run Snyk to check for vulnerabilities
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
    
    - name: Run OWASP Dependency Check
      uses: dependency-check/Dependency-Check_Action@main
      with:
        project: 'DIYAPP'
        path: '.'
        format: 'HTML'
        args: >
          --failOnCVSS 7
          --scan services/
          --scan web-app/

  build-containers:
    runs-on: ubuntu-latest
    needs: [lint-and-test, security-scan]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v2
    
    - name: Login to DockerHub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}
    
    - name: Build and push
      uses: docker/build-push-action@v4
      with:
        context: .
        file: ./services/api-gateway/Dockerfile
        push: true
        tags: |
          ${{ secrets.DOCKER_USERNAME }}/diyapp-api-gateway:${{ github.sha }}
          ${{ secrets.DOCKER_USERNAME }}/diyapp-api-gateway:latest
        cache-from: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/diyapp-api-gateway:buildcache
        cache-to: type=registry,ref=${{ secrets.DOCKER_USERNAME }}/diyapp-api-gateway:buildcache,mode=max
```

**monitoring/prometheus/alert-rules.yml**
```yaml
groups:
  - name: diyapp-alerts
    rules:
      # High CPU Usage
      - alert: HighCPUUsage
        expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
        for: 5m
        labels:
          severity: warning
          environment: staging
        annotations:
          summary: "High CPU usage on {{ $labels.instance }}"
          description: "CPU usage is above 80% for 5 minutes on {{ $labels.instance }}"

      # High Memory Usage
      - alert: HighMemoryUsage
        expr: (node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) / node_memory_MemTotal_bytes * 100 > 85
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High memory usage on {{ $labels.instance }}"
          description: "Memory usage is above 85% for 5 minutes on {{ $labels.instance }}"

      # Disk Space
      - alert: LowDiskSpace
        expr: (node_filesystem_avail_bytes{mountpoint="/"} / node_filesystem_size_bytes{mountpoint="/"}) * 100 < 15
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "Low disk space on {{ $labels.instance }}"
          description: "Disk space is below 15% on {{ $labels.instance }}"

      # Application Health
      - alert: ApplicationDown
        expr: up{job="application"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Application {{ $labels.instance }} is down"
          description: "Application instance {{ $labels.instance }} has been down for more than 1 minute"

      # High Error Rate
      - alert: HighErrorRate
        expr: rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m]) * 100 > 5
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "High error rate on {{ $labels.instance }}"
          description: "Error rate is above 5% for 2 minutes on {{ $labels.instance }}"

      # High Latency
      - alert: HighLatency
        expr: histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m])) > 1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "High latency on {{ $labels.instance }}"
          description: "95th percentile latency is above 1 second for 5 minutes"

      # Database Connections
      - alert: HighDatabaseConnections
        expr: pg_stat_database_numbackends{datname="diyapp"} > 50
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "High database connections"
          description: "Number of database connections is above 50"

      # SLO Error Budget Burn
      - alert: ErrorBudgetBurnRateHigh
        expr: |
          (
            rate(http_requests_total{status=~"5.."}[1h])
            /
            rate(http_requests_total[1h])
          ) > 0.001
        for: 5m
        labels:
          severity: critical
          slo: "error-rate-0.1percent"
        annotations:
          summary: "Error budget burn rate high"
          description: "Error rate exceeds SLO threshold of 0.1%"

### api-gateway/src/middleware/auth.js
```javascript
const jwt = require('jsonwebtoken');
const { createError } = require('../utils/error-handler');

const JWT_SECRET = process.env.JWT_SECRET || 'your-super-secret-jwt-key-change-in-production';
const JWT_ALGORITHM = 'HS256';

/**
 * Middleware de autenticação JWT
 * Valida token antes de qualquer lógica de negócio
 */
const authenticateJWT = (req, res, next) => {
  try {
    // Extrai token do header Authorization
    const authHeader = req.headers.authorization;
    
    if (!authHeader || !authHeader.startsWith('Bearer ')) {
      return createError(res, 401, 'AUTH_TOKEN_REQUIRED', 'Token de autenticação é obrigatório');
    }

    const token = authHeader.split(' ')[1];
    
    // Verifica e decodifica o token
    const decoded = jwt.verify(token, JWT_SECRET, {
      algorithms: [JWT_ALGORITHM],
      clockTolerance: 30 // 30 segundos de tolerância para diferença de relógio
    });

    // Adiciona informações do usuário ao request
    req.user = {
      id: decoded.userId,
      email: decoded.email,
      role: decoded.role,
      permissions: decoded.permissions || []
    };

    // Log estruturado
    req.log.info({
      correlation_id: req.correlationId,
      user_id: req.user.id,
      action: 'AUTH_VALIDATION',
      status: 'SUCCESS'
    });

    next();
  } catch (error) {
    // Log detalhado do erro interno
    req.log.error({
      correlation_id: req.correlationId,
      action: 'AUTH_VALIDATION',
      error: error.message,
      stack: error.stack
    });

    // Retorna erro amigável ao cliente
    if (error.name === 'TokenExpiredError') {
      return createError(res, 401, 'TOKEN_EXPIRED', 'Token expirado. Faça login novamente.');
    }
    
    if (error.name === 'JsonWebTokenError') {
      return createError(res, 401, 'INVALID_TOKEN', 'Token inválido.');
    }

    return createError(res, 401, 'AUTH_FAILED', 'Falha na autenticação');
  }
};

/**
 * Middleware de autorização baseada em roles
 */
const authorize = (...allowedRoles) => {
  return (req, res, next) => {
    if (!req.user) {
      return createError(res, 401, 'UNAUTHORIZED', 'Usuário não autenticado');
    }

    if (!allowedRoles.includes(req.user.role)) {
      req.log.warn({
        correlation_id: req.correlationId,
        user_id: req.user.id,
        action: 'AUTHORIZATION_FAILED',
        required_roles: allowedRoles,
        user_role: req.user.role
      });

      return createError(res, 403, 'FORBIDDEN', 'Acesso negado. Permissões insuficientes.');
    }

    next();
  };
};

/**
 * Middleware de autorização baseada em permissões específicas
 */
const hasPermission = (requiredPermission) => {
  return (req, res, next) => {
    if (!req.user) {
      return createError(res, 401, 'UNAUTHORIZED', 'Usuário não autenticado');
    }

    const userPermissions = req.user.permissions || [];
    
    if (!userPermissions.includes(requiredPermission)) {
      req.log.warn({
        correlation_id: req.correlationId,
        user_id: req.user.id,
        action: 'PERMISSION_DENIED',
        required_permission: requiredPermission,
        user_permissions: userPermissions
      });

      return createError(res, 403, 'FORBIDDEN', 
        `Permissão "${requiredPermission}" é necessária para esta ação`);
    }

    next();
  };
};

module.exports = {
  authenticateJWT,
  authorize,
  hasPermission
};

const authRateLimiter = rateLimit({
  store: new RedisStore({
    sendCommand: (...args) => redisClient.sendCommand(args),
    prefix: 'rl:auth:'
  }),
  windowMs: 60 * 60 * 1000, // 1 hora
  max: 5, // Apenas 5 tentativas por hora
  skipSuccessfulRequests: true, // Não conta tentativas bem-sucedidas
  keyGenerator: (req) => {
    // Usa IP + email para rate limiting mais preciso
    return `${req.ip}:${req.body.email || 'unknown'}`;
  },
  message: {
    error: 'AUTH_RATE_LIMIT_EXCEEDED',
    message: 'Muitas tentativas de login. Tente novamente em 1 hora.'
  }

/**
 * Rate limiting para endpoints de API específicos
 */
const apiRateLimiter = (windowMs, max, keyPrefix) => {

    max,
    standardHeaders: true,
    legacyHeaders: false,
    keyGenerator: (req) => {

      if (req.user && req.user.id) {
        return `user:${req.user.id}`;
      }
      // Para não autenticados, usa IP
      return req.ip;
    }
  });

        } catch {

        }

    validateToken()
  }, [dispatch, token])

    }
  }

      dispatch(setCredentials(response))
      return { success: true }
    } catch (error: any) {
      return { success: false, error: error.message }
    }
  }

  const signOut = () => {
    dispatch(logout())
  }

  return {
    user,
    token,
    isLoading,
    login,
    register,
    signOut,

### 2.2 Política de Criptografia
```javascript
// src/security/encryption.js
const crypto = require('crypto');
const { promisify } = require('util');

class EncryptionManager {
  constructor() {
    this.algorithm = 'aes-256-gcm';
    this.keyLength = 32; // 256 bits
    this.ivLength = 16; // 128 bits
    this.authTagLength = 16;
    this.hashAlgorithm = 'sha256';
  }

  async generateKey() {
    return crypto.randomBytes(this.keyLength);
  }

  async encrypt(text, key) {
    if (!key || key.length !== this.keyLength) {
      throw new Error(`Chave deve ter ${this.keyLength} bytes`);
    }

    const iv = crypto.randomBytes(this.ivLength);
    const cipher = crypto.createCipheriv(this.algorithm, key, iv);
    
    let encrypted = cipher.update(text, 'utf8', 'hex');
    encrypted += cipher.final('hex');
    
    const authTag = cipher.getAuthTag();
    
    return {
      encrypted,
      iv: iv.toString('hex'),
      authTag: authTag.toString('hex')
    };
  }

  async decrypt(encryptedData, key) {
    const { encrypted, iv, authTag } = encryptedData;
    
    if (!key || key.length !== this.keyLength) {
      throw new Error(`Chave deve ter ${this.keyLength} bytes`);
    }

    const decipher = crypto.createDecipheriv(
      this.algorithm,
      key,
      Buffer.from(iv, 'hex')
    );
    
    decipher.setAuthTag(Buffer.from(authTag, 'hex'));
    
    let decrypted = decipher.update(encrypted, 'hex', 'utf8');
    decrypted += decipher.final('utf8');
    
    return decrypted;
  }

  async hashData(data, salt = null) {
    if (!salt) {
      salt = crypto.randomBytes(16).toString('hex');
    }
    
    const hash = crypto.createHmac(this.hashAlgorithm, salt)
      .update(data)
      .digest('hex');
    
    return { hash, salt };
  }

  async verifyHash(data, hash, salt) {
    const newHash = await this.hashData(data, salt);
    return crypto.timingSafeEqual(
      Buffer.from(hash, 'hex'),
      Buffer.from(newHash.hash, 'hex')
    );
  }

  // Para dados sensíveis em banco de dados
  async encryptField(value, context = 'default') {
    const key = await this.getContextKey(context);
    const encrypted = await this.encrypt(JSON.stringify(value), key);
    return {
      ...encrypted,
      context,
      version: '1.0'
    };
  }

  async decryptField(encryptedField) {
    const key = await this.getContextKey(encryptedField.context);
    const decrypted = await this.decrypt(encryptedField, key);
    return JSON.parse(decrypted);
  }

  async getContextKey(context) {
    // Em produção, usar KMS ou HashiCorp Vault
    // Esta é uma implementação simplificada para desenvolvimento
    const masterKey = process.env.ENCRYPTION_MASTER_KEY;
    if (!masterKey) {
      throw new Error('ENCRYPTION_MASTER_KEY não configurada');
    }
    
    return crypto.createHash('sha256')
      .update(masterKey + context)
      .digest()
      .slice(0, this.keyLength);
  }

  // Rotação de chaves
  async rotateKeys(oldKey, newKey, encryptedData) {
    const decrypted = await this.decrypt(encryptedData, oldKey);
    return await this.encrypt(decrypted, newKey);
  }
}

module.exports = EncryptionManager;

      email: Joi.string().email().optional(),

    }));

    }));

  async validateInput(schemaName, data) {
    const schema = this.schemas.get(schemaName);
    if (!schema) {
      throw new Error(`Schema ${schemaName} não encontrado`);
    }

    const { error, value } = schema.validate(data, {
      abortEarly: false,
      stripUnknown: true
    });

    if (error) {
      const errors = error.details.map(detail => ({
        field: detail.path.join('.'),
        message: detail.message,
        type: detail.type
      }));
      
      throw new ValidationError('Validação falhou', errors);
    }

    return value;
  }

  sanitizeInput(input) {
    if (typeof input === 'string') {
      // Remover scripts e tags HTML
      let sanitized = validator.escape(input);
      sanitized = validator.stripLow(sanitized);
      sanitized = validator.trim(sanitized);
      return sanitized;
    }
    
    if (Array.isArray(input)) {
      return input.map(item => this.sanitizeInput(item));
    }
    
    if (typeof input === 'object' && input !== null) {
      const sanitized = {};
      for (const [key, value] of Object.entries(input)) {
        sanitized[key] = this.sanitizeInput(value);
      }
      return sanitized;
    }
    
    return input;

  async checkRateLimit(userId, endpoint, points = 1) {

        points: 10, // 10 requisições
        duration: 60 // por minuto
      }));
    }

    try {
      await limiter.consume(points);
      return { allowed: true, remaining: 0 };
    } catch (rejRes) {
      return {
        allowed: false,
        remaining: Math.floor(rejRes.msBeforeNext / 1000),
        retryAfter: rejRes.msBeforeNext
      };
    }
  }

## 2. Política de Retenção - Serviço Automatizado

**src/compliance/services/RetentionService.js**
```javascript
const { Database } = require('../../core/db');
const { complianceLogger } = require('../../utils/complianceLogger');
const { RETENTION_POLICIES } = require('../../config/complianceConfig');

class RetentionService {
  constructor() {
    this.db = new Database();
    this.logger = complianceLogger;
  }

  /**
   * Aplica política de retenção automaticamente
   */
  async applyRetentionPolicies() {
    try {
      this.logger.info('Iniciando aplicação de políticas de retenção');
      
      const results = {
        anonymized: 0,
        deleted: 0,
        archived: 0,
        errors: []
      };

      // 1. Anonimização de dados de usuários inativos (1 ano)
      results.anonymized += await this.anonymizeInactiveUsers();
      
      // 2. Exclusão de dados de usuários (5 anos)
      results.deleted += await this.deleteOldUserData();
      
      // 3. Rotação de logs de auditoria
      results.archived += await this.rotateAuditLogs();
      
      // 4. Limpeza de dados temporários
      results.deleted += await this.cleanupTemporaryData();
      
      // 5. Backup de dados para exclusão
      await this.backupForDeletion();
      
      this.logger.info('Políticas de retenção aplicadas', results);
      return results;
      
    } catch (error) {
      this.logger.error('Erro ao aplicar políticas de retenção', error);
      throw error;
    }
  }

  /**
   * Anonimiza dados de usuários inativos há mais de 1 ano
   */
  async anonymizeInactiveUsers() {
    const oneYearAgo = new Date();
    oneYearAgo.setFullYear(oneYearAgo.getFullYear() - 1);
    
    const users = await this.db.query(`
      SELECT id, email, cpf, phone 
      FROM users 
      WHERE last_activity < ? 
        AND anonymized_at IS NULL
        AND deletion_scheduled IS NULL
      LIMIT 1000
    `, [oneYearAgo]);
    
    let anonymizedCount = 0;
    
    for (const user of users) {
      try {
        // Anonimização pseudônima (hash com salt)
        const anonymizedEmail = `anon_${this.hashData(user.email)}@anonymized.diyapp`;
        const anonymizedCpf = this.hashData(user.cpf);
        const anonymizedPhone = this.hashData(user.phone);
        
        await this.db.query(`
          UPDATE users 
          SET 
            email = ?,
            cpf = ?,
            phone = ?,
            anonymized_at = NOW(),
            anonymization_reason = 'Retention policy - 1 year inactivity'
          WHERE id = ?
        `, [anonymizedEmail, anonymizedCpf, anonymizedPhone, user.id]);
        
        // Registro de auditoria
        await this.logAnonymization(user.id, 'user_data');
        
        anonymizedCount++;
        
      } catch (error) {
        this.logger.error(`Erro ao anonimizar usuário ${user.id}`, error);
      }
    }
    
    return anonymizedCount;
  }

  /**
   * Exclui dados de usuários após 5 anos de inatividade
   */
  async deleteOldUserData() {
    const fiveYearsAgo = new Date();
    fiveYearsAgo.setFullYear(fiveYearsAgo.getFullYear() - 5);
    
    // Busca usuários para exclusão
    const users = await this.db.query(`
      SELECT id, email 
      FROM users 
      WHERE last_activity < ? 
        AND deletion_scheduled IS NULL
        AND deletion_exempt = 0
      LIMIT 500
    `, [fiveYearsAgo]);
    
    let deletedCount = 0;
    
    for (const user of users) {
      try {
        // Soft delete com marcação temporal
        await this.db.query(`
          UPDATE users 
          SET 
            deletion_scheduled = NOW(),
            deletion_reason = 'Retention policy - 5 years inactivity',
            status = 'pending_deletion'
          WHERE id = ?
        `, [user.id]);
        
        // Agenda exclusão física após 30 dias (período de grace)
        await this.schedulePhysicalDeletion(user.id);
        
        deletedCount++;
        
      } catch (error) {
        this.logger.error(`Erro ao agendar exclusão do usuário ${user.id}`, error);
      }
    }
    
    return deletedCount;
  }

  /**
   * Rotaciona logs de auditoria conforme política
   */
  async rotateAuditLogs() {
    const retentionPeriods = RETENTION_POLICIES.audit_logs;
    
    // Logs de segurança - 6 meses
    const sixMonthsAgo = new Date();
    sixMonthsAgo.setMonth(sixMonthsAgo.getMonth() - 6);
    
    const securityLogs = await this.db.query(`
      SELECT * FROM audit_logs_security 
      WHERE created_at < ?
      LIMIT 10000
    `, [sixMonthsAgo]);
    
    // Arquiva logs antigos
    await this.archiveLogs(securityLogs, 'security_logs_archive');
    
    // Remove logs arquivados
    await this.db.query(`
      DELETE FROM audit_logs_security 
      WHERE created_at < ?
    `, [sixMonthsAgo]);
    
    return securityLogs.length;
  }

  /**
   * Limpa dados temporários
   */
  async cleanupTemporaryData() {
    // Sessões expiradas
    const sessionDays = 30;
    const sessionDate = new Date();
    sessionDate.setDate(sessionDate.getDate() - sessionDays);
    
    const deletedSessions = await this.db.query(`
      DELETE FROM user_sessions 
      WHERE last_activity < ?
    `, [sessionDate]);
    
    // Cache antigo
    const cacheDays = 7;
    const cacheDate = new Date();
    cacheDate.setDate(cacheDate.getDate() - cacheDays);
    
    const deletedCache = await this.db.query(`
      DELETE FROM application_cache 
      WHERE created_at < ?
    `, [cacheDate]);
    
    return deletedSessions.affectedRows + deletedCache.affectedRows;
  }

  /**
   * Cria backup antes da exclusão definitiva
   */
  async backupForDeletion() {
    const usersToDelete = await this.db.query(`
      SELECT u.*, 
             ud.*,
             GROUP_CONCAT(ua.action) as user_actions
      FROM users u
      LEFT JOIN user_data ud ON u.id = ud.user_id
      LEFT JOIN user_audit ua ON u.id = ua.user_id
      WHERE u.deletion_scheduled IS NOT NULL
        AND u.deletion_executed IS NULL
        AND u.deletion_scheduled < DATE_SUB(NOW(), INTERVAL 30 DAY)
      GROUP BY u.id
      LIMIT 100
    `);
    
    for (const user of usersToDelete) {
      // Cria backup em formato estruturado
      const backupData = {
        user_id: user.id,
        backup_date: new Date().toISOString(),
        data: {
          profile: {
            original_email: user.original_email,
            created_at: user.created_at,
            last_activity: user.last_activity
          },
          metadata: {
            anonymized_at: user.anonymized_at,
            deletion_reason: user.deletion_reason
          }
        }
      };
      
      // Armazena backup criptografado
      await this.storeEncryptedBackup(backupData);

### Decisões Arquiteturais (ADRs)

#### ADR-001: Microserviços vs Monolito
**Status**: Aprovado  
**Data**: 2024-01-15  
**Contexto**: Necessidade de escalabilidade independente por componente  
**Decisão**: Arquitetura de microserviços com API Gateway  
**Consequências**:
- ✅ Escalabilidade granular
- ✅ Deploys independentes
- ❌ Complexidade aumentada
- ❌ Latência de rede entre serviços

#### ADR-002: Banco de Dados Principal
**Status**: Aprovado  
**Data**: 2024-01-20  
**Contexto**: Necessidade de ACID e relações complexas  
**Decisão**: PostgreSQL com replicação read-only  
**Alternativas Consideradas**:
- MongoDB: Rejeitado por falta de joins nativos
- MySQL: Rejeitado por performance em JSON operations

const axios = require('axios');
const aiConfig = require('../config/ai-config');

class OpenAIClient {
    constructor() {
        this.config = aiConfig.getActiveConfig();
        this.client = axios.create({
            timeout: 30000,
            headers: {
                'Content-Type': 'application/json'
            }
        });
    }

    async generate(prompt, options = {}) {
        try {
            const provider = this.config.provider;
            
            switch(provider) {
                case 'openai':
                    return await this._callOpenAI(prompt, options);
                case 'anthropic':
                    return await this._callAnthropic(prompt, options);
                case 'local':
                    return await this._callLocalLLM(prompt, options);
                default:
                    throw new Error(`Provedor ${provider} não suportado`);
            }
        } catch (error) {
            console.error('❌ Erro na geração com IA:', error.message);
            return {
                success: false,
                error: error.message,
                fallback: this._getFallbackResponse(prompt)
            };
        }
    }

    async _callOpenAI(prompt, options) {
        const config = this.config.config;
        
        const response = await this.client.post(
            'https://api.openai.com/v1/chat/completions',
            {
                model: options.model || config.model,
                messages: [
                    {
                        role: 'system',
                        content: 'Você é um assistente de código especializado. Responda apenas com código ou análise técnica.'
                    },
                    {
                        role: 'user',
                        content: prompt
                    }
                ],
                max_tokens: options.maxTokens || config.maxTokens,
                temperature: options.temperature || config.temperature
            },
            {
                headers: {
                    'Authorization': `Bearer ${config.apiKey}`
                }
            }
        );

        return {
            success: true,
            content: response.data.choices[0].message.content,
            usage: response.data.usage,
            provider: 'openai'
        };
    }

    async _callAnthropic(prompt, options) {
        const config = this.config.config;
        
        const response = await this.client.post(
            'https://api.anthropic.com/v1/messages',
            {
                model: options.model || config.model,
                max_tokens: options.maxTokens || config.maxTokens,
                messages: [
                    {
                        role: 'user',
                        content: prompt
                    }
                ],
                system: 'Responda apenas com código ou análise técnica.'
            },
            {
                headers: {
                    'x-api-key': config.apiKey,
                    'anthropic-version': '2023-06-01',
                    'Content-Type': 'application/json'
                }
            }
        );

        return {
            success: true,
            content: response.data.content[0].text,
            usage: response.data.usage,
            provider: 'anthropic'
        };
    }

    async _callLocalLLM(prompt, options) {
        const config = this.config.config;
        
        const response = await this.client.post(
            `${config.baseUrl}/api/generate`,
            {
                model: options.model || config.model,
                prompt: `[INST] ${prompt} [/INST]`,
                stream: false,
                options: {
                    temperature: options.temperature || 0.2
                }
            }
        );

        return {
            success: true,
            content: response.data.response,
            provider: 'local'
        };
    }

    _getFallbackResponse(prompt) {
        // Fallback simples quando API falha
        if (prompt.includes('gerar código')) {
            return '// ⚠️ API de IA indisponível\n// Implementação básica de fallback\nconsole.log("Funcionalidade em modo offline");';
        }
        return 'Análise indisponível no momento. Verifique a conexão com a API.';
    }

    async estimateCost(tokens) {
        // Estimativa de custo por provedor
        const costs = {
            openai: {
                'gpt-4-turbo-preview': 0.01 / 1000, // $0.01 por 1K tokens
                'gpt-4': 0.03 / 1000
            },
            anthropic: {
                'claude-3-opus-20240229': 0.015 / 1000
            },
            local: {
                'codellama:7b': 0
            }
        };

        const provider = this.config.provider;
        const model = this.config.config.model;
        
        if (costs[provider] && costs[provider][model]) {
            return tokens * costs[provider][model];
        }
        
        return 0;
    }
}

module.exports = new OpenAIClient();

        console.log(`🤖 Gerando código ${type}...`);
        const startTime = Date.now();
        
        const result = await aiClient.generate(prompt);
        
        const generationTime = Date.now() - startTime;
        console.log(`✅ Geração concluída em ${generationTime}ms`);
        
        if (result.success) {
            return {
                code: result.content,
                metadata: {
                    type,
                    generationTime,
                    tokens: result.usage?.total_tokens || 0,
                    estimatedCost: await aiClient.estimateCost(result.usage?.total_tokens || 0),
                    provider: result.provider
                },
                suggestions: this._extractSuggestions(result.content)
            };
        }
        
        return {
            code: result.fallback,
            metadata: {
                type,
                generationTime,
                error: result.error,
                fallback: true
            },
            suggestions: []
        };
    }

    async analyzeAndRefactor(existingCode, issues = []) {
        const prompt = `Analise este código e sugira melhorias:\n\n${existingCode}\n\nProblemas reportados: ${issues.join(', ')}\n\nSugira refatoração com foco em: legibilidade, performance, boas práticas.`;
        
        const result = await aiClient.generate(prompt);
        
        if (result.success) {
            return {
                analysis: result.content,
                suggestedChanges: this._parseRefactoringSuggestions(result.content),
                beforeStats: this._calculateCodeMetrics(existingCode),
                estimatedImprovement: '20-40%' // Estimativa baseada em padrões
            };
        }
        
        return {
            analysis: 'Análise não disponível',
            suggestedChanges: [],
            beforeStats: this._calculateCodeMetrics(existingCode)
        };
    }

    _buildCodePrompt(type, requirements, context) {
        const template = this.templates[type] || '';
        
        return `Gere código ${type} com os seguintes requisitos:
        
REQUISITOS:
${requirements}

CONTEXTO:
Linguagem: ${context.language || 'JavaScript'}
Framework: ${context.framework || 'Node.js'}
Padrão: ${context.pattern || 'MVC'}

${template}

INSTRUÇÕES:
1. Código completo e funcional
2. Incluir comentários explicativos
3. Seguir boas práticas da linguagem
4. Incluir tratamento de erros
5. Retornar APENAS o código, sem texto adicional`;
    }

    _getControllerTemplate() {
        return `TEMPLATE CONTROLLER:
- Exportar classe ou funções
- Validação de entrada
- Tratamento de erros HTTP
- Logging de operações
- Respostas padronizadas`;
    }

    _getServiceTemplate() {
        return `TEMPLATE SERVICE:
- Lógica de negócio isolada
- Stateless
- Injeção de dependências
- Métodos bem definidos
- Testabilidade`;
    }

    _getModelTemplate() {
        return `TEMPLATE MODEL:
- Definição de schema/entidade
- Validações
- Relacionamentos
- Métodos estáticos para queries comuns
- Hooks se necessário`;
    }

    _getMiddlewareTemplate() {
        return `TEMPLATE MIDDLEWARE:
- Função que recebe (req, res, next)
- Early returns em caso de erro
- Adição de contexto ao request
- Opcional: async/await support`;
    }

    _getTestTemplate() {
        return `TEMPLATE TEST:
- Describe blocks organizados
- Before/After hooks
- Mocking de dependências
- Asserts claros
- Cobertura de casos edge`;
    }

    _extractSuggestions(code) {
        const suggestions = [];
        
        // Análise simples do código gerado
        if (code.includes('console.log(') && !code.includes('// TODO:')) {
            suggestions.push('Considerar usar sistema de logging estruturado');
        }
        
        if (code.includes('.then(') && code.includes('.catch(')) {
            suggestions.push('Pode converter para async/await para melhor legibilidade');
        }
        
        if (code.split('\n').length > 50) {
            suggestions.push('Código muito longo - considerar dividir em funções menores');
        }
        
        return suggestions;
    }

    _parseRefactoringSuggestions(analysis) {
        const changes = [];
        const lines = analysis.split('\n');
        
        for (const line of lines) {
            if (line.includes('✅') || line.includes('✔') || line.includes('Recomendo')) {
                changes.push(line.trim());
            }
        }
        
        return changes.length > 0 ? changes : ['Nenhuma sugestão específica encontrada'];
    }

    _calculateCodeMetrics(code) {
        const lines = code.split('\n');
        return {
            lines: lines.length,
            functions: (code.match(/function\s+\w+|const\s+\w+\s*=\s*\(|async\s+\w+/g) || []).length,
            complexity: this._calculateCyclomaticComplexity(code),
            comments: lines.filter(l => l.trim().startsWith('//')).length
        };
    }

    _calculateCyclomaticComplexity(code) {
        // Simplificação do cálculo de complexidade ciclomática
        const decisions = (code.match(/if|for|while|catch|\?|&&|\|\|/g) || []).length;
        return Math.max(1, decisions + 1);
    }
}

            } else {
