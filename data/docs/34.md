# DIYAPP Evolution - V12 Core - Arquitetura de Microsserviços

## ADR-001: Arquitetura de Microsserviços para V12

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra + Especialista Backend

### CONTEXTO:
A V11 do DIYAPP atingiu limites de escalabilidade devido à arquitetura monolítica. O crescimento do código base (agora ~85k linhas) e a necessidade de deploy independente de funcionalidades específicas (IA, processamento de mídia, notificações) demandam desacoplamento. A squad precisa operar em Modo Hive com refatoração paralela, mantendo 100% de estabilidade durante a transição.

### DECISÃO:
Adotar arquitetura de microsserviços baseada em eventos com padrão CQRS para separação de leitura/escrita. Cada domínio de negócio será um serviço independente com banco de dados dedicado. Comunicação síncrona via gRPC para comandos, assíncrona via Apache Kafka para eventos de domínio.

### OPÇÕES CONSIDERADAS:

**Opção A: Monolito Modularizado (Evolução Incremental)**
- Prós: Menor risco, migração gradual, compartilhamento de código
- Contras: Limites de escalabilidade permanecem, deploy acoplado
- Custo: 40% mais lento em 6 meses

**Opção B: Microsserviços Full (Estrangulamento)**
- Prós: Escalabilidade independente, resiliência por isolamento, deploy contínuo
- Contras: Complexidade operacional, necessidade de SRE dedicado
- Custo: 60% mais rápido após transição de 3 meses

**Opção C: Arquitetura Híbrida (Serviços + Monolito)**
- Prós: Balanceamento risco/benefício, migração seletiva
- Contras: Gerenciamento de duas arquiteturas, pontos de integração complexos

**Opção escolhida: B** - Justificativa: O Modo Hive permite refatoração paralela com mitigação de risco. A maturidade atual da squad (SRE automatizado, CI/CD robusto) suporta complexidade operacional. Benefícios de escalabilidade justificam investimento.

### CONSEQUÊNCIAS:
**Positivas:**
- Deploy independente por domínio
- Escalabilidade horizontal granular
- Isolamento de falhas
- Evolução tecnológica por serviço

**Negativas:**
- Latência de rede entre serviços
- Complexidade de debug distribuído
- Gerenciamento de transações distribuídas
- Overhead de infraestrutura

**Riscos:**
1. **Degradação de performance:** Mitigar com cache distribuído (Redis) e design assíncrono
2. **Perda de consistência:** Implementar SAGA pattern com compensação
3. **Observabilidade reduzida:** Implementar OpenTelemetry + Jaeger + Prometheus

**REVISÃO:** 2024-04-15 (3 meses após implementação)

```typescript
// src/infrastructure/messaging/grpc-client.ts
import * as grpc from '@grpc/grpc-js';
import * as protoLoader from '@grpc/proto-loader';
import { Logger } from '../shared/utils/logger';
import { CircuitBreaker } from './circuit-breaker';

export class GRPCClient {
  private clients: Map<string, any> = new Map();
  private circuitBreakers: Map<string, CircuitBreaker> = new Map();
  private readonly logger = new Logger('GRPCClient');

  constructor(private config: GRPCConfig) {}

  async getClient(serviceName: string): Promise<any> {
    if (this.clients.has(serviceName)) {
      return this.clients.get(serviceName);
    }

    const serviceConfig = this.config.services[serviceName];
    if (!serviceConfig) {
      throw new Error(`Service ${serviceName} not configured`);
    }

    const packageDefinition = protoLoader.loadSync(
      serviceConfig.protoPath,
      {
        keepCase: true,
        longs: String,
        enums: String,
        defaults: true,
        oneofs: true
      }
    );

    const proto = grpc.loadPackageDefinition(packageDefinition);
    const client = new proto[serviceConfig.package][serviceConfig.service](
      serviceConfig.host,
      grpc.credentials.createInsecure(),
      {
        'grpc.keepalive_time_ms': 10000,
        'grpc.keepalive_timeout_ms': 5000,
        'grpc.keepalive_permit_without_calls': 1,
        'grpc.max_connection_age_ms': 30000
      }
    );

    // Setup circuit breaker
    const circuitBreaker = new CircuitBreaker({
      failureThreshold: 3,
      resetTimeout: 10000,
      timeout: 5000
    });

    this.circuitBreakers.set(serviceName, circuitBreaker);
    this.clients.set(serviceName, client);

    return client;
  }

  async call<T>(serviceName: string, method: string, request: any): Promise<T> {
    const circuitBreaker = this.circuitBreakers.get(serviceName);
    
    return circuitBreaker!.execute(async () => {
      const client = await this.getClient(serviceName);
      
      return new Promise((resolve, reject) => {
        const deadline = new Date();
        deadline.setSeconds(deadline.getSeconds() + 5);
        
        client[method](request, { deadline }, (error: any, response: any) => {
          if (error) {
            this.logger.error(`gRPC call failed: ${serviceName}.${method}`, error);
            reject(error);
          } else {
            resolve(response);
          }
        });
      });
    });
  }
}

// src/infrastructure/messaging/kafka-producer.ts
import { Kafka, Producer, ProducerRecord } from 'kafkajs';
import { Logger } from '../shared/utils/logger';
import { Event } from '../../domain/domain-events/event';

export class KafkaProducer {
  private producer: Producer;
  private readonly logger = new Logger('KafkaProducer');

  constructor(private kafka: Kafka) {
    this.producer = this.kafka.producer({
      idempotent: true,
      transactionalId: `producer-${process.env.SERVICE_NAME}`,
      maxInFlightRequests: 1,
      retry: {
        retries: 10,
        initialRetryTime: 100
      }
    });
  }

  async connect(): Promise<void> {
    await this.producer.connect();
    this.logger.info('Kafka producer connected');
  }

  async disconnect(): Promise<void> {
    await this.producer.disconnect();
    this.logger.info('Kafka producer disconnected');
  }

  async publish(event: Event): Promise<void> {
    const record: ProducerRecord = {
      topic: event.topic,
      messages: [
        {
          key: event.aggregateId,
          value: JSON.stringify({
            ...event,
            metadata: {
              ...event.metadata,
              publishedAt: new Date().toISOString(),
              producer: process.env.SERVICE_NAME
            }
          }),
          headers: {
            'event-type': event.type,
            'event-version': event.version.toString(),
            'correlation-id': event.metadata.correlationId
          }
        }
      ]
    };

    try {
      await this.producer.send(record);
      this.logger.debug(`Event published: ${event.type}`, {
        topic: event.topic,
        aggregateId: event.aggregateId
      });
    } catch (error) {
      this.logger.error(`Failed to publish event: ${event.type}`, error);
      throw error;
    }
  }

  async publishInTransaction(events: Event[]): Promise<void> {
    const transaction = await this.producer.transaction();
    
    try {
      for (const event of events) {
        await transaction.send({
          topic: event.topic,
          messages: [
            {
              key: event.aggregateId,
              value: JSON.stringify(event)
            }
          ]
        });
      }
      
      await transaction.commit();
      this.logger.info(`Transaction committed: ${events.length} events`);
    } catch (error) {
      await transaction.abort();
      this.logger.error('Transaction aborted', error);
      throw error;
    }
  }
}
```

```typescript
import { logger } from './logger';

export class Database {
  private pool: Pool;
  private static instances: Map<string, Database> = new Map();

  private constructor(config: PoolConfig) {
    this.pool = new Pool({
      ...config,
      max: 20,
      idleTimeoutMillis: 30000,
      connectionTimeoutMillis: 5000,
    });

    this.pool.on('error', (err) => {
      logger.error('Database pool error', err, { correlation_id: 'database' });
    });
  }

  static getInstance(serviceName: string, config: PoolConfig): Database {
    if (!this.instances.has(serviceName)) {
      this.instances.set(serviceName, new Database(config));
    }
    return this.instances.get(serviceName)!;
  }

  async query<T = any>(text: string, params?: any[]): Promise<T[]> {
    const start = Date.now();
    const correlationId = 'query-' + Date.now();

    try {
      logger.debug('Executing query', { 
        correlation_id: correlationId,
        query: text,
        params: this.maskSensitiveData(params)
      });

      const result = await this.pool.query(text, params);
      const duration = Date.now() - start;

      logger.debug('Query completed', {
        correlation_id: correlationId,
        duration_ms: duration,
        row_count: result.rowCount
      });

      return result.rows;
    } catch (error) {
      const duration = Date.now() - start;
      logger.error('Query failed', error as Error, {
        correlation_id: correlationId,
        query: text,
        duration_ms: duration
      });
      throw error;
    }
  }

  async transaction<T>(callback: (client: any) => Promise<T>): Promise<T> {
    const client = await this.pool.connect();
    const correlationId = 'tx-' + Date.now();

    try {
      await client.query('BEGIN');
      logger.debug('Transaction started', { correlation_id: correlationId });

      const result = await callback(client);
      await client.query('COMMIT');

      logger.debug('Transaction committed', { correlation_id: correlationId });
      return result;
    } catch (error) {
      await client.query('ROLLBACK');
      logger.error('Transaction rolled back', error as Error, { 
        correlation_id: correlationId 
      });
      throw error;
    } finally {
      client.release();
    }
  }

  private maskSensitiveData(params?: any[]): any[] {
    if (!params) return [];
    
    return params.map(param => {
      if (typeof param === 'string') {
        // Mask emails
        if (param.includes('@')) {
          const [local, domain] = param.split('@');
          return `${local[0]}***@${domain}`;
        }
        // Mask tokens (JWT-like)
        if (param.length > 50 && param.includes('.')) {
          return `${param.substring(0, 10)}...`;
        }
      }
      return param;
    });
  }

  async close(): Promise<void> {
    await this.pool.end();
  }
}
```

      correlation_id: 'system',

      service: this.serviceName
    });
  }

    await this.subscriber.quit();
  }

```markdown
# THREAT MODELING - DIYAPP V12 CORE

## Data: $(date)
## Versão: V12.0.0
## Responsável: Especialista em Segurança

## 1. VISÃO GERAL DO SISTEMA

### Componentes Principais:
1. Frontend Web (React/Next.js)
2. API Gateway (Node.js/Express)
3. Serviço de Autenticação (OAuth2.1/JWT)
4. Serviço de Dados (PostgreSQL)
5. Serviço de IA/LLM (OpenAI/Anthropic)
6. Serviço de Cache (Redis)
7. Serviço de Filas (RabbitMQ)
8. Serviço de Monitoramento (ELK Stack)

### Fluxos de Dados Críticos:
1. Autenticação OAuth2.1 → JWT → API Gateway
2. Upload de arquivos → Processamento LLM → Armazenamento
3. Integração WhatsApp → Webhook → Processamento
4. Exportação de dados → Geração de relatórios

## 2. ANÁLISE STRIDE POR COMPONENTE

### 2.1 Serviço de Autenticação

#### Spoofing (S):
- Ameaça: Token JWT roubado ou falsificado
- Impacto: Alto (CVSS 8.2)
- Controles:
  - JWT com assinatura HMAC-SHA256
  - Tokens de curta duração (15min access, 7d refresh)
  - MFA obrigatório para admin
  - Rate limiting (10 tentativas/hora por IP)

#### Tampering (T):
- Ameaça: Manipulação de claims JWT
- Impacto: Alto (CVSS 7.5)
- Controles:
  - Assinatura digital verificada em todas as APIs
  - Claims obrigatórias: iss, aud, exp, iat, sub
  - Blacklist de tokens revogados

#### Repudiation (R):
- Ameaça: Negação de login bem-sucedido
- Impacto: Médio (CVSS 5.5)
- Controles:
  - Logs de auditoria imutáveis (AWS CloudTrail)
  - Timestamp + IP + user agent em todos os logins
  - Assinatura digital nos logs críticos

#### Information Disclosure (I):
- Ameaça: Vazamento de secrets em logs
- Impacto: Alto (CVSS 8.1)
- Controles:
  - Mascaramento de PII em logs
  - Secrets no AWS Secrets Manager
  - Criptografia em trânsito (TLS 1.3)

#### Denial of Service (D):
- Ameaça: Ataque de força bruta no login
- Impacto: Médio (CVSS 6.5)
- Controles:
  - Rate limiting por IP e usuário
  - Circuit breaker após 5 falhas
  - CAPTCHA após 3 tentativas

#### Elevation of Privilege (E):
- Ameaça: Usuário comum acessa endpoints admin
- Impacto: Crítico (CVSS 9.1)
- Controles:
  - RBAC com verificações em cada endpoint
  - Princípio do menor privilégio
  - Revisão periódica de permissões

### 2.2 Serviço de IA/LLM

#### Spoofing (S):
- Ameaça: Prompt injection para assumir identidade do sistema
- Impacto: Alto (CVSS 8.5)
- Controles:
  - Separação rígida system prompt / user input
  - Validação de intenção do usuário
  - Sanitização de inputs com allowlist

#### Tampering (T):
- Ameaça: Manipulação de contexto do LLM
- Impacto: Médio (CVSS 6.8)
- Controles:
  - Hash digital do contexto fornecido
  - Validação de integridade antes do processamento
  - Logs de alterações no contexto

#### Information Disclosure (I):
- Ameaça: Data exfiltration via LLM output
- Impacto: Crítico (CVSS 9.3)
- Controles:
  - Não incluir PII no contexto do LLM
  - Sanitização de outputs (regex + ML)
  - Content filter no output (Azure/OpenAI)

#### Denial of Service (D):
- Ameaça: Prompt flooding para esgotar tokens
- Impacto: Médio (CVSS 5.9)
- Controles:
  - Limite de tokens por usuário/sessão
  - Timeout de 30s por requisição
  - Queue management com priorização

### 2.3 API Gateway

#### Spoofing (S):
- Ameaça: API key comprometida
- Impacto: Alto (CVSS 8.0)
- Controles:
  - Rotação automática a cada 90 dias
  - HMAC signing em todas as requisições
  - IP whitelist para APIs críticas

#### Tampering (T):
- Ameaça: Manipulação de payload em trânsito
- Impacto: Médio (CVSS 6.2)
- Controles:
  - TLS 1.3 com certificados válidos
  - Validação de schema JSON (AJV)
  - Checksum em uploads de arquivo
```

      .digest('base64url');

  // OAuth2.1 Authorization endpoint
  async authorize(req, res) {
    try {
      const {
        client_id,
        response_type,
        redirect_uri,
        scope,
        state,
        code_challenge,
        code_challenge_method = 'S256'
      } = req.query;

      // Validate client
      if (!await this.validateClient(client_id, redirect_uri)) {
        return res.status(401).json({ error: 'invalid_client' });
      }

      // PKCE validation (OAuth2.1 requirement)
      if (!code_challenge || code_challenge_method !== 'S256') {
        return res.status(400).json({ error: 'invalid_request', 
          error_description: 'PKCE code_challenge required with S256 method' });
      }

      // Check if user is authenticated
      if (!req.session.userId) {
        // Store authorization request in session
        req.session.authRequest = {
          client_id,
          redirect_uri,
          scope,
          state,
          code_challenge,
          code_challenge_method
        };
        return res.redirect('/login');
      }

      // Generate authorization code with PKCE binding
      const authCode = uuidv4();
      const codeVerifier = crypto.randomBytes(32).toString('hex');
      
      // Store code verifier for token exchange
      this.codeVerifiers.set(authCode, {
        verifier: codeVerifier,
        challenge: code_challenge,
        clientId: client_id,
        userId: req.session.userId,
        scope,
        expiresAt: Date.now() + 600000 // 10 minutes
      });

      this.cleanupExpiredVerifiers();

  async token(req, res) {

      const {
        grant_type,
        code,
        redirect_uri,
        client_id,
        code_verifier,
        refresh_token
      } = req.body;

      // Validate client credentials
      const clientValid = await this.validateClientCredentials(
        client_id,
        req.headers.authorization?.replace('Basic ', '')
      );
      
      if (!clientValid) {
        return res.status(401).json({ error: 'invalid_client' });
      }

      if (grant_type === 'authorization_code') {
        // PKCE validation
        const codeData = this.codeVerifiers.get(code);
        if (!codeData) {
          return res.status(400).json({ error: 'invalid_grant' });
        }

        // Verify code verifier matches challenge
        const calculatedChallenge = crypto
          .createHash('sha256')
          .update(code_verifier)
          .digest('base64url');
        
        if (calculatedChallenge !== codeData.challenge) {
          return res.status(400).json({ error: 'invalid_grant' });
        }

        // Clean used code
        this.codeVerifiers.delete(code);

        // Generate tokens
        const tokens = await this.generateTokens(
          codeData.userId,
          codeData.clientId,
          codeData.scope
        );

        return res.json({
          access_token: tokens.accessToken,
          token_type: 'Bearer',
          expires_in: 900, // 15 minutes in seconds
          refresh_token: tokens.refreshToken,
          scope: codeData.scope
        });

      } else if (grant_type === 'refresh_token') {
        // Refresh token flow
        const tokens = await this.refreshAccessToken(refresh_token);
        if (!tokens) {
          return res.status(400).json({ error: 'invalid_grant' });
        }

        return res.json({
          access_token: tokens.accessToken,
          token_type: 'Bearer',
          expires_in: 900,
          refresh_token: tokens.refreshToken
        });
      } else {
        return res.status(400).json({ error: 'unsupported_grant_type' });
      }
    } catch (error) {
      console.error('Token error:', error);
      return res.status(500).json({ error: 'server_error' });
    }
  }

      aud: clientId, // Audience

      exp: Math.floor(Date.now() / 1000) + 900, // 15 minutes
      scope: scope || 'read write',
      typ: 'access',
      // Additional security claims
      azp: clientId, // Authorized party
      version: 'v12'
    };

      jti: refreshTokenId,
      sub: userId,
      clientId,
