# DIYAPP Evolution - V12 Core - Arquitetura de Referência

## 1. DECISÕES ARQUITETURAIS PRINCIPAIS (ADRs)

### ADR-001: Stack Tecnológica Principal
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead

**CONTEXTO:**
Precisamos escolher a stack tecnológica principal para o DIYAPP V12 que ofereça:
- Alta produtividade para desenvolvimento rápido
- Estabilidade e maturidade da comunidade
- Boa performance para aplicações web complexas
- Facilidade de contratação e onboarding

**DECISÃO:**
- **Frontend:** React 18 com TypeScript
- **Backend:** Node.js 20+ com Express.js
- **Banco de Dados:** PostgreSQL 15 (principal) + Redis 7 (cache/sessões)
- **Runtime:** Node.js 20+ LTS

**OPÇÕES CONSIDERADAS:**
- **Opção A (React + Node.js):** Prós: Ecossistema maduro, vasta comunidade, TypeScript nativo. Contras: Bundle size maior que Vue.
- **Opção B (Vue 3 + Node.js):** Prós: Curva de aprendizado mais suave, performance excelente. Contras: Ecossistema menor que React.
- **Opção C (Next.js Full Stack):** Prós: SSR nativo, otimizações automáticas. Contras: Vendor lock-in, menos flexibilidade.

**Escolha:** Opção A - Justificativa: Maior mercado, melhor suporte para TypeScript, mais bibliotecas maduras para IA.

**CONSEQUÊNCIAS:**
- Positivas: Facilidade de contratação, vasta documentação, compatibilidade com bibliotecas de IA
- Negativas: Bundle size inicial maior, necessidade de configurar SSR manualmente
- Riscos: Over-engineering com muitas bibliotecas React

### ADR-002: Estrutura de Pastas
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead

**CONTEXTO:**
Necessidade de estrutura escalável que suporte:
- Desenvolvimento paralelo com Modo Hive
- Separação clara de responsabilidades
- Testabilidade
- Deploy independente de módulos

**DECISÃO:**
Estrutura monorepo com workspaces:
```
diyapp-v12/
├── packages/
│   ├── core/           # Utilitários compartilhados
│   ├── api/           # Backend principal
│   ├── web/           # Frontend React
│   ├── agents/        # Agentes de IA
│   └── shared/        # Tipos e contratos compartilhados
├── infrastructure/
│   ├── docker/
│   ├── k8s/
│   └── terraform/
├── docs/
│   ├── adrs/
│   └── api/
└── tools/
    ├── scripts/
    └── generators/
```

**OPÇÕES CONSIDERADAS:**
- **Opção A (Monorepo com Turborepo):** Prós: Compartilhamento de código fácil, builds otimizados. Contras: Complexidade inicial.
- **Opção B (Multi-repo):** Prós: Isolamento total, deploys independentes. Contras: Sincronização difícil, versionamento complexo.
- **Opção C (Monorepo simples):** Prós: Simplicidade. Contras: Builds lentos, dependências desorganizadas.

**Escolha:** Opção A - Justificativa: Turborepo oferece cache inteligente e paralelização ideal para Modo Hive.

### ADR-003: Pipeline CI/CD
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra

**CONTEXTO:**
Pipeline deve garantir:
- 100% de testes antes do merge
- Deploy automático para staging
- Rollback automático em falhas
- Segurança integrada

**DECISÃO:**
GitHub Actions com os seguintes fluxos:
1. **PR Pipeline:** Lint → Testes Unitários → Testes de Integração → Build → Security Scan
2. **Main Pipeline:** Todos acima + Deploy Staging → Testes E2E → Deploy Production (aprovado)
3. **Hotfix Pipeline:** Deploy direto para produção com rollback automático

**CONSEQUÊNCIAS:**
- Positivas: Automação completa, feedback rápido, segurança integrada
- Negativas: Custo de minutos GitHub Actions, complexidade de configuração
- Riscos: Pipeline muito lento pode atrasar entregas

```typescript
const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.errors({ stack: true }),
    winston.format.json()
  ),

  transports: [

      format: winston.format.combine(
        winston.format.colorize(),
        winston.format.simple()
      )
    }),
    new ElasticsearchTransport({
      level: 'info',
      clientOpts: { node: process.env.ELASTICSEARCH_URL },
      index: 'diyapp-logs'
    })
  ]
});

// Logging automático para requests HTTP
export const requestLogger = (req: Request, res: Response, next: NextFunction) => {
  const start = Date.now();
  
  res.on('finish', () => {
    const duration = Date.now() - start;
    
    logger.info('HTTP Request', {
      method: req.method,
      url: req.url,
      statusCode: res.statusCode,
      duration,
      userAgent: req.get('User-Agent'),
      ip: req.ip
    });
  });
  
  next();
};
```

# DIYAPP Evolution - V12 Core - Frontend Base

## Análise da Tarefa

Como Dev Frontend Sênior, vou implementar a base do frontend com:
1. Setup do projeto com Vite (escolha técnica baseada em performance)
2. Estrutura de pastas profissional
3. Sistema de Design integrado com tokens
4. Ponto de entrada obrigatório (dashboard principal)
5. Roteamento básico
6. Foco em estabilidade 100% autônoma

## Decisões Técnicas

**Framework**: Vite + React + TypeScript
- Justificativa: Performance superior (LCP < 2.5s), bundle otimizado, melhor DX
- Alternativa Next.js considerada, mas Vite atende melhor ao requisito de "estabilidade 100% autônoma"

**Sistema de Design**: Criado do zero com tokens CSS custom properties
- Justificativa: Zero dependências externas, total controle sobre tokens

**Estrutura**: Modular com separação clara de responsabilidades

```javascript
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');
const mongoSanitize = require('express-mongo-sanitize');
const xss = require('xss-clean');
const hpp = require('hpp');
const securityConfig = require('../config/security');

/**
 * Middleware de Segurança Completo V12
 * Implementa controles STRIDE
 */
class SecurityMiddleware {
  constructor() {
    this.securityHeaders = this.setupSecurityHeaders();
    this.rateLimiters = this.setupRateLimiters();
  }

  /**
   * S - Spoofing Protection Headers
   */
  setupSecurityHeaders() {
    return helmet({
      contentSecurityPolicy: {
        directives: securityConfig.dataProtection.headers.csp
      },
      hsts: securityConfig.dataProtection.headers.hsts,
      xssFilter: true,
      noSniff: true,
      frameguard: { action: 'deny' },
      hidePoweredBy: true,
      referrerPolicy: { policy: 'strict-origin-when-cross-origin' }
    });
  }

  /**
   * D - Denial of Service Protection
   */
  setupRateLimiters() {
    return {
      global: rateLimit(securityConfig.availability.rateLimiting.global),
      auth: rateLimit(securityConfig.availability.rateLimiting.auth),
      api: rateLimit(securityConfig.availability.rateLimiting.api)
    };
  }

  /**
   * T - Tampering Protection
   */
  inputSanitization() {
    return [
      // Previne NoSQL injection
      mongoSanitize({
        replaceWith: '_',
        onSanitize: ({ req, key }) => {
          console.warn(`[SECURITY] NoSQL injection attempt detected: ${key}`);
        }
      }),
      
      // Previne XSS attacks
      xss(),
      
      // Previne parameter pollution
      hpp({
        whitelist: ['sort', 'limit', 'page'] // Parâmetros permitidos para duplicação
      }),
      
      // Validação de tamanho de payload
      (req, res, next) => {
        const contentLength = parseInt(req.headers['content-length'] || '0');
        if (contentLength > 10 * 1024 * 1024) { // 10MB max
          return res.status(413).json({
            error: 'Payload too large',
            message: 'Request body exceeds 10MB limit'
          });
        }
        next();
      }
    ];
  }

  /**
   * I - Information Disclosure Protection
   */
  dataMasking() {
    return (req, res, next) => {
      // Intercepta respostas para mascarar PII
      const originalSend = res.send;
      res.send = function(data) {
        if (securityConfig.dataProtection.pii.maskInLogs) {
          this.maskPII(data);
        }
        return originalSend.call(this, data);
      };
      next();
    };
  }

  /**
   * E - Elevation of Privilege Protection
   */
  authorizationMiddleware(requiredPermission) {
    return async (req, res, next) => {
      try {
        const userRole = req.user?.role || 'anonymous';
        const userPermissions = securityConfig.authorization.rbac.permissions[userRole];
        
        if (!userPermissions || !userPermissions.includes(requiredPermission)) {
          // STRIDE: E - Log de tentativa de elevação de privilégio
          console.warn(`[SECURITY] Unauthorized access attempt: User ${req.user?.id} tried to access ${requiredPermission}`);
          
          return res.status(403).json({
            error: 'Forbidden',
            message: 'Insufficient permissions'
          });
        }
        
        // Verificação adicional de IDOR (Insecure Direct Object Reference)
        if (req.params.userId && req.params.userId !== req.user.id && !userPermissions.includes('read:all')) {
          return res.status(403).json({
            error: 'Forbidden',
            message: 'Cannot access other user resources'
          });
        }
        
        next();
      } catch (error) {
        console.error('[SECURITY] Authorization error:', error);
        res.status(500).json({ error: 'Internal server error' });
      }
    };
  }

  /**
   * LLM Security Middleware
   */
  llmSecurity() {
    return [
      // Sanitização de input para LLM
      (req, res, next) => {
        if (req.body.prompt) {
          req.body.prompt = this.sanitizeLLMInput(req.body.prompt);
        }
        next();
      },
      
      // Validação de output de LLM
      (req, res, next) => {
        const originalJson = res.json;
        res.json = function(data) {
          if (data.llmResponse) {
            data.llmResponse = this.sanitizeLLMOutput(data.llmResponse);
          }
          return originalJson.call(this, data);
        };
        next();
      }
    ];
  }

  /**
   * Sanitização específica para inputs de LLM
   */
  sanitizeLLMInput(input) {
    // Remove tentativas de prompt injection
    const injectionPatterns = [
      /ignore previous instructions/i,
      /system prompt/i,
      /###(.*?)###/gs,
      /\[INST\].*?\[\/INST\]/gs
    ];
    
    let sanitized = input;
    injectionPatterns.forEach(pattern => {
      sanitized = sanitized.replace(pattern, '[REDACTED]');
    });
    
    // Limita tamanho
    if (sanitized.length > 5000) {
      sanitized = sanitized.substring(0, 5000);
    }
    
    return sanitized;
  }

  /**
   * Sanitização de outputs de LLM
   */
  sanitizeLLMOutput(output) {
    // Remove conteúdo malicioso
    const maliciousPatterns = [
      /<script\b[^<]*(?:(?!<\/script>)<[^<]*)*<\/script>/gi,
      /javascript:/gi,
      /data:/gi,
      /on\w+\s*=/gi
    ];
    
    let sanitized = output;
    maliciousPatterns.forEach(pattern => {
      sanitized = sanitized.replace(pattern, '');
    });
    
    // Escapa HTML
    sanitized = sanitized
      .replace(/&/g, '&amp;')
      .replace(/</g, '&lt;')
      .replace(/>/g, '&gt;')
      .replace(/"/g, '&quot;')
      .replace(/'/g, '&#x27;');
    
    return sanitized;
  }

  /**
   * Middleware completo de segurança
   */
  getMiddleware() {
    return [
      this.securityHeaders,
      ...this.inputSanitization(),
      this.dataMasking(),
      this.rateLimiters.global
    ];
  }
}

module.exports = new SecurityMiddleware();
```

```python
class RedisQueueManager:
    def __init__(self, host: str = "localhost", port: int = 6379, password: Optional[str] = None):
        self.redis = redis.Redis(
            host=host,
            port=port,
            password=password,
            decode_responses=True,
            socket_keepalive=True
        )
        self.pubsub = self.redis.pubsub()
        
    async def connect(self):
        """Connect to Redis"""
        try:
            await self.redis.ping()
            logger.info("Connected to Redis successfully")
        except Exception as e:
            logger.error(f"Failed to connect to Redis: {e}")
            raise
    
    async def publish_task(self, queue_name: str, task: Dict[str, Any]) -> bool:
        """Publish a task to a specific queue"""
        try:
            task_json = json.dumps(task)
            await self.redis.lpush(queue_name, task_json)
            # Also publish to pubsub for real-time monitoring
            await self.redis.publish(f"queue:{queue_name}", task_json)
            logger.debug(f"Published task to queue {queue_name}: {task.get('id')}")
            return True
        except Exception as e:
            logger.error(f"Failed to publish task: {e}")
            return False
    
    async def consume_task(self, queue_name: str, timeout: int = 30) -> Optional[Dict[str, Any]]:
        """Consume a task from a queue with timeout"""
        try:
            result = await self.redis.brpop(queue_name, timeout=timeout)
            if result:
                _, task_json = result
                task = json.loads(task_json)
                logger.debug(f"Consumed task from queue {queue_name}: {task.get('id')}")
                return task
            return None
        except Exception as e:
            logger.error(f"Failed to consume task: {e}")
            return None
    
    async def get_queue_length(self, queue_name: str) -> int:
        """Get the length of a queue"""
        try:
            return await self.redis.llen(queue_name)
        except Exception as e:
            logger.error(f"Failed to get queue length: {e}")
            return 0
    
    async def publish_agent_event(self, agent_id: str, event_type: str, data: Dict[str, Any]):

        try:
            event = {
                "agent_id": agent_id,
                "event_type": event_type,
                "data": data,
                "timestamp": datetime.utcnow().isoformat()
            }
            await self.redis.publish("agent:events", json.dumps(event))
            logger.debug(f"Published agent event: {event_type} for agent {agent_id}")
        except Exception as e:
            logger.error(f"Failed to publish agent event: {e}")

        """Subscribe to agent events"""
        await self.pubsub.subscribe("agent:events")

                    await callback(event)

                    logger.error(f"Failed to process event: {e}")
```

    async def update_agent_status(self, agent_id: str, status: AgentStatus, metrics: Optional[AgentMetrics] = None):
        """Update agent status and metrics"""
        if agent_id in self.agents:
            self.agents[agent_id].status = status
            self.agents[agent_id].last_heartbeat = datetime.utcnow()
            
            if metrics:
                self.agents[agent_id].metrics = metrics

        """Select the best agent for a task based on multiple factors"""
        candidate_agents = []
        
        for agent_id, agent in self.agents.items():
            # Filter by agent type capability
            if not self._agent_can_handle_task(agent, task):
                continue
            
            # Filter by status
            if agent.status != AgentStatus.IDLE:
                continue
            
            # Calculate score for this agent
            score = self._calculate_agent_score(agent, task)
            
            candidate_agents.append({
                "agent_id": agent_id,
                "agent": agent,
                "score": score
            })
        
        if not candidate_agents:
            logger.warning(f"No available agents for task {task.id}")
            return None
        
        # Select agent with highest score
        best_agent = max(candidate_agents, key=lambda x: x["score"])
        
        # Log decision
        decision_log = {
            "timestamp": datetime.utcnow().isoformat(),
            "task_id": task.id,
            "task_type": task.type,
            "selected_agent": best_agent["agent_id"],
            "score": best_agent["score"],
            "candidates": len(candidate_agents),
            "reasoning": self._get_decision_reasoning(best_agent["agent"], task)
        }
        self.decision_log.append(decision_log)
        self.stats["decisions_made"] += 1
        
        logger.info(f"Selected agent {best_agent['agent_id']} for task {task.id} (score: {best_agent['score']:.2f})")
        return best_agent["agent_id"]
    
    def _agent_can_handle_task(self, agent: Agent, task: Task) -> bool:
        """Check if agent can handle the task type"""
        # Simple type matching - can be extended with capability matching
        return agent.type.value == task.type or task.type in agent.capabilities.get("handles", [])
    
    def _calculate_agent_score(self, agent: Agent, task: Task) -> float:
        """Calculate a score for agent selection"""
        score = 0.0
        
        # 1. Priority matching (40%)
        priority_factor = self._calculate_priority_factor(agent, task)
        score += priority_factor * 0.4
        
        # 2. Load balancing (30%)
        load_factor = self._calculate_load_factor(agent)
        score += load_factor * 0.3
        
        # 3. Performance history (20%)
        perf_factor = self._calculate_performance_factor(agent)
        score += perf_factor * 0.2
        
        # 4. Specialization (10%)
        spec_factor = self._calculate_specialization_factor(agent, task)
        score += spec_factor * 0.1
        
        return score

        """Calculate factor based on task priority and agent capability"""
        priority_weights = {
            TaskPriority.LOW: 0.2,

        }
        
        base_score = priority_weights.get(task.priority, 0.5)
        
        # Adjust based on agent's success rate with similar tasks
        if agent.metrics.total_processed > 0:
            success_rate = 1 - agent.metrics.error_rate
            return base_score * success_rate
        
        return base_score

### Exemplo de Código Padrão
```javascript
class SquadComponent {
  /**
   * @param {Object} config - Configuração do componente
   * @throws {SquadError} Se a configuração for inválida
   */
  constructor(config) {
    this.validateConfig(config);
    this.config = config;
    this.logger = createLogger(`squad:${this.constructor.name}`);
  }

  async execute(task) {
    try {
      this.logger.info('Executing task', { taskId: task.id });
      const result = await this.processTask(task);
      this.logger.info('Task completed', { taskId: task.id });
      return result;
    } catch (error) {
      this.logger.error('Task failed', { taskId: task.id, error });
      throw new SquadError('TASK_EXECUTION_FAILED', { cause: error });
    }
  }
}
```

# Arquitetura do Sistema - DIYAPP Evolution V12

## Visão Geral

O DIYAPP Evolution V12 implementa uma arquitetura de **squads autônomas** onde cada componente do sistema é operado por um agente especializado com responsabilidades bem definidas. O sistema é projetado para operação 100% autônoma com capacidade de refatoração paralela através do **Modo Hive**.

"""
Sistema de Compliance para Rastreamento de Dados de Execução
Conformidade com LGPD (Brasil) e GDPR (UE)
Versão: 1.0.0
"""

import json
import hashlib
import sqlite3
from datetime import datetime, timedelta
from typing import Dict, List, Optional, Tuple
import logging
from dataclasses import dataclass, asdict
from enum import Enum
import re

# Configuração de logging
logging.basicConfig(

    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',

        logging.FileHandler('compliance_audit.log'),

class DataCategory(Enum):
    """Categorias de dados pessoais conforme LGPD/GDPR"""
    IDENTIFIER = "identificador"  # Nome, email, CPF, ID único
    CONTACT = "contato"  # Telefone, endereço
    BEHAVIORAL = "comportamental"  # Histórico de uso, cliques
    TECHNICAL = "técnico"  # IP, user-agent, device ID
    SENSITIVE = "sensível"  # Dados de saúde, biometria, opinião política
    ANONYMIZED = "anonimizado"  # Dados irreversivelmente anonimizados

class LegalBasis(Enum):
    """Base legal para processamento conforme LGPD Art. 7 / GDPR Art. 6"""
    CONSENT = "consentimento"
    CONTRACT = "contrato"
    LEGAL_OBLIGATION = "obrigação_legal"
    VITAL_INTEREST = "interesse_vital"
    PUBLIC_INTEREST = "interesse_público"
    LEGITIMATE_INTEREST = "interesse_legítimo"

@dataclass

    """Registro de Atividade de Processamento (ROPA)"""
    id: str
    data_category: DataCategory
    purpose: str
    legal_basis: LegalBasis
    retention_days: int
    collection_timestamp: datetime
    anonymization_date: Optional[datetime] = None
    deletion_date: Optional[datetime] = None
    data_subject_id: Optional[str] = None
    is_anonymized: bool = False
    
    def to_dict(self):
        return {
            **asdict(self),
            'collection_timestamp': self.collection_timestamp.isoformat(),
            'anonymization_date': self.anonymization_date.isoformat() if self.anonymization_date else None,
            'deletion_date': self.deletion_date.isoformat() if self.deletion_date else None
        }

class GDPRComplianceManager:
    """Gerenciador central de conformidade LGPD/GDPR"""
    
    def __init__(self, db_path: str = 'compliance_data.db'):

        """Carrega políticas de retenção por categoria de dados"""
        return {

    def track_execution_data(self, data: Dict) -> str:
        """
        Rastreia dados de execução com conformidade LGPD/GDPR
        
        Args:
            data: Dicionário com dados da execução
            
        Returns:
            ID do registro criado
        """
        # Classificação automática de dados
        data_category = self._classify_data(data)
        
        # Determina base legal
        legal_basis = self._determine_legal_basis(data, data_category)
        
        # Define política de retenção
        retention_days = self.retention_policies.get(

        )
        
        # Cria registro ROPA
        record = DataProcessingRecord(
            id=hashlib.sha256(
                f"{datetime.now().isoformat()}{json.dumps(data)}".encode()
            ).hexdigest()[:32],
            data_category=data_category,
            purpose=data.get('purpose', 'execution_tracking'),
            legal_basis=legal_basis,
            retention_days=retention_days,
            collection_timestamp=datetime.now(),
            data_subject_id=data.get('user_id')
        )
        
        # Anonimiza dados sensíveis antes de armazenar
        anonymized_data = self._anonymize_sensitive_data(data)
        
        # Armazena no banco
        self._store_processing_record(record)
        self._store_execution_log(record.id, anonymized_data)
        
        logger.info(f"Dados de execução rastreados: {record.id} - Categoria: {data_category.value}")
        return record.id
    
    def _classify_data(self, data: Dict) -> DataCategory:
        """Classifica dados em categorias LGPD/GDPR"""
        # Verifica dados sensíveis
        sensitive_keywords = ['health', 'medical', 'biometric', 'political', 'religion', 'racial']
        if any(keyword in json.dumps(data).lower() for keyword in sensitive_keywords):
            return DataCategory.SENSITIVE
        
        # Verifica identificadores
        if data.get('user_id') or data.get('email') or data.get('cpf'):
            return DataCategory.IDENTIFIER
        
        # Verifica dados de contato
        if data.get('phone') or data.get('address'):
            return DataCategory.CONTACT
        
        # Verifica dados comportamentais
        if data.get('action') in ['click', 'view', 'purchase', 'search']:
            return DataCategory.BEHAVIORAL
        
        # Dados técnicos por padrão
        return DataCategory.TECHNICAL
    
    def _determine_legal_basis(self, data: Dict, category: DataCategory) -> LegalBasis:
        """Determina base legal para processamento"""
        # Dados técnicos/anônimos: interesse legítimo
        if category in [DataCategory.TECHNICAL, DataCategory.ANONYMIZED]:
            return LegalBasis.LEGITIMATE_INTEREST
        
        # Dados comportamentais com consentimento
        if category == DataCategory.BEHAVIORAL and data.get('consent_given'):
            return LegalBasis.CONSENT
        
        # Dados para execução de contrato
        if data.get('is_contractual'):
            return LegalBasis.CONTRACT
        
        # Default: interesse legítimo (com avaliação de impacto)
        return LegalBasis.LEGITIMATE_INTEREST
    
    def _anonymize_sensitive_data(self, data: Dict) -> Dict:
        """Anonimiza dados sensíveis de forma irreversível"""
        anonymized = data.copy()
        
        # Anonimiza identificadores
        if 'user_id' in anonymized:
            anonymized['user_id'] = hashlib.sha256(
                f"{anonymized['user_id']}{self._get_salt()}".encode()
            ).hexdigest()[:16]
        
        if 'email' in anonymized:
            anonymized['email'] = hashlib.sha256(
                anonymized['email'].encode()
            ).hexdigest()[:16] + '@anonymized.local'
        
        if 'cpf' in anonymized:
            anonymized['cpf'] = '***' + anonymized['cpf'][-3:] if len(anonymized['cpf']) > 3 else '***'
        
        # Anonimiza IP (mantém apenas prefixo)
        if 'ip_address' in anonymized:
            parts = anonymized['ip_address'].split('.')
            if len(parts) == 4:
                anonymized['ip_address'] = f"{parts[0]}.{parts[1]}.*.*"
        
        # Remove dados sensíveis explícitos
        sensitive_fields = ['password', 'credit_card', 'health_record', 'biometric_data']
        for field in sensitive_fields:
            if field in anonymized:
                del anonymized[field]
        
        return anonymized
    
    def _get_salt(self) -> str:
        """Gera salt para anonimização determinística"""
        return "lgpd_compliance_v12"
    
    def _map_to_retention_category(self, category: DataCategory) -> str:
        """Mapeia categoria de dados para política de retenção"""
        mapping = {
            DataCategory.TECHNICAL: 'technical_logs',
            DataCategory.BEHAVIORAL: 'behavioral_logs',
            DataCategory.IDENTIFIER: 'user_identifiers',
            DataCategory.SENSITIVE: 'sensitive_data',
            DataCategory.CONTACT: 'user_identifiers',
            DataCategory.ANONYMIZED: 'technical_logs'
        }
        return mapping.get(category, 'technical_logs')
    
    def _store_processing_record(self, record: DataProcessingRecord):
