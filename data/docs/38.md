# DIYAPP Evolution - V11 Core - Arquitetura de Microsserviços

## 1. ARQUITETURA DE SISTEMA - ADR-001

**ADR-001: Arquitetura Baseada em Eventos com Microsserviços**

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead

**CONTEXTO:**
DIYAPP V11 precisa escalar para suportar 100k usuários simultâneos com 99.9% de disponibilidade. A arquitetura monolítica atual não permite:
- Deploy independente de componentes
- Escala seletiva de serviços com alta carga
- Isolamento de falhas
- Evolução tecnológica por serviço

**DECISÃO:**
Adotar arquitetura baseada em eventos com microsserviços, usando:
1. **API Gateway** como ponto único de entrada
2. **Message Broker** (RabbitMQ) para comunicação assíncrona
3. **Microsserviços especializados** por domínio
4. **Event Store** para auditoria e replay de eventos
5. **Service Discovery** para auto-registro de serviços

**OPÇÕES CONSIDERADAS:**
- **Opção A:** Arquitetura de eventos com RabbitMQ - Prós: Maturidade, confiabilidade, suporte a padrões complexos. Contras: Overhead de configuração.
- **Opção B:** Kafka como message broker - Prós: Alta throughput, retenção de eventos. Contras: Complexidade operacional, overkill para nosso volume.
- **Opção C:** Comunicação síncrona REST - Prós: Simplicidade. Contras: Acoplamento temporal, cascata de falhas.

**Opção escolhida:** A - Justificativa: RabbitMQ oferece balanço ideal entre robustez e simplicidade operacional para nosso volume (até 10k msg/segundo).

**CONSEQUÊNCIAS:**
**Positivas:**
- Desacoplamento temporal entre serviços
- Escala independente por serviço
- Tolerância a falhas parcial
- Evolução tecnológica independente

**Negativas:**
- Complexidade distribuída aumentada
- Necessidade de monitoramento distribuído
- Garantias de consistência eventual

**Riscos:**
- Eventos perdidos: Mitigar com confirmações e dead letter queues
- Ordem de eventos: Usar correlation IDs e sequenciamento por aggregate

**REVISÃO:** 2024-07-15

## 2. STACK TECNOLÓGICA - ADR-002

**ADR-002: Stack Tecnológica Padronizada**

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra

**CONTEXTO:**
Necessidade de padronizar tecnologias para reduzir curva de aprendizado, facilitar manutenção e garantir compatibilidade entre serviços.

**DECISÃO:**
Stack padrão para todos os novos microsserviços:
- **Runtime:** Node.js 18+ (LTS)
- **Framework Backend:** Express.js + TypeScript
- **Framework Frontend:** React 18+ com Vite + TypeScript
- **Banco de Dados:** PostgreSQL 15 (dados relacionais) + Redis 7 (cache/sessões)
- **Message Broker:** RabbitMQ 3.12
- **Containerização:** Docker + Docker Compose
- **Orquestração:** Kubernetes (futuro)
- **Monitoramento:** Prometheus + Grafana + ELK Stack

**OPÇÕES CONSIDERADAS:**
- **Opção A:** Node.js/Express - Prós: Ecossistema rico, performance I/O, mesma linguagem no front/back. Contras: Single-threaded.
- **Opção B:** Go - Prós: Performance, concorrência nativa. Contras: Curva de aprendizado, ecossistema menor.
- **Opção C:** Python/FastAPI - Prós: Produtividade, ML integration. Contras: Performance, tipagem fraca.

**Opção escolhida:** A - Justificativa: Consistência com stack existente, produtividade da equipe, adequação para I/O bound applications.

**CONSEQUÊNCIAS:**
**Positivas:**
- Uniformidade de código entre equipes
- Reutilização de componentes
- Facilidade de contratação/manutenção

**Negativas:**
- Não otimizado para CPU-intensive tasks
- Dependência do ecossistema npm

**REVISÃO:** 2024-04-15

```yaml
name: CI Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        service: ['api-gateway', 'service-auth', 'service-users', 'lib-common']
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
        cache-dependency-path: 'packages/${{ matrix.service }}/package-lock.json'
    
    - name: Install dependencies
      run: |
        cd packages/${{ matrix.service }}
        npm ci
    
    - name: Lint
      run: |
        cd packages/${{ matrix.service }}
        npm run lint
    
    - name: Type check
      run: |
        cd packages/${{ matrix.service }}
        npm run type-check
    
    - name: Run tests
      run: |
        cd packages/${{ matrix.service }}
        npm test -- --coverage
      
    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        file: ./packages/${{ matrix.service }}/coverage/lcov.info
        flags: ${{ matrix.service }}
  
  security:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Run Snyk to check for vulnerabilities
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --all-projects
    
    - name: Run dependency check
      uses: dependency-review-action@v3
  
  build-docker:
    needs: [test, security]
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v2
    
    - name: Login to DockerHub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}
    
    - name: Build and push
      uses: docker/build-push-action@v4
      with:
        context: .
        file: ./packages/api-gateway/Dockerfile
        push: true
        tags: |
          ${{ secrets.DOCKER_USERNAME }}/diyapp-api-gateway:latest
          ${{ secrets.DOCKER_USERNAME }}/diyapp-api-gateway:${{ github.sha }}
        cache-from: type=gha
        cache-to: type=gha,mode=max
```

### 4.2. `.github/workflows/cd.yml`

```yaml
name: CD Pipeline

on:
  workflow_run:
    workflows: ["CI Pipeline"]
    branches: [main]
    types:
      - completed

jobs:
  deploy-staging:
    if: ${{ github.event.workflow_run.conclusion == 'success' }}
    runs-on: ubuntu-latest
    environment: staging
    
    steps:
    - name: Deploy to Staging
      uses: appleboy/ssh-action@master
      with:
        host: ${{ secrets.STAGING_HOST }}
        username: ${{ secrets.STAGING_USER }}
        key: ${{ secrets.STAGING_SSH_KEY }}
        script: |
          cd /opt/diyapp-v11
          docker-compose pull
          docker-compose up -d --build
          docker system prune -f
    
    - name: Run smoke tests
      run: |
        curl -f ${{ secrets.STAGING_URL }}/health || exit 1
  
  deploy-production:
    needs: [deploy-staging]
    runs-on: ubuntu-latest
    environment: production
    
    steps:
    - name: Approve production deployment
      uses: trstringer/manual-approval@v1
      with:
        secret: ${{ github.TOKEN }}
        approvers: ${{ vars.PRODUCTION_APPROVERS }}
        minimum-approvals: 2
    
    - name: Deploy to Production
      uses: appleboy/ssh-action@master
      with:
        host: ${{ secrets.PRODUCTION_HOST }}
        username: ${{ secrets.PRODUCTION_USER }}
        key: ${{ secrets.PRODUCTION_SSH_KEY }}
        script: |
          cd /opt/diyapp-v11
          docker-compose -f docker-compose.prod.yml pull
          docker-compose -f docker-compose.prod.yml up -d --build
          docker system prune -f
    
    - name: Verify deployment
      run: |
        for i in {1..10}; do
          if curl -f ${{ secrets.PRODUCTION_URL }}/health; then
            echo "Deployment successful"
            exit 0
          fi
          sleep 10
        done
        echo "Deployment failed"
        exit 1
```

```dockerfile
# Build stage
FROM node:18-alpine AS builder

WORKDIR /app

# Copy package files
COPY package*.json ./
COPY tsconfig.json ./

# Install dependencies
RUN npm ci --only=production

# Copy source code
COPY src/ ./src/

# Build TypeScript
RUN npm run build

# Production stage
FROM node:18-alpine

WORKDIR /app

# Create non-root user
RUN addgroup -g 1001 -S nodejs
RUN adduser -S nodejs -u 1001

# Copy from builder
COPY --from=builder --chown=nodejs:nodejs /app/node_modules ./node_modules
COPY --from=builder --chown=nodejs:nodejs /app/dist ./dist
COPY --from=builder --chown=nodejs:nodejs /app/package*.json ./

# Switch to non-root user
USER nodejs

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:${PORT:-3000}/health', (r) => {if(r.statusCode !== 200) throw new Error()})"

# Expose port
EXPOSE 3000

# Start application
CMD ["node", "dist/index.js"]
```

from typing import List, Optional, Dict, Any
from sqlalchemy.orm import Session
from sqlalchemy import and_, or_, desc, asc
from ..models.project import ProjectDB, ProjectStatus
import logging
from uuid import uuid4

logger = logging.getLogger(__name__)

class ProjectRepository:
    def __init__(self, db: Session):
        self.db = db
    
    def create(self, project_data: Dict[str, Any], user_id: int) -> ProjectDB:
        """Create a new project with parameterized query."""
        try:
            project = ProjectDB(
                name=project_data["name"],
                description=project_data.get("description"),
                config=project_data.get("config", {}),
                created_by=user_id,
                status="active"
            )
            self.db.add(project)
            self.db.flush()  # Get ID without commit
            return project
        except Exception as e:
            logger.error(f"Error creating project: {e}", exc_info=True)
            raise
    
    def get_by_id(self, project_id: int, user_id: Optional[int] = None) -> Optional[ProjectDB]:
        """Get project by ID with optional user authorization."""
        try:
            query = self.db.query(ProjectDB).filter(
                ProjectDB.id == project_id,
                ProjectDB.is_deleted == False
            )
            
            if user_id:
                query = query.filter(ProjectDB.created_by == user_id)
            
            return query.first()
        except Exception as e:
            logger.error(f"Error getting project {project_id}: {e}", exc_info=True)
            raise
    
    def get_all(self, user_id: int, skip: int = 0, limit: int = 100, 
                status: Optional[str] = None) -> List[ProjectDB]:
        """Get all projects for a user with pagination."""
        try:
            query = self.db.query(ProjectDB).filter(
                ProjectDB.created_by == user_id,
                ProjectDB.is_deleted == False
            )
            
            if status:
                query = query.filter(ProjectDB.status == status)
            
            return query.order_by(desc(ProjectDB.created_at)).offset(skip).limit(limit).all()
        except Exception as e:
            logger.error(f"Error getting projects for user {user_id}: {e}", exc_info=True)
            raise
    
    def update(self, project_id: int, update_data: Dict[str, Any], user_id: int) -> Optional[ProjectDB]:
        """Update project with authorization check."""
        try:
            project = self.get_by_id(project_id, user_id)
            if not project:
                return None
            
            for key, value in update_data.items():
                if value is not None:
                    setattr(project, key, value)
            
            return project
        except Exception as e:
            logger.error(f"Error updating project {project_id}: {e}", exc_info=True)
            raise
    
    def delete(self, project_id: int, user_id: int) -> bool:
        """Soft delete project."""
        try:
            project = self.get_by_id(project_id, user_id)
            if not project:
                return False
            
            project.is_deleted = True
            project.status = "archived"
            return True
        except Exception as e:
            logger.error(f"Error deleting project {project_id}: {e}", exc_info=True)
            raise
    
    def count_by_user(self, user_id: int) -> int:
        """Count active projects for a user."""
        try:
            return self.db.query(ProjectDB).filter(
                ProjectDB.created_by == user_id,
                ProjectDB.is_deleted == False,
                ProjectDB.status == "active"
            ).count()
        except Exception as e:
            logger.error(f"Error counting projects for user {user_id}: {e}", exc_info=True)
            raise

```sql
-- Projetos criados pelos usuários
CREATE TABLE raw_projects (
    project_id UUID PRIMARY KEY,

    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,

    source_system VARCHAR(50) DEFAULT 'diyapp_api',
    ingested_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE raw_agent_executions (
    execution_id UUID PRIMARY KEY,
    project_id UUID NOT NULL,
    agent_type VARCHAR(50) NOT NULL, -- 'data_engineer', 'backend_dev', 'frontend_dev', 'ai_specialist'
    execution_status VARCHAR(20) NOT NULL, -- 'pending', 'running', 'success', 'failed', 'cancelled'
    input_data JSONB,
    output_data JSONB,
    error_message TEXT,
    started_at TIMESTAMP WITH TIME ZONE,

);

    log_id BIGSERIAL PRIMARY KEY,

    log_message TEXT NOT NULL,
    log_context JSONB,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    source_system VARCHAR(50) DEFAULT 'agent_logger',
    ingested_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE raw_performance_metrics (
    metric_id BIGSERIAL PRIMARY KEY,
    execution_id UUID NOT NULL,

    metric_value DECIMAL(15,6) NOT NULL,
    metric_unit VARCHAR(20), -- 'ms', 'tokens', 'percentage', 'count'

);

);
```

## 1. Threat Modeling STRIDE - V11

### Assets Críticos:
1. **Dados de usuário**: PII, credenciais, tokens de sessão
2. **Sistema de mensagens**: WhatsApp Business API, histórico de conversas

### Análise STRIDE:

#### **Spoofing**:
- **Ameaça**: Ataque de força bruta em login, token JWT roubado
- **Controles**: MFA obrigatório para admin, rate limiting em login, tokens JWT com expiração curta (15min), refresh tokens com expiração (7dias)

#### **Tampering**:
- **Ameaça**: Manipulação de dados em trânsito, SQL injection
- **Controles**: TLS 1.3 obrigatório, validação de input rigorosa, prepared statements, assinatura de payloads críticos

#### **Repudiation**:
- **Ameaça**: Negação de ações administrativas críticas
- **Controles**: Logs imutáveis de auditoria (CloudTrail-like), assinatura digital de logs críticos

#### **Information Disclosure**:
- **Ameaça**: Vazamento de secrets, dados sensíveis em logs, erro messages detalhadas
- **Controles**: Secret management (Vault), mascaramento de PII em logs, sanitização de erros

#### **Denial of Service**:
- **Ameaça**: Ataque DDoS em endpoints públicos, sobrecarga de LLM
- **Controles**: Rate limiting por IP/usuario, circuit breakers, quotas de uso de LLM

#### **Elevation of Privilege**:
- **Ameaça**: IDOR, falha de autorização, prompt injection em LLM

### 2.1. Rate Limiter por Usuário

**src/security/middleware/rate-limiter.js**:
```javascript
const rateLimit = require('express-rate-limit');
const Redis = require('ioredis');
const { createHash } = require('crypto');

class RateLimiter {
    constructor() {
        this.redisClient = new Redis({
            host: process.env.REDIS_HOST || 'localhost',
            port: process.env.REDIS_PORT || 6379,
            password: process.env.REDIS_PASSWORD
        });

        this.limits = {
            // Limites por endpoint
            'auth-login': { windowMs: 15 * 60 * 1000, max: 5 }, // 5 tentativas em 15min
            'api-general': { windowMs: 60 * 1000, max: 60 }, // 60 req/min
            'llm-requests': { windowMs: 60 * 1000, max: 10 }, // 10 req/min por usuário
            'whatsapp-send': { windowMs: 60 * 1000, max: 30 } // 30 msg/min
        };
    }

    getUserIdentifier(req) {
        // Prioridade: user ID > IP > session ID
        if (req.user && req.user.id) {
            return `user:${req.user.id}`;
        }
        
        const ip = req.headers['x-forwarded-for'] || req.connection.remoteAddress;
        const ipHash = createHash('sha256').update(ip).digest('hex');
        
        return `ip:${ipHash}`;
    }

    createLimiter(endpointKey) {
        const limitConfig = this.limits[endpointKey] || this.limits['api-general'];
        
        return rateLimit({
            windowMs: limitConfig.windowMs,
            max: limitConfig.max,
            keyGenerator: (req) => {
                return `${endpointKey}:${this.getUserIdentifier(req)}`;
            },
            handler: (req, res) => {
                res.status(429).json({
                    error: 'Too many requests',
                    message: `Rate limit exceeded for ${endpointKey}. Please try again later.`,
                    retryAfter: Math.ceil(limitConfig.windowMs / 1000)
                });
            },
            skip: (req) => {
                // Skip rate limiting for internal health checks
                return req.path === '/health' || req.path === '/metrics';
            }
        });
    }

    async getRateLimitStatus(userId) {
        const keys = await this.redisClient.keys(`*:user:${userId}:*`);
        const status = {};
        
        for (const key of keys) {
            const [endpoint] = key.split(':');
            const count = await this.redisClient.get(key);
            const ttl = await this.redisClient.ttl(key);
            
            status[endpoint] = {
                current: parseInt(count) || 0,
                ttl: ttl,
                limit: this.limits[endpoint]?.max || 60
            };
        }
        
        return status;
    }

    async resetRateLimit(userId, endpoint) {
        const key = `${endpoint}:user:${userId}`;
        await this.redisClient.del(key);
    }
}

module.exports = new RateLimiter();
```

**src/security/middleware/input-sanitizer.js**:

```javascript
const { body, query, param, validationResult } = require('express-validator');
const { escape } = require('validator');
const DOMPurify = require('isomorphic-dompurify');

class InputSanitizer {
    constructor() {
        this.sanitizationRules = {
            // Regras por endpoint/field
            'user-input': {
                name: body('name').trim().escape().isLength({ min: 2, max: 100 }),
                email: body('email').trim().normalizeEmail().isEmail(),
                password: body('password').isLength({ min: 8 }).matches(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$/)
            },
            'llm-prompt': {
                prompt: body('prompt')
                    .trim()
                    .isLength({ min: 1, max: 5000 })
                    .customSanitizer(value => this.sanitizeLLMPrompt(value))
            },
            'whatsapp-message': {
                to: body('to').trim().matches(/^\+[1-9]\d{1,14}$/), // E.164 format
                message: body('message')
                    .trim()
                    .isLength({ min: 1, max: 1600 })
                    .customSanitizer(value => this.sanitizeWhatsAppMessage(value))
            }
        };
    }

    sanitizeLLMPrompt(prompt) {
        // Remove tentativas de prompt injection
        const injectionPatterns = [
            /ignore previous instructions/i,
            /system prompt/i,
            /roleplay as/i,
            /you are now/i,
            /forget that/i,
            /disregard/i
        ];
        
        let sanitized = prompt;
        injectionPatterns.forEach(pattern => {
            sanitized = sanitized.replace(pattern, '[removed]');
        });
        
        // Limita comprimento
        sanitized = sanitized.substring(0, 5000);
        
        return sanitized;
    }

    sanitizeWhatsAppMessage(message) {
        // Remove caracteres perigosos para WhatsApp API
        const dangerousChars = /[\x00-\x08\x0B\x0C\x0E-\x1F\x7F]/g;
        let sanitized = message.replace(dangerousChars, '');
        
        // Escapa HTML se necessário
        sanitized = DOMPurify.sanitize(sanitized, {
            ALLOWED_TAGS: [], // No HTML allowed
            ALLOWED_ATTR: []
        });
        
        return sanitized;
    }

    sanitizeSQLInput(input) {
        // Remove SQL injection patterns (simplificado - usar prepared statements é essencial)
        const sqlPatterns = [
            /(\b(SELECT|INSERT|UPDATE|DELETE|DROP|UNION|EXEC|ALTER)\b)/gi,
            /('|"|`|--|\/\*|\*\/|;)/g
        ];
        
        let sanitized = input;
        sqlPatterns.forEach(pattern => {
            sanitized = sanitized.replace(pattern, '');
        });
        
        return sanitized;
    }

    getValidationRules(endpoint) {
        const rules = this.sanitizationRules[endpoint];
        if (!rules) {
            return Object.values(this.sanitizationRules['user-input']);
        }
        
        return Object.values(rules);
    }

    validate(req, res, next) {
        const errors = validationResult(req);
        if (!errors.isEmpty()) {
            return res.status(400).json({
                error: 'Validation failed',
                details: errors.array().map(err => ({
                    field: err.param,
                    message: err.msg,
                    value: err.value
                }))
            });
        }
        next();
    }

    // Middleware para sanitização automática
    middleware(endpoint) {
        return [
            ...this.getValidationRules(endpoint),
            this.validate.bind(this)
        ];
    }
}

module.exports = new InputSanitizer();
```

```javascript
const jwt = require('jsonwebtoken');
const jwksClient = require('jwks-rsa');
const crypto = require('crypto');

class JWTValidator {
    constructor() {
        this.jwksClient = jwksClient({
            jwksUri: process.env.JWKS_URI || 'https://your-auth-server/.well-known/jwks.json',
            cache: true,
            rateLimit: true,
            jwksRequestsPerMinute: 10
        });

        this.algorithms = ['RS256', 'RS512']; // Nunca usar HS256 em produção
        this.issuer = process.env.JWT_ISSUER || 'diyapp-auth';
        this.audience = process.env.JWT_AUDIENCE || 'diyapp-api';
    }

    getKey(header, callback) {
        this.jwksClient.getSigningKey(header.kid, (err, key) => {
            if (err) {
                return callback(err);
            }
            const signingKey = key.getPublicKey();
            callback(null, signingKey);
        });
    }

    async verifyToken(token) {
        return new Promise((resolve, reject) => {
            const verifyOptions = {
                algorithms: this.algorithms,
                issuer: this.issuer,
                audience: this.audience,
                clockTolerance: 30, // 30 segundos de tolerância
                maxAge: '15m' // Access tokens expiram em 15 minutos
            };

            jwt.verify(token, this.getKey.bind(this), verifyOptions, (err, decoded) => {
                if (err) {
                    return reject(this.mapJWTError(err));
                }
                resolve(decoded);
            });
        });
    }

    mapJWTError(err) {
        const errors = {
            'TokenExpiredError': { code: 'TOKEN_EXPIRED', status: 401 },
            'JsonWebTokenError': { code: 'INVALID_TOKEN', status: 401 },
            'NotBeforeError': { code: 'TOKEN_NOT_ACTIVE', status: 401 }
        };
        
        return errors[err.name] || { code: 'AUTH_ERROR', status: 401 };
    }

    async generateTokens(user) {
        // Access token (15 minutos)
        const accessToken = jwt.sign(
            {
                sub: user.id,
                email: user.email,
                roles: user.roles,
                permissions: user.permissions
            },
            process.env.JWT_PRIVATE_KEY,
            {
                algorithm: 'RS256',
                expiresIn: '15m',
                issuer: this.issuer,
                audience: this.audience,
                jwtid: crypto.randomBytes(16).toString('hex') // JTI único
            }
        );

        // Refresh token (7 dias)
        const refreshToken = jwt.sign(
            {
                sub: user.id,
                type: 'refresh'
            },
            process.env.JWT_REFRESH_SECRET,
            {
                algorithm: 'HS256',
                expiresIn: '7d',
                issuer: this.issuer,
                jwtid: crypto.randomBytes(16).toString('hex')
            }
        );

        return {
            access_token: accessToken,
            refresh_token: refreshToken,
            token_type: 'Bearer',
            expires_in: 900 // 15 minutos em segundos
        };
    }

    async refreshAccessToken(refreshToken) {
        try {
            const decoded = jwt.verify(refreshToken, process.env.JWT_REFRESH_SECRET, {
                issuer: this.issuer
            });

            if (decoded.type !== 'refresh') {
                throw new Error('Invalid token type');
            }

            // Verificar se o refresh token não foi revogado
            const isRevoked = await this.isTokenRevoked(decoded.jti);
            if (isRevoked) {
                throw new Error('Token revoked');
            }

            // Gerar novo access token
            const user = await this.getUserById(decoded.sub);
            return this.generateTokens(user);
        } catch (error) {
            throw this.mapJWTError(error);
        }
    }

    async revokeToken(jti) {
        // Armazenar JTI revogado no Redis por 7 dias
        const redisKey = `revoked:${jti}`;
        await this.redisClient.setex(redisKey, 7 * 24 * 60 * 60, '1');
    }

    async isTokenRevoked(jti) {
        const redisKey = `revoked:${jti}`;
        const result = await this.redisClient.get(redisKey);
        return result === '1';
    }

    // Middleware para validação JWT
    middleware() {
        return async (req, res, next) => {
            try {
                const authHeader = req.headers.authorization;
                if (!authHeader || !authHeader.startsWith('Bearer ')) {
                    return res.status(401).json({ error: 'No token provided' });
                }

                const token = authHeader.substring(7);
                const decoded = await this.verifyToken(token);

                // Verificar se o token foi revogado
                if (await this.isTokenRevoked(decoded.jti)) {
                    return res.status(401).json({ error: 'Token revoked' });
                }

                // Adicionar usuário ao request
                req.user = {
                    id: decoded.sub,
                    email: decoded.email,
                    roles: decoded.roles || [],
                    permissions: decoded.permissions || []
                };

                // Log de auditoria
                this.logAccess(req);

                next();
            } catch (error) {
                res.status(error.status || 401).json({
                    error: error.code || 'AUTH_ERROR',
                    message: 'Invalid or expired token'
                });
            }
        };
    }

    logAccess(req) {
        console.log(`[AUDIT] User ${req.user.id} accessed ${req.method} ${req.path} at ${new Date().toISOString()}`);
    }
}
```

```javascript
/**
 * DIYAPP Evolution - V11 Core
 * Product Owner: Backlog Refiner e Critérios de Aceite Generator
 * 
 * Este módulo transforma intenções de negócio em especificações executáveis
 * com critérios de aceite no formato Gherkin.
 */

class BacklogRefiner {
  constructor() {
    this.dod = {
      universal: [
        "Cobertura de testes automatizados ≥ 80% para código novo",
        "Nenhum warning crítico no linter ou análise estática",
        "PR revisado e aprovado pelo Tech Lead",
        "Documentação atualizada pelo Tech Writer",
        "Sem regressões nos testes existentes em staging",
        "Especialista de Segurança validou (quando aplicável)",
        "Card no Kanban atualizado com link para PR mergeado"
      ],
      squadSpecific: {
        frontend: ["Lighthouse performance score ≥ 90", "Cross-browser testing concluído"],
        backend: ["API documentada no Swagger", "Métricas de performance coletadas"],
        ai: ["Modelo versionado no MLflow", "Drift detection implementado"]
      }
    };
  }

  /**
   * Refina uma história de usuário com critérios de aceite Gherkin
   */
  refineUserStory(pmCard) {
    if (!pmCard.businessContext || !pmCard.priority) {
      throw new Error("Card incompleto: falta contexto de negócio ou prioridade. Devolver ao PM.");
    }

    const refinedStory = {
      id: `STORY-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`,
      title: pmCard.title,
      businessContext: pmCard.businessContext,
      priority: pmCard.priority,
      
      // Template de história de usuário
      userStory: {
        asA: this.determineUserType(pmCard),
        iWant: pmCard.featureDescription,
        soThat: pmCard.businessValue
      },
      
      // Critérios de aceite em Gherkin
      acceptanceCriteria: {
        happyPath: this.generateHappyPathCriteria(pmCard),
        edgeCases: this.generateEdgeCases(pmCard),
        errorScenarios: this.generateErrorScenarios(pmCard),
        performanceCriteria: this.generatePerformanceCriteria(pmCard)
      },
      
      // Dependências
      dependencies: this.identifyDependencies(pmCard),
      
      // Artefatos de referência
      references: {
        mockup: pmCard.mockupUrl || "N/A",
        apiSpec: pmCard.apiSpecUrl || "N/A",
        designSystem: "https://design.diyapp.com/v11"
      },
      
      // Notas técnicas
      technicalNotes: this.generateTechnicalNotes(pmCard),
      
      // Definition of Done aplicável
      applicableDoD: this.getApplicableDoD(pmCard)
    };

    return refinedStory;
  }

  /**
   * Gera critérios de aceite para caminho feliz
   */
  generateHappyPathCriteria(pmCard) {
    const criteria = [];
    
    // Exemplo para criação de projeto
    if (pmCard.title.includes("Criar Projeto")) {
      criteria.push({
        scenario: "Criação de projeto bem-sucedida",
        gherkin: `Dado que estou autenticado como usuário premium
                  E estou na página "Meus Projetos"
                  Quando preencho o formulário com:
                    | Campo       | Valor               |
                    | Nome        | Meu Novo Projeto    |
                    | Descrição   | Descrição do projeto|
                    | Template    | React + Node        |
                  E clico em "Criar Projeto"
                  Então o sistema deve:
                    - Persistir o projeto no banco de dados
                    - Retornar status HTTP 201
                    - Exibir o projeto na lista "Meus Projetos"
                    - Enviar notificação de confirmação por email
                    - Registrar log de auditoria`
      });
    }
    
    // Exemplo para deploy automático
    if (pmCard.title.includes("Deploy Automático")) {
      criteria.push({
        scenario: "Deploy automático bem-sucedido",

    }

    return criteria;
  }

  /**
   * Gera edge cases
   */
  generateEdgeCases(pmCard) {
    const edgeCases = [];
    
    if (pmCard.title.includes("Criar Projeto")) {
      edgeCases.push({

  /**

  generateErrorScenarios(pmCard) {

   * Gera critérios de performance

  generatePerformanceCriteria(pmCard) {

  /**
   * Determina o tipo de usuário baseado no card
   */
  determineUserType(pmCard) {
    const userTypes = {
      admin: "administrador do sistema",
      premium: "usuário premium",
      free: "usuário free",
      developer: "desenvolvedor integrando com API",
      system: "sistema automatizado"

    return userTypes[pmCard.userType] || "usuário autenticado";
  }

  /**
   * Retorna DoD aplicável ao card
   */
  getApplicableDoD(pmCard) {
    const applicable = [...this.dod.universal];
    
    if (pmCard.team === "frontend") {
      applicable.push(...this.dod.squadSpecific.frontend);
    } else if (pmCard.team === "backend") {
      applicable.push(...this.dod.squadSpecific.backend);
    } else if (pmCard.team === "ai") {
      applicable.push(...this.dod.squadSpecific.ai);
    }

    return applicable;
  }

  /**
   * Valida uma entrega contra os critérios de aceite
   */
  validateDelivery(story, delivery) {
    const validation = {
      storyId: story.id,
      deliveryId: delivery.id,
      timestamp: new Date().toISOString(),
      validatedBy: "PO-Senior",
      results: [],
      status: "pending",
      comments: []
    };

    // Valida cada critério
    let allPassed = true;
    
    // Happy Path
    story.acceptanceCriteria.happyPath.forEach((criterion, index) => {
      const result = this.validateCriterion(criterion, delivery);
      validation.results.push({
        type: "happyPath",
        scenario: criterion.scenario,
        passed: result.passed,
        evidence: result.evidence
      });
      
      if (!result.passed) {
        allPassed = false;
        validation.comments.push(`CRITÉRIO ${index + 1} FALHOU: ${criterion.scenario}`);
      }
    });

    // Edge Cases
    story.acceptanceCriteria.edgeCases.forEach((criterion, index) => {
      const result = this.validateCriterion(criterion, delivery);
      validation.results.push({
        type: "edgeCase",
        scenario: criterion.scenario,
        passed: result.passed,
        evidence: result.evidence
      });
      
      if (!result.passed) {
        allPassed = false;
        validation.comments.push(`EDGE CASE ${index + 1} FALHOU: ${criterion.scenario}`);
      }
    });

    // Valida DoD
    const dodValidation = this.validateDoD(story.applicableDoD, delivery);
    validation.dodValidation = dodValidation;
    
    if (!dodValidation.allPassed) {
      allPassed = false;
      validation.comments.push("DEFINITION OF DONE NÃO ATENDIDO");
    }

    validation.status = allPassed ? "accepted" : "rejected";
    
    if (validation.status === "rejected") {
      validation.rejectionReason = validation.comments.join("; ");
      validation.nextSteps = "Corrigir critérios falhados e reenviar para validação";
    }

    return validation;
  }

  /**
   * Valida um critério individual
   */
  validateCriterion(criterion, delivery) {
    // Implementação real validaria contra evidências de entrega
    // Por enquanto, simula validação
    return {
      passed: Math.random() > 0.1, // 90% de sucesso para demonstração
      evidence: {
        testResults: delivery.testResults || [],
        logs: delivery.logs || [],
        screenshots: delivery.screenshots || []
      }
    };
  }

  /**
   * Valida Definition of Done
   */
  validateDoD(dodItems, delivery) {
    const results = dodItems.map(item => ({
      item,
      passed: this.checkDoDItem(item, delivery)
    }));

    return {
      results,
      allPassed: results.every(r => r.passed),
      passedCount: results.filter(r => r.passed).length,
      totalCount: results.length
    };
  }

  /**
   * Verifica um item específico do DoD
   */
  checkDoDItem(item, delivery) {
    // Lógica de verificação real
    if (item.includes("testes automatizados")) {
      return delivery.testCoverage >= 80;
    }
    
    if (item.includes("linter")) {
      return delivery.linterWarnings === 0;
    }
    
    if (item.includes("PR revisado")) {
      return delivery.prApproved === true;
    }
    
    return true; // Para demonstração
  }

  /**
   * Orquestra o Modo Hive para paralelização
   */
  orchestrateHiveMode(stories) {
    const backlog = {
      tasks: [],
      sprintId: `SPRINT-${Date.now()}`,
      startDate: new Date().toISOString(),
      endDate: new Date(Date.now() + 14 * 24 * 60 * 60 * 1000).toISOString(), // 2 semanas
      dependenciesMap: this.buildDependenciesMap(stories)
    };

    // Distribui tarefas para agentes baseado no tipo de história
    stories.forEach(story => {
      const agentTasks = this.assignToAgents(story);
      backlog.tasks.push(...agentTasks);
    });

    return backlog;
  }

  /**
   * Atribui tarefas a agentes específicos
   */
  assignToAgents(story) {
    const tasks = [];

    if (story.userStory.iWant.includes("interface") || story.userStory.iWant.includes("página")) {

        ],

      });
    }

    // QA
    tasks.push({
      agent: "QA",
      desc: `Criar plano de testes para: ${story.title}`,
      storyId: story.id,
      acceptanceCriteria: [
        "Testes automatizados baseados nos critérios Gherkin",

        "Testes de performance e carga",
        "Testes de segurança OWASP",
        "Relatório de cobertura de testes"
      ],
      estimatedHours: 12
    });

    // SRE (se aplicável)
    if (story.title.includes("Deploy") || story.title.includes("Infra")) {
      tasks.push({
        agent: "SRE",
        desc: `Provisionar infraestrutura para: ${story.title}`,
        storyId: story.id,

          "Alertas configurados no PagerDuty",
          "Backup e disaster recovery testado",
          "Documentação de runbooks"
        ],
        estimatedHours: 20
      });
    }

    return tasks;
  }

  /**
   * Constrói mapa de dependências
   */
  buildDependenciesMap(stories) {
    const map = {};
    
    stories.forEach(story => {
      map[story.id] = {
        story: story.title,
        dependencies: story.dependencies,
        blocks: story.dependencies.blocks,
        blockedBy: story.dependencies.blockedBy
      };
    });

    return map;
  }
```
