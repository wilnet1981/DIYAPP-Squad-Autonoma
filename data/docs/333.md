# DIYAPP Evolution - V12 Core - Documentação de Arquitetura

## ADR-001: Estrutura Arquitetural - Monólito Modular vs Microserviços

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V12 Core

### CONTEXTO:
O DIYAPP está evoluindo para a versão V12 com requisitos de:
- Escalabilidade horizontal para suportar 100k usuários concorrentes
- Alta disponibilidade (99.99% uptime)
- Manutenção de múltiplas equipes trabalhando em paralelo
- Deployment independente de componentes
- Performance consistente (P95 < 200ms)

### DECISÃO:
Adotar arquitetura **Monólito Modular com decomposição por domínio**, preparada para evolução para microserviços quando necessário.

### OPÇÕES CONSIDERADAS:

**Opção A: Microserviços Puros**
- Prós: Escalabilidade granular, deploy independente, isolamento de falhas
- Contras: Complexidade operacional, latência de rede, consistência distribuída difícil
- Custo estimado: 3x em infraestrutura, 2x em desenvolvimento

**Opção B: Monólito Tradicional**
- Prós: Simplicidade, debug fácil, transações ACID
- Contras: Escalabilidade limitada, acoplamento alto, deploy tudo-ou-nada

**Opção C: Monólito Modular (Escolhida)**
- Prós: Benefícios de monólito com modularidade de microserviços
- Contras: Requer disciplina de equipe, acoplamento ainda possível
- Justificativa: Balanceia agilidade com estabilidade, permite evolução gradual

### CONSEQUÊNCIAS:
**Positivas:**
- Time to market mais rápido para V12
- Debug simplificado com logs centralizados
- Transações ACID mantidas
- Evolução gradual para microserviços quando métricas justificarem

**Negativas:**
- Requer disciplina rigorosa de boundaries de módulo
- Deploy ainda é monolítico (mas com hot reload de módulos)

**Riscos:**
- Acoplamento acidental entre módulos
- Mitigação: Contract tests e reviews de PR rigorosos

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V12 Core + Especialista Infra

### CONTEXTO:
Módulos precisam se comunicar de forma eficiente, com baixa latência e alta confiabilidade.

### DECISÃO:
Implementar comunicação híbrida:
1. **Síncrona intra-processo** para operações críticas

## ADR-003: Estratégia de Cache - Redis

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V12 Core + Especialista Infra

### CONTEXTO:
Necessidade de reduzir latência para <200ms P95, reduzir carga no banco.

### DECISÃO:
Implementar cache multi-camada com Redis:
1. **L1 Cache:** In-memory (Node.js) para dados ultra-quentes
2. **L2 Cache:** Redis Cluster para dados compartilhados
3. **Cache-Aside pattern** com invalidation por evento

### IMPLEMENTAÇÃO:
```javascript
// src/infrastructure/cache/RedisManager.js
const Redis = require('ioredis');
const logger = require('../logging/Logger');

class RedisManager {
  constructor() {
    this.cluster = new Redis.Cluster([
      { host: process.env.REDIS_NODE_1, port: 6379 },
      { host: process.env.REDIS_NODE_2, port: 6379 },
      { host: process.env.REDIS_NODE_3, port: 6379 }
    ]);
    
    this.localCache = new Map();
    this.ttl = {
      SHORT: 60, // 1 minuto
      MEDIUM: 300, // 5 minutos
      LONG: 3600 // 1 hora
    };
  }

  async getWithCache(key, fetchFn, ttl = this.ttl.MEDIUM) {
    // 1. Verifica cache local
    const local = this.localCache.get(key);
    if (local && Date.now() - local.timestamp < 10000) {
      logger.debug('Cache hit L1', { key });
      return local.data;
    }

    // 2. Verifica Redis
    const cached = await this.cluster.get(key);
    if (cached) {
      const data = JSON.parse(cached);
      // Atualiza cache local
      this.localCache.set(key, {
        data,
        timestamp: Date.now()
      });
      logger.debug('Cache hit L2', { key });
      return data;
    }

    // 3. Cache miss - busca original
    logger.debug('Cache miss', { key });
    const freshData = await fetchFn();
    
    // 4. Atualiza caches
    await this.cluster.setex(key, ttl, JSON.stringify(freshData));
    this.localCache.set(key, {
      data: freshData,
      timestamp: Date.now()
    });

    return freshData;
  }

  async invalidate(pattern) {
    const keys = await this.cluster.keys(pattern);
    if (keys.length > 0) {
      await this.cluster.del(...keys);
      // Limpa cache local correspondente
      for (const key of this.localCache.keys()) {
        if (key.includes(pattern.replace('*', ''))) {
          this.localCache.delete(key);
        }
      }
    }
  }
}

module.exports = new RedisManager();
```

## ADR-004: Sistema de Filas - RabbitMQ

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V12 Core + Especialista Infra

### CONTEXTO:
Processamento assíncrono de tarefas pesadas, garantia de entrega, retry com backoff.

### DECISÃO:
RabbitMQ com:
- **DLX (Dead Letter Exchange)** para retry automático
- **Persistent messages** para não perder tarefas
- **Priority queues** para tarefas críticas

### IMPLEMENTAÇÃO:
```javascript
// src/infrastructure/queue/QueueManager.js
const amqp = require('amqplib');
const logger = require('../logging/Logger');

class QueueManager {
  constructor() {
    this.connection = null;
    this.channel = null;
    this.queues = new Map();
  }

  async connect() {
    this.connection = await amqp.connect(process.env.RABBITMQ_URL);
    this.channel = await this.connection.createChannel();
    
    // Configura exchange principal
    await this.channel.assertExchange('diyapp.main', 'direct', { durable: true });
    
    // Configura DLX para retry
    await this.channel.assertExchange('diyapp.dlx', 'direct', { durable: true });
    
    logger.info('RabbitMQ connected');
  }

  async createQueue(name, options = {}) {
    const { priority = 0, maxRetries = 3, retryDelay = 5000 } = options;
    
    // Queue principal
    const queueResult = await this.channel.assertQueue(name, {
      durable: true,
      maxPriority: 10,
      arguments: {
        'x-dead-letter-exchange': 'diyapp.dlx',
        'x-dead-letter-routing-key': `${name}.retry`
      }
    });

    // Queue de retry
    const retryQueue = `${name}.retry`;
    await this.channel.assertQueue(retryQueue, {
      durable: true,
      arguments: {
        'x-dead-letter-exchange': 'diyapp.main',
        'x-dead-letter-routing-key': name,
        'x-message-ttl': retryDelay,
        'x-max-retries': maxRetries
      }
    });

    // Bindings
    await this.channel.bindQueue(queueResult.queue, 'diyapp.main', name);
    await this.channel.bindQueue(retryQueue, 'diyapp.dlx', `${name}.retry`);

    this.queues.set(name, {
      main: queueResult.queue,
      retry: retryQueue,
      options
    });

    return queueResult.queue;
  }

  async publish(queueName, message, priority = 0) {
    const queue = this.queues.get(queueName);
    if (!queue) {
      throw new Error(`Queue ${queueName} not found`);
    }

    return this.channel.sendToQueue(queue.main, Buffer.from(JSON.stringify(message)), {
      persistent: true,
      priority,
      contentType: 'application/json'
    });
  }

  async consume(queueName, handler, options = {}) {
    const { prefetch = 10, noAck = false } = options;
    
    await this.channel.prefetch(prefetch);
    
    return this.channel.consume(queueName, async (msg) => {
      try {
        const content = JSON.parse(msg.content.toString());
        await handler(content);
        
        if (!noAck) {
          this.channel.ack(msg);
        }
      } catch (error) {
        logger.error('Queue processing error', { error, queueName });
        
        if (!noAck) {
          // Rejeita e manda para DLX
          this.channel.nack(msg, false, false);
        }
      }
    });
  }
}

module.exports = new QueueManager();
```

3. **Traces** (Jaeger)
4. **Health Checks** (Kubernetes)

```javascript
// src/infrastructure/monitoring/MetricsCollector.js
const promClient = require('prom-client');
const responseTime = require('response-time');
const logger = require('../logging/Logger');

class MetricsCollector {
  constructor() {
    promClient.collectDefaultMetrics();
    
    this.httpRequestDuration = new promClient.Histogram({
      name: 'http_request_duration_seconds',
      help: 'Duration of HTTP requests in seconds',
      labelNames: ['method', 'route', 'status_code'],
      buckets: [0.1, 0.2, 0.5, 1, 2, 5]
    });

    this.businessMetrics = {
      activeUsers: new promClient.Gauge({
        name: 'diyapp_active_users',
        help: 'Number of active users'
      }),
      queueLength: new promClient.Gauge({
        name: 'diyapp_queue_length',
        help: 'Length of processing queues',
        labelNames: ['queue_name']
      }),
      cacheHitRate: new promClient.Gauge({
        name: 'diyapp_cache_hit_rate',
        help: 'Cache hit rate percentage'
      })
    };
  }

  middleware() {
    return responseTime((req, res, time) => {
      const route = req.route?.path || req.path;
      this.httpRequestDuration
        .labels(req.method, route, res.statusCode)
        .observe(time / 1000);
      
      // Alerta se P95 > 200ms
      if (time > 190) { // Threshold com margem
        logger.warn('Slow request detected', {
          method: req.method,
          route,
          duration: time,
          threshold: 200
        });
      }
    });
  }

  async getMetrics() {
    return await promClient.register.metrics();
  }

  updateBusinessMetric(metric, value, labels = {}) {
    if (this.businessMetrics[metric]) {
      if (labels && Object.keys(labels).length > 0) {
        this.businessMetrics[metric].labels(labels).set(value);
      } else {
        this.businessMetrics[metric].set(value);
      }
    }
  }
}

// Health Check Endpoint
app.get('/health', async (req, res) => {
  const health = {
    status: 'healthy',
    timestamp: new Date().toISOString(),
    checks: {
      database: await checkDatabase(),
      redis: await checkRedis(),
      rabbitmq: await checkRabbitMQ(),
      memory: process.memoryUsage(),
      uptime: process.uptime()
    }
  };

  // Se qualquer check falhar, status = 'unhealthy'
  const allHealthy = Object.values(health.checks).every(check => 
    typeof check === 'boolean' ? check : true
  );
  
  if (!allHealthy) {
    health.status = 'unhealthy';
    res.status(503);
  }

  res.json(health);
});

// Métricas endpoint para Prometheus
app.get('/metrics', async (req, res) => {
  res.set('Content-Type', promClient.register.contentType);
  res.end(await metricsCollector.getMetrics());
});

module.exports = new MetricsCollector();
```

```javascript
// src/core/resilience/CircuitBreaker.js
class CircuitBreaker {
  constructor(options = {}) {
    this.failureThreshold = options.failureThreshold || 5;
    this.resetTimeout = options.resetTimeout || 60000; // 60 seconds
    this.halfOpenTimeout = options.halfOpenTimeout || 30000; // 30 seconds
    this.failureCount = 0;
    this.lastFailureTime = null;
    this.state = 'CLOSED'; // CLOSED, OPEN, HALF_OPEN
    this.successCount = 0;
    this.halfOpenSuccessThreshold = options.halfOpenSuccessThreshold || 2;
    
    // Métricas para observabilidade
    this.metrics = {
      totalRequests: 0,
      failedRequests: 0,
      successfulRequests: 0,
      stateChanges: []
    };
  }

  async call(serviceFunction, ...args) {
    this.metrics.totalRequests++;
    
    if (this.state === 'OPEN') {
      const timeSinceFailure = Date.now() - this.lastFailureTime;
      
      if (timeSinceFailure > this.resetTimeout) {
        this.state = 'HALF_OPEN';
        this.metrics.stateChanges.push({
          timestamp: new Date().toISOString(),
          from: 'OPEN',
          to: 'HALF_OPEN'
        });
      } else {
        throw new CircuitBreakerOpenError(
          `Service unavailable. Circuit breaker is OPEN. Retry in ${Math.ceil((this.resetTimeout - timeSinceFailure) / 1000)}s`
        );
      }
    }

    try {
      const result = await serviceFunction(...args);
      
      if (this.state === 'HALF_OPEN') {
        this.successCount++;
        
        if (this.successCount >= this.halfOpenSuccessThreshold) {
          this.reset();
        }
      } else {
        this.failureCount = 0;
      }
      
      this.metrics.successfulRequests++;
      return result;
    } catch (error) {
      this.metrics.failedRequests++;
      
      if (this.state === 'HALF_OPEN') {
        this.state = 'OPEN';
        this.lastFailureTime = Date.now();
        this.metrics.stateChanges.push({
          timestamp: new Date().toISOString(),
          from: 'HALF_OPEN',
          to: 'OPEN'
        });
      } else {
        this.failureCount++;
        
        if (this.failureCount >= this.failureThreshold) {
          this.state = 'OPEN';
          this.lastFailureTime = Date.now();
          this.metrics.stateChanges.push({
            timestamp: new Date().toISOString(),
            from: 'CLOSED',
            to: 'OPEN'
          });
        }
      }
      
      throw error;
    }
  }

  reset() {
    this.state = 'CLOSED';
    this.failureCount = 0;
    this.successCount = 0;
    this.lastFailureTime = null;
    this.metrics.stateChanges.push({
      timestamp: new Date().toISOString(),
      from: 'HALF_OPEN',
      to: 'CLOSED'
    });
  }

  getMetrics() {
    return {
      ...this.metrics,
      currentState: this.state,
      failureCount: this.failureCount,
      successRate: this.metrics.totalRequests > 0 
        ? (this.metrics.successfulRequests / this.metrics.totalRequests * 100).toFixed(2)
        : 0
    };
  }
}

class CircuitBreakerOpenError extends Error {
  constructor(message) {
    super(message);
    this.name = 'CircuitBreakerOpenError';
    this.statusCode = 503;
  }
}
```

```javascript
// src/core/resilience/RetryManager.js
class RetryManager {
  constructor(options = {}) {
    this.maxRetries = options.maxRetries || 3;
    this.baseDelay = options.baseDelay || 100; // 100ms
    this.maxDelay = options.maxDelay || 10000; // 10 seconds
    this.retryableStatusCodes = options.retryableStatusCodes || [408, 429, 500, 502, 503, 504];
    this.retryableErrors = options.retryableErrors || ['ECONNRESET', 'ETIMEDOUT', 'ECONNREFUSED'];
    
    this.metrics = {
      totalAttempts: 0,
      successfulRetries: 0,
      failedRetries: 0,
      totalDelayApplied: 0
    };
  }

  async execute(operation, context = {}) {
    let lastError;
    let attempt = 0;
    
    while (attempt <= this.maxRetries) {
      attempt++;
      this.metrics.totalAttempts++;
      
      try {
        const result = await operation();
        
        if (attempt > 1) {
          this.metrics.successfulRetries++;
        }
        
        return result;
      } catch (error) {
        lastError = error;
        
        // Verifica se o erro é retryable
        const shouldRetry = this.shouldRetry(error, attempt);
        
        if (!shouldRetry) {
          this.metrics.failedRetries++;
          throw error;
        }
        
        // Calcula delay com backoff exponencial + jitter
        const delay = this.calculateDelay(attempt);
        this.metrics.totalDelayApplied += delay;
        
        // Adiciona contexto ao log
        console.warn(`Retry attempt ${attempt}/${this.maxRetries} after ${delay}ms`, {
          error: error.message,
          context,
          delay
        });
        
        // Aguarda antes da próxima tentativa
        await this.sleep(delay);
      }
    }
    
    this.metrics.failedRetries++;
    throw lastError;
  }

  shouldRetry(error, attempt) {
    if (attempt >= this.maxRetries) return false;
    
    // Verifica códigos de status HTTP
    if (error.statusCode && this.retryableStatusCodes.includes(error.statusCode)) {
      return true;
    }
    
    // Verifica códigos de erro do sistema
    if (error.code && this.retryableErrors.includes(error.code)) {
      return true;
    }
    
    // Verifica se é um timeout
    if (error.name === 'TimeoutError' || error.message.includes('timeout')) {
      return true;
    }
    
    return false;
  }

  calculateDelay(attempt) {
    const exponentialDelay = this.baseDelay * Math.pow(2, attempt - 1);
    const jitter = Math.random() * 100; // Adiciona até 100ms de jitter
    const delay = Math.min(exponentialDelay + jitter, this.maxDelay);
    
    return Math.floor(delay);
  }

  sleep(ms) {
    return new Promise(resolve => setTimeout(resolve, ms));
  }

  getMetrics() {
    return { ...this.metrics };
  }
}

module.exports = RetryManager;
```

```javascript
// src/core/auth/JWTManager.js
const jwt = require('jsonwebtoken');
const crypto = require('crypto');

class JWTManager {
  constructor(options = {}) {
    this.secret = options.secret || process.env.JWT_SECRET;
    this.accessTokenExpiry = options.accessTokenExpiry || '15m';
    this.refreshTokenExpiry = options.refreshTokenExpiry || '7d';
    this.algorithm = options.algorithm || 'HS256';
    
    // Store para refresh tokens (em produção usar Redis)
    this.refreshTokens = new Map();
    
    this.metrics = {
      tokensIssued: 0,
      tokensRefreshed: 0,
      tokensRevoked: 0,
      failedValidations: 0
    };
  }

  generateAccessToken(payload) {
    this.metrics.tokensIssued++;
    
    return jwt.sign(
      {
        ...payload,
        type: 'access',
        jti: crypto.randomUUID()
      },
      this.secret,
      {
        expiresIn: this.accessTokenExpiry,
        algorithm: this.algorithm
      }
    );
  }

  generateRefreshToken(userId, deviceInfo = {}) {
    const refreshTokenId = crypto.randomUUID();
    const refreshToken = jwt.sign(
      {
        sub: userId,
        type: 'refresh',
        jti: refreshTokenId,
        device: deviceInfo
      },
      this.secret,
      {
        expiresIn: this.refreshTokenExpiry,
        algorithm: this.algorithm
      }
    );
    
    // Armazena o refresh token (em produção usar Redis com TTL)
    this.refreshTokens.set(refreshTokenId, {
      userId,
      deviceInfo,
      issuedAt: new Date(),
      expiresAt: new Date(Date.now() + this.parseTime(this.refreshTokenExpiry)),
      blacklisted: false
    });
    
    return refreshToken;
  }

  async verifyToken(token, type = 'access') {
    try {
      const decoded = jwt.verify(token, this.secret, { algorithms: [this.algorithm] });
      
      if (decoded.type !== type) {
        throw new Error(`Invalid token type. Expected ${type}, got ${decoded.type}`);
      }
      
      // Para refresh tokens, verifica se não está blacklisted
      if (type === 'refresh' && this.refreshTokens.has(decoded.jti)) {
        const tokenData = this.refreshTokens.get(decoded.jti);
        
        if (tokenData.blacklisted) {
          throw new Error('Refresh token has been revoked');
        }
      }
      
      return decoded;
    } catch (error) {
      this.metrics.failedValidations++;
      throw error;
    }
  }

  async refreshAccessToken(refreshToken, deviceInfo = {}) {
    try {
      const decoded = await this.verifyToken(refreshToken, 'refresh');
      
      // Verifica se o refresh token ainda está armazenado
      if (!this.refreshTokens.has(decoded.jti)) {
        throw new Error('Refresh token not found');
      }
      
      // Invalida o refresh token antigo (rotation)
      this.refreshTokens.delete(decoded.jti);
      this.metrics.tokensRevoked++;
      
      // Gera novos tokens
      const newAccessToken = this.generateAccessToken({ sub: decoded.sub });
      const newRefreshToken = this.generateRefreshToken(decoded.sub, deviceInfo);
      
      this.metrics.tokensRefreshed++;
      
      return {
        accessToken: newAccessToken,
        refreshToken: newRefreshToken,
        expiresIn: this.parseTime(this.accessTokenExpiry)
      };
    } catch (error) {
      throw new Error(`Token refresh failed: ${error.message}`);
    }
  }

  revokeRefreshToken(tokenId) {
    if (this.refreshTokens.has(tokenId)) {
      this.refreshTokens.delete(tokenId);
      this.metrics.tokensRevoked++;
      return true;
    }
    return false;
  }

  revokeAllUserTokens(userId) {
    let revokedCount = 0;
    
    for (const [tokenId, tokenData] of this.refreshTokens.entries()) {
      if (tokenData.userId === userId) {
        this.refreshTokens.delete(tokenId);
        revokedCount++;
      }
    }
    
    this.metrics.tokensRevoked += revokedCount;
    return revokedCount;
  }

  parseTime(timeString) {
    const units = {
      s: 1000,
      m: 60000,
      h: 3600000,
      d: 86400000
    };
    
    const match = timeString.match(/^(\d+)([smhd])$/);
    if (!match) return 3600000; // Default 1 hour
    
    const [, value, unit] = match;
    return parseInt(value) * units[unit];
  }

  getMetrics() {
    return {
      ...this.metrics,
      activeRefreshTokens: this.refreshTokens.size
    };
  }
}

module.exports = JWTManager;
```

## ADR-001: Microservices vs Modular Monolith

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Infra Specialist + Backend Lead

### CONTEXTO:
DIYAPP V11 cresceu para 50+ funcionalidades com equipes distribuídas. Necessidade de deploy independente, escalabilidade por componente, e isolamento de falhas. Time atual: 15 desenvolvedores, 3 squads.

### DECISÃO:
Adotar arquitetura híbrida: **Monólito Modular com capacidade de desacoplamento progressivo**. 
- Fase 1 (V12): Monólito modular com boundaries claros
- Fase 2 (V12.5): Serviços críticos isolados como microserviços
- Fase 3 (V13): Arquitetura completa de microserviços

### OPÇÕES CONSIDERADAS:
- **Opção A: Microserviços puros** - Prós: Deploy independente, escalabilidade granular. Contras: Complexidade operacional alta, latência de rede, consistência distribuída.
- **Opção B: Monólito modular** - Prós: Simplicidade operacional, consistência forte. Contras: Acoplamento, deploy único, escalabilidade limitada.
- **Opção C: Híbrida (escolhida)** - Prós: Transição controlada, aprendizado progressivo. Contras: Gerenciamento de duas arquiteturas temporariamente.

### CONSEQUÊNCIAS:
**Positivas:**
- Transição suave sem downtime
- Time aprende microserviços gradualmente
- Pode escalar componentes críticos primeiro

**Negativas:**
- Complexidade temporária aumentada
- Necessidade de compatibilidade entre modos

**Riscos:**
- Acoplamento não resolvido na fase 1
- Mitigação: Boundaries rigorosos desde o início

### REVISÃO: 2024-04-15

## ADR-002: Padrões de Comunicação

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Backend Lead

### CONTEXTO:
Necessidade de comunicação entre módulos no monólito e futuramente entre microserviços. Requisitos: baixa latência (<50ms), alta disponibilidade, resiliência a falhas.

### DECISÃO:
Implementar **Event-Driven Architecture com fallback síncrono**:
1. Comunicação intra-módulo: Chamadas diretas de função
2. Comunicação inter-módulo: Eventos assíncronos via RabbitMQ
3. Comunicação crítica síncrona: REST/GraphQL com circuit breaker

### OPÇÕES CONSIDERADAS:
- **REST puro**: Simples mas acoplamento temporal
- **GraphQL**: Flexível mas complexidade no servidor
- **gRPC**: Performance mas tooling limitado
- **Event-Driven (escolhida)**: Desacoplamento, resiliência, escalabilidade

### CONSEQUÊNCIAS:
**Positivas:**
- Desacoplamento temporal
- Resiliência a falhas
- Escalabilidade independente

**Negativas:**
- Complexidade de consistência eventual
- Debug mais difícil

**Riscos:**
- Perda de eventos
- Mitigação: DLQ (Dead Letter Queue) e retry policies

### REVISÃO: 2024-03-15

## ADR-003: Estratégia de Cache (Redis)

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Infra Specialist

### CONTEXTO:
Performance requirements: P95 < 200ms, 10k req/segundo. Dados frequentemente acessados: user sessions, configurations, product catalog.

### DECISÃO:
Implementar **cache distribuído em 3 níveis**:
1. **L1**: Cache em memória (Node.js) - 5 minutos TTL
2. **L2**: Redis cluster - 30 minutos TTL
3. **L3**: Database com query optimization

**Patterns:**
- Cache-Aside para dados voláteis
- Write-Through para dados críticos
- Redis Cluster: 3 master, 3 replicas

### OPÇÕES CONSIDERADAS:
- **Redis único**: Simples mas SPOF
- **Memcached**: Mais rápido mas menos features
- **Redis Cluster (escolhido)**: Alta disponibilidade, persistência

### CONSEQUÊNCIAS:
**Positivas:**
- Performance consistentemente baixa
- Alta disponibilidade
- Escalabilidade horizontal

**Negativas:**
- Complexidade de consistência
- Custo operacional

**Riscos:**
- Cache stampede
- Mitigação: Locking distribuído e early expiration

### REVISÃO: 2024-02-15

## ADR-004: Estratégia de Filas (RabbitMQ)

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Infra Specialist

### CONTEXTO:
Processamento assíncrono de: emails, notificações, relatórios, image processing. Volume: 100k mensagens/dia, picos de 1k/segundo.

### DECISÃO:
**RabbitMQ com cluster mirroring**:
- **Exchange Types**: Direct (routing preciso), Topic (pattern matching), Fanout (broadcast)
- **Queues**: Durable, auto-delete=false, DLQ configurada
- **Cluster**: 3 nodes com quorum queues

**Padrões:**
- Publisher Confirms para garantia de entrega
- Consumer Prefetch para balanceamento
- Dead Letter Exchanges para retry

### OPÇÕES CONSIDERADAS:
- **Kafka**: Alta throughput mas complexo
- **AWS SQS**: Gerenciado mas vendor lock-in
- **RabbitMQ (escolhido)**: Balanceamento bom, feature-rich, open source

### CONSEQUÊNCIAS:
**Positivas:**
- Garantia de entrega
- Alta disponibilidade
- Flexibilidade de routing

**Negativas:**
- Overhead de gerenciamento
- Learning curve

**Riscos:**
- Perda de mensagens em falha
- Mitigação: Persistent messages, publisher confirms

### REVISÃO: 2024-02-28

## ADR-005: Métricas de Performance

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Infra Specialist + DevOps

### CONTEXTO:
SLA requerido: P95 < 200ms, uptime 99.99%. Necessidade de monitoramento proativo, alertas automáticos, dashboards em tempo real.

### DECISÃO:
**Stack de observabilidade completa**:
1. **Métricas**: Prometheus + Grafana
2. **Tracing**: Jaeger
3. **Logs**: ELK Stack (Elasticsearch, Logstash, Kibana)
4. **Alertas**: Alertmanager com escalonamento

**KPIs obrigatórios:**
- Latência P50, P95, P99
- Error rate (< 0.1%)
- Throughput (req/seg)
- Uptime (99.99%)
- Resource utilization (< 80%)

### OPÇÕES CONSIDERADAS:
- **Datadog**: Completo mas caro
- **New Relic**: Boa integração mas vendor lock-in
- **OSS Stack (escolhida)**: Flexível, custo controlável, sem lock-in

### CONSEQUÊNCIAS:
**Positivas:**
- Visibilidade completa
- Alertas proativos
- Capacidade de debugging

**Negativas:**
- Complexidade de setup
- Manutenção requerida

**Riscos:**
- Alert fatigue
- Mitigação: Alertas hierárquicos, suppression rules

```javascript
export function getContrastColor(backgroundColor) {
  // Calcula luminância e retorna cor de texto apropriada
  const rgb = backgroundColor.replace(/^#/, '');
  const r = parseInt(rgb.substr(0, 2), 16);
  const g = parseInt(rgb.substr(2, 2), 16);
  const b = parseInt(rgb.substr(4, 2), 16);
  
  const luminance = (0.299 * r + 0.587 * g + 0.114 * b) / 255;
  return luminance > 0.5 ? colors.text.light.primary : colors.text.dark.primary;
}
```

```javascript
const NodeCache = require('node-cache');
const Redis = require('ioredis');
const logger = require('../../utils/logger');

class MultiLayerCache {
  constructor() {
    // Cache L1: In-memory LRU
    this.l1Cache = new NodeCache({
      stdTTL: 300, // 5 minutos default
      checkperiod: 60,
      useClones: false,
      maxKeys: 10000
    });
    
    // Cache L2: Redis
    this.redisClient = null;
    this.redisEnabled = process.env.REDIS_URL !== undefined;
    
    // Estatísticas
    this.stats = {
      l1Hits: 0,
      l1Misses: 0,
      l2Hits: 0,
      l2Misses: 0,
      errors: 0
    };
  }
  
  async initialize() {
    if (this.redisEnabled) {
      this.redisClient = new Redis(process.env.REDIS_URL, {
        retryStrategy: (times) => {
          const delay = Math.min(times * 50, 2000);
          return delay;
        },
        maxRetriesPerRequest: 3,
        enableReadyCheck: true,
        lazyConnect: true
      });
      
      this.redisClient.on('error', (err) => {
        logger.error('Redis connection error', { error: err.message });
        this.stats.errors++;
      });
      
      this.redisClient.on('connect', () => {
        logger.info('Redis connected successfully');
      });
      
      await this.redisClient.connect();
    }
  }
  
  async get(key, options = {}) {
    const { ttl = 300, fallback = null, fallbackTtl = 60 } = options;
    
    try {
      // Tentar L1 primeiro
      const l1Value = this.l1Cache.get(key);
      if (l1Value !== undefined) {
        this.stats.l1Hits++;
        return l1Value;
      }
      this.stats.l1Misses++;
      
      // Tentar L2 (Redis)
      if (this.redisEnabled && this.redisClient) {
        try {
          const l2Value = await this.redisClient.get(key);
          if (l2Value !== null) {
            this.stats.l2Hits++;
            const parsedValue = JSON.parse(l2Value);
            // Popular L1
            this.l1Cache.set(key, parsedValue, ttl);
            return parsedValue;
          }
          this.stats.l2Misses++;
        } catch (redisError) {
          logger.warn('Redis get failed, falling back', {
            key,
            error: redisError.message
          });
        }
      }
      
      // Executar fallback se fornecido
      if (fallback && typeof fallback === 'function') {
        try {
          const fallbackValue = await fallback();
          await this.set(key, fallbackValue, { ttl: fallbackTtl });
          return fallbackValue;
        } catch (fallbackError) {
          logger.error('Fallback function failed', {
            key,
            error: fallbackError.message
          });
          throw fallbackError;
        }
      }
      
      return null;
    } catch (error) {
      this.stats.errors++;
      logger.error('Cache get error', {
        key,
        error: error.message
      });
      return null;
    }
  }
  
  async set(key, value, options = {}) {
    const { ttl = 300 } = options;
    
    try {
      // Set L1
      this.l1Cache.set(key, value, ttl);
      
      // Set L2 (Redis) se disponível
      if (this.redisEnabled && this.redisClient) {
        try {
          await this.redisClient.setex(
            key,
            ttl,
            JSON.stringify(value)
          );
        } catch (redisError) {
          logger.warn('Redis set failed, continuing with L1 only', {
            key,
            error: redisError.message
          });
        }
      }
      
      return true;
    } catch (error) {
      this.stats.errors++;
      logger.error('Cache set error', {
        key,
        error: error.message
      });
      return false;
    }
  }
  
  async del(key) {
    try {
      // Remover de L1
      this.l1Cache.del(key);
      
      // Remover de L2
      if (this.redisEnabled && this.redisClient) {
        await this.redisClient.del(key);
      }
      
      return true;
    } catch (error) {
      this.stats.errors++;
      logger.error('Cache delete error', {
        key,
        error: error.message
      });
      return false;
    }
  }
  
  async mget(keys) {
    const results = {};
    
    for (const key of keys) {
      results[key] = await this.get(key);
    }
    
    return results;
  }
  
  async clearPattern(pattern) {
    try {
      // Limpar L1 (não suporta pattern, limpar tudo)
      this.l1Cache.flushAll();
      
      // Limpar L2 por pattern
      if (this.redisEnabled && this.redisClient) {
        const keys = await this.redisClient.keys(pattern);
        if (keys.length > 0) {
          await this.redisClient.del(...keys);
        }
      }
      
      return true;
    } catch (error) {
      this.stats.errors++;
      logger.error('Cache clear pattern error', {
        pattern,
        error: error.message
      });
      return false;
    }
  }
  
  getStats() {
    return {
      ...this.stats,
      l1Size: this.l1Cache.keys().length,
      l1Stats: this.l1Cache.getStats(),
      redisEnabled: this.redisEnabled,
      redisConnected: this.redisEnabled && this.redisClient?.status === 'ready'
    };
  }
}

async function initializeCache() {
  await cacheInstance.initialize();
  logger.info('Multi-layer cache initialized');
  return cacheInstance;
}
```

const EventEmitter = require('events');

class CircuitBreaker extends EventEmitter {
  constructor(options = {}) {
    super();
    
    this.name = options.name || 'default';
    this.failureThreshold = options.failureThreshold || 5;
    this.resetTimeout = options.resetTimeout || 60000; // 60 segundos
    this.timeout = options.timeout || 10000; // 10 segundos
    
    this.state = 'CLOSED'; // CLOSED, OPEN, HALF_OPEN
    this.failureCount = 0;
    this.lastFailureTime = null;
    this.nextAttempt = null;
    
    this.stats = {
      totalRequests: 0,
      successfulRequests: 0,
      failedRequests: 0,
      shortCircuited: 0,
      stateChanges: 0
    };
    
    // Métricas
    this.metrics = {
      latency: [],
      errorRate: 0
    };
  }
  
  async call(serviceFunction, ...args) {
    this.stats.totalRequests++;
    
    // Se o circuito está aberto, verificar se pode tentar reabrir
    if (this.state === 'OPEN') {
      if (Date.now() >= this.nextAttempt) {
        this.state = 'HALF_OPEN';
        this.stats.stateChanges++;
        logger.info(`Circuit ${this.name} moving to HALF_OPEN`);
      } else {
        this.stats.shortCircuited++;
        this.emit('shortCircuit', this.name);
        throw new Error(`Circuit ${this.name} is OPEN`);
      }
    }
    
    // Executar com timeout
    const startTime = Date.now();
    const timeoutPromise = new Promise((_, reject) => {
      setTimeout(() => {
        reject(new Error(`Circuit ${this.name} timeout after ${this.timeout}ms`));
      }, this.timeout);
    });
    
    try {
      const result = await Promise.race([
        serviceFunction(...args),
        timeoutPromise
      ]);
      
      const latency = Date.now() - startTime;
      this.recordSuccess(latency);
      
      // Se estava em HALF_OPEN, fechar o circuito

        this.state = 'CLOSED';
        this.failureCount = 0;

-- DIYAPP V12 - Data Warehouse Schema
-- Camada RAW (Bronze) - Dados brutos preservados

CREATE SCHEMA IF NOT EXISTS raw;
CREATE SCHEMA IF NOT EXISTS staging;
CREATE SCHEMA IF NOT EXISTS marts;

-- Tabelas RAW
CREATE TABLE raw.user_events (
    event_id UUID PRIMARY KEY,
    user_id VARCHAR(255),
    session_id VARCHAR(255),
    event_type VARCHAR(100),
    event_name VARCHAR(200),
    event_timestamp TIMESTAMP WITH TIME ZONE,
    page_url TEXT,
    referrer TEXT,
    user_agent TEXT,
    device_type VARCHAR(50),
    browser VARCHAR(100),
    os VARCHAR(100),
    country VARCHAR(2),
    region VARCHAR(100),
    city VARCHAR(100),
    event_properties JSONB,
    raw_payload JSONB,
    ingested_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    source_system VARCHAR(100),
    batch_id VARCHAR(100)
);

CREATE TABLE raw.llm_calls (
    call_id UUID PRIMARY KEY,
    request_id VARCHAR(255),
    user_id VARCHAR(255),
    session_id VARCHAR(255),
    feature_name VARCHAR(200),
    model_name VARCHAR(100),
    provider VARCHAR(50),
    prompt_tokens INTEGER,
    completion_tokens INTEGER,
    total_tokens INTEGER,
    cost_usd DECIMAL(10,6),
    latency_ms INTEGER,
    success BOOLEAN,
    error_message TEXT,
    request_payload JSONB,
    response_payload JSONB,
    request_timestamp TIMESTAMP WITH TIME ZONE,
    response_timestamp TIMESTAMP WITH TIME ZONE,
    ingested_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    source_system VARCHAR(100)
);

CREATE TABLE raw.system_metrics (
    metric_id UUID PRIMARY KEY,
    metric_name VARCHAR(200),
    metric_value DECIMAL(15,4),
    metric_unit VARCHAR(50),
    resource_type VARCHAR(100),
    resource_id VARCHAR(255),
    environment VARCHAR(50),
    timestamp TIMESTAMP WITH TIME ZONE,
    tags JSONB,
    ingested_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()

    event_id UUID PRIMARY KEY,
    user_id VARCHAR(255),

    event_name VARCHAR(200),
