# DIYAPP Evolution - V11 Core - Arquitetura de Microsserviços

## 1. Documento de Arquitetura (ADR-001)

```
# ADR-001: Arquitetura de Microsserviços para DIYAPP V11

Data: 2024-01-15
Status: Aceita
Autores: Tech Lead, Especialista Infra, Especialista Backend

## CONTEXTO:
DIYAPP V10 opera como uma aplicação monolítica que está enfrentando:
1. Dificuldades de escalabilidade horizontal
2. Acoplamento alto entre funcionalidades
3. Tempos de deploy longos (>15 minutos)
4. Dificuldade em adotar novas tecnologias por partes

Forças em jogo:
- Necessidade de manter 100% de uptime durante transição
- Equipe distribuída trabalhando em paralelo (Modo Hive)
- Requisitos de performance variados por domínio
- Orçamento limitado para infraestrutura

## DECISÃO:
Adotar arquitetura de microsserviços com os seguintes princípios:
1. 6 serviços independentes com bounded contexts claros
2. Comunicação via eventos assíncronos (80%) e APIs síncronas (20%)
3. API Gateway como único ponto de entrada público
4. Service Mesh para comunicação interna
5. Banco de dados por serviço (Database per Service)

## OPÇÕES CONSIDERADAS:

### Opção A: Monólito modularizado
- Prós: Menor complexidade operacional, transação ACID fácil
- Contras: Não resolve problemas de escalabilidade, deploy ainda lento
- Custo: Baixo

### Opção B: Microsserviços completos (escolhida)
- Prós: Escalabilidade independente, deploy rápido, resiliência
- Contras: Complexidade distribuída, eventual consistency
- Custo: Médio-Alto

### Opção C: Arquitetura de camadas
- Prós: Separação de responsabilidades
- Contras: Não resolve acoplamento
- Custo: Baixo

## CONSEQUÊNCIAS:

### Positivas:
- Deploy independente por serviço (<2 minutos cada)
- Escalabilidade granular (ex: serviço de IA pode ter mais recursos)
- Isolamento de falhas
- Equipes podem trabalhar com stacks diferentes

### Negativas:
- Complexidade de transações distribuídas
- Necessidade de monitoramento distribuído
- Overhead de comunicação entre serviços
- Maior consumo de recursos

### Riscos:
1. Latência aumentada em chamadas entre serviços
   - Mitigação: Cache estratégico, padrões CQRS
2. Dados inconsistentes temporariamente
   - Mitigação: Sagas, compensação, idempotência
3. Debug complexo
   - Mitigação: Correlation IDs, logs centralizados, tracing distribuído

## REVISÃO: 2024-04-15 (3 meses)
```

## 2. Padrões de Código

### 2.1. Linguagens e Frameworks
- Backend: Node.js 18+ com TypeScript, NestJS
- Frontend: React 18, TypeScript, Vite
- Mobile: React Native (compartilha 70% do código com web)

### 2.2. Estrutura de Serviço
```
service-name/
├── src/
│   ├── application/     # Casos de uso
│   ├── domain/         # Entidades, value objects
│   ├── infrastructure/ # Implementações concretas
│   ├── interfaces/     # Controllers, GraphQL resolvers
│   └── shared/         # Utilitários do serviço
├── tests/
│   ├── unit/
│   ├── integration/
│   └── e2e/
├── Dockerfile
├── docker-compose.yml
└── package.json
```

### 2.3. Convenções de Nomenclatura
- Classes: PascalCase (UserService)
- Métodos/funções: camelCase (getUserById)
- Variáveis: camelCase (userRepository)
- Constantes: UPPER_SNAKE_CASE (MAX_RETRY_COUNT)
- Arquivos: kebab-case (user-controller.ts)
- Interfaces: Prefixo I (IUserRepository)
- Types: Sufixo Type (UserResponseType)

### 2.4. Commits
- Usar Conventional Commits
- Formato: `type(scope): description`
- Exemplo: `feat(auth): add MFA support`
- Tipos: feat, fix, docs, style, refactor, test, chore

### 2.5. Testes
- Cobertura mínima: 80%
- Unit tests: Jest
- Integration tests: Supertest + Testcontainers
- E2E: Cypress (frontend), Jest (backend)

### 2.6. Documentação
- Swagger/OpenAPI para APIs REST
- Schema-first para GraphQL
- JSDoc para funções complexas
- README por serviço com:
  - Descrição do domínio
  - Como executar localmente
  - Variáveis de ambiente
  - Endpoints principais

## 3. Padrões de API

### 3.1. REST
- Versionamento: `/api/v1/resource`
- Métodos HTTP semânticos
- Paginação: `?page=1&limit=20`
- Filtros: `?status=active&created_after=2024-01-01`
- Ordenação: `?sort=-created_at,name`
- HATEOAS opcional

### 3.2. GraphQL
- Schema-first development
- Resolvers organizados por domínio
- N+1 prevention: DataLoader
- Depth limiting: max 10 níveis
- Complexity limiting

### 3.3. Eventos
- Formato: CloudEvents
- Schema registry: Apache Avro
- Dead letter queues para eventos falhos
- Idempotência obrigatória

## 4. Segurança

### 4.1. Autenticação
- JWT com expiração curta (15min)
- Refresh tokens com rotação
- MFA obrigatório para admin

### 4.2. Autorização
- RBAC com roles e permissions
- Policy-based access control
- Attribute-based onde necessário

### 4.3. Dados
- PII: criptografia em repouso
- Senhas: bcrypt (cost 12)
- Secrets: HashiCorp Vault ou AWS Secrets Manager

## 5. Performance

### 5.1. Cache
- Redis para cache distribuído
- Estratégias: Cache-Aside, Write-Through
- TTL: configurável por domínio

### 5.2. Banco de Dados
- Indexes para queries frequentes
- Connection pooling
- Read replicas para cargas pesadas de leitura

### 5.3. API
- Rate limiting: 100 req/min por usuário
- Response compression: gzip
- Paginação obrigatória para listas > 100 itens

## 6. Observabilidade

### 6.1. Logs
- Estrutura JSON
- Níveis: error, warn, info, debug, trace
- Correlation ID obrigatório
- Contexto: userId, requestId, serviceName

### 6.2. Métricas
- Prometheus format
- Latência por endpoint
- Taxa de erro por serviço
- Throughput

### 6.3. Tracing
- OpenTelemetry
- 100% sample rate para erros
- 10% sample rate para sucesso

## 7. Deploy e Infra

### 7.1. Containers
- Multi-stage Docker builds
- Non-root user
- Health checks obrigatórios
- Resource limits definidos

### 7.2. Kubernetes
- Namespace por ambiente
- HPA configurado
- PDB (Pod Disruption Budget)
- Network policies

## 8. Qualidade de Código

### 8.1. Linting
- ESLint com regras personalizadas
- Prettier para formatação
- Husky pre-commit hooks

### 8.2. Análise Estática
- SonarQube/SonarCloud
- Dependabot para security updates
- Snyk para vulnerabilidades

### 8.3. Review
- 2 approvals mínimo
- Tech Lead approval obrigatório para:
  - Mudanças arquiteturais
  - Novas dependências
  - Mudanças de segurança
```

### 3.1. GitHub Actions Workflow (`/.github/workflows/ci-cd.yml`)

```yaml
name: DIYAPP V11 CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # Job 1: Validação e Testes
  validate-and-test:
    name: Validate and Test
    runs-on: ubuntu-latest
    strategy:
      matrix:
        service: [api-gateway, auth-service, user-service, project-service, ai-service, notification-service, file-service]
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
      with:
        fetch-depth: 0
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18.x'
        cache: 'npm'
        cache-dependency-path: ${{ matrix.service }}/package-lock.json
    
    - name: Install dependencies
      run: |
        cd ${{ matrix.service }}
        npm ci --audit=false
      env:
        NODE_ENV: development
    
    - name: Lint code
      run: |
        cd ${{ matrix.service }}
        npm run lint
    
    - name: Type check
      run: |
        cd ${{ matrix.service }}
        npx tsc --noEmit
    
    - name: Run unit tests
      run: |
        cd ${{ matrix.service }}
        npm test -- --coverage --passWithNoTests
      env:
        NODE_ENV: test
    
    - name: Upload coverage reports
      uses: codecov/codecov-action@v3
      with:
        directory: ${{ matrix.service }}/coverage
        flags: ${{ matrix.service }}
        name: ${{ matrix.service }}-coverage
    
    - name: Security scan
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
        directory: ${{ matrix.service }}
    
    - name: Build Docker image
      run: |
        cd ${{ matrix.service }}
        docker build -t ${{ matrix.service }}:test .
    
    - name: Run integration tests
      run: |
        cd ${{ matrix.service }}
        docker-compose -f docker-compose.test.yml up --abort-on-container-exit --exit-code-from test
      env:
        NODE_ENV: test

  # Job 2: Análise de Qualidade
  quality-gate:
    name: Quality Gate
    runs-on: ubuntu-latest
    needs: validate-and-test
    if: github.event_name == 'pull_request'
    
    steps:
    - name: SonarCloud Scan
      uses: SonarSource/sonarcloud-github-action@master
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        SONAR_TOKEN: ${{ secrets.SONAR_TOKEN }}
      with:
        args: >
          -Dsonar.organization=diyapp
          -Dsonar.projectKey=diyapp-v11
          -Dsonar.sources=.
          -Dsonar.exclusions=**/node_modules/**,**/dist/**,**/coverage/**
          -Dsonar.tests=.
          -Dsonar.test.inclusions=**/*.test.ts,**/*.spec.ts
          -Dsonar.javascript.lcov.reportPaths=coverage/lcov.info
          -Dsonar.coverage.exclusions=**/*.d.ts,**/test/**,**/mock/**
          -Dsonar.qualitygate.wait=true

  # Job 3: Build e Push de Imagens
  build-and-push:
    name: Build and Push Docker Images
    runs-on: ubuntu-latest
    needs: [validate-and-test, quality-gate]
    if: github.ref == 'refs/heads/main' || github.ref == 'refs/heads/develop'
    
    strategy:
      matrix:
        service: [api-gateway, auth-service, user-service, project-service, ai-service, notification-service, file-service]
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v2
    
    - name: Log in to Container Registry
      uses: docker/login-action@v2
      with:
        registry: ${{ env.REGISTRY }}
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}
    
    - name: Extract metadata
      id: meta
      uses: docker/metadata-action@v4
      with:
        images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}/${{ matrix.service }}
        tags: |
          type=ref,event=branch
          type=ref,event=pr
          type=semver,pattern={{version}}
          type=semver,pattern={{major}}.{{minor}}
          type=sha,prefix={{branch}}-
    
    - name: Build and push
      uses: docker/build-push-action@v4
      with:
        context: ./${{ matrix.service }}
        push: ${{ github.ref == 'refs/heads/main' || github.ref == 'refs/heads/develop' }}
        tags: ${{ steps.meta.outputs.tags }}
        labels: ${{ steps.meta.outputs.labels }}
        cache-from: type=gha
        cache-to: type=gha,mode=max

  # Job 4: Deploy para Ambiente
  deploy:
    name: Deploy to Environment
    runs-on: ubuntu-latest
    needs: build-and-push
    if: github.ref == 'refs/heads/main' || github.ref == 'refs/heads/develop'
    environment:
      name: ${{ github.ref == 'refs/heads/main' && 'production' || 'staging' }}
      url: ${{ github.ref == 'refs/heads/main' && 'https://app.diyapp.com' || 'https://staging.diyapp.com' }}
    
    steps:
    - name: Checkout infrastructure
      uses: actions/checkout@v3
      with:
        path: infrastructure
    
    - name: Configure AWS credentials
      uses: aws-actions/configure-aws-credentials@v1
      with:
        aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
        aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
        aws-region: us-east-1
    
    - name: Setup kubectl
      uses: azure/setup-kubectl@v3
      with:
        version: 'v1.26.0'
    
    - name: Configure kubeconfig
      run: |
        aws eks update-kubeconfig --name diyapp-${{ github.ref == 'refs/heads/main' && 'prod' || 'staging' }} --region us-east-1
    
    - name: Deploy with Helm
      run: |
        cd infrastructure/helm
        helm upgrade --install diyapp-v11 . \
          --namespace diyapp-${{ github.ref == 'refs/heads/main' && 'prod' || 'staging' }} \
          --set image.tag=${{ github.sha }} \
          --set environment=${{ github.ref == 'refs/heads/main' && 'production' || 'staging' }} \
          --atomic \
          --timeout 10m \
          --wait
    
    - name: Run smoke tests
      run: |
        cd infrastructure/tests
        npm ci
        npm run smoke-test -- --base-url=${{ github.ref == 'refs/heads/main' && 'https://app.diyapp.com' || 'https://staging.diyapp.com' }}
    
    - name: Notify Slack
      uses: 8398a7/action-slack@v3
      with:
        status: ${{ job.status }}
        author_name: CI/CD Pipeline
        fields: repo,message,commit,author,action
      env:
        SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
      if: always()

  # Job 5: Rollback Automático (em caso de falha)
  auto-rollback:
    name: Auto Rollback on Failure
    runs-on: ubuntu-latest
    needs: deploy
    if: failure() && github.ref == 'refs/heads/main'
    
    steps:
    - name: Checkout infrastructure
      uses: actions/checkout@v3
      with:
        path: infrastructure
    
    - name: Configure AWS credentials
      uses: aws-actions/

  const [isSidebarOpen, setIsSidebarOpen] = React.useState(true);

              isMobile={isMobile}

          </Suspense>

const CircuitBreaker = require('opossum');
const config = require('./index');
const logger = require('../utils/logger');

class CircuitBreakerManager {
  constructor() {
    this.breakers = new Map();
    this.defaultOptions = {
      timeout: config.circuitBreaker.timeout,
      errorThresholdPercentage: config.circuitBreaker.errorThresholdPercentage,
      resetTimeout: config.circuitBreaker.resetTimeout,
      rollingCountTimeout: config.circuitBreaker.rollingCountTimeout,
      rollingCountBuckets: config.circuitBreaker.rollingCountBuckets,
    };
  }

  create(name, action, options = {}) {
    const breakerOptions = { ...this.defaultOptions, ...options };
    const breaker = new CircuitBreaker(action, breakerOptions);
    
    // Event listeners for monitoring
    breaker.on('open', () => {
      logger.warn(`Circuit breaker "${name}" opened`, { 
        name, 
        stats: breaker.stats 
      });
    });
    
    breaker.on('close', () => {
      logger.info(`Circuit breaker "${name}" closed`, { 
        name, 
        stats: breaker.stats 
      });
    });
    
    breaker.on('halfOpen', () => {
      logger.info(`Circuit breaker "${name}" half-open`, { 
        name, 
        stats: breaker.stats 
      });
    });
    
    breaker.on('failure', (error) => {
      logger.error(`Circuit breaker "${name}" failure`, { 
        name, 
        error: error.message,
        stats: breaker.stats 
      });
    });
    
    breaker.on('success', () => {
      logger.debug(`Circuit breaker "${name}" success`, { 
        name, 
        stats: breaker.stats 
      });
    });
    
    this.breakers.set(name, breaker);
    return breaker;
  }

  get(name) {
    return this.breakers.get(name);
  }

  async fire(name, ...args) {
    const breaker = this.breakers.get(name);
    if (!breaker) {
      throw new Error(`Circuit breaker "${name}" not found`);
    }
    return breaker.fire(...args);
  }

  getStats(name) {
    const breaker = this.breakers.get(name);
    return breaker ? breaker.stats : null;
  }

  getAllStats() {
    const stats = {};
    for (const [name, breaker] of this.breakers.entries()) {
      stats[name] = breaker.stats;
    }
    return stats;
  }
}

module.exports = new CircuitBreakerManager();

const jwt = require('jsonwebtoken');
const crypto = require('crypto');
const { createHash } = require('crypto');

class JWTSecurity {
  constructor() {
    this.secret = process.env.JWT_SECRET || this.generateSecret();
    this.accessExpiry = '15m';
    this.refreshExpiry = '7d';
    this.issuer = 'diyapp-v11';
    this.audience = 'diyapp-users';
    
    // Blacklist para tokens invalidados
    this.tokenBlacklist = new Set();
    
    // Rate limiting por usuário
    this.loginAttempts = new Map();
  }

  generateSecret() {
    return crypto.randomBytes(64).toString('hex');
  }

  async generateTokens(user) {
    const accessToken = jwt.sign(
      {
        userId: user.id,
        email: user.email,
        roles: user.roles,
        permissions: user.permissions
      },
      this.secret,
      {
        expiresIn: this.accessExpiry,
        issuer: this.issuer,
        audience: this.audience,
        algorithm: 'HS256'
      }
    );

    const refreshToken = jwt.sign(
      {
        userId: user.id,
        type: 'refresh'
      },
      this.secret,
      {
        expiresIn: this.refreshExpiry,
        issuer: this.issuer,
        audience: this.audience
      }
    );

    // Hash do refresh token para armazenamento seguro
    const refreshTokenHash = createHash('sha256')
      .update(refreshToken)
      .digest('hex');

    return {
      accessToken,
      refreshToken,
      refreshTokenHash,
      expiresIn: 900 // 15 minutos em segundos
    };
  }

  verifyToken(token, options = {}) {
    try {
      // Verificar se token está na blacklist
      const tokenHash = createHash('sha256').update(token).digest('hex');
      if (this.tokenBlacklist.has(tokenHash)) {
        throw new Error('Token revoked');
      }

      const decoded = jwt.verify(token, this.secret, {
        issuer: this.issuer,
        audience: this.audience,
        algorithms: ['HS256'],
        ...options
      });

      return decoded;
    } catch (error) {
      throw new Error(`Token verification failed: ${error.message}`);

  }

### 2.2 Rate Limiting Avançado

**src/security/auth/rate-limiter.js**
```javascript
const rateLimit = require('express-rate-limit');
const Redis = require('ioredis');
const { RateLimiterRedis } = require('rate-limiter-flexible');

class SecurityRateLimiter {
  constructor() {
    this.redisClient = new Redis({
      host: process.env.REDIS_HOST || 'localhost',
      port: process.env.REDIS_PORT || 6379,
      enableOfflineQueue: false
    });

    this.setupLimiters();
  }

  setupLimiters() {
    // Rate limiter para autenticação
    this.authLimiter = new RateLimiterRedis({
      storeClient: this.redisClient,
      keyPrefix: 'auth',
      points: 5, // 5 tentativas
      duration: 900, // 15 minutos
      blockDuration: 1800 // Bloqueia por 30 minutos após exceder
    });

    // Rate limiter para API geral
    this.apiLimiter = rateLimit({
      windowMs: 15 * 60 * 1000, // 15 minutos
      max: 100, // Limite de 100 requisições por IP
      standardHeaders: true,
      legacyHeaders: false,
      skipSuccessfulRequests: false,
      handler: (req, res) => {
        res.status(429).json({
          error: 'Too many requests',
          retryAfter: Math.ceil(req.rateLimit.resetTime - Date.now()) / 1000
        });
      }
    });

    // Rate limiter para endpoints específicos
    this.sensitiveEndpointsLimiter = new RateLimiterRedis({
      storeClient: this.redisClient,
      keyPrefix: 'sensitive',
      points: 10, // 10 requisições
      duration: 300, // 5 minutos
      blockDuration: 600 // 10 minutos de bloqueio
    });
  }

  async checkAuthLimit(ip, username) {
    const key = `auth:${ip}:${username}`;
    
    try {
      await this.authLimiter.consume(key);
      return { allowed: true };
    } catch (rejRes) {
      return {
        allowed: false,
        retryAfter: rejRes.msBeforeNext / 1000,
        remainingPoints: rejRes.remainingPoints
      };
    }
  }

  async checkSensitiveEndpoint(userId, endpoint) {
    const key = `sensitive:${userId}:${endpoint}`;
    
    try {
      await this.sensitiveEndpointsLimiter.consume(key);
      return { allowed: true };
    } catch (rejRes) {
      return {
        allowed: false,
        retryAfter: rejRes.msBeforeNext / 1000
      };
    }
  }

  middleware() {
    return async (req, res, next) => {
      // Aplicar rate limiting baseado no tipo de endpoint
      const ip = req.ip;
      const userId = req.user?.id || 'anonymous';
      const endpoint = req.path;

      // Endpoints de autenticação
      if (req.path.includes('/auth/login')) {
        const result = await this.checkAuthLimit(ip, req.body.email);
        if (!result.allowed) {
          return res.status(429).json({
            error: 'Too many login attempts',
            retryAfter: result.retryAfter
          });
        }
      }

      // Endpoints sensíveis
      if (this.isSensitiveEndpoint(endpoint)) {
        const result = await this.checkSensitiveEndpoint(userId, endpoint);
        if (!result.allowed) {
          return res.status(429).json({
            error: 'Rate limit exceeded for sensitive operation',
            retryAfter: result.retryAfter
          });
        }
      }

      next();
    };
  }

  isSensitiveEndpoint(endpoint) {
    const sensitivePatterns = [
      '/api/users',
      '/api/admin',
      '/api/settings',
      '/api/payments',
      '/api/export'
    ];
    
    return sensitivePatterns.some(pattern => endpoint.startsWith(pattern));
  }

  async resetLimits(ip, username) {
    const key = `auth:${ip}:${username}`;
    await this.authLimiter.delete(key);
  }
}

module.exports = new SecurityRateLimiter();

### app/src/middleware/observability.js
```javascript
const { v4: uuidv4 } = require('uuid');
const metrics = require('../utils/metrics');
const winston = require('winston');
const { ElasticsearchTransport } = require('winston-elasticsearch');

// Configurar logger Winston com Elasticsearch
const esTransport = new ElasticsearchTransport({
  level: 'info',
  clientOpts: { node: process.env.ELASTICSEARCH_HOST || 'http://localhost:9200' },
  indexPrefix: 'diyapp-logs'
});

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    new winston.transports.Console(),
    esTransport
  ]
});

// Middleware de observabilidade
const observabilityMiddleware = (req, res, next) => {
  const start = Date.now();
  const requestId = uuidv4();
  const traceId = req.headers['x-trace-id'] || uuidv4();

    url: req.url,

      method: req.method,
      url: req.url,
      statusCode: res.statusCode,
      duration: `${duration}ms`,
      contentLength: res.get('Content-Length') || 0
    });
  });

};

- **Spoofing**: Token roubado ou falsificado
- **Tampering**: Modificação de claims no JWT

- **Information Disclosure**: Token exposto em logs/erros
- **Denial of Service**: Ataque de força bruta em login
- **Elevation of Privilege**: Token com privilégios elevados

- **Repudiation**: Falta de logs de auditoria

- **Denial of Service**: Rate limiting inadequado
- **Elevation of Privilege**: IDOR (Insecure Direct Object Reference)

#### C. Banco de Dados
- **Spoofing**: Credenciais de DB comprometidas
- **Tampering**: SQL Injection
- **Repudiation**: Falta de audit trail
- **Information Disclosure**: Dados não criptografados
- **Denial of Service**: Queries maliciosas
- **Elevation of Privilege**: Acesso excessivo ao DB

#### D. Integração WhatsApp

**src/security/auth/jwt.js**
```javascript
const jwt = require('jsonwebtoken');
const crypto = require('crypto');
const { promisify } = require('util');

class SecureJWTManager {
    constructor() {
        this.secret = process.env.JWT_SECRET || this.generateSecureSecret();
        this.accessTokenExpiry = '15m';
        this.refreshTokenExpiry = '7d';
        this.algorithm = 'HS256';
        this.tokenBlacklist = new Set();
        this.failedAttempts = new Map();
    }

    generateSecureSecret() {
        return crypto.randomBytes(64).toString('hex');
    }

    async generateTokens(user) {
        const accessToken = jwt.sign(
            {
                userId: user.id,
                role: user.role,
                permissions: user.permissions,
                iss: 'diyapp-v11',
                aud: 'diyapp-client',
                jti: crypto.randomBytes(16).toString('hex')
            },
            this.secret,
            {
                expiresIn: this.accessTokenExpiry,
                algorithm: this.algorithm
            }
        );

        const refreshToken = jwt.sign(
            {
                userId: user.id,
                type: 'refresh',
                jti: crypto.randomBytes(16).toString('hex')
            },
            this.secret,
            {
                expiresIn: this.refreshTokenExpiry,
                algorithm: this.algorithm
            }
        );

        // Store refresh token hash in DB
        await this.storeRefreshToken(user.id, refreshToken);

        return { accessToken, refreshToken };
    }

    async verifyToken(token, options = {}) {
        try {
            // Check blacklist
            if (this.tokenBlacklist.has(token)) {
                throw new Error('Token revoked');
            }

            const decoded = jwt.verify(token, this.secret, {
                algorithms: [this.algorithm],
                ...options
            });

            // Additional security checks
            if (decoded.iss !== 'diyapp-v11') {
                throw new Error('Invalid issuer');
            }

            if (decoded.aud !== 'diyapp-client') {
                throw new Error('Invalid audience');
            }

            return decoded;
        } catch (error) {
            this.logSecurityEvent('TOKEN_VERIFICATION_FAILED', {
                error: error.message,
                token: token.substring(0, 20) + '...'
            });
            throw error;

const rateLimit = require('express-rate-limit');
const Redis = require('ioredis');
const crypto = require('crypto');

class AdvancedRateLimiter {
    constructor() {
        this.redis = new Redis(process.env.REDIS_URL || 'redis://localhost:6379');
        this.windows = {
            auth: 15 * 60 * 1000, // 15 minutes
            api: 60 * 60 * 1000,  // 1 hour
            global: 24 * 60 * 60 * 1000 // 24 hours
        };
    }

    getAuthLimiter() {
        return rateLimit({
            windowMs: this.windows.auth,
            max: 5, // 5 attempts per 15 minutes
            message: 'Too many login attempts, please try again later',
            standardHeaders: true,
            legacyHeaders: false,
            keyGenerator: (req) => {
                // Use IP + user agent for fingerprinting
                const fingerprint = crypto
                    .createHash('sha256')
                    .update(req.ip + req.headers['user-agent'])
                    .digest('hex');
                return `auth:${fingerprint}`;
            },
            handler: async (req, res) => {
                // Log failed attempt
                await this.logFailedAttempt(req);
                
                // Check if we should block this IP
                const shouldBlock = await this.checkForBlock(req);
                if (shouldBlock) {
                    return res.status(429).json({
                        error: 'IP temporarily blocked due to suspicious activity',
                        retryAfter: '1 hour'
                    });
                }

                res.status(429).json({
                    error: 'Too many login attempts',
                    retryAfter: '15 minutes'
                });
            }
        });
    }

    getAPILimiter() {
        return rateLimit({
            windowMs: this.windows.api,
            max: 100, // 100 requests per hour per user
            message: 'Too many requests, please try again later',
            standardHeaders: true,
            legacyHeaders: false,
            keyGenerator: (req) => {
                return req.user ? `user:${req.user.id}` : `ip:${req.ip}`;
            },
            skip: (req) => {
                // Skip rate limiting for certain endpoints
                const skipPaths = ['/health', '/metrics'];
                return skipPaths.includes(req.path);
            }
        });
    }

    async logFailedAttempt(req) {
        const key = `failed_attempts:${req.ip}`;
        const attempts = await this.redis.incr(key);
        
        if (attempts === 1) {
            await this.redis.expire(key, 3600); // Expire after 1 hour
        }

        // Log detailed information
        const attemptData = {
            ip: req.ip,
            userAgent: req.headers['user-agent'],
            timestamp: new Date().toISOString(),
            path: req.path,
            method: req.method
        };

        await this.redis.lpush('security_logs', JSON.stringify(attemptData));
        await this.redis.ltrim('security_logs', 0, 999); // Keep last 1000 logs
    }

    async checkForBlock(req) {
        const key = `failed_attempts:${req.ip}`;
        const attempts = await this.redis.get(key);
        
        if (attempts && parseInt(attempts) > 10) {
            // Block IP for 1 hour
            await this.redis.setex(`blocked:${req.ip}`, 3600, 'true');
            return true;
        }
        
        return false;
    }
}

const crypto = require('crypto');
const jwt = require('jsonwebtoken');

class SecureJWTManager {
    constructor() {
        this.secret = process.env.JWT_SECRET || crypto.randomBytes(64).toString('hex');
        this.refreshSecret = process.env.JWT_REFRESH_SECRET || crypto.randomBytes(64).toString('hex');
        this.tokenBlacklist = new Set();
        
        // Rotação automática de secrets (24h)
        setInterval(() => this.rotateSecrets(), 24 * 60 * 60 * 1000);
    }
    
    rotateSecrets() {
        const oldSecret = this.secret;
        this.secret = crypto.randomBytes(64).toString('hex');
        this.refreshSecret = crypto.randomBytes(64).toString('hex');
        
        // Mantém oldSecret por 1h para tokens ainda válidos
        setTimeout(() => {
            delete this.oldSecret;
        }, 60 * 60 * 1000);
        
        this.oldSecret = oldSecret;
    }
    
    generateAccessToken(user) {
        return jwt.sign(
            {
                userId: user.id,
                email: user.email,
                role: user.role,
                permissions: user.permissions
            },
            this.secret,
            {
                expiresIn: '15m', // Curta duração
                algorithm: 'HS256',
                issuer: 'diyapp-v11',
                audience: 'diyapp-users'
            }
        );
    }
    
    generateRefreshToken(user) {
        const refreshToken = jwt.sign(
            {
                userId: user.id,
                type: 'refresh'
            },
            this.refreshSecret,
            {
                expiresIn: '7d',
                algorithm: 'HS256'
            }
        );
        
        // Armazena hash do refresh token no DB
        const tokenHash = crypto.createHash('sha256').update(refreshToken).digest('hex');
        this.storeRefreshTokenHash(user.id, tokenHash);
        
        return refreshToken;
    }
    
    verifyToken(token, isRefresh = false) {
        try {
            // Verifica blacklist
            if (this.tokenBlacklist.has(token)) {
                throw new Error('Token revoked');
            }
            
            const secret = isRefresh ? this.refreshSecret : this.secret;
            const decoded = jwt.verify(token, secret, {
                algorithms: ['HS256'],
                issuer: 'diyapp-v11',
                audience: 'diyapp-users'
            });
            
            // Verificação adicional de timestamp
            const now = Math.floor(Date.now() / 1000);
            if (decoded.exp < now) {
                throw new Error('Token expired');
            }
            
            // Verifica se é refresh token quando esperado
            if (isRefresh && decoded.type !== 'refresh') {
                throw new Error('Invalid token type');
            }
            
            return decoded;
        } catch (error) {
            // Tenta com oldSecret se disponível
            if (!isRefresh && this.oldSecret) {
                try {
                    return jwt.verify(token, this.oldSecret, {
                        algorithms: ['HS256'],
                        issuer: 'diyapp-v11',
                        audience: 'diyapp-users'
                    });
                } catch {
                    throw error;
                }
            }
            throw error;
        }
    }
    
    revokeToken(token) {
        this.tokenBlacklist.add(token);
        
        // Limpeza automática de tokens expirados
        setTimeout(() => {
            this.tokenBlacklist.delete(token);
        }, 16 * 60 * 1000); // 16 minutos (mais que expiração)
    }
    
    async storeRefreshTokenHash(userId, tokenHash) {
        // Implementação com DB
        const db = require('../../config/database');
        await db.query(
            'INSERT INTO refresh_tokens (user_id, token_hash, expires_at) VALUES (?, ?, DATE_ADD(NOW(), INTERVAL 7 DAY))',
            [userId, tokenHash]
        );
    }
    
    async validateRefreshToken(userId, refreshToken) {
        const tokenHash = crypto.createHash('sha256').update(refreshToken).digest('hex');
        const db = require('../../config/database');
        
        const result = await db.query(
            'SELECT * FROM refresh_tokens WHERE user_id = ? AND token_hash = ? AND expires_at > NOW() AND revoked = FALSE',
            [userId, tokenHash]
        );
        
        return result.length > 0;
    }
}

module.exports = new SecureJWTManager();

### 2.2. Rate Limiting com Redis

**src/security/auth/rate-limiter.js**
```javascript
const Redis = require('ioredis');
const crypto = require('crypto');

class SecurityRateLimiter {
    constructor() {
        this.redis = new Redis(process.env.REDIS_URL || 'redis://localhost:6379');
        
        this.limits = {
            // Por IP
            'auth-login': { windowMs: 15 * 60 * 1000, max: 5 }, // 5 tentativas em 15min
            'auth-register': { windowMs: 60 * 60 * 1000, max: 3 }, // 3 registros por hora
            'api-general': { windowMs: 60 * 1000, max: 100 }, // 100 requests por minuto
            'api-critical': { windowMs: 60 * 1000, max: 30 }, // 30 requests por minuto
            
            // Por usuário
            'user-actions': { windowMs: 60 * 1000, max: 50 }
        };
    }
    
    async checkLimit(key, limitConfig, userId = null) {
        const now = Date.now();
        const windowStart = now - limitConfig.windowMs;
        
        // Chave composta para identificar usuário/IP
        const identifier = userId ? `user:${userId}` : `ip:${key}`;
        const redisKey = `ratelimit:${identifier}:${limitConfig.windowMs}`;
        
        // Remove registros antigos
        await this.redis.zremrangebyscore(redisKey, 0, windowStart);
        
        // Conta requests na janela
        const requestCount = await this.redis.zcard(redisKey);
        
        if (requestCount >= limitConfig.max) {
            // Usuário/IP excedeu limite
            const oldest = await this.redis.zrange(redisKey, 0, 0, 'WITHSCORES');
            const retryAfter = Math.ceil((parseInt(oldest[1]) + limitConfig.windowMs - now) / 1000);
            
            return {
                allowed: false,
                retryAfter,
                remaining: 0,
                limit: limitConfig.max
            };
        }
        
        // Adiciona novo request
        await this.redis.zadd(redisKey, now, `${now}-${crypto.randomBytes(4).toString('hex')}`);
        await this.redis.expire(redisKey, Math.ceil(limitConfig.windowMs / 1000) + 1);
        
        return {
            allowed: true,
            retryAfter: 0,
            remaining: limitConfig.max - requestCount - 1,
            limit: limitConfig.max
        };
    }
    
    async middleware(req, res, next) {
        const ip = req.ip || req.connection.remoteAddress;
        const userId = req.user ? req.user.id : null;
        
        // Determina qual limite aplicar
        let limitType = 'api-general';
        
        if (req.path.includes('/auth/login')) {
            limitType = 'auth-login';
        } else if (req.path.includes('/auth/register')) {
            limitType = 'auth-register';
        } else if (req.path.includes('/admin/') || req.path.includes('/llm/')) {
            limitType = 'api-critical';
        } else if (userId && req.path.includes('/user/')) {
            limitType = 'user-actions';
        }
        
        const limitResult = await this.checkLimit(
            ip,
            this.limits[limitType],
            userId
        );
        
        if (!limitResult.allowed) {
            // Log de tentativa de ataque
            console.warn(`Rate limit exceeded: ${ip} ${userId} ${req.path}`);
            
            // Resposta com headers RFC 6585
            res.setHeader('Retry-After', limitResult.retryAfter);
            res.setHeader('X-RateLimit-Limit', limitResult.limit);
            res.setHeader('X-RateLimit-Remaining', 0);
            res.setHeader('X-RateLimit-Reset', new Date(Date.now() + limitResult.retryAfter * 1000).toISOString());
            
            return res.status(429).json({
                error: 'Too Many Requests',
                message: `Rate limit exceeded. Try again in ${limitResult.retryAfter} seconds.`,
                retryAfter: limitResult.retryAfter
            });
        }
        
        // Adiciona headers de rate limit
        res.setHeader('X-RateLimit-Limit', limitResult.limit);
        res.setHeader('X-RateLimit-Remaining', limitResult.remaining);
        
        next();
    }
    
    // Circuit breaker para endpoints críticos
    async circuitBreaker(endpoint, fn) {
        const failureKey = `circuit:${endpoint}:failures`;
        const stateKey = `circuit:${endpoint}:state`;
        
        const state = await this.redis.get(stateKey) || 'CLOSED';
        const failures = parseInt(await this.redis.get(failureKey) || '0');
        
        if (state === 'OPEN') {
            const openedAt = await this.redis.get(`circuit:${endpoint}:opened_at`);
            const halfOpenTime = parseInt(openedAt) + 30000; // 30s depois
            
            if (Date.now() > halfOpenTime) {
                // Tenta half-open
                await this.redis.set(stateKey, 'HALF_OPEN');
                try {
                    const result = await fn();
                    // Sucesso - fecha circuito
                    await this.redis.set(stateKey, 'CLOSED');
                    await this.redis.set(failureKey, '0');
                    return result;
                } catch (error) {
                    // Falha - reabre circuito
                    await this.redis.set(stateKey, 'OPEN');
                    await this.redis.set(`circuit:${endpoint}:opened_at`, Date.now());
                    throw error;
                }
            }
            
            throw new Error(`Circuit breaker open for ${endpoint}`);
        }
        
        try {
            const result = await fn();
            
            // Sucesso - reseta falhas se houver
            if (failures > 0) {
                await this.redis.set(failureKey, '0');
            }
            
            return result;
        } catch (error) {
            // Incrementa falhas
            const newFailures = failures + 1;
            await this.redis.set(failureKey, newFailures);
            
            // Se muitas falhas, abre circuito
            if (newFailures >= 5) {
                await this.redis.set(stateKey, 'OPEN');
                await this.redis.set(`circuit:${endpoint}:opened_at`, Date.now());
                
                // Fecha automaticamente depois de 30s
                setTimeout(async () => {
                    await this.redis.set(stateKey, 'HALF_OPEN');
                }, 30000);
            }
            
            throw error;
        }
    }
}

function calculateTrend(data: number[]): { slope: number; r2: number } {
  const n = data.length;
  const x = Array.from({ length: n }, (_, i) => i);
  
  const sumX = x.reduce((a, b) => a + b, 0);
  const sumY = data.reduce((a, b) => a + b, 0);
  const sumXY = x.reduce((sum, xi, i) => sum + xi * data[i], 0);
  const sumX2 = x.reduce((sum, xi) => sum + xi * xi, 0);
  const sumY2 = data.reduce((sum, yi) => sum + yi * yi, 0);
  
  const slope = (n * sumXY - sumX * sumY) / (n * sumX2 - sumX * sumX);
  const intercept = (sumY - slope * sumX) / n;
  
  // Calcula R²
  const yMean = sumY / n;
  const ssTot = data.reduce((sum, yi) => sum + Math.pow(yi - yMean, 2), 0);
  const ssRes = data.reduce((sum, yi, i) => sum + Math.pow(yi - (slope * x[i] + intercept), 2), 0);
  const r2 = 1 - (ssRes / ssTot);
  
  return { slope, r2 };
}

# Guia de Autenticação - DIYAPP V11

## Visão Geral

A API V11 usa autenticação baseada em JWT (JSON Web Tokens). Cada requisição deve incluir um token de acesso válido no header `Authorization`.

## Fluxo de Autenticação

### 1. Login
```http
POST /auth/login
Content-Type: application/json

{
  "email": "usuario@empresa.com",
  "password": "sua_senha"
}
```

**Resposta de Sucesso:**
```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 3600,
  "user": {
    "id": "user_123",
    "email": "usuario@empresa.com",
    "name": "João Silva",
    "role": "admin"
  }
}
```

### 2. Usando o Token de Acesso
Inclua o token no header de todas as requisições protegidas:

```http
GET /workflows
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### 3. Renovando o Token
Quando o token de acesso expirar (1 hora), use o refresh token:

```http
POST /auth/refresh
Content-Type: application/json

{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

## Tipos de Tokens

| Token | Duração | Propósito |
|-------|---------|-----------|
| Access Token | 1 hora | Autenticar requisições à API |
| Refresh Token | 7 dias | Renovar access token |

## Códigos de Erro

| Código | Significado | Ação Recomendada |
|--------|-------------|------------------|
| 401 | Token inválido ou expirado | Renovar token via `/auth/refresh` |
| 403 | Permissão insuficiente | Verificar permissões do usuário |
| 429 | Muitas requisições | Aguardar antes de tentar novamente |
