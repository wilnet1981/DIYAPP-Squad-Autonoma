# DIYAPP Evolution - V12 Core - Arquitetura de Microsserviços

## ADR-001: Arquitetura de Microsserviços V12

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra

**CONTEXTO:**
A V11 do DIYAPP atingiu limites de escalabilidade devido à arquitetura monolítica. O crescimento do código base (agora >50k linhas) e a necessidade de deploy independente de funcionalidades exigem transição para microsserviços. A squad opera em Modo Hive com 5 desenvolvedores trabalhando em paralelo.

**DECISÃO:**
Adotar arquitetura de microsserviços baseada em domínios (Domain-Driven Design) com os seguintes princípios:
1. Cada microsserviço é responsável por um domínio de negócio específico
2. Comunicação síncrona via gRPC para operações críticas
3. Comunicação assíncrona via Apache Kafka para eventos de domínio
4. API Gateway único para clientes externos (GraphQL)
5. Cada serviço tem seu próprio banco de dados (Database per Service)

**OPÇÕES CONSIDERADAS:**
- **Opção A:** Arquitetura monolítica com módulos - Prós: Simplicidade, deploy único. Contras: Escalabilidade limitada, acoplamento alto.
- **Opção B:** Microsserviços baseados em funcionalidade - Prós: Separação clara. Contras: Pode criar serviços muito pequenos (nanoservices).
- **Opção C:** Microsserviços baseados em domínios (escolhida) - Prós: Alinhamento com negócio, autonomia por squad. Contras: Complexidade inicial maior.

**CONSEQUÊNCIAS:**
**Positivas:** Escalabilidade horizontal independente, deploy contínuo por serviço, tecnologia heterogênea permitida.
**Negativas:** Complexidade operacional aumentada, necessidade de observabilidade distribuída, consistência eventual.
**Riscos:** Latência em chamadas entre serviços, dificuldade de debugging distribuído. Mitigação: Implementar tracing distribuído (OpenTelemetry) e circuit breakers.

**REVISÃO:** 2024-04-15

## CI/CD Pipeline Integrado (GitHub Actions)

```yaml
# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:14
        env:
          POSTGRES_PASSWORD: postgres
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
      kafka:
        image: confluentinc/cp-kafka:latest
        env:
          KAFKA_ZOOKEEPER_CONNECT: localhost:2181
        ports:
          - 9092:9092

    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
      
    - name: Lint
      run: npm run lint
      
    - name: Type check
      run: npm run type-check
      
    - name: Unit tests
      run: npm test -- --coverage
      
    - name: Integration tests
      run: npm run test:integration
      env:
        DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_db
        KAFKA_BROKERS: localhost:9092
        
    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        file: ./coverage/lcov.info

  security:
    runs-on: ubuntu-latest
    needs: test
    steps:
    - uses: actions/checkout@v3
    
    - name: Run Snyk to check for vulnerabilities
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
        
    - name: Run OWASP Dependency Check
      uses: dependency-check/Dependency-Check_Action@main
      with:
        project: '${{ github.event.repository.name }}'
        path: '.'
        format: 'HTML'
        args: >
          --failOnCVSS 7
          --scan .
          
    - name: Run Semgrep
      uses: returntocorp/semgrep-action@v1
      with:
        config: p/security-audit

  build:
    runs-on: ubuntu-latest
    needs: security
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v2
      
    - name: Log in to Container Registry
      uses: docker/login-action@v2
      with:
        registry: ${{ env.REGISTRY }}
        username: ${{ github.actor }}
        password: ${{ secrets.GITHUB_TOKEN }}
        
    - name: Extract metadata
      id: meta
      uses: docker/metadata-action@v4
      with:
        images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
        tags: |
          type=sha,prefix={{branch}}-
          type=ref,event=branch
          type=ref,event=pr
          type=semver,pattern={{version}}
          type=semver,pattern={{major}}.{{minor}}
          
    - name: Build and push
      uses: docker/build-push-action@v4
      with:
        context: .
        push: true
        tags: ${{ steps.meta.outputs.tags }}
        labels: ${{ steps.meta.outputs.labels }}
        cache-from: type=gha
        cache-to: type=gha,mode=max

  deploy:
    runs-on: ubuntu-latest
    needs: build
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    environment: production
    
    steps:
    - name: Deploy to Kubernetes
      uses: azure/k8s-deploy@v4
      with:
        namespace: 'diyapp-production'
        manifests: |
          k8s/deployment.yaml
          k8s/service.yaml
          k8s/ingress.yaml
        images: |
          ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
        kubectl-version: 'latest'
        
    - name: Run smoke tests
      run: |
        npm run test:smoke
        --url https://api.diyapp.example.com/health
        
    - name: Notify deployment
      uses: rtCamp/action-slack-notify@v2
      env:
        SLACK_WEBHOOK: ${{ secrets.SLACK_WEBHOOK }}
        SLACK_CHANNEL: deployments
        SLACK_COLOR: good
        SLACK_TITLE: "Deployment Successful"
        SLACK_MESSAGE: "Service ${{ github.event.repository.name }} deployed to production"
```

## Protocolo de Comunicação gRPC

```proto
// protobuf/service.proto
syntax = "proto3";

package diyapp.v12;

import "google/protobuf/timestamp.proto";
import "google/protobuf/empty.proto";

service UserService {
  rpc GetUser(GetUserRequest) returns (UserResponse);
  rpc CreateUser(CreateUserRequest) returns (UserResponse);
  rpc UpdateUser(UpdateUserRequest) returns (UserResponse);
  rpc DeleteUser(DeleteUserRequest) returns (google.protobuf.Empty);
}

message GetUserRequest {
  string user_id = 1;
}

message CreateUserRequest {
  string email = 1;
  string name = 2;
  UserRole role = 3;
}

message UpdateUserRequest {
  string user_id = 1;
  optional string email = 2;
  optional string name = 3;
  optional UserRole role = 4;
}

message DeleteUserRequest {
  string user_id = 1;
}

message UserResponse {
  string id = 1;
  string email = 2;
  string name = 3;
  UserRole role = 4;
  google.protobuf.Timestamp created_at = 5;
  google.protobuf.Timestamp updated_at = 6;
}

enum UserRole {
  USER = 0;
  ADMIN = 1;
  MODERATOR = 2;
}

// Event streaming
service EventService {
  rpc StreamEvents(StreamEventsRequest) returns (stream Event);
}

message StreamEventsRequest {
  string service_name = 1;
  repeated string event_types = 2;
}

message Event {
  string id = 1;
  string type = 2;
  string aggregate_id = 3;
  google.protobuf.Timestamp occurred_at = 4;
  bytes payload = 5;
  string metadata = 6; // JSON string
}
```

## Configuração Docker para Desenvolvimento

```dockerfile
# docker/Dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

COPY package*.json ./
COPY tsconfig.json ./
COPY .npmrc ./

RUN npm ci --only=production && npm cache clean --force

COPY . .

RUN npm run build

FROM node:18-alpine AS production

WORKDIR /app

RUN apk add --no-cache tini

COPY --from=builder /app/package*.json ./
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/.env.production ./

USER node

ENTRYPOINT ["/sbin/tini", "--"]

CMD ["node", "dist/src/index.js"]

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:3000/health', (r) => {process.exit(r.statusCode === 200 ? 0 : 1)})"
```

```typescript
interface AppState {

  deleteTask: (id: string) => void;
  
  // Notifications state
  notifications: Notification[];
  addNotification: (notification: Omit<Notification, 'id' | 'createdAt' | 'read'>) => void;
  markAsRead: (id: string) => void;
  clearNotifications: () => void;
  
  // UI state
  sidebarOpen: boolean;
  toggleSidebar: () => void;
  
  // Loading states
  loading: boolean;
  setLoading: (loading: boolean) => void;
}

export const useStore = create<AppState>()(
  persist(
    (set) => ({
      // Initial state
      user: null,
      theme: 'system',
      projects: [],
      tasks: [],
      notifications: [],
      sidebarOpen: true,
      loading: false,
      
      // Actions
      setUser: (user) => set({ user }),
      
      setTheme: (theme) => set({ theme }),
      
      addProject: (project) =>
        set((state) => ({ projects: [...state.projects, project] })),

            project.id === id ? { ...project, ...updates, updatedAt: new Date() } : project
          ),
        })),
      
      deleteProject: (id) =>
        set((state) => ({
          projects: state.projects.filter((project) => project.id !== id),
        })),
      
      addTask: (task) =>
        set((state) => ({ tasks: [...state.tasks, task] })),
      
      updateTask: (id, updates) =>
```

  constructor() {
    super({
      // Timeout explícito para queries
      datasources: {
        db: {
          url: process.env.DATABASE_URL,
        },
      },
    });
  }

  async onModuleInit() {
    await this.$connect();
  }

  async onModuleDestroy() {
    await this.$disconnect();
  }

  // Helper para transações com timeout
  async transactionWithTimeout<T>(
    callback: (prisma: PrismaClient) => Promise<T>,
    timeoutMs = 10000,
  ): Promise<T> {
    return Promise.race([
      this.$transaction(callback),
      new Promise<never>((_, reject) =>
        setTimeout(() => reject(new Error('Transaction timeout')), timeoutMs),
      ),
    ]);
  }

  // Helper para queries parametrizadas (segurança)
  async safeQuery<T>(
    model: any,
    action: string,
    args: any,
  ): Promise<T> {
    try {
      return await model[action](args);
    } catch (error) {
      // Log seguro - nunca expõe dados sensíveis
      console.error({
        message: `Database error in ${action}`,
        model: model.name,
        errorCode: error.code,
        // NUNCA loga args completos - pode conter dados sensíveis
        hasArgs: !!args,
      });
      throw error;
    }
  }
}

```sql
-- Function to create monthly partitions for users table
CREATE OR REPLACE FUNCTION create_users_monthly_partition(month_date DATE)
RETURNS VOID AS $$
DECLARE
    partition_name TEXT;
    start_date DATE;
    end_date DATE;
BEGIN
    start_date := DATE_TRUNC('month', month_date);
    end_date := start_date + INTERVAL '1 month';
    partition_name := 'users_y' || EXTRACT(YEAR FROM start_date) || 'm' || LPAD(EXTRACT(MONTH FROM start_date)::TEXT, 2, '0');
    
    EXECUTE format(
        'CREATE TABLE IF NOT EXISTS %I PARTITION OF users FOR VALUES FROM (%L) TO (%L)',
        partition_name,
        start_date,
        end_date
    );
    
    RAISE NOTICE 'Created partition % for users table', partition_name;
END;
$$ LANGUAGE plpgsql;

-- Function to create daily partitions for events table
CREATE OR REPLACE FUNCTION create_events_daily_partition(day_date DATE)
RETURNS VOID AS $$
DECLARE
    partition_name TEXT;
    start_date DATE;
    end_date DATE;
BEGIN
    start_date := day_date;
    end_date := start_date + INTERVAL '1 day';
    partition_name := 'user_events_y' || EXTRACT(YEAR FROM start_date) || 'm' || 
                     LPAD(EXTRACT(MONTH FROM start_date)::TEXT, 2, '0') || 'd' ||
                     LPAD(EXTRACT(DAY FROM start_date)::TEXT, 2, '0');
    
    EXECUTE format(
        'CREATE TABLE IF NOT EXISTS %I PARTITION OF user_events FOR VALUES FROM (%L) TO (%L)',
        partition_name,
        start_date,
        end_date
    );
    
    RAISE NOTICE 'Created partition % for user_events table', partition_name;
END;
$$ LANGUAGE plpgsql;

-- Function to drop old partitions (retention policy)
CREATE OR REPLACE FUNCTION drop_old_partitions(retention_months INTEGER DEFAULT 12)
RETURNS INTEGER AS $$
DECLARE
    partition_record RECORD;
    dropped_count INTEGER := 0;
    cutoff_date DATE;
BEGIN
    cutoff_date := CURRENT_DATE - (retention_months || ' months')::INTERVAL;
    
    -- Drop old user partitions
    FOR partition_record IN
        SELECT tablename
        FROM pg_tables
        WHERE tablename LIKE 'users_y%'
        AND tablename ~ '^users_y(\d{4})m(\d{2})$'
        AND (
            SUBSTRING(tablename FROM '^users_y(\d{4})m(\d{2})$')::DATE + INTERVAL '1 month'
        ) < cutoff_date
    LOOP
        EXECUTE format('DROP TABLE IF EXISTS %I', partition_record.tablename);
        dropped_count := dropped_count + 1;
        RAISE NOTICE 'Dropped old partition: %', partition_record.tablename;
    END LOOP;
    
    -- Drop old event partitions
    FOR partition_record IN
        SELECT tablename
        FROM pg_tables
        WHERE tablename LIKE 'user_events_y%'
        AND tablename ~ '^user_events_y(\d{4})m(\d{2})d(\d{2})$'
        AND SUBSTRING(tablename FROM '^user_events_y(\d{4})m(\d{2})d(\d{2})$')::DATE < cutoff_date
    LOOP
        EXECUTE format('DROP TABLE IF EXISTS %I', partition_record.tablename);
        dropped_count := dropped_count + 1;
        RAISE NOTICE 'Dropped old partition: %', partition_record.tablename;
    END LOOP;
    
    RETURN dropped_count;
END;
$$ LANGUAGE plpgsql;
```

### 1.2 Análise STRIDE

#### **Spoofing (Falsificação)**
- **Ameaça:** Ataque de força bruta em endpoints de autenticação
- **Ameaça:** Token JWT roubado/reutilizado
- **Ameaça:** Phishing de credenciais de administrador
- **Controles Implementados:**
  - MFA obrigatório para contas admin
  - Rate limiting por IP/usuario
  - Tokens JWT com expiração curta (15min)
  - Refresh tokens com revogação
  - Monitoramento de logins suspeitos

#### **Tampering (Manipulação)**
- **Ameaça:** Manipulação de dados em trânsito (MITM)
- **Ameaça:** Injeção SQL/NoSQL
- **Ameaça:** Manipulação de prompts de LLM
- **Controles Implementados:**
  - TLS 1.3 em todas as comunicações
  - Validação de schema em todas as APIs
  - Prepared statements/ORM sanitizado
  - Sanitização de inputs de LLM
  - Assinatura digital de payloads críticos

#### **Repudiation (Não-repúdio)**
- **Ameaça:** Usuário nega ação realizada
- **Ameaça:** Falta de logs de operações críticas
- **Controles Implementados:**
  - Logs imutáveis em SIEM centralizado
  - Auditoria completa de operações admin
  - Assinatura de timestamp em transações
  - Retenção de logs por 365 dias

#### **Information Disclosure (Divulgação)**
- **Ameaça:** Vazamento de dados sensíveis
- **Ameaça:** Exposição de secrets no código
- **Ameaça:** Logs contendo PII
- **Controles Implementados:**
  - Criptografia AES-256-GCM em repouso
  - Mascaramento de PII em logs
  - Secrets gerenciados por Vault
  - Política de least privilege em DB

#### **Denial of Service (Negação)**
- **Ameaça:** Ataque DDoS na API
- **Ameaça:** Exaustão de recursos do LLM
- **Ameaça:** Rate limiting agressivo
- **Controles Implementados:**
  - WAF com proteção DDoS
  - Rate limiting por usuário/API key
  - Circuit breakers em microsserviços
  - Auto-scaling com limites de custo

#### **Elevation of Privilege (Elevação)**
- **Ameaça:** Exploração de vulnerabilidade para acesso admin
- **Ameaça:** IDOR (Insecure Direct Object Reference)
- **Ameaça:** Manipulação de parâmetros de URL
- **Controles Implementados:**
  - RBAC com verificação por recurso
  - Validação de autorização em cada endpoint
  - Revisão periódica de permissões
  - Segregação de ambientes (dev/staging/prod)

### 1.3 Riscos Aceitos Conscientemente
1. **Risco:** Uso de LLM de terceiros pode expor dados via API
   **Justificativa:** Necessário para funcionalidade core, mitigado com contrato de processamento de dados e criptografia em trânsito
   
2. **Risco:** Retenção de logs por 365 dias pode aumentar superfície de ataque
   **Justificativa:** Requerido por compliance (LGPD), mitigado com criptografia e acesso restrito

### 2.2 Middleware WAF em Node.js

```javascript
// src/security/waf-middleware.js
const rateLimit = require('express-rate-limit');
const { body, validationResult } = require('express-validator');
const mongoSanitize = require('express-mongo-sanitize');
const helmet = require('helmet');
const xss = require('xss-clean');
const hpp = require('hpp');

class WAFMiddleware {
    constructor() {
        this.setupSecurityHeaders();
        this.setupRateLimiting();
        this.setupInputValidation();
    }

    setupSecurityHeaders() {
        return helmet({
            contentSecurityPolicy: {
                directives: {
                    defaultSrc: ["'self'"],
                    styleSrc: ["'self'", "'unsafe-inline'", "https://cdn.jsdelivr.net"],
                    scriptSrc: ["'self'", "'unsafe-inline'", "https://cdn.jsdelivr.net"],
                    imgSrc: ["'self'", "data:", "https:"],
                    connectSrc: ["'self'", "https://api.openai.com"],
                    fontSrc: ["'self'", "https://cdn.jsdelivr.net"],
                    objectSrc: ["'none'"],
                    mediaSrc: ["'self'"],
                    frameSrc: ["'none'"]
                }
            },
            hsts: {
                maxAge: 31536000,
                includeSubDomains: true,
                preload: true
            }
        });
    }

    setupRateLimiting() {
        const apiLimiter = rateLimit({
            windowMs: 15 * 60 * 1000, // 15 minutos
            max: 100, // limite por IP
            message: {
                error: 'Muitas requisições deste IP, tente novamente após 15 minutos'
            },
            standardHeaders: true,
            legacyHeaders: false
        });

        const authLimiter = rateLimit({
            windowMs: 60 * 60 * 1000, // 1 hora
            max: 5, // 5 tentativas de login por hora
            message: {
                error: 'Muitas tentativas de login, tente novamente após 1 hora'
            }
        });

        return { apiLimiter, authLimiter };
    }

    setupInputValidation() {
        return [
            body().customSanitizer(value => {
                // Sanitize all inputs
                if (typeof value === 'string') {
                    return value.replace(/[<>]/g, '');
                }
                return value;
            }),
            mongoSanitize(),
            xss(),
            hpp()
        ];
    }

    // Detecção de ataques comuns
    static detectMaliciousInput(input) {
        const patterns = {
            sqlInjection: /(\%27)|(\')|(\-\-)|(\%23)|(#)|(\%3B)|(;)/i,
            xss: /((\%3C)|<)((\%2F)|\/)*[a-z0-9\%]+((\%3E)|>)/i,
            pathTraversal: /\.\.\//i,
            commandInjection: /(\||\&|\;|\`|\$\(|\n|\r)/i,
            nosqlInjection: /\$where|\$ne|\$gt|\$lt|\$in|\$nin/i
        };

        for (const [type, pattern] of Object.entries(patterns)) {
            if (pattern.test(input)) {
                return { malicious: true, type };
            }
        }

        return { malicious: false };
    }

    // Validação específica para LLM inputs
    static validateLLMInput(prompt) {
        const threats = {
            promptInjection: [
                /ignore previous instructions/i,
                /system prompt/i,
                /forget your instructions/i,
                /you are now/i,
                /act as/i,
                /disregard your/i
            ],
            dataExfiltration: [
                /send.*data.*to/i,
                /email.*to/i,
                /post.*to.*http/i,
                /base64.*encode/i
            ],
            privilegeEscalation: [
                /admin.*password/i,
                /root.*access/i,
                /sudo.*command/i,
                /execute.*as.*admin/i
            ]
        };

        const detectedThreats = [];
        
        for (const [threatType, patterns] of Object.entries(threats)) {
            for (const pattern of patterns) {
                if (pattern.test(prompt)) {
                    detectedThreats.push(threatType);
                    break;
                }
            }
        }

        return {
            safe: detectedThreats.length === 0,
            threats: detectedThreats,
            sanitized: detectedThreats.length > 0 ? 
                prompt.replace(/[<>\[\]{}()\\\/|&;`$]/g, '') : prompt
        };
    }
}

module.exports = WAFMiddleware;
```

### 2.3 Configuração do WAF Principal

```javascript
// src/security/waf-config.js
const WAFMiddleware = require('./waf-middleware');
const winston = require('winston');
const { createClient } = require('redis');

class WAFConfig {
    constructor() {
        this.waf = new WAFMiddleware();
        this.logger = this.setupLogger();
        this.redisClient = this.setupRedis();
        this.blockedIPs = new Set();
    }

    setupLogger() {
        return winston.createLogger({
            level: 'info',
            format: winston.format.combine(
                winston.format.timestamp(),
                winston.format.json()
            ),
            transports: [
                new winston.transports.File({ 
                    filename: 'logs/waf-security.log',
                    level: 'warn'
                }),
                new winston.transports.Console({
                    format: winston.format.simple()
                })
            ]
        });
    }

    setupRedis() {
        const client = createClient({
            url: process.env.REDIS_URL || 'redis://localhost:6379'
        });

        client.on('error', (err) => {
            this.logger.error('Redis Client Error', { error: err.message });
        });

        client.connect();
        return client;
    }

    async blockIP(ip, reason, duration = 3600) {
        try {
            await this.redisClient.setEx(
                `blocked:ip:${ip}`,
                duration,
                JSON.stringify({ reason, timestamp: new Date().toISOString() })
            );
            this.blockedIPs.add(ip);
            this.logger.warn('IP Blocked', { ip, reason, duration });
        } catch (error) {
            this.logger.error('Error blocking IP', { ip, error: error.message });
        }
    }

    async isIPBlocked(ip) {
        try {
            const blocked = await this.redisClient.get(`blocked:ip:${ip}`);
            return blocked !== null;
        } catch (error) {
            this.logger.error('Error checking blocked IP', { ip, error: error.message });
            return false;
        }
    }

    // Middleware de segurança principal
    securityMiddleware() {
        return async (req, res, next) => {
            const clientIP = req.ip || req.connection.remoteAddress;
            
            // Verifica se IP está bloqueado
            if (await this.isIPBlocked(clientIP)) {
                return res.status(403).json({
                    error: 'Access denied',
                    code: 'IP_BLOCKED'
                });
            }

            // Detecta input malicioso
            const bodyString = JSON.stringify(req.body);
            const detection = WAFMiddleware.detectMaliciousInput(bodyString);
            
            if (detection.malicious) {
                await this.blockIP(clientIP, `Malicious input detected: ${detection.type}`, 7200);
                
                this.logger.warn('Malicious request blocked', {
                    ip: clientIP,
                    type: detection.type,
                    path: req.path,
                    userAgent: req.get('User-Agent')
                });

                return res.status(403).json({
                    error: 'Invalid request',
                    code: 'MALICIOUS_INPUT'
                });
            }

            // Validação específica para endpoints de LLM
            if (req.path.includes('/api/llm/') || req.path.includes('/api/chat/')) {
                const validation = WAFMiddleware.validateLLMInput(bodyString);
                
                if (!validation.safe) {
                    this.logger.warn('LLM threat detected', {
                        ip: clientIP,
                        threats: validation.threats,
                        path: req.path
                    });

                    // Usa versão sanitizada
                    req.body = typeof req.body === 'string' ? 
                        validation.sanitized : 
                        JSON.parse(validation.sanitized);
                }
            }

            next();
        };
    }

    // Monitoramento em tempo real
    async monitorTraffic() {
        setInterval(async () => {
            const stats = {
                blockedIPs: this.blockedIPs.size,
                timestamp: new Date().toISOString(),
                memoryUsage: process.memoryUsage()
            };

            await this.redisClient.set('waf:stats', JSON.stringify(stats));
            
            this.logger.info('WAF Statistics', stats);
        }, 60000); // A cada minuto
    }
}

module.exports = WAFConfig;
```

# compliance_manager.py
"""
Sistema de Gestão de Compliance LGPD/GDPR para DIYAPP Evolution V12
Implementa: Consent Management, Política de Retenção, Auditorias Automatizadas
"""

import json

    CONTACT = "contact"
    FINANCIAL = "financial"
    LOCATION = "location"
    BEHAVIORAL = "behavioral"
    SENSITIVE = "sensitive"  # Dados sensíveis - LGPD Art. 5º II
    ANONYMIZED = "anonymized"

class RetentionPolicy(Enum):
    """Políticas de retenção por finalidade"""
    ACCOUNT_DATA = 365 * 5  # 5 anos para dados de conta
    TRANSACTIONAL = 365 * 7  # 7 anos para transações financeiras
    MARKETING = 365 * 2  # 2 anos para marketing

    SUPPORT_TICKETS = 365 * 3  # 3 anos para suporte

class ComplianceDatabase:
    """Gerencia o banco de dados de compliance"""
    
    def __init__(self, db_path: str = "compliance.db"):
        self.db_path = db_path
        self.init_database()
    
    def init_database(self):
        """Inicializa o banco de dados com todas as tabelas necessárias"""
        with sqlite3.connect(self.db_path) as conn:
            cursor = conn.cursor()
            
            # Tabela de titulares de dados
            cursor.execute('''
                CREATE TABLE IF NOT EXISTS data_subjects (
                    id TEXT PRIMARY KEY,
                    email TEXT UNIQUE NOT NULL,
                    created_at TIMESTAMP NOT NULL,
                    last_updated TIMESTAMP NOT NULL,
                    consent_version TEXT NOT NULL,
                    status TEXT DEFAULT 'active'
                )
            ''')
            
            # Tabela ROPA (Registro de Atividades de Processamento)
            cursor.execute('''
                CREATE TABLE IF NOT EXISTS ropa_records (
                    id TEXT PRIMARY KEY,
                    purpose TEXT NOT NULL,
                    data_categories TEXT NOT NULL,
                    legal_basis TEXT NOT NULL,
                    retention_days INTEGER NOT NULL,
                    created_at TIMESTAMP NOT NULL,
                    updated_at TIMESTAMP NOT NULL,
                    is_active BOOLEAN DEFAULT 1
                )
            ''')
            
            # Tabela de consentimentos
            cursor.execute('''
                CREATE TABLE IF NOT EXISTS consent_records (

            # Tabela de regras de retenção
            cursor.execute('''
                CREATE TABLE IF NOT EXISTS retention_rules (

# GERENCIADOR DE CONSENTIMENTO

class ConsentManager:
    """Gerencia consentimentos LGPD/GDPR"""
    
    def __init__(self, db: ComplianceDatabase):
        self.db = db
        self.current_version = "v1.2"
    
    def request_consent(self, data_subject_id: str, purpose: str, 
                       ip_address: str, user_agent: str) -> ConsentRecord:
        """Solicita consentimento para uma finalidade específica"""
        
        # Verifica se já existe consentimento ativo
        existing = self.get_active_consent(data_subject_id, purpose)
        if existing and existing.granted:
            return existing
        
        # Cria novo registro de consentimento (inicialmente não concedido)
        consent = ConsentRecord(
            id=str(uuid.uuid4()),
            data_subject_id=data_subject_id,
            purpose=purpose,
            granted=False,
            timestamp=datetime.now(),
            version=self.current_version,
            ip_address=ip_address,
            user_agent=user_agent
        )
        
        self._save_consent(consent)
        self.db.log_audit_event(
            event_type="CONSENT_REQUESTED",
            description=f"Consentimento solicitado para: {purpose}",
            data_subject_id=data_subject_id,
            ip_address=ip_address,
            user_agent=user_agent,
            metadata={"purpose": purpose, "version": self.current_version}
        )
        
        return consent
    
    def grant_consent(self, consent_id: str, ip_address: str, user_agent: str) -> bool:
        """Registra concessão de consentimento"""
        with sqlite3.connect(self.db.db_path) as conn:
            cursor = conn.cursor()
            cursor.execute('''
                UPDATE consent_records 
                SET granted = 1, timestamp = ?, ip_address = ?, user_agent = ?
                WHERE id = ? AND granted = 0
            ''', (datetime.now().isoformat(), ip_address, user_agent, consent_id))
            
            if cursor.rowcount > 0:
                cursor.execute('SELECT data_subject_id, purpose FROM consent_records WHERE id = ?', (consent_id,))
                result = cursor.fetchone()
                if result:
                    self.db.log_audit_event(
                        event_type="CONSENT_GRANTED",
                        description=f"Consentimento concedido para: {result[1]}",
                        data_subject_id=result[0],
                        ip_address=ip_address,
                        user_agent=user_agent
                    )
                conn.commit()
                return True
        return False
    
    def withdraw_consent(self, data_subject_id: str, purpose: str) -> bool:
        """Revoga consentimento"""
        with sqlite3.connect(self.db.db_path) as conn:
            cursor = conn.cursor()
            cursor.execute('''
                UPDATE consent_records 
                SET granted = 0, withdrawal_timestamp = ?
                WHERE data_subject_id = ? AND purpose = ? AND granted = 1
            ''', (datetime.now().isoformat(), data_subject_id, purpose))
            
            if cursor.rowcount > 0:
                self.db.log_audit_event(
                    event_type="CONSENT_WITHDRAWN",
                    description=f"Consentimento revogado para: {purpose}",
                    data_subject_id=data_subject_id
                )
                conn.commit()
                return True
        return False
    
    def get_active_consent(self, data_subject_id: str, purpose: str) -> Optional[ConsentRecord]:
        """Obtém consentimento ativo para uma finalidade"""
        with sqlite3.connect(self.db.db_path) as conn:
            cursor = conn.cursor()
            cursor.execute('''
                SELECT * FROM consent_records 
                WHERE data_subject_id = ? AND purpose = ? AND granted = 1
                ORDER BY timestamp DESC LIMIT 1
            ''', (data_subject_id, purpose))
            
            result = cursor.fetchone()
            if result:
                return self._row_to_consent(result)
        return None
    
    def _save_consent(self, consent: ConsentRecord):
        """Salva registro de consentimento"""
        with sqlite3.connect(self.db.db_path) as conn:
            cursor = conn.cursor()
            cursor.execute('''
                INSERT INTO consent_records 
                (id, data_subject_id, purpose, granted, timestamp, version, ip_address, user_agent, withdrawal_timestamp)
                VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
            ''', (
                consent.id,
                consent.data_subject_id,
                consent.purpose,
                1 if consent.granted else 0,
                consent.timestamp.isoformat(),
                consent.version,
                consent.ip_address,
                consent.user_agent,
                consent.withdrawal_timestamp.isoformat() if consent.withdrawal_timestamp else None
            ))
            conn.commit()
    
    def _row_to_consent(self, row) -> ConsentRecord:
        """Converte linha do banco para objeto ConsentRecord"""
        return ConsentRecord(
            id=row[0],
            data_subject_id=row[1],
            purpose=row[2],
            granted=bool(row[3]),
            timestamp=datetime.fromisoformat(row[4]),
            version=row[5],
            ip_address=row[6],
            user_agent=row[7],
            withdrawal_timestamp=datetime.fromisoformat(row[8]) if row[8] else None
        )

class DataRetentionManager:
    """Implementa política de retenção e exclusão de dados"""
    
    def __init__(self, db: ComplianceDatabase):
        self.db = db
        self._init_default_rules()
    
    def _init_default_rules(self):
        """Inicializa regras padrão de retenção"""
        default_rules = [
            DataRetentionRule(
                id=str(uuid.uuid4()),
                table_name="user_sessions",
                column_name="session_data",
                data_category=DataCategory.BEHAVIORAL,
                retention_days=30,
                anonymization_method="delete"
            ),
            DataRetentionRule(
                id=str(uuid.uuid4()),
                table_name="audit_logs",
                column_name="*",
                data_category=DataCategory.IDENTIFICATION,
                retention_days=365 * 6,
                anonymization_method="keep"
            ),
            DataRetentionRule(
                id=str(uuid.uuid4()),
                table_name="user_activity",
                column_name="ip_address",
                data_category=DataCategory.IDENTIFICATION,
                retention_days=90,
                anonymization_method="anonymize_ip"
            )
        ]
        
        with sqlite3.connect(self.db.db_path) as conn:
            cursor = conn.cursor()
            for rule in default_rules:
                cursor.execute('''
                    INSERT OR IGNORE INTO retention_rules 
                    (id, table_name, column_name, data_category, retention_days, anonymization_method)
                    VALUES (?, ?, ?, ?, ?, ?)
                ''', (
                    rule.id,
                    rule.table_name,
                    rule.column_name,
                    rule.data_category.value,
                    rule.retention_days,
                    rule.anonymization_method
                ))
            conn.commit()
    
    def execute_retention_policy(self):
        """Executa política de retenção em todas as tabelas"""
        logger.info("Iniciando execução da política de retenção de dados")
        
        with sqlite3.connect(self.db.db_path) as conn:
            cursor = conn.cursor()
            cursor.execute('SELECT * FROM retention_rules')
            rules = cursor.fetchall()
            
            for rule in rules:
                self._apply_retention_rule(rule, conn)
        
        logger.info("Política de retenção executada com sucesso")
    
    def _apply_retention_rule(self, rule, conn):
        """Aplica uma regra de retenção específica"""
        rule_id, table_name, column_name, data_category, retention_days, method, last_exec = rule
        
        cutoff_date = datetime.now() - timedelta(days=retention_days)
        
        if method == "delete":
            cursor = conn.cursor()
            # Assume que a tabela tem uma coluna 'created_at'
            cursor.execute(f'''
                DELETE FROM {table_name} 
                WHERE created_at < ?
            ''', (cutoff_date.isoformat(),))
            deleted_count = cursor.rowcount
            
            if deleted_count > 0:
                logger.info(f"Excluídos {deleted_count} registros de {table_name}")
                self.db.log_audit_event(
                    event_type="DATA_RETENTION_EXECUTED",
                    description=f"Exclusão de {deleted_count} registros de {table_name}",
                    metadata={
                        "table": table_name,
                        "method": method,
                        "cutoff_date": cutoff_date.isoformat(),
                        "count": deleted_count
                    }
                )
        
        elif method == "anonymize_ip":
            cursor = conn.cursor()
            cursor.execute(f'''
                UPDATE {table_name} 
                SET {column_name} = SUBSTR({column_name}, 1, INSTR({column_name}, '.')-1) || '.0.0.0'
                WHERE {column_name} IS NOT NULL 
                AND created_at < ?
                AND {column_name} NOT LIKE '%.0.0.0'
            ''', (cutoff_date.isoformat(),))
            anonymized_count = cursor.rowcount
            
            if anonymized_count > 0:
                logger.info(f"Anonimizados {anonymized_count} IPs em {table_name}")
        
        # Atualiza data da última execução
        cursor.execute('''
            UPDATE retention_rules 
            SET last_execution = ?
            WHERE id = ?
        ''', (datetime.now().isoformat(), rule_id))
