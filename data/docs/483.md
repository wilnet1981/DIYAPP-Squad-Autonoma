```markdown
# Como Usar o Modo Hive para Refatoração Paralela

O **Modo Hive** permite que você refatore fluxos de trabalho de forma paralela, garantindo **estabilidade 100% autônoma**. Ideal para:
- Atualizar múltiplos agentes simultaneamente.
- Testar mudanças sem interromper o fluxo principal.
- Escalar operações sem depender de intervenção manual.

## Pré-requisitos
✅ **Agentes em estado estável**: Verifique o status dos agentes em **Dashboard > Agents**.

## Passo a Passo

### 1. Ativar o Modo Hive
1. Acesse o painel do DIYAPP: [https://app.diyapp.io](https://app.diyapp.io).
2. Vá para **Agents** e selecione o agente desejado.
3. Clique em **Ativar Modo Hive**.
   ![Ativar Modo Hive](https://diyapp.io/assets/hive-mode-button.png)
4. Confirme a ativação.

> ⚠️ **Atenção**: O Modo Hive só pode ser ativado se o agente estiver em estado `active` e sem erros pendentes.

---

### 2. Configurar Refatoração Paralela
1. No painel do Modo Hive, defina:
   - **Escopo da refatoração**: Selecione os fluxos de trabalho a serem atualizados.

2. Clique em **Iniciar Refatoração**.

- Acompanhe o progresso em **Dashboard > Hive Mode**.
- Se um erro ocorrer, o sistema tentará **auto-recuperação**. Caso falhe, você será notificado.

### 4. Finalizar o Modo Hive
1. Após a conclusão, revise os logs em **Dashboard > Logs**.
2. Clique em **Desativar Modo Hive** para retornar ao modo normal.
```

## Breaking Changes
### 1. Remoção do endpoint `/agents/{id}/refactor`
**O que mudou?**

**Como migrar?**

```bash
# Antes (V11)

  -H "Authorization: Bearer YOUR_TOKEN"

# Depois (V12)
curl -X POST https://api.diyapp.io/v12/agents/agent-123/hive-mode \
  -H "Authorization: Bearer YOUR_TOKEN"
```

  "name": "Data Analyst Agent",

```python
from .utils.metrics_processor import fetch_metrics

def run_diagnostics():

    features = np.array([[metrics['cpu_usage'], metrics['memory_usage'], 
                          metrics['disk_io'], metrics['network_latency']]])
    
    prediction = model.predict(features)

        return "System is running optimally"

diagnostics_result = run_diagnostics()
```
