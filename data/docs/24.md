# DIYAPP Evolution - V10 Core - Arquitetura Técnica

## 1. ADR-001: Stack Tecnológica Principal

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V10

### CONTEXTO:
Precisamos definir uma stack tecnológica que suporte:
1. Autonomia completa da squad (zero dependência externa para operação)
2. Estabilidade em produção com auto-recuperação
3. Escalabilidade modular (microsserviços/módulos)
4. Observabilidade nativa
5. Pipeline CI/CD totalmente automatizado

### DECISÃO:
Adotar Node.js 20+ com TypeScript como runtime principal, com arquitetura baseada em módulos autônomos que podem operar como microsserviços ou módulos monolíticos.

### OPÇÕES CONSIDERADAS:
- **Opção A:** Node.js + TypeScript + Express + PostgreSQL
  - Prós: Ecossistema maduro, tipagem forte, performance excelente para I/O
  - Contras: Necessita gerenciamento de processos
- **Opção B:** Python + FastAPI + PostgreSQL
  - Prós: Excelente para IA/ML, sintaxe clara
  - Contras: Performance inferior em I/O intensivo, ecossistema menos consistente
- **Opção C:** Go + Gin + PostgreSQL
  - Prós: Performance excepcional, concorrência nativa
  - Contras: Curva de aprendizado, ecossistema menor

**Opção escolhida:** A - Justificativa: Node.js oferece melhor ecossistema para automação, maior comunidade, e alinha com expertise existente da squad.

### CONSEQUÊNCIAS:
**Positivas:**
- Tipagem estática reduz bugs em produção
- Ecossistema NPM rico para automação
- Suporte nativo a async/await para operações concorrentes

**Negativas:**
- Necessidade de transpilação TypeScript
- Gerenciamento de memória requer atenção

**Riscos:**
- Callback hell se não usar async/await corretamente
- Mitigação: Adoção estrita de async/await e tratamento de erros

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V10 + Especialista Infra

### CONTEXTO:
Pipeline que opera sem intervenção humana, com rollback automático e validação contínua.

### ARQUITETURA DO PIPELINE:
```
Pipeline V10 - Fully Autonomous CI/CD
=====================================

1. PRECIÇÃO (Pre-commit):
   - Husky hooks: lint, type-check, tests unitários
   - Commit message validation (Conventional Commits)
   - Security scanning (npm audit, snyk)

2. INTEGRAÇÃO CONTÍNUA (GitHub Actions):
   - On push to feature/*:
     a. Build all affected modules
     b. Run unit tests (min 80% coverage)
     c. Run integration tests
     d. Security scan (SAST)
     e. Container image build
   
   - On push to main:
     a. All above +
     b. Performance benchmarking
     c. Compatibility testing
     d. Deploy to staging

3. ENTREGA CONTÍNUA (Auto-deploy):
   - Staging validation (24h observability)
   - Canary deployment (5% traffic)
   - Health checks auto-rollback
   - Metrics validation (error rate < 0.1%, p95 latency < 200ms)
   - Full rollout (100% traffic)

4. PÓS-IMPLANTAÇÃO:
   - Auto-monitoring (24/7)
   - Anomaly detection (auto-alerts)
   - Self-healing procedures
   - Performance optimization suggestions
```

```typescript
// packages/core/src/lib/autonomous-engine.ts
export interface AutonomousConfig {
  selfHealing: boolean;
  maxRetries: number;
  healthCheckInterval: number;
  metricsCollection: boolean;
}

export class AutonomousEngine {
  private config: AutonomousConfig;
  private healthMonitor: HealthMonitor;
  private metricsCollector: MetricsCollector;
  private recoverySystem: RecoverySystem;
  
  constructor(config: Partial<AutonomousConfig> = {}) {
    this.config = {
      selfHealing: true,
      maxRetries: 3,
      healthCheckInterval: 30000, // 30 seconds
      metricsCollection: true,
      ...config
    };
    
    this.healthMonitor = new HealthMonitor();
    this.metricsCollector = new MetricsCollector();
    this.recoverySystem = new RecoverySystem();
    
    this.initializeAutonomy();
  }
  
  private initializeAutonomy(): void {
    // Iniciar monitoramento de saúde
    setInterval(() => {
      this.performHealthCheck();
    }, this.config.healthCheckInterval);
    
    // Iniciar coleta de métricas
    if (this.config.metricsCollection) {
      this.startMetricsCollection();
    }
  }
  
  private async performHealthCheck(): Promise<void> {
    const health = await this.healthMonitor.check();
    
    if (health.status !== 'healthy') {
      this.metricsCollector.increment('health_check_failed');
      
      if (this.config.selfHealing) {
        await this.recoverySystem.attemptRecovery(health.issues);
      }
    }
  }
  
  public async executeWithResilience<T>(
    operation: () => Promise<T>,
    context: string
  ): Promise<T> {
    let lastError: Error;
    
    for (let attempt = 1; attempt <= this.config.maxRetries; attempt++) {
      try {
        this.metricsCollector.increment(`${context}_attempt`);
        const result = await operation();
        this.metricsCollector.increment(`${context}_success`);
        return result;
      } catch (error) {
        lastError = error as Error;
        this.metricsCollector.increment(`${context}_failure`);
        
        logger.warn(`Attempt ${attempt} failed for ${context}`, {
          error: error.message,
          attempt
        });
        
        if (attempt < this.config.maxRetries) {
          await this.delay(Math.pow(2, attempt) * 1000); // Exponential backoff
        }
      }
    }
    
    throw new AutonomousError(
      `Operation failed after ${this.config.maxRetries} attempts`,
      'OPERATION_MAX_RETRIES_EXCEEDED',
      { context, lastError: lastError?.message }
    );
  }
  
  private delay(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
  
  private startMetricsCollection(): void {
    // Coleta periódica de métricas
    setInterval(() => {
      const metrics = this.metricsCollector.getMetrics();
      // Enviar para sistema de monitoramento
      this.emitMetrics(metrics);
    }, 60000); // Cada minuto
  }
  
  private emitMetrics(metrics: Record<string, number>): void {
    // Implementação de envio de métricas
    console.log('Metrics emitted:', metrics);
  }
}

export class AutonomousError extends Error {
  constructor(
    message: string,
    public code: string,
    public details?: any
  ) {
    super(message);
    this.name = 'AutonomousError';
  }
}

// packages/core/src/lib/health-monitor.ts
export interface HealthStatus {
  status: 'healthy' | 'degraded' | 'unhealthy';
  issues: HealthIssue[];
  timestamp: Date;
  metrics: SystemMetrics;
}

export interface HealthIssue {
  component: string;
  severity: 'low' | 'medium' | 'high' | 'critical';
  description: string;
  suggestion?: string;
}

export class HealthMonitor {
  private checks: HealthCheck[] = [];
  
  registerCheck(check: HealthCheck): void {
    this.checks.push(check);
  }
  
  async check(): Promise<HealthStatus> {
    const results = await Promise.all(
      this.checks.map(check => check.execute())
    );
    
    const issues = results
      .filter(result => !result.healthy)
      .map(result => result.issue!);
    
    const status = this.determineStatus(issues);
    
    return {
      status,
      issues,
      timestamp: new Date(),
      metrics: this.collectMetrics()
    };
  }
  
  private determineStatus(issues: HealthIssue[]): HealthStatus['status'] {
    if (issues.length === 0) return 'healthy';
    
    const hasCritical = issues.some(i => i.severity === 'critical');
    const hasHigh = issues.some(i => i.severity === 'high');
    
    if (hasCritical) return 'unhealthy';
    if (hasHigh) return 'degraded';
    return 'healthy';
  }
  
  private collectMetrics(): SystemMetrics {
    return {
      memoryUsage: process.memoryUsage(),
      cpuUsage: process.cpuUsage(),
      uptime: process.uptime(),
      activeHandles: (process as any)._getActiveHandles?.()?.length || 0
    };
  }
}

    def format(self, record: logging.LogRecord) -> str:
        log_data: Dict[str, Any] = {
            "timestamp": datetime.utcnow().isoformat() + "Z",
            "level": record.levelname,
            "logger": record.name,
            "message": record.getMessage(),
            "correlation_id": getattr(record, 'correlation_id', 'unknown'),
            "user_id": getattr(record, 'user_id', None),
            "endpoint": getattr(record, 'endpoint', None),
            "duration_ms": getattr(record, 'duration_ms', None),
            "status": getattr(record, 'status', None),
        }
        
        # Mask sensitive data
        if hasattr(record, 'sensitive_data'):
            log_data["sensitive_data"] = "[MASKED]"
        
        # Add exception info if present
        if record.exc_info:
            log_data["exception"] = self.formatException(record.exc_info)
        
        return json.dumps(log_data)

import redis
from typing import Optional, Any
import json
from tenacity import retry, stop_after_attempt, wait_exponential
from circuitbreaker import circuit

from core.config import settings
from core.logging import logger

class RedisCache:
    def __init__(self):
        self.client = redis.Redis.from_url(
            settings.REDIS_URL,
            decode_responses=True,
            socket_timeout=5,
            socket_connect_timeout=5,
            retry_on_timeout=True
        )
    
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=4, max=10)
    )
    @circuit(
        failure_threshold=settings.CIRCUIT_BREAKER_FAILURE_THRESHOLD,
        recovery_timeout=settings.CIRCUIT_BREAKER_RECOVERY_TIMEOUT
    )
    def get(self, key: str) -> Optional[Any]:
        """Get value from cache with circuit breaker and retry"""
        try:
            value = self.client.get(key)
            if value:
                return json.loads(value)
            return None
        except redis.RedisError as e:
            logger.error(f"Redis get error for key {key}: {str(e)}")
            raise
    
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=4, max=10)
    )
    @circuit(
        failure_threshold=settings.CIRCUIT_BREAKER_FAILURE_THRESHOLD,
        recovery_timeout=settings.CIRCUIT_BREAKER_RECOVERY_TIMEOUT
    )
    def set(self, key: str, value: Any, ttl: int = None) -> bool:
        """Set value in cache with circuit breaker and retry"""
        try:
            if ttl is None:
                ttl = settings.CACHE_TTL
            
            serialized = json.dumps(value)
            return self.client.setex(key, ttl, serialized)
        except redis.RedisError as e:
            logger.error(f"Redis set error for key {key}: {str(e)}")
            raise
    
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=4, max=10)
    )
    @circuit(
        failure_threshold=settings.CIRCUIT_BREAKER_FAILURE_THRESHOLD,
        recovery_timeout=settings.CIRCUIT_BREAKER_RECYCLE_TIMEOUT
    )
    def delete(self, key: str) -> bool:
        """Delete key from cache"""
        try:
            return bool(self.client.delete(key))
        except redis.RedisError as e:
            logger.error(f"Redis delete error for key {key}: {str(e)}")
            raise
    
    def generate_cache_key(self, prefix: str, **kwargs) -> str:
        """Generate consistent cache key"""
        sorted_params = sorted(kwargs.items())
        param_str = ":".join(f"{k}={v}" for k, v in sorted_params)
        return f"{prefix}:{param_str}"

import httpx
from typing import Optional, Dict, Any
from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type
from circuitbreaker import circuit
import logging

from core.config import settings
from core.exceptions import ExternalServiceException
from core.logging import logger

class ExternalServiceClient:
    def __init__(self, base_url: str, service_name: str):
        self.base_url = base_url.rstrip('/')
        self.service_name = service_name
        self.timeout = httpx.Timeout(30.0, connect=5.0)
        
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=2, max=10),
        retry=retry_if_exception_type((httpx.TimeoutException, httpx.NetworkError))
    )
    @circuit(
        failure_threshold=settings.CIRCUIT_BREAKER_FAILURE_THRESHOLD,
        recovery_timeout=settings.CIRCUIT_BREAKER_RECOVERY_TIMEOUT
    )
    async def request(
        self,
        method: str,
        endpoint: str,
        headers: Optional[Dict[str, str]] = None,
        json: Optional[Dict[str, Any]] = None,
        params: Optional[Dict[str, Any]] = None
    ) -> Dict[str, Any]:
        """Make HTTP request with retry and circuit breaker"""
        url = f"{self.base_url}/{endpoint.lstrip('/')}"
        
        async with httpx.AsyncClient(timeout=self.timeout) as client:
            try:
                response = await client.request(
                    method=method,
                    url=url,
                    headers=headers,
                    json=json,
                    params=params
                )
                
                response.raise_for_status()
                return response.json()
                
            except httpx.TimeoutException:
                logger.error(f"{self.service_name} timeout: {url}")
                raise ExternalServiceException(
                    service=self.service_name,
                    message="Service timeout",
                    status_code=504
                )
            except httpx.HTTPStatusError as e:
                logger.error(f"{self.service_name} error {e.response.status_code}: {url}")
                raise ExternalServiceException(
                    service=self.service_name,
                    message=f"Service error: {e.response.status_code}",
                    status_code=e.response.status_code
                )
            except httpx.RequestError as e:
                logger.error(f"{self.service_name} connection error: {str(e)}")
                raise ExternalServiceException(
                    service=self.service_name,
                    message="Service unavailable",
                    status_code=503
                )

import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import { immer } from 'zustand/middleware/immer';

// Types
export interface User {
  id: string;
  email: string;
  name: string;
  avatar?: string;
  role: 'user' | 'admin';
}

export interface AppState {
  // User state
  user: User | null;
  isAuthenticated: boolean;
  
  // UI state
  theme: 'light' | 'dark' | 'system';
  sidebarOpen: boolean;
  notifications: Notification[];
  
  // Loading states
  loading: Record<string, boolean>;
  
  // Actions
  setUser: (user: User | null) => void;
  setTheme: (theme: 'light' | 'dark' | 'system') => void;
  toggleSidebar: () => void;
  addNotification: (notification: Omit<Notification, 'id' | 'timestamp'>) => void;
  removeNotification: (id: string) => void;
  setLoading: (key: string, isLoading: boolean) => void;
}

export interface Notification {
  id: string;
  type: 'success' | 'error' | 'warning' | 'info';
  title: string;
  message: string;
  timestamp: number;
  read: boolean;
}

export const useAppStore = create<AppState>()(
  persist(
    immer((set) => ({
      // Initial state
      user: null,
      isAuthenticated: false,
      theme: 'system',
      sidebarOpen: false,
      notifications: [],
      loading: {},
      
      // Actions
      setUser: (user) =>
        set((state) => {
          state.user = user;
          state.isAuthenticated = !!user;
        }),
      
      setTheme: (theme) =>
        set((state) => {
          state.theme = theme;
        }),
      
      toggleSidebar: () =>
        set((state) => {
          state.sidebarOpen = !state.sidebarOpen;
        }),
      
      addNotification: (notification) =>
        set((state) => {
          const newNotification: Notification = {
            ...notification,
            id: crypto.randomUUID(),
            timestamp: Date.now(),
            read: false,
          };
          state.notifications.unshift(newNotification);
          // Keep only last 50 notifications
          if (state.notifications.length > 50) {
            state.notifications = state.notifications.slice(0, 50);
          }
        }),
      
      removeNotification: (id) =>
        set((state) => {
          state.notifications = state.notifications.filter(
            (n) => n.id !== id
          );
        }),
      
      setLoading: (key, isLoading) =>
        set((state) => {
          if (isLoading) {
            state.loading[key] = true;
          } else {
            delete state.loading[key];
          }
        }),
    })),
    {
      name: 'diyapp-store',
      storage: createJSONStorage(() => localStorage),
      partialize: (state) => ({
        user: state.user,
        theme: state.theme,
      }),
    }
  )
);

// Selectors for optimized re-renders
export const useUser = () => useAppStore((state) => state.user);
export const useIsAuthenticated = () => useAppStore((state) => state.isAuthenticated);
export const useTheme = () => useAppStore((state) => state.theme);
export const useSidebarOpen = () => useAppStore((state) => state.sidebarOpen);
export const useNotifications = () => useAppStore((state) => state.notifications);
export const useIsLoading = (key: string) =>
  useAppStore((state) => !!state.loading[key]);

        """Process a single telemetry event"""

            # Validate event
            validated_event = self._validate_event(event_data)
            
            # Enrich with additional data
            enriched_event = self._enrich_event(validated_event)
            
            # Add to batch buffer
            self.batch_buffer.append(enriched_event)
            
            # Check if we should flush
            if (len(self.batch_buffer) >= self.batch_size or 
                time.time() - self.last_flush >= self.batch_timeout):
                await self._flush_batch()

                await self._update_realtime_counters(enriched_event)

# DIYAPP Evolution - V10 Core - Arquitetura Técnica

## ADR-001: Stack Tecnológica e Padrões Arquiteturais V10

**Data:** 2024-01-15

### CONTEXTO:
A DIYAPP evoluiu para uma fábrica de software autônoma com múltiplas squads especializadas. A versão V10 precisa suportar:
1. Execução 100% autônoma com estabilidade de produção

3. Microsserviços/módulos independentes mas coordenados
4. Pipeline CI/CD que garante qualidade sem intervenção humana
5. Observabilidade completa para diagnóstico automático

### DECISÃO:
Adotar arquitetura de microsserviços com core monolítico inteligente, usando:
- **Backend:** Node.js 20+ com TypeScript 5.0+
- **Frontend:** React 18 com Vite + TypeScript
- **Banco de Dados:** PostgreSQL (principal) + Redis (cache/sessão) + SQLite (local/backup)
- **Mensageria:** RabbitMQ para comunicação entre serviços
- **Containerização:** Docker + Docker Compose para desenvolvimento
- **Orquestração:** Kubernetes (produção) / Docker Swarm (staging)
- **Monitoramento:** Prometheus + Grafana + ELK Stack
- **CI/CD:** GitHub Actions com testes automatizados e deploy progressivo

### OPÇÕES CONSIDERADAS:

**Opção A: Microsserviços Puros (12-Factor Apps)**
- Prós: Escalabilidade independente, deploy isolado, tecnologia heterogênea
- Contras: Complexidade operacional, latência de rede, consistência distribuída difícil
- **Rejeitada:** Over-engineering para nosso contexto atual

**Opção B: Monolito Modular (Modulith)**
- Prós: Simplicidade operacional, consistência forte, debug fácil
- Contras: Acoplamento potencial, escalabilidade vertical limitada
- **Rejeitada:** Não suporta bem o Modo Hive de refatoração paralela

**Opção C: Core Monolítico + Serviços Especializados (Escolhida)**
- Prós: Balanceia simplicidade e escalabilidade, suporta Modo Hive, deploy progressivo
- Contras: Requer disciplina de boundaries, mais complexo que monólito puro
- **Justificativa:** Alinha com necessidade de squads autônomas trabalhando em paralelo

### CONSEQUÊNCIAS:
**Positivas:**
- Squads podem desenvolver/testar/deployar módulos independentemente
- Modo Hive suportado via boundaries claros entre módulos
- Observabilidade granular por módulo/serviço
- Rollback seletivo por serviço em caso de falha

**Negativas:**
- Overhead de comunicação entre serviços
- Necessidade de gateway API unificado
- Gerenciamento de transações distribuídas mais complexo

**Riscos:**
1. Degradação de performance por chamadas entre serviços
   - Mitigação: Cache agressivo, padrão CQRS onde aplicável
2. Dificuldade de debug em produção
   - Mitigação: Correlation IDs em todas as requisições, logs centralizados
3. Configuração distribuída complexa
   - Mitigação: Config Server central com atualização em tempo real

# src/shared/resilience/circuit_breaker.py

from enum import Enum
from typing import Callable, Any
import time as ttime
from dataclasses import dataclass
from functools import wraps

class CircuitState(Enum):
    CLOSED = "CLOSED"      # Operação normal
    OPEN = "OPEN"          # Circuito aberto - falhas
    HALF_OPEN = "HALF_OPEN" # Testando recuperação

@dataclass
class CircuitBreakerConfig:
    failure_threshold: int = 5          # Falhas consecutivas para abrir
    reset_timeout: int = 60             # Segundos para tentar reabrir
    half_open_max_calls: int = 3        # Chamadas no estado HALF_OPEN
    excluded_exceptions: tuple = ()     # Exceções que não contam como falha

class CircuitBreaker:
    def __init__(self, name: str, config: CircuitBreakerConfig = None):
        self.name = name
        self.config = config or CircuitBreakerConfig()
        self.state = CircuitState.CLOSED
        self.failure_count = 0
        self.last_failure_time = 0
        self.half_open_success_count = 0
        
    def record_success(self):
        if self.state == CircuitState.HALF_OPEN:
            self.half_open_success_count += 1
            if self.half_open_success_count >= self.config.half_open_max_calls:
                self._close_circuit()
        elif self.state == CircuitState.CLOSED:
            self.failure_count = 0
            
    def record_failure(self):
        self.failure_count += 1
        self.last_failure_time = ttime.time()
        
        if self.state == CircuitState.CLOSED and self.failure_count >= self.config.failure_threshold:
            self._open_circuit()
        elif self.state == CircuitState.HALF_OPEN:
            self._open_circuit()
            
    def _open_circuit(self):
        self.state = CircuitState.OPEN
        self.failure_count = 0
        logger.warning(f"Circuit {self.name} OPENED")
        
    def _close_circuit(self):
        self.state = CircuitState.CLOSED
        self.failure_count = 0
        self.half_open_success_count = 0
        logger.info(f"Circuit {self.name} CLOSED")
        
    def _half_open(self):
        self.state = CircuitState.HALF_OPEN
        self.half_open_success_count = 0
        logger.info(f"Circuit {self.name} HALF_OPEN")
        
    def can_execute(self) -> bool:
        if self.state == CircuitState.OPEN:
            if ttime.time() - self.last_failure_time > self.config.reset_timeout:
                self._half_open()
                return True
            return False
        return True
    
    def execute(self, func: Callable, *args, **kwargs) -> Any:
        if not self.can_execute():
            raise CircuitOpenException(f"Circuit {self.name} is OPEN")
            
        try:
            result = func(*args, **kwargs)
            self.record_success()
            return result
        except Exception as e:
            if not isinstance(e, self.config.excluded_exceptions):
                self.record_failure()
            raise

class CircuitOpenException(Exception):
    pass

def circuit_breaker(name: str, config: CircuitBreakerConfig = None):
    def decorator(func):
        cb = CircuitBreaker(name, config)
        
        @wraps(func)
        def wrapper(*args, **kwargs):
            return cb.execute(func, *args, **kwargs)
            
        wrapper.circuit_breaker = cb
        return wrapper
    return decorator

# src/shared/resilience/retry.py

import random
from typing import Type, Tuple, Callable
from functools import wraps

class RetryConfig:
    def __init__(
        self,
        max_attempts: int = 3,
        base_delay: float = 0.1,
        max_delay: float = 10.0,
        jitter: bool = True,
        retry_on_exceptions: Tuple[Type[Exception], ...] = (Exception,)
    ):
        self.max_attempts = max_attempts
        self.base_delay = base_delay
        self.max_delay = max_delay
        self.jitter = jitter
        self.retry_on_exceptions = retry_on_exceptions

def retry(config: RetryConfig = None):
    config = config or RetryConfig()
    
    def decorator(func: Callable):
        @wraps(func)
        async def async_wrapper(*args, **kwargs):
            last_exception = None
            
            for attempt in range(config.max_attempts):
                try:
                    return await func(*args, **kwargs)
                except config.retry_on_exceptions as e:
                    last_exception = e
                    
                    if attempt == config.max_attempts - 1:
                        break
                        
                    # Backoff exponencial com jitter
                    delay = min(
                        config.base_delay * (2 ** attempt),
                        config.max_delay
                    )
                    
                    if config.jitter:
                        delay = random.uniform(delay * 0.5, delay * 1.5)
                    
                    logger.warning(
                        f"Retry attempt {attempt + 1}/{config.max_attempts} "
                        f"for {func.__name__} after {delay:.2f}s. Error: {str(e)}"
                    )
                    
                    await asyncio.sleep(delay)
            
            raise last_exception
            
        @wraps(func)
        def sync_wrapper(*args, **kwargs):
            last_exception = None
            
            for attempt in range(config.max_attempts):
                try:
                    return func(*args, **kwargs)
                except config.retry_on_exceptions as e:
                    last_exception = e
                    
                    if attempt == config.max_attempts - 1:
                        break
                        
                    delay = min(
                        config.base_delay * (2 ** attempt),
                        config.max_delay
                    )
                    
                    if config.jitter:
                        delay = random.uniform(delay * 0.5, delay * 1.5)
                    
                    logger.warning(
                        f"Retry attempt {attempt + 1}/{config.max_attempts} "
                        f"for {func.__name__} after {delay:.2f}s"
                    )
                    
                    ttime.sleep(delay)
            
            raise last_exception
        
        return async_wrapper if asyncio.iscoroutinefunction(func) else sync_wrapper
    
    return decorator

from typing import Optional, Union
import pickle
import hashlib

class CacheService:
    def __init__(self, redis_client=None, default_ttl: int = 300):
        self.redis = redis_client
        self.default_ttl = default_ttl
        
    def _generate_key(self, prefix: str, *args, **kwargs) -> str:
        """Gera chave de cache consistente"""
        key_data = f"{prefix}:{str(args)}:{str(sorted(kwargs.items()))}"
        return f"cache:{hashlib.md5(key_data.encode()).hexdigest()}"
    
    async def get(self, key: str) -> Optional[Any]:
        if not self.redis:
            return None
            
        try:
            data = await self.redis.get(key)
            if data:
                return pickle.loads(data)
        except Exception as e:
            logger.error(f"Cache get error: {e}")
            return None
            
    async def set(self, key: str, value: Any, ttl: Optional[int] = None):
        if not self.redis:
            return
            
        try:
            ttl = ttl or self.default_ttl
            await self.redis.setex(
                key,
                ttl,
                pickle.dumps(value)
            )
        except Exception as e:
            logger.error(f"Cache set error: {e}")
            
    async def delete(self, key: str):
        if not self.redis:
            return
            
        try:
            await self.redis.delete(key)
        except Exception as e:
            logger.error(f"Cache delete error: {e}")
            
    async def delete_pattern(self, pattern: str):
        if not self.redis:
            return
            
        try:
            keys = await self.redis.keys(pattern)
            if keys:
                await self.redis.delete(*keys)
        except Exception as e:
            logger.error(f"Cache delete pattern error: {e}")

def cache_key(prefix: str, ttl: int = 300):
    def decorator(func):
        @wraps(func)
        async def async_wrapper(*args, **kwargs):
            cache_service = kwargs.get('cache_service') or get_cache_service()
            if not cache_service:
                return await func(*args, **kwargs)
                
            key = cache_service._generate_key(prefix, *args, **kwargs)
            
            # Tentar cache
            cached = await cache_service.get(key)
            if cached is not None:
                logger.debug(f"Cache hit for {key}")
                return cached
                
            # Cache miss - executar função
            result = await func(*args, **kwargs)
            
            # Armazenar em cache
            await cache_service.set(key, result, ttl)
            
            return result
            
        @wraps(func)
        def sync_wrapper(*args, **kwargs):
            cache_service = kwargs.get('cache_service') or get_cache_service()
            if not cache_service:
                return func(*args, **kwargs)
                
            key = cache_service._generate_key(prefix, *args, **kwargs)
            
            # Para sync, precisamos de implementação sync do cache
            # Esta é uma versão simplificada
            return func(*args, **kwargs)
        
        return async_wrapper if asyncio.iscoroutinefunction(func) else sync_wrapper
    
    return decorator

CREATE MATERIALIZED VIEW daily_active_users AS
SELECT 
    metric_date,
    COUNT(DISTINCT user_id) as active_users,
    COUNT(DISTINCT session_id) as total_sessions,
    SUM(CASE WHEN metric_name = 'page_view' THEN metric_value ELSE 0 END) as page_views

WHERE metric_date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY metric_date
WITH DATA;

    metric_date,
    component,
    operation,

    success_rate,
    total_calls,

## Análise de Ameaças (Threat Modeling) - STRIDE

### 1. Ativos Críticos Identificados
- **Dados de Usuário**: Credenciais, tokens de sessão, dados pessoais
- **Sistema de Autenticação**: Endpoints de login, registro, OAuth
- **API Backend**: Endpoints REST/GraphQL com dados sensíveis
- **Banco de Dados**: Armazenamento de dados criptografados
- **Integrações Externas**: Serviços de terceiros (WhatsApp, APIs)
- **Sistema de Arquivos**: Uploads de usuários, logs

### 2. Análise STRIDE por Componente

#### Sistema de Autenticação
- **Spoofing**: Ataques de força bruta, credenciais vazadas
- **Tampering**: Manipulação de tokens JWT, cookies
- **Repudiation**: Falta de logs de autenticação
- **Info Disclosure**: Tokens em logs, headers não seguros
- **DoS**: Rate limiting ausente em endpoints de login
- **Elevation**: Falha na verificação de escopos/roles

#### API Backend
- **Spoofing**: CSRF, falta de validação de origem
- **Tampering**: SQL/NoSQL injection, manipulação de parâmetros
- **Repudiation**: Logs de auditoria insuficientes
- **Info Disclosure**: Erros detalhados em produção
- **DoS**: Falta de rate limiting, queries pesadas
- **Elevation**: IDOR, falhas de autorização

#### Banco de Dados
- **Spoofing**: Credenciais de DB em código
- **Tampering**: Acesso direto ao DB não autorizado
- **Repudiation**: Falta de logs de alterações
- **Info Disclosure**: Dados não criptografados
- **DoS**: Queries maliciosas
- **Elevation**: Privilégios excessivos no DB

      algorithm: this.algorithm
    });

      if (this.blacklist.has(this.getTokenHash(token))) {
        throw new Error('Token revoked');
      }

      const decoded = jwt.verify(token, this.secret, {
        algorithms: [this.algorithm],
        ...options
      });

      this.validateTokenClaims(decoded);

      if (options.audit) {
        await this.auditLogger.log({
          action: 'TOKEN_VERIFIED',
          userId: decoded.sub,
          timestamp: new Date(),
          metadata: { jti: decoded.jti }
        });
      }

      return decoded;
    } catch (error) {

## 3. Serviços de Conformidade

### `src/compliance/services/ComplianceService.js`

```javascript
const complianceConfig = require('../../config/complianceConfig');
const retentionPolicies = require('../../config/retentionPolicies');

class ComplianceService {
  constructor(models) {
    this.models = models;
  }

  // Avaliar necessidade de DPIA
  async assessDPIANecessity(featureData) {
    const highRiskIndicators = [
      'sensitive' in featureData.dataCategories,
      'largeScale' in featureData.scope,
      'systematicMonitoring' in featureData.characteristics,
      'automatedDecisionMaking' in featureData.characteristics,
      'vulnerableSubjects' in featureData.dataSubjects
    ];

    const requiresDPIA = highRiskIndicators.some(indicator => indicator === true);

    return {
      requiresDPIA,
      riskLevel: requiresDPIA ? 'HIGH' : 'LOW',
      indicators: highRiskIndicators.filter(Boolean),
      recommendation: requiresDPIA 
        ? 'Conduzir DPIA antes do desenvolvimento' 
        : 'DPIA não obrigatória'
    };
  }

  // Validar base legal
