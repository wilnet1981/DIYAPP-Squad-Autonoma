# DIYAPP Evolution - V12 Core - Arquitetura de Microsserviços

## 1. ADR-001: Arquitetura de Microsserviços V12

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra + Especialista Backend

### CONTEXTO:
A V11 do DIYAPP atingiu limites de escalabilidade devido à arquitetura monolítica. O sistema precisa:
- Escalar componentes independentemente
- Permitir deploy contínuo sem downtime
- Suportar times distribuídos trabalhando em paralelo
- Manter 100% de disponibilidade com failover automático
- Integrar múltiplos provedores de IA com isolamento de falhas

### DECISÃO:
Adotar arquitetura de microsserviços baseada em eventos com os seguintes princípios:
1. **12-Factor App** para todos os serviços
2. **Comunicação**: gRPC para chamadas síncronas, Apache Kafka para eventos assíncronas
3. **Orquestração**: Kubernetes com service mesh (Istio)
4. **Banco de dados**: Database-per-service com CQRS pattern
5. **Observabilidade**: OpenTelemetry + Prometheus + Grafana
6. **API Gateway**: Kong com rate limiting e autenticação centralizada

### OPÇÕES CONSIDERADAS:
- **Opção A**: Monolito modularizado (atual) - Prós: Simples deploy, debugging fácil. Contras: Não escala, single point of failure.
- **Opção B**: Microsserviços com REST/HTTP - Prós: Simples implementação. Contras: Overhead de serialização, sem streaming nativo.
- **Opção C**: Microsserviços com gRPC + Eventos - Prós: Alta performance, tipagem forte, streaming bidirecional. Contras: Curva de aprendizado, tooling mais complexo.
- **Opção escolhida: C** - Justificativa: Performance crítica para IA, necessidade de streaming para respostas longas, tipagem forte previne erros em runtime.

### CONSEQUÊNCIAS:
**Positivas:**
- Escalabilidade horizontal ilimitada
- Deploy independente por serviço
- Isolamento de falhas (um serviço quebrado não derruba o sistema)
- Polyglot persistence (cada serviço escolhe o banco ideal)

**Negativas:**
- Complexidade operacional aumentada
- Latência de rede entre serviços
- Consistency eventual em alguns cenários
- Debugging distribuído mais complexo

**Riscos:**
- Network partitioning: Mitigado com retries exponenciais + circuit breakers
- Data consistency: Mitigado com sagas pattern e compensação
- Service discovery: Mitigado com service mesh

### 3. Protocol Buffers Definitions

**`src/proto/service/v1/diyapp.proto`**
```proto
syntax = "proto3";

package diyapp.v1;

option go_package = "github.com/diyapp/v12/proto/service/v1";
option java_multiple_files = true;
option java_package = "com.diyapp.v12.grpc";

import "google/protobuf/timestamp.proto";
import "google/api/annotations.proto";

service DIYAppService {
  // Task Management
  rpc CreateTask(CreateTaskRequest) returns (TaskResponse) {
    option (google.api.http) = {
      post: "/v1/tasks"
      body: "*"
    };
  }
  
  rpc GetTask(GetTaskRequest) returns (TaskResponse) {
    option (google.api.http) = {
      get: "/v1/tasks/{task_id}"
    };
  }
  
  rpc StreamTaskUpdates(StreamTaskUpdatesRequest) returns (stream TaskUpdate);
  
  // AI Processing
  rpc ProcessWithAI(ProcessAIRequest) returns (stream AIResponse);
  
  // Health Check
  rpc HealthCheck(HealthCheckRequest) returns (HealthCheckResponse);
}

message CreateTaskRequest {
  string title = 1;
  string description = 2;
  TaskType type = 3;
  map<string, string> metadata = 4;
}

message TaskResponse {
  string task_id = 1;
  string title = 2;
  TaskStatus status = 3;
  google.protobuf.Timestamp created_at = 4;
  google.protobuf.Timestamp updated_at = 5;
  float progress = 6;
  string result_url = 7;
}

message StreamTaskUpdatesRequest {
  string task_id = 1;
}

message TaskUpdate {
  string task_id = 1;
  TaskStatus status = 2;
  float progress = 3;
  string message = 4;
  google.protobuf.Timestamp timestamp = 5;
}

message ProcessAIRequest {
  string task_id = 1;
  string prompt = 2;
  AIModel model = 3;
  map<string, string> parameters = 4;
}

message AIResponse {
  string chunk = 1;
  bool is_final = 2;
  map<string, string> metadata = 3;
}

enum TaskType {
  TASK_TYPE_UNSPECIFIED = 0;
  TASK_TYPE_CODE_GENERATION = 1;
  TASK_TYPE_DOCUMENTATION = 2;
  TASK_TYPE_TEST_GENERATION = 3;
  TASK_TYPE_REFACTORING = 4;
}

enum TaskStatus {
  TASK_STATUS_PENDING = 0;
  TASK_STATUS_PROCESSING = 1;
  TASK_STATUS_COMPLETED = 2;
  TASK_STATUS_FAILED = 3;
  TASK_STATUS_CANCELLED = 4;
}

enum AIModel {
  AI_MODEL_UNSPECIFIED = 0;
  AI_MODEL_GPT4 = 1;
  AI_MODEL_CLAUDE3 = 2;
  AI_MODEL_GEMINI_PRO = 3;
  AI_MODEL_LLAMA3 = 4;
}
```

### 4. Dockerfile Otimizado

**`deployments/docker/Dockerfile`**
```dockerfile
# Build stage
FROM golang:1.21-alpine AS builder

WORKDIR /app

# Install dependencies
RUN apk add --no-cache git ca-certificates tzdata

# Copy go mod files
COPY go.mod go.sum ./
RUN go mod download

# Copy source code
COPY . .

# Build
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo \
    -ldflags="-w -s -X main.version=$(git describe --tags --always)" \
    -o /go/bin/service ./src/cmd/service

# Runtime stage
FROM alpine:3.18

# Install runtime dependencies
RUN apk add --no-cache ca-certificates tzdata && \
    addgroup -S app && adduser -S app -G app

WORKDIR /home/app

# Copy binary and configs
COPY --from=builder /go/bin/service /usr/local/bin/service
COPY --chown=app:app .env.example .env

# Create necessary directories
RUN mkdir -p /home/app/logs /home/app/data && \
    chown -R app:app /home/app

USER app

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD ["/usr/local/bin/service", "health"]

EXPOSE 8080 9090

CMD ["/usr/local/bin/service"]
```

    this.serviceName = serviceName;

      winston.format.timestamp(),

      winston.format.json()
    );

    this.logger = winston.createLogger({
      level: process.env.LOG_LEVEL || 'info',
      format,
      defaultMeta: { service: serviceName },
      transports: [
        new winston.transports.Console(),
        new winston.transports.File({ 
          filename: `logs/${serviceName}-error.log`, 
          level: 'error' 
        }),
        new winston.transports.File({ 
          filename: `logs/${serviceName}-combined.log` 
        })
      ]
    });
  }

import axios, { AxiosError } from 'axios';
import { HealthCheckResult, ServiceInstance } from '../types';
import { StructuredLogger } from '../logger';

export class HealthChecker {
  private logger: StructuredLogger;

  constructor(serviceName: string) {
    this.logger = new StructuredLogger(serviceName);
  }

  public async checkServiceHealth(service: ServiceInstance): Promise<HealthCheckResult> {
    const startTime = Date.now();
    
    try {
      const response = await axios.get(`${service.url}${service.healthEndpoint || '/health'}`, {
        timeout: 5000,
        validateStatus: (status) => status < 500
      });

      const duration = Date.now() - startTime;

      if (response.status === 200 && response.data?.status === 'healthy') {
        return {
          status: 'healthy',
          timestamp: new Date(),
          details: {
            responseTime: duration,
            ...response.data.details
          }
        };
      }

      return {
        status: 'unhealthy',
        timestamp: new Date(),
        details: {
          responseTime: duration,
          statusCode: response.status,
          response: response.data
        }
      };
    } catch (error) {
      const duration = Date.now() - startTime;
      const axiosError = error as AxiosError;

      this.logger.error('Health check failed', {
        correlationId: `health-${service.id}`,
        endpoint: service.healthEndpoint,
        durationMs: duration,
        error: axiosError.message
      });

      return {
        status: 'unhealthy',
        timestamp: new Date(),
        details: {
          responseTime: duration,
          error: axiosError.message,
          code: axiosError.code
        }
      };
    }
  }

  public async checkDependencies(dependencies: ServiceInstance[]): Promise<Record<string, HealthCheckResult>> {
    const results: Record<string, HealthCheckResult> = {};

    await Promise.all(
      dependencies.map(async (dep) => {
        results[dep.name] = await this.checkServiceHealth(dep);
      })
    );

    return results;
  }
}

    fallback?: () => Promise<T>

        correlationId: `cb-${this.serviceName}`,
        endpoint: 'circuit-breaker'
      });

    }

        fn(),
        new Promise<never>((_, reject) =>
          setTimeout(() => reject(new Error('Timeout')), this.timeout)
        )
      ]);

      this.onSuccess();
      return result;
    } catch (error) {

          correlationId: `cb-${this.serviceName}`,

### Análise STRIDE:

#### **S - Spoofing (Falsificação)**
- **Ameaça:** Ataque de replay de tokens OAuth2
- **Controles:** Tokens JWT com expiração curta (15min), nonce em fluxos OIDC, validação de issuer
- **Risco:** ⚠️ MÉDIO - Mitigado com controles implementados

#### **T - Tampering (Manipulação)**
- **Ameaça:** Manipulação de tokens JWT no cliente
- **Controles:** Assinatura HMAC SHA-256, validação de assinatura no backend
- **Risco:** ✅ BAIXO - Tokens assinados e verificados

#### **R - Repudiation (Repúdio)**
- **Ameaça:** Usuário negar ações realizadas
- **Controles:** Logs imutáveis de autenticação, auditoria de operações críticas
- **Risco:** ⚠️ MÉDIO - Requer implementação de sistema de auditoria

#### **I - Information Disclosure (Divulgação)**
- **Ameaça:** Vazamento de tokens via logs
- **Controles:** Mascaramento de tokens em logs, criptografia de PII em repouso
- **Risco:** 🔴 ALTO - Dados sensíveis em trânsito

#### **D - Denial of Service (Negação)**
- **Ameaça:** Ataque de força bruta em endpoints de auth
- **Controles:** Rate limiting por IP e usuário, circuit breakers
- **Risco:** 🔴 ALTO - Endpoint de login exposto

#### **E - Elevation of Privilege (Escalação)**
- **Ameaça:** Token com escopo maior que o necessário
- **Controles:** Principle of least privilege, escopos granulares OAuth2
- **Risco:** ⚠️ MÉDIO - Controle de escopos necessário

---

### Riscos Específicos de LLM/IA:
1. **Prompt Injection:** Usuário injeta comandos no system prompt
2. **Data Exfiltration:** LLM extrai dados sensíveis do contexto
3. **Indirect Injection:** Conteúdo externo contém instruções maliciosas

- Sanitização de inputs de usuário

- Validação de outputs antes de renderização

**src/security/auth/oauth2.js**
```javascript
/**
 * Módulo OAuth2/OIDC Seguro
 * CVSS: Critical (9.3) - Autenticação é superfície crítica
 */

const crypto = require('crypto');
const jwt = require('jsonwebtoken');
const { securityConfig } = require('../../config/security-config');
const rateLimiter = require('./rate-limiter');
const auditLogger = require('../monitoring/audit-logger');

class SecureOAuth2 {
  constructor() {
    this.config = securityConfig.oauth2;
    this.stateStore = new Map(); // Em produção, usar Redis
    this.nonceStore = new Map();
    
    // Inicializar rate limiter para endpoints de auth
    this.authLimiter = rateLimiter.createLimiter({
      windowMs: 15 * 60 * 1000,
      max: securityConfig.rateLimiting.maxRequests.auth,
      keyGenerator: (req) => req.ip || req.connection.remoteAddress
    });
  }
  
  /**
   * Gerar URL de autorização OAuth2 com proteções
   * STRIDE: S - Prevenção de CSRF com state
   */
  generateAuthUrl(req) {
    const state = crypto.randomBytes(32).toString('hex');
    const nonce = crypto.randomBytes(16).toString('hex');
    
    // Armazenar state e nonce com timestamp
    this.stateStore.set(state, {
      nonce,
      timestamp: Date.now(),
      ip: req.ip,
      userAgent: req.headers['user-agent']
    });
    
    // Limpar states antigos (5 minutos)
    this.cleanupOldStates();
    
    const params = new URLSearchParams({
      response_type: 'code',
      client_id: this.config.clientId,
      redirect_uri: this.config.redirectUri,
      scope: this.config.scopes.join(' '),
      state,
      nonce,
      prompt: 'consent', // Sempre pedir consentimento
      access_type: 'offline'
    });
    
    return `${this.config.issuer}/authorize?${params.toString()}`;
  }
  
  /**
   * Processar callback OAuth2 com validações
   * STRIDE: S, T - Validação de state e assinatura
   */
  async handleCallback(req, res) {
    const { code, state, error } = req.query;
    
    // Log de auditoria
    auditLogger.logAuthAttempt({
      ip: req.ip,
      userAgent: req.headers['user-agent'],
      state,
      timestamp: new Date().toISOString()
    });
    
    // Validar state
    if (!state || !this.stateStore.has(state)) {
      auditLogger.logSecurityEvent({
        type: 'INVALID_OAUTH_STATE',
        severity: 'HIGH',
        ip: req.ip,
        details: 'State inválido ou expirado'
      });
      throw new Error('State inválido ou expirado');
    }
    
    const stateData = this.stateStore.get(state);
    this.stateStore.delete(state);
    
    // Verificar timestamp (5 minutos máximo)
    if (Date.now() - stateData.timestamp > 5 * 60 * 1000) {
      auditLogger.logSecurityEvent({
        type: 'EXPIRED_OAUTH_STATE',
        severity: 'MEDIUM',
        ip: req.ip
      });
      throw new Error('Sessão expirada');
    }
    
    if (error) {
      auditLogger.logSecurityEvent({
        type: 'OAUTH_ERROR',
        severity: 'MEDIUM',
        ip: req.ip,
        details: error
      });
      throw new Error(`Erro OAuth: ${error}`);
    }
    
    // Trocar código por token
    const tokens = await this.exchangeCodeForToken(code, stateData.nonce);
    
    // Validar ID Token
    const validatedUser = await this.validateIdToken(tokens.id_token, stateData.nonce);
    
    auditLogger.logAuthSuccess({
      userId: validatedUser.sub,
      ip: req.ip,
      timestamp: new Date().toISOString()
    });
    
    return {
      accessToken: tokens.access_token,
      refreshToken: tokens.refresh_token,
      idToken: tokens.id_token,
      user: validatedUser
    };
  }
  
  /**
   * Trocar código por token com rate limiting
   * STRIDE: D - Proteção contra abuso
   */
  async exchangeCodeForToken(code, nonce) {
    // Aplicar rate limiting
    const limitKey = `token_exchange_${code.substring(0, 10)}`;
    if (!await rateLimiter.checkLimit(limitKey, 3, 60000)) { // 3 tentativas por minuto
      auditLogger.logSecurityEvent({
        type: 'RATE_LIMIT_EXCEEDED',
        severity: 'HIGH',
        action: 'token_exchange'
      });
      throw new Error('Limite de requisições excedido');
    }
    
    const tokenResponse = await fetch(`${this.config.issuer}/token`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/x-www-form-urlencoded',
        'Authorization': `Basic ${Buffer.from(`${this.config.clientId}:${this.config.clientSecret}`).toString('base64')}`
      },
      body: new URLSearchParams({
        grant_type: 'authorization_code',
        code,
        redirect_uri: this.config.redirectUri
      })
    });
    
    if (!tokenResponse.ok) {
      const error = await tokenResponse.text();
      auditLogger.logSecurityEvent({
        type: 'TOKEN_EXCHANGE_FAILED',
        severity: 'HIGH',
        details: error
      });
      throw new Error(`Falha na troca de token: ${error}`);
    }
    
    return await tokenResponse.json();
  }
  
  /**
   * Validar ID Token com nonce
   * STRIDE: S, T - Validação de assinatura e nonce
   */
  async validateIdToken(idToken, expectedNonce) {
    // Obter chaves públicas do issuer
    const jwksResponse = await fetch(`${this.config.issuer}${this.config.token.jwksUri}`);
    const jwks = await jwksResponse.json();
    
    // Decodificar e validar JWT
    const decoded = jwt.decode(idToken, { complete: true });
    
    if (!decoded) {
      throw new Error('Token JWT inválido');
    }
    
    // Encontrar a chave correspondente
    const key = jwks.keys.find(k => k.kid === decoded.header.kid);
    if (!key) {
      throw new Error('Chave pública não encontrada');
    }
    
    // Validar assinatura
    const publicKey = crypto.createPublicKey({
      key: key,
      format: 'jwk'
    });
    
    const verified = jwt.verify(idToken, publicKey, {
      algorithms: [this.config.token.algorithm],
      issuer: this.config.issuer,
      audience: this.config.clientId
    });
    
    // Validar nonce
    if (verified.nonce !== expectedNonce) {
      auditLogger.logSecurityEvent({
        type: 'INVALID_NONCE',
        severity: 'HIGH',
        details: 'Nonce não corresponde'
      });
      throw new Error('Nonce inválido');
    }
    
    // Validar timestamp
    const now = Math.floor(Date.now() / 1000);
    if (verified.exp < now) {
      throw new Error('Token expirado');
    }
    
    return verified;
  }
  
  /**
   * Refresh token com validações
   */
  async refreshToken(refreshToken) {
    const refreshResponse = await fetch(`${this.config.issuer}/token`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/x-www-form-urlencoded'
      },
      body: new URLSearchParams({
        grant_type: 'refresh_token',
        refresh_token: refreshToken,
        client_id: this.config.clientId,
        client_secret: this.config.clientSecret
      })
    });
    
    if (!refreshResponse.ok) {
      auditLogger.logSecurityEvent({
        type: 'REFRESH_TOKEN_FAILED',
        severity: 'MEDIUM'
      });
      throw new Error('Falha no refresh token');
    }
    
    return await refreshResponse.json();
  }
  
  /**
   * Limpar states antigos
   */
  cleanupOldStates() {
    const now = Date.now();
    for (const [state, data] of this.stateStore.entries()) {
      if (now - data.timestamp > 5 * 60 * 1000) { // 5 minutos
        this.stateStore.delete(state);
      }
    }
  }
  
  /**
   * Middleware de autenticação para rotas protegidas
   */
  authenticate() {
    return async (req, res, next) => {
      try {
        const authHeader = req.headers.authorization;
        
        if (!authHeader || !authHeader.startsWith('Bearer ')) {
          return res.status(401).json({ error: 'Token não fornecido' });
        }
        
        const token = authHeader.substring(7);
        
        // Validar token
        const decoded = jwt.verify(token, process.env.JWT_SECRET, {
          algorithms: [securityConfig.jwt.algorithm]
        });
        
        // Adicionar usuário ao request
        req.user = decoded;
        
        // Log de acesso
        auditLogger.logApiAccess({
          userId: decoded.sub,
          endpoint: req.path,
          method: req.method,
          ip: req.ip,
          timestamp: new Date().toISOString()
        });
        
        next();
      } catch (error) {
        auditLogger.logSecurityEvent({
          type: 'INVALID_TOKEN',
          severity: 'MEDIUM',
          ip: req.ip,
          endpoint: req.path
        });
        
        if (error.name === 'TokenExpiredError') {
          return res.status(401).json({ error: 'Token expirado' });
        }
        
        return res.status(401).json({ error: 'Token inválido' });
      }
    };
  }
}

module.exports = new SecureOAuth2();
```

```markdown
# DIYAPP V12 - Visão Geral da Arquitetura

**Data de criação:** 2024-01-15  
**Última atualização:** 2024-01-15  
**Owner:** Tech Lead  
**Status:** Ativo

## 1. Visão Geral

A arquitetura V12 do DIYAPP evoluiu para um sistema de agentes autônomos baseado em microserviços com orquestração de IA. O sistema é projetado para operação 100% autônoma com capacidade de auto-recuperação.

## 2. Princípios Arquiteturais

### 2.1. Autonomia Total
- Cada agente opera independentemente
- Decisões descentralizadas com consenso distribuído
- Auto-recuperação de falhas sem intervenção humana

### 2.2. Resiliência
- Circuit breakers em todas as comunicações externas
- Retry com backoff exponencial
- Fallbacks degradados elegantes

### 2.3. Observabilidade
- Métricas em tempo real para todos os agentes
- Tracing distribuído com OpenTelemetry
- Logs estruturados com contexto

## 3. Componentes Principais

### 3.1. Orchestrator Service
- **Função:** Coordena a execução de agentes
- **Tecnologia:** Node.js + TypeScript
- **Escalabilidade:** Horizontal com auto-scaling
- **Persistência:** Redis para estado distribuído

### 3.2. Agent Pool
- **Dev Agent:** Desenvolvimento e refatoração de código
- **QA Agent:** Testes automatizados e validação
- **SRE Agent:** Monitoramento e operações
- **Tech Writer Agent:** Documentação automática

### 3.3. Message Bus
- **Implementação:** Apache Kafka
- **Tópicos:** agent-commands, agent-responses, system-events
- **Retenção:** 7 dias para debugging

### 3.4. Knowledge Base
- **Armazenamento:** PostgreSQL + Vector embeddings
- **Conteúdo:** Decisões arquiteturais, padrões, incidentes
- **Acesso:** REST API com autenticação JWT

## 4. Fluxo de Dados

```
Usuário → API Gateway → Orchestrator → Message Bus → Agent → Resultado → Knowledge Base
```

## 5. Decisões de Design

### 5.1. Comunicação Assíncrona
**Problema:** Agentes bloqueando uns aos outros  
**Solução:** Event-driven architecture com filas  
**Resultado:** Maior resiliência e escalabilidade

### 5.2. Estado Distribuído
**Problema:** Single point of failure  
**Solução:** Redis Cluster com replicação  
**Resultado:** 99.99% disponibilidade

### 5.3. Auto-descoberta
**Problema:** Configuração manual de agentes  
**Solução:** Service discovery com Consul  
**Resultado:** Deploy zero-touch de novos agentes

## 6. Métricas de Saúde

| Métrica | Alvo | Alerta |
|---------|------|--------|
| Uptime | 99.95% | < 99.9% |
| Latência p95 | < 200ms | > 500ms |
| Taxa de erro | < 0.1% | > 1% |
| Uso de CPU | < 70% | > 85% |
| Uso de memória | < 80% | > 90% |

## 7. Dependências Externas

- **GitHub API:** Para operações de repositório
- **OpenAI API:** Para capacidades de IA
- **AWS S3:** Para armazenamento de artefatos
- **SendGrid:** Para notificações

## 8. Considerações de Segurança

- Todos os agentes rodam com permissões mínimas
- Comunicação entre serviços com mTLS
- Secrets gerenciados pelo HashiCorp Vault
- Auditoria de todas as operações

## 9. Plano de Evolução

### Fase 1 (Atual): Estabilidade básica
- Auto-recuperação de agentes individuais
- Monitoramento proativo

### Fase 2 (Q1 2024): Inteligência coletiva
- Agentes aprendendo com incidentes passados
- Otimização automática de recursos
```

## 1. Visão Geral

## 3. Procedimento de Resposta

### Fase 1: Identificação (0-5 minutos)

```bash
./scripts/tail-logs.sh --service=all --minutes=5

# Verificar métricas
curl -H "Authorization: Bearer $TOKEN" \

```

#### Passo 2: Classificar severidade
- **P1:** Sistema completamente inacessível
- **P2:** Funcionalidade crítica degradada
- **P3:** Funcionalidade não-crítica afetada
- **P4:** Problema menor, sem impacto direto

#### Passo 3: Criar canal de comunicação
- Slack: #incident-response

```bash
  -H "Authorization: Bearer $TOKEN" \
  https://orchestrator.diyapp.com/api/circuit-breaker/enable \
  -d '{"service": "failing-service"}'
```

#### Passo 5: Implementar mitigação
**Cenário A: Alto uso de CPU**
```bash
# Escalar horizontalmente
./scripts/scale-service.sh --service=high-cpu --count=+2

# Reduzir carga
./scripts/reduce-traffic.sh --percentage=50
```

**Cenário B: Vazamento de memória**
```bash
# Reiniciar instâncias afetadas
./scripts/restart-service.sh --service=leaking-service --instances=affected

# Coletar heap dump
./scripts/collect-heapdump.sh --service=leaking-service
```

**Cenário C: Erros de banco de dados**
```bash
# Verificar conexões
./scripts/check-db-connections.sh

# Failover para réplica
./scripts/failover-database.sh --primary-to=replica-2
```

#### Passo 6: Corrigir a causa raiz
- Analisar logs detalhados
- Revisar mudanças recentes (último deploy)
- Testar hipóteses em staging

#### Passo 7: Implementar correção

```bash
# Reverter deploy se necessário

./scripts/deploy-hotfix.sh
```

```bash
./scripts/run-smoke-tests.sh

# Verificar métricas
./scripts/verify-recovery.sh --duration=10m
```

#### Passo 9: Restaurar operação normal

```bash
  -H "Authorization: Bearer $TOKEN" \
  https://orchestrator.diyapp.com/api/circuit-breaker/disable \
  -d '{"service": "failing-service"}'

# Normalizar escala
./scripts/scale-service.sh --service=all --count=normal
```
