# DIYAPP Evolution - V13 Core - Arquitetura Técnica

## 1. DECISÕES ARQUITETURAIS (ADRs)

### ADR-001: Arquitetura de Microsserviços com Containers
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead, Especialista Infra

**CONTEXTO:**
DIYAPP V13 precisa escalar para 100k usuários ativos com 99.9% uptime. A arquitetura monolítica atual não suporta deploy independente de componentes e cria acoplamento técnico.

**DECISÃO:**
Adotar arquitetura de microsserviços baseada em containers Docker, com:
- 6 serviços independentes (API Gateway, Auth Service, User Service, Content Service, Notification Service, Analytics Service)
- Comunicação via HTTP/REST para síncrono e RabbitMQ para assíncrono
- Banco de dados por serviço (pattern Database-per-Service)
- Service Discovery via Consul
- API Gateway como único ponto de entrada

**OPÇÕES CONSIDERADAS:**
- **Opção A:** Monólito com escalamento vertical - Prós: Simplicidade. Contras: Limite de escalabilidade, deploy acoplado.
- **Opção B:** Microsserviços com Kubernetes - Prós: Escalabilidade automática. Contras: Complexidade operacional alta.
- **Opção C:** Microsserviços com Docker Compose + Swarm - Prós: Balanceamento entre simplicidade e escalabilidade. Contras: Menos automático que K8s.

**Opção escolhida:** C - Justificativa: Time atual tem expertise em Docker, necessidade de maturidade gradual, custo operacional controlado.

**CONSEQUÊNCIAS:**
- Positivas: Deploy independente, escalabilidade horizontal, resiliência por isolamento
- Negativas: Complexidade distribuída, necessidade de observabilidade robusta
- Riscos: Latência em chamadas entre serviços, consistência eventual

**REVISÃO:** 2024-04-15

### ADR-002: Stack Tecnológica Principal
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead, Dev Backend, Dev Frontend

**CONTEXTO:**
Necessidade de padronização tecnológica para garantir consistência, manutenibilidade e contratação de talentos.

**DECISÃO:**
- **Backend:** Node.js 18+ com TypeScript, Express.js, Prisma ORM
- **Frontend:** React 18 com TypeScript, Vite, TailwindCSS
- **Banco de Dados:** PostgreSQL (dados relacionais), Redis (cache), MongoDB (dados não estruturados)
- **Message Queue:** RabbitMQ com exchanges topic-based
- **Cache:** Redis com estratégia Cache-Aside
- **Search:** Elasticsearch para busca full-text

**OPÇÕES CONSIDERADAS:**
- **Backend:** NestJS vs Express - Express escolhido por simplicidade e controle
- **ORM:** Prisma vs TypeORM - Prisma escolhido por type safety e migrações
- **Frontend:** Next.js vs React + Vite - Vite escolhido por build mais rápido e flexibilidade

**CONSEQUÊNCIAS:**
- Positivas: Stack moderna, ampla comunidade, type safety
- Negativas: Curva de aprendizado para Prisma, necessidade de múltiplos bancos

### ADR-003: Estratégia de Observabilidade
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead, Especialista Infra

**DECISÃO:**
Implementar observabilidade em três pilares:
1. **Logs:** Structured logging com Winston + Elastic Stack (ELK)
2. **Métricas:** Prometheus + Grafana para monitoramento
3. **Tracing:** Jaeger para distributed tracing

Cada serviço expõe endpoint `/metrics` para Prometheus e envia traces para Jaeger.

FROM node:18-alpine AS builder

RUN npm run build

FROM node:18-alpine

    adduser -S nodejs -u 1001

COPY --from=builder --chown=nodejs:nodejs /app/node_modules ./node_modules

COPY --from=builder --chown=nodejs:nodejs /app/package*.json ./

EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node healthcheck.js

CMD ["node", "dist/index.js"]

## 4. CI/CD PIPELINE (GitHub Actions)

### CI Pipeline
```yaml
# .github/workflows/ci.yml
name: CI Pipeline

on:
  pull_request:
    branches: [main, develop]
  push:
    branches: [main, develop]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        service: [api-gateway, auth-service, user-service, content-service, notification-service, analytics-service]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
        cache-dependency-path: services/${{ matrix.service }}/package-lock.json
    
    - name: Install dependencies
      working-directory: services/${{ matrix.service }}
      run: npm ci
    
    - name: Lint
      working-directory: services/${{ matrix.service }}
      run: npm run lint
    
    - name: Type check
      working-directory: services/${{ matrix.service }}
      run: npm run type-check
    
    - name: Run tests
      working-directory: services/${{ matrix.service }}
      run: npm test -- --coverage
      env:
        DATABASE_URL: postgresql://test:test@localhost:5432/test
        NODE_ENV: test
    
    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        file: services/${{ matrix.service }}/coverage/lcov.info
        flags: ${{ matrix.service }}

  security-scan:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    
    - name: Run Snyk to check for vulnerabilities
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
    
    - name: Run Trivy vulnerability scanner
      uses: aquasecurity/trivy-action@master
      with:
        scan-type: 'fs'
        scan-ref: '.'
        format: 'sarif'
        output: 'trivy-results.sarif'
    
    - name: Upload Trivy scan results to GitHub Security tab
      uses: github/codeql-action/upload-sarif@v2
      with:
        sarif_file: 'trivy-results.sarif'

  build-containers:
    runs-on: ubuntu-latest
    needs: [lint-and-test, security-scan]
    strategy:
      matrix:
        service: [api-gateway, auth-service, user-service, content-service, notification-service, analytics-service]
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v2
    
    - name: Build Docker image
      uses: docker/build-push-action@v4
      with:
        context: ./services/${{ matrix.service }}
        push: false
        tags: diyapp/${{ matrix.service }}:pr-${{ github.event.pull_request.number }}
        cache-from: type=gha
        cache-to: type=gha,mode=max
```

### CD Pipeline
```yaml
# .github/workflows/cd.yml
name: CD Pipeline

on:
  push:
    branches: [main]
    tags: ['v*']

jobs:
  deploy-staging:
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    environment: staging
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Configure AWS credentials
      uses: aws-actions/configure-aws-credentials@v1
      with:
        aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
        aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
        aws-region: us-east-1
    
    - name: Login to Amazon ECR
      id: login-ecr
      uses: aws-actions/amazon-ecr-login@v1
    
    - name: Build and push to ECR
      env:
        ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
        ECR_REPOSITORY: diyapp
      run: |
        for service in api-gateway auth-service user-service content-service notification-service analytics-service; do
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$service-${{ github.sha }} ./services/$service
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:$service-${{ github.sha }}
        done
    
    - name: Deploy to ECS
      run: |
        aws ecs update-service \
          --cluster diyapp-staging \
          --service ${{ matrix.service }} \
          --force-new-deployment \
          --region us-east-1

  deploy-production:
    runs-on: ubuntu-latest
    if: startsWith(github.ref, 'refs/tags/v')
    needs: deploy-staging
    environment: production
    
    steps:
    - name: Promote to production
      run: |
        # Implementar blue-green deployment ou canary release
        ./scripts/deploy-production.sh ${{ github.ref_name }}
```

// shared/libs/logger.ts
import winston from 'winston';
import { ElasticsearchTransport } from 'winston-elasticsearch';

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.errors({ stack: true }),
    winston.format.json()
  ),
  defaultMeta: { service: process.env.SERVICE_NAME },
  transports: [
    new winston.transports.Console({
      format: winston.format.simple()
    }),
    new ElasticsearchTransport({
      level: 'info',
      clientOpts: { node: process.env.ELASTICSEARCH_URL },
      index: 'diyapp-logs'
    })
  ]
});

export const requestLogger = (req: Request, res: Response, next: NextFunction) => {
  const start = Date.now();
  
  res.on('finish', () => {
    const duration = Date.now() - start;
    logger.info('HTTP Request', {
      method: req.method,
      url: req.url,
      statusCode: res.statusCode,
      duration,
      userAgent: req.get('User-Agent'),
      ip: req.ip
    });
  });
  
  next();
};

// src/components/ui/button.tsx
import * as React from 'react'
import { cn } from '@/lib/utils'
import { Loader2 } from 'lucide-react'

export interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'outline' | 'ghost' | 'danger'
  size?: 'sm' | 'md' | 'lg'
  isLoading?: boolean
  leftIcon?: React.ReactNode
  rightIcon?: React.ReactNode
}

const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  (
    {
      className,
      variant = 'primary',
      size = 'md',
      isLoading = false,
      leftIcon,
      rightIcon,
      children,
      disabled,
      ...props
    },
    ref
  ) => {
    const baseStyles = 'inline-flex items-center justify-center font-medium rounded-DEFAULT transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary-600 disabled:opacity-50 disabled:pointer-events-none'
    
    const variants = {
      primary: 'bg-primary-600 text-white hover:bg-primary-700 active:bg-primary-800',
      secondary: 'bg-neutral-100 text-neutral-900 hover:bg-neutral-200 active:bg-neutral-300',
      outline: 'border border-neutral-300 bg-transparent hover:bg-neutral-50 active:bg-neutral-100',
      ghost: 'bg-transparent hover:bg-neutral-100 active:bg-neutral-200',
      danger: 'bg-error-500 text-white hover:bg-error-600 active:bg-error-700',
    }

    const sizes = {
      sm: 'h-8 px-3 text-sm',
      md: 'h-10 px-4 text-base',
      lg: 'h-12 px-6 text-lg',
    }

    return (
      <button
        className={cn(
          baseStyles,
          variants[variant],
          sizes[size],
          className
        )}
        ref={ref}
        disabled={disabled || isLoading}
        aria-busy={isLoading}
        {...props}
      >
        {isLoading && (
          <Loader2 className="mr-2 h-4 w-4 animate-spin" aria-hidden="true" />
        )}
        {!isLoading && leftIcon && (
          <span className="mr-2" aria-hidden="true">{leftIcon}</span>
        )}
        {children}
        {!isLoading && rightIcon && (
          <span className="ml-2" aria-hidden="true">{rightIcon}</span>
        )}
      </button>
    )
  }
)

export { Button }

    """Hash a password"""

def create_access_token(
    data: Dict[str, Any],
    expires_delta: Optional[timedelta] = None,
    secret_key: str = None,
    algorithm: str = None
) -> str:
    """Create JWT access token"""
    settings = get_settings()
    
    to_encode = data.copy()
    if expires_delta:
        expire = datetime.utcnow() + expires_delta
    else:
        expire = datetime.utcnow() + timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    
    to_encode.update({"exp": expire, "type": "access"})
    
    encoded_jwt = jwt.encode(
        to_encode,
        secret_key or settings.SECRET_KEY,
        algorithm=algorithm or settings.ALGORITHM
    )
    return encoded_jwt

def create_refresh_token(
    data: Dict[str, Any],
    expires_delta: Optional[timedelta] = None
) -> str:
    """Create JWT refresh token"""
    settings = get_settings()
    
    to_encode = data.copy()
    if expires_delta:
        expire = datetime.utcnow() + expires_delta
    else:

def mask_sensitive_data(data: str) -> str:
    """Mask sensitive data for logging"""
    if not data or len(data) < 4:
        return "***"
    
    if "@" in data:  # Email
        parts = data.split("@")
        if len(parts[0]) > 2:
            masked = parts[0][:2] + "*" * (len(parts[0]) - 2)
            return f"{masked}@{parts[1]}"
    
    # Generic masking
    return data[:2] + "*" * (len(data) - 4) + data[-2:]

class CircuitBreaker:
    """Circuit breaker pattern for Redis"""
    
    def __init__(self, failure_threshold=5, recovery_timeout=30):
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.failure_count = 0
        self.last_failure_time = 0
        self.state = "CLOSED"  # CLOSED, OPEN, HALF_OPEN
    
    def record_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        
        if self.failure_count >= self.failure_threshold:
            self.state = "OPEN"
            logger.error("Circuit breaker OPEN for Redis")
    
    def record_success(self):
        self.failure_count = 0
        self.state = "CLOSED"
    
    def can_execute(self) -> bool:
        if self.state == "CLOSED":
            return True
        
        if self.state == "OPEN":
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = "HALF_OPEN"
                return True
            return False
        
        return True  # HALF_OPEN

class RedisCache:
    """Redis cache with circuit breaker and retry logic"""
    
    def __init__(self):
        self.circuit_breaker = CircuitBreaker()

                socket_connect_timeout=settings.REDIS_TIMEOUT / 1000,

                max_connections=20

            self.circuit_breaker.record_success()

BEGIN
    -- Delete existing metrics for the date

    -- 1. Daily Active Users
    INSERT INTO daily_aggregates (date, metric_name, metric_value)

        p_date,
        'daily_active_users',

    AND event_type = 'session';

    INSERT INTO daily_aggregates (date, metric_name, metric_value)
    SELECT 
        p_date,
        'new_users',
        COUNT(*)
    FROM users

    INSERT INTO daily_aggregates (date, metric_name, metric_value, dimensions)
    SELECT 
        p_date,
        'llm_cost_by_provider',
        SUM(cost_usd),
        jsonb_build_object('provider', provider)
    FROM llm_calls
    WHERE DATE(created_at) = p_date
    GROUP BY provider;
    
    -- 4. Feature Usage
    INSERT INTO daily_aggregates (date, metric_name, metric_value, dimensions)
    SELECT 
        p_date,
        'feature_usage',
        COUNT(*),
        jsonb_build_object('feature_type', feature_type)
    FROM features f
    WHERE DATE(last_used_at) = p_date
    AND last_used_at IS NOT NULL
    GROUP BY feature_type;
    
    -- 5. Project Creation
    INSERT INTO daily_aggregates (date, metric_name, metric_value)
    SELECT 
        p_date,
        'projects_created',
        COUNT(*)
    FROM projects
    WHERE DATE(created_at) = p_date;
END;
