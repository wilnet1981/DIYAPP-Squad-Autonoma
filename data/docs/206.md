## ADR-001: Arquitetura de Microsserviços V12

```markdown
# ADR-001: Arquitetura de Microsserviços para V12

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Arquitetura Squad

## CONTEXTO:
DIYAPP evoluiu para uma plataforma complexa com múltiplos domínios:
- Gerenciamento de usuários e autenticação
- Processamento de pagamentos
- Sistema de notificações
- Analytics e relatórios
- Integração com APIs externas

A arquitetura monolítica atual apresenta:
- Acoplamento alto entre domínios
- Dificuldade de deploy independente
- Escalabilidade limitada
- Single point of failure

## DECISÃO:
Adotar arquitetura de microsserviços com os seguintes princípios:
1. Cada domínio de negócio vira um microsserviço independente
2. Comunicação via eventos assíncronos (Kafka)
3. API Gateway como ponto único de entrada
4. Service Discovery com Consul
5. Circuit Breaker para resiliência

## OPÇÕES CONSIDERADAS:

### Opção A: Arquitetura Hexagonal (Ports & Adapters)
- Prós: Desacoplamento total, testabilidade, flexibilidade
- Contras: Complexidade inicial maior, curva de aprendizado

### Opção B: Clean Architecture
- Prós: Separação clara de responsabilidades, independência de frameworks
- Contras: Overhead de boilerplate

### Opção C: CQRS + Event Sourcing
- Prós: Escalabilidade extrema, consistência eventual garantida
- Contras: Complexidade operacional, difícil debugging

**Opção escolhida:** A (Arquitetura Hexagonal) com elementos de CQRS para serviços críticos

## CONSEQUÊNCIAS:

### Positivas:
- Deploy independente por serviço
- Escalabilidade granular
- Stack tecnológica heterogênea
- Falhas isoladas por serviço
- Evolução independente

### Negativas:
- Complexidade distribuída
- Latência de rede
- Consistência eventual
- Monitoramento mais complexo

### Riscos:
1. **Network partitioning:** Mitigação com retry policies e circuit breakers
2. **Data consistency:** Mitigação com SAGA pattern e compensação
3. **Debugging distribuído:** Mitigação com distributed tracing (Jaeger/Tempo)

## REVISÃO: 2024-04-15
```

## Engineering Standards

```markdown
# Engineering Standards - DIYAPP V12

## 1. Padrões de Código

### 1.1 Linguagens Principais
- Backend: Node.js 18+ (TypeScript)
- Frontend: React 18+ (TypeScript)
- Mobile: React Native 0.72+

### 1.2 Estrutura de Projeto
```
src/
├── domain/          # Entidades e regras de negócio
├── application/     # Casos de uso
├── infrastructure/  # Implementações concretas
├── interfaces/      # Controllers, GraphQL resolvers
└── shared/         # Utilitários compartilhados
```

### 1.3 Convenções de Nomenclatura
- Classes: PascalCase (UserService)
- Funções/métodos: camelCase (getUserById)
- Variáveis: camelCase (userRepository)
- Constantes: UPPER_SNAKE_CASE (MAX_RETRY_COUNT)
- Interfaces: I + PascalCase (IUserRepository)
- Types: PascalCase + "Type" (UserType)

### 1.4 Commits
- Usar Conventional Commits
- Exemplos:
  - feat: adiciona autenticação JWT
  - fix: corrige vazamento de memória no cache
  - docs: atualiza README
  - test: adiciona testes para paginação
  - refactor: extrai lógica de validação
  - chore: atualiza dependências

## 2. Padrões de API

### 2.1 REST
- Versionamento: `/api/v1/resource`
- Métodos HTTP semânticos
- Paginação: `?page=1&limit=20`
- Filtros: `?status=active&created_after=2024-01-01`
- Ordenação: `?sort=-created_at,name`

### 2.2 GraphQL
- Schema-first development
- Resolvers organizados por domínio
- N+1 query prevention (DataLoader)
- Rate limiting por query complexity

### 2.3 Eventos
- Formato: CloudEvents 1.0
- Tópicos: `domain.entity.action`
- Exemplo: `payment.transaction.completed`

## 3. Segurança

### 3.1 Autenticação
- JWT com RS256
- Refresh tokens rotativos
- OAuth2 para integrações

### 3.2 Autorização
- RBAC (Role-Based Access Control)
- Policies por recurso
- Attribute-Based Access Control para casos complexos

### 3.3 Dados Sensíveis
- Nunca logar tokens, senhas ou PII
- Criptografia em repouso (AES-256)
- Criptografia em trânsito (TLS 1.3)

## 4. Banco de Dados

### 4.1 Escolha por Serviço
- Serviços transacionais: PostgreSQL
- Cache: Redis
- Search: Elasticsearch
- Analytics: ClickHouse
- Eventos: Kafka

### 4.2 Migrações
- Versionadas e reversíveis
- Testadas em staging antes de produção
- Rollback automático em caso de falha

## 5. Testes

### 5.1 Cobertura Mínima
- Unitários: 80%
- Integração: 70%
- E2E: 50%

### 5.2 Pirâmide de Testes
```
      [E2E] - 10%
     /      \
[Integração] - 20%
     \
      [Unitários] - 70%
```

## 6. CI/CD

### 6.1 Pipeline
1. Lint e formatação
2. Testes unitários
3. Build da imagem Docker
4. Testes de integração
5. Scan de segurança
6. Deploy para staging
7. Testes E2E
8. Deploy para produção (canary)

### 6.2 Branch Strategy
- main: produção
- staging: pré-produção
- feature/*: novas funcionalidades
- hotfix/*: correções críticas

## 7. Observabilidade

### 7.1 Métricas
- Latência por percentil (p50, p95, p99)
- Taxa de erro
- Taxa de requisições
- Uso de recursos (CPU, memória)

### 7.2 Logs
- Structured logging (JSON)
- Níveis: error, warn, info, debug
- Correlation IDs para tracing

### 7.3 Tracing
- W3C Trace Context
- Sampling adaptativo
- Span attributes para business context
```

**templates/microservice-template/src/infrastructure/database/database.module.ts**
```typescript
import { Module } from '@nestjs/common';
import { ConfigModule, ConfigService } from '@nestjs/config';
import { TypeOrmModule } from '@nestjs/typeorm';
import { SnakeNamingStrategy } from 'typeorm-naming-strategies';

@Module({
  imports: [
    TypeOrmModule.forRootAsync({
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: (configService: ConfigService) => ({
        type: 'postgres',
        host: configService.get('DB_HOST'),
        port: configService.get('DB_PORT'),
        username: configService.get('DB_USERNAME'),
        password: configService.get('DB_PASSWORD'),
        database: configService.get('DB_DATABASE'),
        entities: [__dirname + '/../**/*.entity{.ts,.js}'],
        migrations: [__dirname + '/migrations/*{.ts,.js}'],
        namingStrategy: new SnakeNamingStrategy(),
        synchronize: configService.get('NODE_ENV') === 'development',
        logging: configService.get('NODE_ENV') === 'development',
        extra: {
          max: 20,
          connectionTimeoutMillis: 5000,
        },
      }),
    }),
  ],
})
export class DatabaseModule {}
```

**templates/microservice-template/src/domain/entities/base.entity.ts**
```typescript
import {
  PrimaryGeneratedColumn,
  CreateDateColumn,
  UpdateDateColumn,
  DeleteDateColumn,
  VersionColumn,
} from 'typeorm';

export abstract class BaseEntity {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @CreateDateColumn({ type: 'timestamptz' })
  createdAt: Date;

  @UpdateDateColumn({ type: 'timestamptz' })
  updatedAt: Date;

  @DeleteDateColumn({ type: 'timestamptz', nullable: true })
  deletedAt?: Date;

  @VersionColumn()
  version: number;
}
```

**templates/microservice-template/src/application/services/base.service.ts**
```typescript
import { Injectable, Logger } from '@nestjs/common';
import { EventEmitter2 } from '@nestjs/event-emitter';

@Injectable()
export abstract class BaseService {
  protected readonly logger = new Logger(this.constructor.name);

  constructor(protected readonly eventEmitter: EventEmitter2) {}

  protected emitEvent(eventName: string, payload: any): void {
    this.eventEmitter.emit(eventName, payload);
    this.logger.debug(`Event emitted: ${eventName}`, { payload });
  }

  protected handleError(error: Error, context: string): never {
    this.logger.error(`Error in ${context}: ${error.message}`, error.stack);
    throw error;
  }
}
```

```dockerfile
# Build stage
FROM node:18-alpine AS builder

WORKDIR /app

# Install dependencies
COPY package*.json ./
RUN npm ci --only=production

# Copy source
COPY . .

# Build TypeScript
RUN npm run build

# Production stage
FROM node:18-alpine

WORKDIR /app

# Install runtime dependencies
RUN apk add --no-cache tini

# Copy built application
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/package.json ./

# Create non-root user
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001 && \
    chown -R nodejs:nodejs /app

USER nodejs

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:${PORT:-3000}/health', (r) => {process.exit(r.statusCode === 200 ? 0 : 1)})"

# Entry point
ENTRYPOINT ["/sbin/tini", "--"]
CMD ["node", "dist/main.js"]
```

    """Add correlation ID to every request"""
    request_id = str(uuid.uuid4())
    request.state.request_id = request_id
    
    # Start timer for latency metrics
    start_time = time.time()
    
    # Structured logging
    logger.info(
        "request_start",
        request_id=request_id,
        method=request.method,
        url=str(request.url),
        client_ip=request.client.host if request.client else None
    )
    
    try:
        response = await call_next(request)
        
        # Calculate duration
        duration = time.time() - start_time
        
        # Add request ID to response headers
        response.headers["X-Request-ID"] = request_id
        response.headers["X-Response-Time"] = f"{duration:.3f}"
        
        # Log completion
        logger.info(
            "request_complete",
            request_id=request_id,
            method=request.method,
            url=str(request.url),
            status_code=response.status_code,
            duration_ms=round(duration * 1000, 2)
        )
        
        # Record metrics
        if metrics:
            metrics.record_request(
                endpoint=request.url.path,
                method=request.method,
                status_code=response.status_code,
                duration=duration
            )
        
        return response
        
    except Exception as e:
        duration = time.time() - start_time
        logger.error(
            "request_error",
            request_id=request_id,
            method=request.method,
            url=str(request.url),
            error=str(e),
            duration_ms=round(duration * 1000, 2),
            exc_info=True
        )
        
        if metrics:
            metrics.record_error(
                endpoint=request.url.path,
                method=request.method,
                error_type=type(e).__name__
            )
        
        # Return user-friendly error without stack trace
        return JSONResponse(
            status_code=500,
            content={
                "error": "Internal server error",
                "request_id": request_id,
                "message": "An unexpected error occurred"
            }

"""
API Gateway with routing, rate limiting, and circuit breaking
"""

import asyncio
from typing import Dict, List, Optional, Any
from dataclasses import dataclass
from urllib.parse import urljoin

import httpx
from fastapi import Request, HTTPException
import structlog

logger = structlog.get_logger(__name__)

@dataclass
class ServiceConfig:
    """Service configuration for API Gateway"""
    name: str
    base_url: str
    endpoints: List[str]
    circuit_breaker_config: Dict[str, Any]
    timeout: float = 5.0
    retry_attempts: int = 3
    retry_backoff: float = 0.1

class APIGateway:
    """Main API Gateway implementation"""
    
    def __init__(
        self,
        rate_limiter: 'RateLimiter',
        circuit_breakers: 'CircuitBreakerRegistry',
        metrics: 'MetricsCollector'
    ):
        self.rate_limiter = rate_limiter
        self.circuit_breakers = circuit_breakers
        self.metrics = metrics
        self.services: Dict[str, ServiceConfig] = {}
        self.http_client = httpx.AsyncClient(
            timeout=httpx.Timeout(10.0),
            limits=httpx.Limits(max_keepalive_connections=100, max_connections=1000)
        )
    
    async def register_service(self, **kwargs):
        """Register a microservice with the gateway"""
        config = ServiceConfig(**kwargs)
        self.services[config.name] = config
        
        # Initialize circuit breaker for this service
        self.circuit_breakers.register(
            name=config.name,
            **config.circuit_breaker_config
        )
        
        logger.info("service_registered", service=config.name, base_url=config.base_url)
    
    async def route_request(self, request: Request) -> httpx.Response:
        """Route incoming request to appropriate service with all middleware"""
        # Extract request info
        client_ip = request.client.host if request.client else "unknown"
        endpoint = request.url.path
        method = request.method
        
        # 1. Rate limiting check
        if not await self.rate_limiter.check_limit(client_ip, endpoint):
            logger.warning("rate_limit_exceeded", client_ip=client_ip, endpoint=endpoint)
            raise HTTPException(status_code=429, detail="Rate limit exceeded")
        
        # 2. Find target service
        service_config = self._find_service_for_endpoint(endpoint)
        if not service_config:
            logger.warning("service_not_found", endpoint=endpoint)
            raise HTTPException(status_code=404, detail="Service not found")
        
        # 3. Check circuit breaker
        circuit_breaker = self.circuit_breakers.get(service_config.name)
        if not circuit_breaker.allow_request():
            logger.warning("circuit_open", service=service_config.name)
            
            # Record metrics for circuit breaker open
            if self.metrics:
                self.metrics.record_circuit_open(service_config.name)
            
            # Return fallback response
            return await self._get_fallback_response(service_config, request)
        
        # 4. Prepare request to backend service
        target_url = self._build_target_url(service_config, request)
        headers = self._prepare_headers(request)
        
        try:
            # Execute request with circuit breaker
            response = await circuit_breaker.execute(
                self._make_service_request,
                target_url,
                method,
                headers,
                await request.body(),
                service_config
            )
            
            # Record success
            if self.metrics:
                self.metrics.record_service_request(
                    service=service_config.name,
                    success=True,
                    latency=response.elapsed.total_seconds() if hasattr(response, 'elapsed') else 0
                )
            
            return response
            
        except Exception as e:
            logger.error(
                "service_request_failed",
                service=service_config.name,
                endpoint=endpoint,
                error=str(e)
            )
            
            # Record failure
            if self.metrics:
                self.metrics.record_service_request(
                    service=service_config.name,
                    success=False,
                    error_type=type(e).__name__
                )
            
            # Return fallback
            return await self._get_fallback_response(service_config, request)
    
    def _find_service_for_endpoint(self, endpoint: str) -> Optional[ServiceConfig]:
        """Find which service handles this endpoint"""
        for service in self.services.values():
            for pattern in service.endpoints:
                if self._endpoint_matches_pattern(endpoint, pattern):
                    return service
        return None
    
    def _endpoint_matches_pattern(self, endpoint: str, pattern: str) -> bool:
        """Simple pattern matching for endpoints"""
        if pattern.endswith("*"):
            return endpoint.startswith(pattern[:-1])
        return endpoint == pattern
    
    def _build_target_url(self, service_config: ServiceConfig, request: Request) -> str:
        """Build target URL for backend service"""
        # Remove API version prefix if present
        path = request.url.path
        if path.startswith("/api/v1"):
            path = path[7:]  # Remove "/api/v1"
        
        return urljoin(service_config.base_url, path + ("?" + request.url.query if request.url.query else ""))
    
    def _prepare_headers(self, request: Request) -> Dict[str, str]:
        """Prepare headers for backend request"""
        headers = dict(request.headers)
        
        # Remove hop-by-hop headers
        hop_by_hop = [
            'connection', 'keep-alive', 'proxy-authenticate',
            'proxy-authorization', 'te', 'trailers', 'transfer-encoding', 'upgrade'
        ]
        for header in hop_by_hop:
            headers.pop(header, None)
        
        # Add X-Forwarded headers
        headers['X-Forwarded-For'] = request.client.host if request.client else 'unknown'
        headers['X-Forwarded-Proto'] = request.url.scheme
        headers['X-Forwarded-Host'] = request.headers.get('host', '')
        
        # Add request ID if available
        if hasattr(request.state, 'request_id'):
            headers['X-Request-ID'] = request.state.request_id
        
        return headers
    
    async def _make_service_request(
        self,
        url: str,
        method: str,
        headers: Dict[str, str],
        body: bytes,
        service_config: ServiceConfig
    ) -> httpx.Response:
        """Make actual HTTP request to backend service"""
        # Create request with timeout
        timeout = httpx.Timeout(service_config.timeout)
        
        # Execute request with retries
        for attempt in range(service_config.retry_attempts):
            try:
                response = await self.http_client.request(
                    method=method,
                    url=url,
                    headers=headers,
                    content=body,
                    timeout=timeout
                )
                
                # Only retry on 5xx errors or timeouts
                if response.status_code < 500:
                    return response
                
                # Exponential backoff
                if attempt < service_config.retry_attempts - 1:
                    await asyncio.sleep(service_config.retry_backoff * (2 ** attempt))
                    
            except (httpx.TimeoutException, httpx.NetworkError) as e:
                if attempt == service_config.retry_attempts - 1:
                    raise
                await asyncio.sleep(service_config.retry_backoff * (2 ** attempt))
        
        # This should not be reached
        raise httpx.HTTPError("Max retries exceeded")
    
    async def _get_fallback_response(
        self,
        service_config: ServiceConfig,
        request: Request
    ) -> httpx.Response:
        """Generate fallback response when service is unavailable"""
        # In a real implementation, this could return cached data,
        # default response, or service degradation
        
        fallback_data = {
            "status": "service_unavailable",
            "service": service_config.name,
            "message": "Service is temporarily unavailable",
            "request_id": getattr(request.state, 'request_id', 'unknown')
        }
        
        return httpx.Response(
            status_code=503,
            json=fallback_data,
            headers={"Content-Type": "application/json"}
        )
    
    async def close(self):
        """Cleanup resources"""
        await self.http_client.aclose()

```markdown
# THREAT MODELING - DIYAPP V12 CORE
## Data: $(date)
## Sistema: DIYAPP Evolution V12
## Arquitetura: Microserviços com API Gateway, LLM Integration, Multi-tenant

### ATIVOS CRÍTICOS:
1. **Dados de Usuário**: PII, credenciais, tokens de sessão
2. **Modelos de IA**: LLM fine-tuned, prompts do sistema, dados de treinamento
3. **Infraestrutura**: Kubernetes cluster, banco de dados, filas de mensagens
4. **Integrações**: WhatsApp Business API, provedores de pagamento, serviços externos

### ANÁLISE STRIDE POR COMPONENTE:

#### 1. API GATEWAY
**Spoofing**: Autenticação JWT comprometida → Implementar validação assinatura, refresh tokens curtos
**Tampering**: Manipulação de requests/responses → HTTPS obrigatório, assinatura HMAC
**Repudiation**: Logs de acesso insuficientes → Log estruturado com correlation ID
**Info Disclosure**: Headers sensíveis expostos → Sanitizar headers, CSP
**Denial of Service**: Rate limiting ausente → WAF com rate limiting por IP/user
**Elevation of Privilege**: Falha na validação de escopo → RBAC granular, validação por recurso

#### 2. MICROSERVICOS DE LLM
```

```python
        }

        """Log WAF event and check for automated threats"""
        # Store event in Redis with TTL
        event_key = f"waf:event:{datetime.utcnow().timestamp()}"
        event_data = {
            "timestamp": event.timestamp.isoformat(),
            "client_ip": event.client_ip,
            "threat_level": event.threat_level.value,
            "rule_id": event.rule_id,
            "message": event.message,
            "request_path": event.request_path,
            "user_agent": event.user_agent,
            "blocked": event.blocked
        }
        
        self.redis.setex(event_key, timedelta(hours=24), json.dumps(event_data))
        
        # Update IP threat score
        self._update_ip_threat_score(event.client_ip, event.threat_level)
        
        # Check if IP should be blocked
        if self._should_block_ip(event.client_ip):
            self._block_ip(event.client_ip, f"Exceeded threat threshold: {event.threat_level}")
    
    def _update_ip_threat_score(self, ip: str, threat_level: ThreatLevel) -> None:
        """Update threat score for IP address"""
        score_key = f"waf:ip_score:{ip}"
        window_key = f"waf:ip_window:{ip}"
        
        # Get current window
        current_time = datetime.utcnow()
        window_start = self.redis.get(window_key)
        
        if not window_start:
            # Start new window
            self.redis.setex(window_key, timedelta(hours=1), current_time.isoformat())
            self.redis.setex(score_key, timedelta(hours=1), 0)
        
        # Add score based on threat level
        score_map = {
            ThreatLevel.CRITICAL: 10,
            ThreatLevel.HIGH: 5,
            ThreatLevel.MEDIUM: 2,
            ThreatLevel.LOW: 1
        }
        
        self.redis.incrby(score_key, score_map[threat_level])
    
    def _should_block_ip(self, ip: str) -> bool:
        """Check if IP should be blocked based on threat score"""
        score_key = f"waf:ip_score:{ip}"
        score = int(self.redis.get(score_key) or 0)
        
        # Block if score exceeds threshold
        if score >= 20:  # Critical threshold
            return True

        events_key = f"waf:ip_events:{ip}"
        recent_events = self.redis.lrange(events_key, 0, -1)
        
        critical_count = sum(1 for e in recent_events
```
