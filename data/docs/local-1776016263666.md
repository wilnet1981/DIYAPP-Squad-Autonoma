# DIYAPP Evolution - V11 Core - Arquitetura e Configuração

## Arquitetura V11 - Documento Técnico

### 1. DECISÃO ARQUITETURAL PRINCIPAL: Monorepo com Microsserviços Leves

**ADR-001: Arquitetura Monorepo com Microsserviços Leves**
**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V11

**CONTEXTO:**
DIYAPP precisa evoluir para uma arquitetura que suporte:
- Desenvolvimento paralelo de múltiplas squads (Modo Hive)
- Escalabilidade independente de componentes
- Manutenção simplificada de dependências compartilhadas
- 100% de estabilidade autônoma
- Deploy independente de serviços críticos

**DECISÃO:**
Adotar arquitetura de monorepo com microsserviços leves usando:
- Monorepo principal com Lerna/Yarn Workspaces
- Microsserviços como pacotes independentes
- API Gateway para roteamento e composição
- Banco de dados por serviço (Database-per-service)
- Contratos de API versionados (OpenAPI 3.0)

**OPÇÕES CONSIDERADAS:**
- Opção A: Monorepo com microsserviços leves (escolhida)
  - Prós: Compartilhamento de código fácil, CI/CD simplificado, dependências consistentes
  - Contras: Acoplamento no repositório, build times maiores
- Opção B: Microsserviços completos com repositórios separados
  - Prós: Desacoplamento total, deploys independentes
  - Contras: Complexidade de gerenciamento, versionamento de dependências compartilhadas difícil
- Opção C: Monolito modular
  - Prós: Simplicidade, debug fácil
  - Contras: Escalabilidade limitada, deploy tudo-ou-nada

**CONSEQUÊNCIAS:**
- Positivas: Desenvolvimento paralelo eficiente, reutilização de código, deploys independentes
- Negativas: Complexidade inicial maior, necessidade de ferramentas de monorepo
- Riscos: Build times longos, acoplamento inadvertido entre serviços

```bash
#!/usr/bin/env bash
# wait-for-it.sh

set -e

host="$1"
shift
port="$1"
shift
timeout="${WAITFORIT_TIMEOUT:-60}"
cmd="$@"

echo "Waiting for $host:$port..."

for i in $(seq $timeout); do
    if nc -z "$host" "$port"; then
        echo "$host:$port is available!"
        exec $cmd
        exit 0
    fi
    sleep 1
done

echo "Timeout waiting for $host:$port"
exit 1
```

const { Sequelize } = require('sequelize');
const config = require('./environment');
const logger = require('./logger');

class Database {
  constructor() {
    this.sequelize = null;
    this.isConnected = false;
  }

  async connect() {
    try {
      this.sequelize = new Sequelize(
        config.database.name,
        config.database.user,
        config.database.password,
        {
          host: config.database.host,
          port: config.database.port,
          dialect: 'postgres',
          logging: (msg) => logger.debug(msg),
          pool: {
            max: 10,
            min: 0,
            acquire: 30000,
            idle: 10000
          },
          retry: {
            max: 3,
            timeout: 10000,
            match: [
              /SequelizeConnectionError/,
              /SequelizeConnectionRefusedError/,
              /SequelizeHostNotFoundError/,
              /SequelizeHostNotReachableError/,
              /SequelizeInvalidConnectionError/,
              /SequelizeConnectionTimedOutError/
            ],
            backoffBase: 1000,
            backoffExponent: 1.5
          }
        }
      );

      await this.sequelize.authenticate();
      this.isConnected = true;
      
      logger.info('Database connection established successfully', {
        database: config.database.name,
        host: config.database.host
      });

      return this.sequelize;
    } catch (error) {
      logger.error('Unable to connect to the database:', {
        error: error.message,
        stack: error.stack,
        database: config.database.name,
        host: config.database.host
      });
      
      throw new Error(`Database connection failed: ${error.message}`);
    }
  }

  async disconnect() {
    if (this.sequelize) {
      await this.sequelize.close();
      this.isConnected = false;
      logger.info('Database connection closed');
    }
  }

  async healthCheck() {
    try {
      if (!this.isConnected) {
        return { status: 'DOWN', message: 'Not connected to database' };
      }
      
      await this.sequelize.query('SELECT 1');
      return { status: 'UP', message: 'Database is healthy' };
    } catch (error) {
      logger.error('Database health check failed:', {
        error: error.message
      });
      return { status: 'DOWN', message: error.message };
    }
  }
}

class BaseModel extends Model {
  static async safeCreate(data, options = {}) {
    try {
      const instance = await this.create(data, options);
      logger.info(`${this.name} created successfully`, {
        model: this.name,
        id: instance.id
      });
      return instance;
    } catch (error) {
      logger.error(`Error creating ${this.name}:`, {
        error: error.message,
        data: this.maskSensitiveData(data),
        stack: error.stack
      });
      throw error;
    }
  }

  static async safeUpdate(id, data, options = {}) {
    try {
      const [affectedRows] = await this.update(data, {
        where: { id },
        ...options
      });
      
      if (affectedRows === 0) {
        throw new Error(`${this.name} with id ${id} not found`);
      }
      
      logger.info(`${this.name} updated successfully`, {
        model: this.name,
        id
      });
      
      return affectedRows;
    } catch (error) {
      logger.error(`Error updating ${this.name}:`, {
        error: error.message,
        id,
        data: this.maskSensitiveData(data),
        stack: error.stack
      });
      throw error;
    }
  }

  static async safeDelete(id, options = {}) {
    try {
      const affectedRows = await this.destroy({
        where: { id },
        ...options
      });
      
      if (affectedRows === 0) {
        throw new Error(`${this.name} with id ${id} not found`);
      }
      
      logger.info(`${this.name} deleted successfully`, {
        model: this.name,
        id
      });
      
      return affectedRows;
    } catch (error) {
      logger.error(`Error deleting ${this.name}:`, {
        error: error.message,
        id,
        stack: error.stack
      });
      throw error;
    }
  }

  static maskSensitiveData(data) {
    if (!data || typeof data !== 'object') return data;
    
    const sensitiveFields = ['password', 'token', 'secret', 'credit_card', 'cpf'];
    const masked = { ...data };
    
    sensitiveFields.forEach(field => {
      if (masked[field]) {
        masked[field] = '***MASKED***';
      }
    });
    
    return masked;
  }

  toJSON() {
    const values = Object.assign({}, this.get());
    
    // Remove sensitive fields from JSON output
    const sensitiveFields = ['password', 'token', 'secret', 'credit_card'];
    sensitiveFields.forEach(field => {
      delete values[field];
    });
    
    return values;
  }
}

```python
    pool_size: int = Field(default=10, env="DB_POOL_SIZE")
    max_overflow: int = Field(default=20, env="DB_MAX_OVERFLOW")
    pool_timeout: int = Field(default=30, env="DB_POOL_TIMEOUT")
    pool_recycle: int = 3600  # Recycle connections after 1 hour
    
    # SQLite configuration
    sqlite_path: str = Field(default="data/diyapp.db", env="SQLITE_PATH")
    
    # Application
    app_env: str = Field(default="development", env="APP_ENV")
    
    class Config:
        env_file = ".env"
        case_sensitive = False

    def connection_string(self) -> str:
        """Generate connection string based on database type"""
        if self.db_type == DatabaseType.POSTGRESQL:
            return f"postgresql://{self.db_user}:{self.db_password}@{self.db_host}:{self.db_port}/{self.db_name}"
        else:
            # SQLite
            return f"sqlite:///{self.sqlite_path}"
    
    @property
    def engine_options(self) -> Dict[str, Any]:
        """Get SQLAlchemy engine options"""
        options = {
            "pool_size": self.pool_size,
            "max_overflow": self.max_overflow,
            "pool_timeout": self.pool_timeout,
            "pool_recycle": self.pool_recycle,
            "echo": self.app_env == "development",
        }
        
        if self.db_type == DatabaseType.POSTGRESQL:
            options.update({
                "connect_args": {
                    "connect_timeout": 10,
                    "application_name": "diyapp_v11",
                    "options": "-c timezone=UTC"
                }
            })
        else:
            # SQLite specific options
            options.update({
                "connect_args": {"check_same_thread": False}
            })

                "Database configuration validated",
                db_type=self.db_type,
                db_name=self.db_name,
                env=self.app_env
            )
            return True
            
        except Exception as e:
            logger.error("Database configuration validation failed", error=str(e))
```

```python
"""
Database connection management with connection pooling and monitoring
"""
from contextlib import contextmanager
from typing import Generator, Optional
from sqlalchemy import create_engine, Engine
from sqlalchemy.orm import sessionmaker, Session, scoped_session
from sqlalchemy.pool import QueuePool
import structlog
import time
from prometheus_client import Counter, Histogram, Gauge

from .config import config

logger = structlog.get_logger(__name__)

# Prometheus metrics
DB_CONNECTION_TOTAL = Counter(
    'db_connection_total',
    'Total database connections',
    ['operation']
)

DB_QUERY_DURATION = Histogram(
    'db_query_duration_seconds',
    'Database query duration',
    ['query_type']
)

DB_CONNECTIONS_ACTIVE = Gauge(
    'db_connections_active',
    'Active database connections'
)

DB_CONNECTIONS_IN_POOL = Gauge(
    'db_connections_in_pool',
    'Connections in pool'
)


class DatabaseConnection:
    """Database connection manager with connection pooling"""
    
    _engine: Optional[Engine] = None
    _session_factory: Optional[sessionmaker] = None
    _scoped_session: Optional[scoped_session] = None
    
    @classmethod
    def initialize(cls) -> bool:
        """Initialize database connection pool"""
        try:
            if not config.validate_config():
                logger.error("Invalid database configuration")
                return False
            
            # Create engine with connection pooling
            cls._engine = create_engine(
                config.connection_string,
                **config.engine_options,
                poolclass=QueuePool
            )
            
            # Create session factory
            cls._session_factory = sessionmaker(
                bind=cls._engine,
                autocommit=False,
                autoflush=False,
                expire_on_commit=False
            )
            
            # Create scoped session for thread safety
            cls._scoped_session = scoped_session(cls._session_factory)
            
            # Test connection
            with cls._engine.connect() as conn:
                conn.execute("SELECT 1")
            
            logger.info(
                "Database connection pool initialized",
                db_type=config.db_type,
                db_name=config.db_name,
                pool_size=config.pool_size
            )
            
            # Initialize metrics
            cls._setup_metrics()
            
            return True
            
        except Exception as e:
            logger.error("Failed to initialize database connection", error=str(e))
            return False
    
    @classmethod
    def _setup_metrics(cls):
        """Setup database metrics monitoring"""
        if cls._engine:
            # Monitor pool status
            @event.listens_for(cls._engine, 'checkout')
            def receive_checkout(dbapi_connection, connection_record, connection_proxy):
                DB_CONNECTIONS_ACTIVE.inc()
                DB_CONNECTIONS_IN_POOL.dec()
                DB_CONNECTION_TOTAL.labels(operation='checkout').inc()
            
            @event.listens_for(cls._engine, 'checkin')
            def receive_checkin(dbapi_connection, connection_record):
                DB_CONNECTIONS_ACTIVE.dec()
                DB_CONNECTIONS_IN_POOL.inc()
                DB_CONNECTION_TOTAL.labels(operation='checkin').inc()
    
    @classmethod
    def get_engine(cls) -> Engine:
        """Get database engine instance"""
        if cls._engine is None:
            cls.initialize()
        return cls._engine
    
    @classmethod
    def get_session(cls) -> Session:
        """Get a new database session"""
        if cls._scoped_session is None:
            cls.initialize()
        return cls._scoped_session()
    
    @classmethod
    @contextmanager
    def session_scope(cls) -> Generator[Session, None, None]:
        """Provide a transactional scope around a series of operations"""
        session = cls.get_session()
        try:
            start_time = time.time()
            yield session
            session.commit()
            DB_QUERY_DURATION.labels(query_type='transaction').observe(time.time() - start_time)
            logger.debug("Database transaction committed")
        except Exception as e:
            session.rollback()
            logger.error("Database transaction rolled back", error=str(e))
            raise
        finally:
            session.close()
    
    @classmethod
    def close_all(cls):
        """Close all database connections"""
        if cls._scoped_session:
            cls._scoped_session.remove()
        if cls._engine:
            cls._engine.dispose()
        logger.info("All database connections closed")
    
    @classmethod
    def get_pool_status(cls) -> dict:
        """Get connection pool status"""
        if not cls._engine:
            return {"status": "not_initialized"}
        
        pool = cls._engine.pool
        return {
            "status": "active",
            "checked_out": pool.checkedout(),
            "checked_in": pool.checkedin(),
            "size": pool.size(),
            "overflow": pool.overflow(),
            "connections": pool.status()
        }


# Initialize on import
```

```javascript
const promClient = require('prom-client');
const config = require('../../config/observability');

// Inicializar registro de métricas
const register = new promClient.Registry();

// Configurar labels padrão
promClient.register.setDefaultLabels(config.metrics.defaultLabels);

// Coletar métricas padrão do Node.js
if (config.metrics.collectDefaultMetrics) {
  promClient.collectDefaultMetrics({ register });
}

// Métricas personalizadas
class MetricsCollector {
  constructor() {
    // Contador de requisições HTTP
    this.httpRequestsTotal = new promClient.Counter({
      name: 'http_requests_total',
      help: 'Total de requisições HTTP',
      labelNames: ['method', 'route', 'status_code']
    });

    // Histograma de duração das requisições
    this.httpRequestDuration = new promClient.Histogram({
      name: 'http_request_duration_seconds',
      help: 'Duração das requisições HTTP em segundos',
      labelNames: ['method', 'route', 'status_code'],
      buckets: [0.1, 0.5, 1, 2, 5, 10]
    });

    // Contador de erros
    this.errorsTotal = new promClient.Counter({
      name: 'errors_total',
      help: 'Total de erros por tipo',
      labelNames: ['type', 'service']
    });

    // Métricas de negócio específicas do DIYAPP
    this.llmRequestsTotal = new promClient.Counter({
      name: 'llm_requests_total',
      help: 'Total de requisições para LLMs',
      labelNames: ['provider', 'model', 'feature', 'status']
    });

    this.llmTokensTotal = new promClient.Counter({
      name: 'llm_tokens_total',
      help: 'Total de tokens processados',
      labelNames: ['provider', 'model', 'feature', 'token_type']
    });

    this.llmCostTotal = new promClient.Counter({
      name: 'llm_cost_total',
      help: 'Custo total de LLMs em dólares',
      labelNames: ['provider', 'model', 'feature']
    });

    this.llmLatency = new promClient.Histogram({
      name: 'llm_latency_seconds',
      help: 'Latência das chamadas de LLM',
      labelNames: ['provider', 'model', 'feature'],
      buckets: [0.1, 0.5, 1, 2, 5, 10, 30]
    });

    // Métricas de qualidade
    this.llmQualityScore = new promClient.Gauge({
      name: 'llm_quality_score',
      help: 'Score de qualidade dos outputs de LLM',
      labelNames: ['feature', 'model', 'evaluation_type']
    });

    // Métricas de sistema
    this.activeConnections = new promClient.Gauge({
      name: 'active_connections',
      help: 'Número de conexões ativas'
    });

    this.queueSize = new promClient.Gauge({
      name: 'queue_size',
      help: 'Tamanho da fila de processamento',
      labelNames: ['queue_name']
    });

    // Registrar todas as métricas
    [
      this.httpRequestsTotal,
      this.httpRequestDuration,
      this.errorsTotal,
      this.llmRequestsTotal,
      this.llmTokensTotal,
      this.llmCostTotal,
      this.llmLatency,
      this.llmQualityScore,
      this.activeConnections,
      this.queueSize
    ].forEach(metric => register.registerMetric(metric));
  }

  // Métodos para registrar métricas de LLM
  recordLLMRequest(provider, model, feature, status = 'success') {
    this.llmRequestsTotal.inc({ provider, model, feature, status });
  }

  recordLLMTokens(provider, model, feature, tokenType, count) {
    this.llmTokensTotal.inc({ provider, model, feature, token_type: tokenType }, count);
  }

  recordLLMCost(provider, model, feature, cost) {
    this.llmCostTotal.inc({ provider, model, feature }, cost);
  }

  recordLLMLatency(provider, model, feature, latencySeconds) {
    this.llmLatency.observe({ provider, model, feature }, latencySeconds);
  }

  setLLMQualityScore(feature, model, evaluationType, score) {
    this.llmQualityScore.set({ feature, model, evaluation_type: evaluationType }, score);
  }

  // Métodos para métricas HTTP
  recordHTTPRequest(method, route, statusCode, durationSeconds) {
    this.httpRequestsTotal.inc({ method, route, status_code: statusCode });
    this.httpRequestDuration.observe({ method, route, status_code: statusCode }, durationSeconds);
  }

  // Métodos para erros
  recordError(type, service) {
    this.errorsTotal.inc({ type, service });
  }

  // Métricas de sistema
  setActiveConnections(count) {
    this.activeConnections.set(count);
  }

  setQueueSize(queueName, size) {
    this.queueSize.set({ queue_name: queueName }, size);
  }

  // Obter métricas no formato Prometheus
  async getMetrics() {
    return register.metrics();
  }

  // Obter métricas em formato JSON para APIs
  async getMetricsJSON() {
    const metrics = await register.getMetricsAsJSON();
    return metrics;
  }
}

// Singleton
module.exports = new MetricsCollector();
```

```javascript
          }`;
        })
      )
    })
  );

// File transport
if (config.logging.transports.file.enabled) {

class DIYAPPLogger {

    this.info('LLM Request', {

      model,
      feature,

      latency_ms: latency,
      cost_usd: cost,
      status,
      timestamp: new Date().toISOString()

      timestamp: new Date().toISOString()

      category: 'llm_quality',

      model,
      score,

      details,
      timestamp: new Date().toISOString()
    });

      feature,
      baseline_score: baselineScore,
      current_score: currentScore,
      percentage_decrease: percentageDecrease,
      threshold: 0.1, // 10%
      timestamp: new Date().toISOString()
    });
  }
```

## 1. Threat Model Analysis (STRIDE) para V11

### Ativos Críticos Identificados:
1. **Dados de Usuário**: Perfis, histórico de interações, preferências
2. **Sistema de Autenticação**: Tokens JWT, credenciais
3. **Integração com LLMs**: Prompts, contextos, respostas
4. **Banco de Dados**: SQLite/PostgreSQL com dados sensíveis
5. **APIs Externas**: WhatsApp, serviços de terceiros

### Análise STRIDE:

#### **Spoofing (Falsificação)**
- **Ameaça**: Ataque de força bruta em endpoints de login
- **Controles**: Rate limiting, MFA, tokens com expiração curta
- **Risco**: Alto - Requer mitigação imediata

#### **Tampering (Manipulação)**
- **Ameaça**: Manipulação de dados em trânsito ou prompts de LLM
- **Controles**: TLS, validação de inputs, sanitização de prompts
- **Risco**: Médio-Alto

#### **Repudiation (Repúdio)**
- **Ameaça**: Negação de ações realizadas no sistema
- **Controles**: Logs de auditoria imutáveis, timestamps assinados
- **Risco**: Médio

#### **Information Disclosure (Divulgação)**
- **Ameaça**: Vazamento de dados via logs, erros ou LLM
- **Controles**: Mascaramento de PII, CSP headers, sanitização de outputs
- **Risco**: Alto

#### **Denial of Service (Negação)**
- **Ameaça**: Sobrecarga de APIs ou LLM endpoints
- **Controles**: Rate limiting, circuit breakers, quotas
- **Risco**: Médio

#### **Elevation of Privilege (Elevação)**
- **Ameaça**: Acesso não autorizado a funcionalidades administrativas
- **Controles**: RBAC granular, princípio do menor privilégio
- **Risco**: Alto

```javascript
const helmet = require('helmet');
const hpp = require('hpp');
const rateLimit = require('express-rate-limit');
const crypto = require('crypto');

// Gerar nonce para CSP
const generateNonce = () => {
  return crypto.randomBytes(16).toString('base64');
};

// Rate limiting por tipo de endpoint
const apiLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutos
  max: 100, // limite por IP
  message: 'Too many requests from this IP, please try again later.',
  standardHeaders: true,
  legacyHeaders: false,
});

const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 5, // limite mais restrito para auth
  message: 'Too many login attempts, please try again later.',
});

// Middleware de segurança principal
const securityMiddleware = (app) => {
  // Helmet com configurações customizadas
  app.use(helmet({
    contentSecurityPolicy: {
      directives: {
        defaultSrc: ["'self'"],
        styleSrc: ["'self'", "'unsafe-inline'", "https://fonts.googleapis.com"],
        scriptSrc: ["'self'", (req, res) => `'nonce-${res.locals.nonce}'`],
        fontSrc: ["'self'", "https://fonts.gstatic.com"],
        imgSrc: ["'self'", "data:", "https:"],
        connectSrc: ["'self'"],
        frameSrc: ["'none'"],
        objectSrc: ["'none'"],
        baseUri: ["'self'"],
        formAction: ["'self'"],
        frameAncestors: ["'none'"],
      },
    },
    hsts: {
      maxAge: 31536000, // 1 ano
      includeSubDomains: true,
      preload: true
    },
    referrerPolicy: { policy: 'strict-origin-when-cross-origin' },
    xContentTypeOptions: true,
    xFrameOptions: { action: 'deny' },
    xXssProtection: true,
  }));

  // Prevenir Parameter Pollution
  app.use(hpp());

  // Headers customizados adicionais
  app.use((req, res, next) => {
    // Gerar nonce para cada request
    res.locals.nonce = generateNonce();
    
    // Headers adicionais de segurança
    res.setHeader('X-Permitted-Cross-Domain-Policies', 'none');
    res.setHeader('X-DNS-Prefetch-Control', 'off');
    res.setHeader('Expect-CT', 'max-age=86400, enforce');
    
    // Prevenir caching de dados sensíveis
    if (req.path.includes('/api/auth') || req.path.includes('/api/user')) {
      res.setHeader('Cache-Control', 'no-store, max-age=0');
    }
    
    next();
  });

  // Aplicar rate limiting
  app.use('/api/auth/', authLimiter);
  app.use('/api/', apiLimiter);
};

module.exports = { securityMiddleware, generateNonce };
```

## 3. Auditoria de Dependências e Vulnerabilidades

### Script de Auditoria (`scripts/security-audit.js`):

```javascript
#!/usr/bin/env node

const { execSync } = require('child_process');
const fs = require('fs');
const path = require('path');
const axios = require('axios');

class SecurityAuditor {
  constructor() {
    this.vulnerabilities = [];
    this.criticalCount = 0;
    this.highCount = 0;
  }

  async runAudit() {
    console.log('🔍 Iniciando auditoria de segurança V11...\n');
    
    try {
      // 1. Auditoria NPM
      await this.auditNpmDependencies();
      
      // 2. Auditoria Python (se aplicável)
      if (fs.existsSync('requirements.txt')) {
        await this.auditPythonDependencies();
      }
      
      // 3. Verificar arquivos sensíveis
      this.checkSensitiveFiles();
      
      // 4. Verificar configurações de ambiente
      this.checkEnvironmentConfig();
      
      // 5. Gerar relatório
      this.generateReport();
      
    } catch (error) {
      console.error('❌ Erro durante auditoria:', error.message);
      process.exit(1);
    }
  }

  async auditNpmDependencies() {
    console.log('📦 Auditando dependências NPM...');
    
    try {
      // Executar npm audit
      const auditResult = execSync('npm audit --json', { encoding: 'utf8' });
      const auditData = JSON.parse(auditResult);
      
      if (auditData.metadata && auditData.metadata.vulnerabilities) {
        const vulns = auditData.metadata.vulnerabilities;
        
        this.criticalCount = vulns.critical || 0;
        this.highCount = vulns.high || 0;
        
        if (this.criticalCount > 0 || this.highCount > 0) {
          console.log(`⚠️  Vulnerabilidades encontradas: ${this.criticalCount} críticas, ${this.highCount} altas`);
          
          // Analisar vulnerabilidades críticas/altas
          if (auditData.advisories) {
            Object.values(auditData.advisories).forEach(adv => {
              if (adv.severity === 'critical' || adv.severity === 'high') {
                this.vulnerabilities.push({
                  type: 'npm',
                  package: adv.module_name,
                  severity: adv.severity,
                  title: adv.title,
                  patched_versions: adv.patched_versions,
                  cves: adv.cves || []
                });
              }
            });
          }
        } else {
          console.log('✅ Nenhuma vulnerabilidade crítica/alta encontrada');
        }
      }
      
      // Verificar dependências desatualizadas
      const outdated = execSync('npm outdated --json', { encoding: 'utf8' });
      const outdatedData = JSON.parse(outdated || '{}');
      
      if (Object.keys(outdatedData).length > 0) {
        console.log('📅 Dependências desatualizadas encontradas:');
        Object.entries(outdatedData).forEach(([pkg, info]) => {
          console.log(`  ${pkg}: ${info.current} → ${info.latest}`);
        });
      }
      
    } catch (error) {
      console.log('ℹ️  NPM audit não disponível ou sem vulnerabilidades');
    }
  }

  async auditPythonDependencies() {
    console.log('🐍 Auditando dependências Python...');
    
    try {
      // Verificar se safety está instalado
      execSync('safety --version', { stdio: 'ignore' });
      
      const safetyResult = execSync('safety check --json', { encoding: 'utf8' });
      const safetyData = JSON.parse(safetyResult);
      
      if (safetyData.vulnerabilities && safetyData.vulnerabilities.length > 0) {
        safetyData.vulnerabilities.forEach(vuln => {
          if (vuln.severity === 'CRITICAL' || vuln.severity === 'HIGH') {
            this.vulnerabilities.push({
              type: 'python',
              package: vuln.package_name,
              severity: vuln.severity.toLowerCase(),
              title: vuln.vulnerability_id,
              description: vuln.advisory,
              patched_versions: vuln.patched_versions
            });
          }
        });
      }
    } catch (error) {
      console.log('ℹ️  Safety não disponível para auditoria Python');
    }
  }

  checkSensitiveFiles() {
    console.log('📁 Verificando arquivos sensíveis...');
    
    const sensitivePatterns = [
      /\.env$/,
      /config\.(json|yml|yaml)$/,
      /secret/i,
      /private_key/i,
      /certificate/i,
      /\.pem$/,
      /\.key$/
    ];
    
    const checkDirectory = (dir) => {
      const files = fs.readdirSync(dir, { withFileTypes: true });
      
      files.forEach(file => {
        const fullPath = path.join(dir, file.name);
        
        if (file.isDirectory()) {
          // Ignorar node_modules e .git
          if (!file.name.includes('node_modules') && !file.name.includes('.git')) {
            checkDirectory(fullPath);
          }
        } else {
          // Verificar se o arquivo corresponde a padrões sensíveis
          sensitivePatterns.forEach(pattern => {
            if (pattern.test(file.name)) {
              try {
                const content = fs.readFileSync(fullPath, 'utf8');
                // Verificar por credenciais hardcoded
                const credentialPatterns = [
                  /password\s*[:=]\s*['"][^'"]+['"]/i,
                  /api[_-]?key\s*[:=]\s*['"][^'"]+['"]/i,
                  /secret\s*[:=]\s*['"][^'"]+['"]/i,
                  /token\s*[:=]\s*['"][^'"]+['"]/i
                ];
                
                credentialPatterns.forEach(credPattern => {
                  if (credPattern.test(content)) {
                    console.warn(`⚠️  Possível credencial hardcoded em: ${fullPath}`);
                  }
                });
              } catch (e) {
                // Ignorar arquivos binários
              }
            }
          });
        }
      });
    };
    
    checkDirectory('.');
  }

  checkEnvironmentConfig() {
    console.log('⚙️  Verificando configurações de ambiente...');
    
    const requiredEnvVars = [
      'NODE_ENV',
      'SESSION_SECRET',
      'DATABASE_URL',
      'JWT_SECRET'
    ];
    
    const missingVars = requiredEnvVars.filter(varName => !process.env[varName]);
    
    if (missingVars.length > 0) {
      console.warn(`⚠️  Variáveis de ambiente ausentes: ${missingVars.join(', ')}`);
    }
    
    // Verificar se secrets são fracos
    if (process.env.SESSION_SECRET && process.env.SESSION_SECRET.length < 32) {
      console.warn('⚠️  SESSION_SECRET muito curto (mínimo 32 caracteres recomendado)');
    }
  }

  generateReport() {
    console.log('\n📊 RELATÓRIO DE SEGURANÇA V11');
    console.log('=' .repeat(50));
    
    if (this.vulnerabilities.length === 0) {
      console.log('✅ Nenhuma vulnerabilidade crítica/alta encontrada');
    } else {
      console.log(`❌ ${this.vulnerabilities.length} vulnerabilidades críticas/altas encontradas:\n`);
      
      this.vulnerabilities.forEach((vuln, index) => {
        console.log(`${index + 1}. [${vuln.type.toUpperCase()}] ${vuln.package}`);
        console.log(`   Severidade: ${vuln.severity.toUpperCase()}`);
        console.log(`   Descrição: ${vuln.title}`);
        console.log(`   Versões corrigidas: ${vuln.patched_versions || 'N/A'}`);
        console.log('');
      });
      
      // Gerar arquivo de relatório
      const report = {
        timestamp: new Date().toISOString(),
        vulnerabilities: this.vulnerabilities,
        summary: {
          total: this.vulnerabilities.length,
          critical: this.vulnerabilities.filter(v => v.severity === 'critical').length,
          high: this.vulnerabilities.filter(v => v.severity === 'high').length
        },
        recommendations: this.generateRecommendations()
      };
      
      fs.writeFileSync('security-audit-report.json', JSON.stringify(report, null, 2));
      console.log('📄 Relatório salvo em: security-audit-report.json');
    }
    
    console.log('\n🔒 RECOMENDAÇÕES:');
    console.log('1. Execute "npm audit fix" para correções automáticas');
    console.log('2. Revise dependências com vulnerabilidades críticas/altas');
    console.log('3. Atualize .env.example com todas as variáveis necessárias');
    console.log('4. Implemente varredura de segurança no pipeline CI/CD');
  }

  generateRecommendations() {
    const recommendations = [];
    
    if (this.criticalCount > 0) {
      recommendations.push({
        priority: 'CRITICAL',
        action: 'Corrigir vulnerabilidades críticas em até 24h',
        command: 'npm audit fix --force'
      });
    }
    
    if (this.highCount > 0) {
      recommendations.push({
        priority: 'HIGH',
        action: 'Corrigir vulnerabilidades altas em até 72h',
        command: 'npm update <package-name>'
      });
    }
    
    recommendations.push({
      priority: 'MEDIUM',
      action: 'Implementar dependabot para atualizações automáticas',
      description: 'Configurar GitHub Dependabot para monitorar e atualizar dependências'
    });
    
    recommendations.push({
      priority: 'LOW',
      action: 'Adicionar SAST ao pipeline CI/CD',
      description: 'Integrar Snyk ou SonarQube para análise estática contínua'
    });
    
    return recommendations;
  }
}

// Executar auditoria
const auditor = new SecurityAuditor();
auditor.runAudit().catch(console.error);
```

## 📝 Padrões de Commit

### Convenção de Mensagens
```
<tipo>(<escopo>): <descrição>

[corpo opcional]

[rodapé opcional]
```

### Tipos de Commit
- `feat`: Nova funcionalidade
- `fix`: Correção de bug
- `docs`: Documentação
- `style`: Formatação, ponto e vírgula, etc (não altera código)
- `refactor`: Refatoração de código
- `test`: Adição ou correção de testes
- `chore`: Atualização de tarefas de build, configurações, etc
- `perf`: Melhorias de performance
- `ci`: Mudanças na integração contínua
- `build`: Mudanças no sistema de build
- `revert`: Reversão de commit anterior

### Exemplos
```bash
# Nova funcionalidade
feat(api): adiciona endpoint para criação de squads

# Correção de bug
fix(auth): corrige validação de token JWT expirado

# Documentação
docs(readme): adiciona guia de setup com Docker

# Refatoração
refactor(hive): otimiza algoritmo de distribuição de tarefas

# Testes
test(agents): adiciona testes unitários para health check

# Configuração
chore(deps): atualiza dependências para versões seguras
```

### Validação de Commits
O projeto utiliza Husky + commitlint para validar mensagens de commit:
```json
{
  "extends": ["@commitlint/config-conventional"],
  "rules": {
    "type-enum": [2, "always", [
      "feat", "fix", "docs", "style", "refactor", 
      "test", "chore", "perf", "ci", "build", "revert"
    ]],
    "subject-case": [2, "never", ["sentence-case", "start-case", "pascal-case", "upper-case"]]
  }
}
```

## 🔄 Fluxo de Trabalho da Squad (Git Flow Adaptado)

### Branches Principais
- `main`: Produção (sempre estável)
- `develop`: Desenvolvimento integrado
- `release/*`: Preparação para release
- `feature/*`: Novas funcionalidades
- `bugfix/*`: Correções de bugs
- `hotfix/*`: Correções críticas de produção

### Processo de Desenvolvimento

#### 1. Iniciar Nova Feature
```bash
# Sincronizar com develop
git checkout develop
git pull origin develop

# Criar branch da feature
git checkout -b feature/nome-da-feature

# Desenvolver e commitar
git add .
git commit -m "feat(component): descrição da feature"

# Publicar branch
git push -u origin feature/nome-da-feature
```

#### 2. Code Review via Pull Request
- Criar PR de `feature/*` para `develop`
- Requer aprovação de pelo menos 2 membros da squad
- Todos os testes devem passar
- Cobertura de código mantida ou melhorada
- Documentação atualizada

#### 3. Integração Contínua
```yaml
# .github/workflows/ci.yml
name: CI Pipeline
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: postgres
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          
      - name: Install dependencies
        run: npm ci
        
      - name: Run linting
        run: npm run lint
        
      - name: Run unit tests
        run: npm test
        
      - name: Run integration tests
        run: npm run test:integration
        
      - name: Build application
        run: npm run build
        
      - name: Upload coverage
        uses: codecov/codecov-action@v3
```

#### 4. Release Process
```bash
# Criar branch de release
git checkout -b release/v1.2.0 develop

# Atualizar versionamento
npm version patch --no-git-tag-version

# Commitar mudanças de versionamento
git commit -am "chore(release): v1.2.0"

# Merge para main e develop
git checkout main
git merge --no-ff release/v1.2.0
git tag -a v1.2.0 -m "Release v1.2.0"

git checkout develop
git merge --no-ff release/v1.2.0

# Deletar branch de release
git branch -d release/v1.2.0
```

#### 5. Hotfix Process
```bash
# Criar branch de hotfix a partir de main
git checkout -b hotfix/critical-bug main

# Aplicar correção
git commit -am "fix(component): corrige bug crítico"

# Merge para main e develop
git checkout main
git merge --no-ff hotfix/critical-bug
git tag -a v1.2.1 -m "Hotfix v1.2.1"

git checkout develop
git merge --no-ff hotfix/critical-bug
```
