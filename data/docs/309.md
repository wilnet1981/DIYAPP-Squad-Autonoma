# DIYAPP Evolution - V12 Core - Arquitetura Técnica

## 1. ADR-001: Arquitetura de Microsserviços com Core Monolítico

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead + Especialista Infra + Especialista Backend

### CONTEXTO:
O DIYAPP cresceu para um sistema complexo com múltiplos módulos (WhatsApp, Telegram, Email, Dashboard, IA). A arquitetura monolítica atual dificulta:
- Deploy independente por módulo
- Escalabilidade seletiva
- Manutenção paralela por squads
- Isolamento de falhas

### DECISÃO:
Adotar arquitetura híbrida: Core monolítico + Microsserviços especializados.

**Core Monolítico:**
- Autenticação/autorização
- Gerenciamento de usuários
- Dashboard principal
- Configurações globais
- Gateway de API

**Microsserviços:**
- whatsapp-service
- telegram-service 
- email-service
- ai-orchestrator-service
- notification-service
- analytics-service

### OPÇÕES CONSIDERADAS:
- **Opção A:** Monolito puro - Prós: Simplicidade, deploy único. Contras: Acoplamento alto, dificuldade de escalar.
- **Opção B:** Microsserviços puros - Prós: Desacoplamento máximo. Contras: Complexidade operacional, latência de rede.
- **Opção C:** Híbrido (escolhida) - Prós: Balanceia simplicidade e desacoplamento. Contras: Gerenciamento misto.

### CONSEQUÊNCIAS:
**Positivas:**
- Deploy independente dos serviços de comunicação
- Escalabilidade horizontal dos serviços mais demandados
- Isolamento de falhas (WhatsApp down não afeta Telegram)
- Squads podem trabalhar em serviços diferentes simultaneamente

**Negativas:**
- Complexidade aumentada de deploy
- Necessidade de orquestração de containers
- Comunicação via rede entre serviços

**Riscos:**
- Latência entre serviços → Mitigação: Cache Redis compartilhado
- Consistência de dados → Mitigação: Event sourcing com Kafka
- Debug distribuído → Mitigação: Jaeger para tracing distribuído

## 4. Dockerfile Padrão para Serviços

```dockerfile
# Dockerfile para serviços Node.js
FROM node:20-alpine AS builder

WORKDIR /app

# Copiar dependências
COPY package*.json ./
COPY tsconfig.json ./

# Instalar dependências
RUN npm ci --only=production

# Copiar código
COPY src/ ./src/
COPY prisma/ ./prisma/

# Gerar Prisma client
RUN npx prisma generate

# Build TypeScript
RUN npm run build

# Imagem de produção
FROM node:20-alpine

WORKDIR /app

# Instalar apenas runtime necessário
RUN apk add --no-cache tini

# Copiar do builder
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/package.json ./

# Configurar usuário não-root
USER node

# Health check
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:${PORT:-3000}/health', (r) => {process.exit(r.statusCode === 200 ? 0 : 1)})"

ENTRYPOINT ["/sbin/tini", "--"]
CMD ["node", "dist/index.js"]
```

## 6. GitHub Actions CI/CD Pipeline

```yaml
# .github/workflows/ci-core.yml
name: CI Core Service

on:
  push:
    paths:
      - 'packages/core/**'
      - '.github/workflows/ci-core.yml'
  pull_request:
    paths:
      - 'packages/core/**'

jobs:
  test:
    runs-on: ubuntu-latest
    
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: postgres
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 5432:5432
      redis:
        image: redis:7-alpine
        options: >-
          --health-cmd "redis-cli ping"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5
        ports:
          - 6379:6379

    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '20.x'
        cache: 'npm'
        
    - name: Install dependencies
      run: npm ci
      
    - name: Generate Prisma client
      run: npx prisma generate
      working-directory: packages/core
      
    - name: Run linting
      run: npm run lint
      working-directory: packages/core
      
    - name: Run tests with coverage
      run: npm test -- --coverage
      working-directory: packages/core
      env:
        DATABASE_URL: postgresql://postgres:postgres@localhost:5432/diyapp_test
        REDIS_URL: redis://localhost:6379
        
    - name: Upload coverage to Codecov
      uses: codecov/codecov-action@v3
      with:
        file: ./packages/core/coverage/lcov.info
        
    - name: Security scan
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
        
  build:
    needs: test
    runs-on: ubuntu-latest
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Docker Buildx
      uses: docker/setup-buildx-action@v2
      
    - name: Login to DockerHub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}
        
    - name: Build and push
      uses: docker/build-push-action@v4
      with:
        context: ./packages/core
        push: true
        tags: |
          ${{ secrets.DOCKER_USERNAME }}/diyapp-core:latest
          ${{ secrets.DOCKER_USERNAME }}/diyapp-core:${{ github.sha }}
        cache-from: type=gha
        cache-to: type=gha,mode=max
```

  format: winston.format.json(),
  defaultMeta: { service: 'redis' },
  transports: [
    new winston.transports.File({ filename: 'logs/redis-error.log', level: 'error' }),
    new winston.transports.File({ filename: 'logs/redis-combined.log' })
  ]
});

if (process.env.NODE_ENV !== 'production') {
  logger.add(new winston.transports.Console({
    format: winston.format.simple()
  }));

class RedisClient {
  constructor() {
    this.client = null;
    this.isConnected = false;
    this.connectionAttempts = 0;
    this.maxAttempts = 5;
  }

  async connect() {
    if (this.isConnected) return this.client;

    try {
      this.client = createClient({
        url: `redis://${process.env.REDIS_HOST}:${process.env.REDIS_PORT}`,
        password: process.env.REDIS_PASSWORD || undefined,
        socket: {
          reconnectStrategy: (retries) => {
            this.connectionAttempts = retries;
            if (retries > this.maxAttempts) {
              logger.error('Max Redis reconnection attempts reached');
              return new Error('Max reconnection attempts reached');
            }
            // Exponential backoff with jitter
            const delay = Math.min(1000 * Math.pow(2, retries) + Math.random() * 1000, 10000);
            logger.warn(`Redis reconnection attempt ${retries}, waiting ${delay}ms`);
            return delay;
          }
        }
      });

      this.client.on('error', (err) => {
        logger.error('Redis Client Error:', {
          error: err.message,
          stack: err.stack
        });
        this.isConnected = false;
      });

      this.client.on('connect', () => {
        logger.info('Redis connected successfully');
        this.isConnected = true;
        this.connectionAttempts = 0;
      });

      this.client.on('reconnecting', () => {
        logger.info('Redis reconnecting...');
      });

      await this.client.connect();
      return this.client;
    } catch (error) {
      logger.error('Failed to connect to Redis:', {
        error: error.message,
        stack: error.stack
      });
      throw error;
    }
  }

  async get(key) {
    try {
      if (!this.isConnected) {
        await this.connect();
      }
      return await this.client.get(key);
    } catch (error) {
      logger.error('Redis GET error:', {
        key,
        error: error.message
      });
      return null; // Graceful degradation
    }
  }

  async set(key, value, ttl = 3600) {
    try {
      if (!this.isConnected) {
        await this.connect();
      }
      await this.client.set(key, value, { EX: ttl });
    } catch (error) {
      logger.error('Redis SET error:', {
        key,
        error: error.message
      });
      // Graceful degradation - cache miss is acceptable
    }
  }

  async del(key) {
    try {
      if (!this.isConnected) {
        await this.connect();
      }
      await this.client.del(key);
    } catch (error) {
      logger.error('Redis DEL error:', {
        key,
        error: error.message
      });
    }
  }

  async disconnect() {
    if (this.client && this.isConnected) {
      await this.client.quit();
      this.isConnected = false;
    }
  }
}

module.exports = new RedisClient();

import React, { Suspense, lazy, useEffect } from 'react';
import { BrowserRouter as Router, Routes, Route } from 'react-router-dom';
import { DesignSystemProvider } from './design-system/DesignSystemProvider';
import { PerformanceMonitor } from './performance/PerformanceMonitor';
import { LoadingSpinner } from './components/LoadingSpinner';
import { ErrorBoundary } from './components/ErrorBoundary';
import { usePerformanceMetrics } from './hooks/usePerformanceMetrics';

// Lazy loading de rotas principais
const Dashboard = lazy(() => import('./pages/Dashboard').then(module => ({ default: module.Dashboard })));
const Analytics = lazy(() => import('./pages/Analytics').then(module => ({ default: module.Analytics })));
const Settings = lazy(() => import('./pages/Settings').then(module => ({ default: module.Settings })));
const AIStudio = lazy(() => import('./pages/AIStudio').then(module => ({ default: module.AIStudio })));

// Componente de fallback para lazy loading
const PageLoader: React.FC = () => (
  <div className="page-loader" role="status" aria-label="Carregando página">
    <LoadingSpinner size="large" />
    <span className="visually-hidden">Carregando conteúdo...</span>
  </div>
);

const App: React.FC = () => {
  const { trackLCP, trackINP, trackCLS } = usePerformanceMetrics();

  useEffect(() => {
    // Monitoramento inicial de performance
    const observer = new PerformanceObserver((list) => {
      for (const entry of list.getEntries()) {
        if (entry.entryType === 'largest-contentful-paint') {
          trackLCP(entry.startTime);
        } else if (entry.entryType === 'layout-shift') {
          trackCLS((entry as LayoutShift).value);
        }
      }
    });

    observer.observe({ entryTypes: ['largest-contentful-paint', 'layout-shift'] });

    return () => observer.disconnect();
  }, [trackLCP, trackCLS]);

  return (
    <ErrorBoundary>
      <DesignSystemProvider>
        <PerformanceMonitor>
          <Router>
            <div className="app-container" data-testid="app-container">
              <Suspense fallback={<PageLoader />}>
                <Routes>
                  <Route path="/" element={<Dashboard />} />
                  <Route path="/analytics" element={<Analytics />} />
                  <Route path="/settings" element={<Settings />} />
                  <Route path="/ai-studio" element={<AIStudio />} />
                </Routes>
              </Suspense>
            </div>
          </Router>
        </PerformanceMonitor>
      </DesignSystemProvider>
    </ErrorBoundary>
  );
};

export default App;

import React, { createContext, useContext } from 'react';
import { GlobalStyles } from './GlobalStyles';
import { tokens } from './tokens';

interface DesignSystemContextType {
  tokens: typeof tokens;
  breakpoints: {
    mobile: number;
    tablet: number;
    desktop: number;
  };
}

const DesignSystemContext = createContext<DesignSystemContextType | undefined>(undefined);

export const useDesignSystem = (): DesignSystemContextType => {
  const context = useContext(DesignSystemContext);
  if (!context) {
    throw new Error('useDesignSystem must be used within DesignSystemProvider');
  }
  return context;
};

interface DesignSystemProviderProps {
  children: React.ReactNode;
}

export const DesignSystemProvider: React.FC<DesignSystemProviderProps> = ({ children }) => {
  const value: DesignSystemContextType = {
    tokens,
    breakpoints: {
      mobile: 320,
      tablet: 768,
      desktop: 1024,
    },
  };

  return (
    <DesignSystemContext.Provider value={value}>
      <GlobalStyles />
      {children}
    </DesignSystemContext.Provider>
  );
};

WITH raw_events AS (
    SELECT 
        event_id,
        event_timestamp,
        event_type,
        user_id,
        session_id,
        device_info->>'family' as device_family,
        device_info->>'os' as device_os,
        SPLIT_PART(page_url, '?', 1) as page_path,
        CASE 
            WHEN POSITION('?' IN page_url) > 0 
            THEN jsonb_build_object('query', SPLIT_PART(page_url, '?', 2))
            ELSE '{}'::jsonb
        END as page_query_params,
        event_properties,
        app_version,
        source_system,
        ingested_at,
        -- Validações
        CASE 
            WHEN event_timestamp > NOW() + INTERVAL '1 day' THEN false
            WHEN event_type IS NULL THEN false
            WHEN event_timestamp IS NULL THEN false
            ELSE true
        END as is_valid,
        ARRAY[
            CASE WHEN event_timestamp > NOW() + INTERVAL '1 day' THEN 'future_timestamp' END,
            CASE WHEN event_type IS NULL THEN 'missing_event_type' END
        ] as validation_errors
    FROM {{ source('raw', 'product_events') }}
    WHERE ingested_at >= DATE_TRUNC('day', CURRENT_DATE - INTERVAL '7 days')
)

SELECT 
    event_id,
    event_timestamp,
    event_type,
    NULLIF(TRIM(user_id), '') as user_id,
    NULLIF(TRIM(session_id), '') as session_id,
    NULLIF(TRIM(device_family), '') as device_family,
    NULLIF(TRIM(device_os), '') as device_os,
    page_path,
    page_query_params,
    event_properties,
    app_version,
    source_system,
    ingested_at,
    is_valid as _valid,
    ARRAY_REMOVE(validation_errors, NULL) as _validation_errors
FROM raw_events

WITH daily_calls AS (
    SELECT 
        DATE(request_timestamp) as cost_date,
        feature_category,
        feature_name,
        model_provider,
        model_name,
        COUNT(*) as total_calls,
        SUM(input_tokens) as total_input_tokens,
        SUM(output_tokens) as total_output_tokens,
        SUM(total_tokens) as total_tokens,
        SUM(cost_usd) as total_cost_usd,
        AVG(latency_ms) as avg_latency_ms,
        AVG(CASE WHEN success THEN 1.0 ELSE 0.0 END) as success_rate
    FROM {{ ref('stg_llm_calls') }}
    WHERE request_timestamp >= DATE_TRUNC('day', CURRENT_DATE - INTERVAL '30 days')
      AND _valid = true
    GROUP BY 1, 2, 3, 4, 5
)

SELECT 
    cost_date,
    feature_category,
    feature_name,
    model_provider,
    model_name,
    total_calls,
    total_input_tokens,
    total_output_tokens,
    total_tokens,
    total_cost_usd,
    avg_latency_ms,
    success_rate,
    CURRENT_TIMESTAMP as calculated_at
FROM daily_calls
