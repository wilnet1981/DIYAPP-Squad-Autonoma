# DIYAPP Evolution - V12 Core - Arquitetura de Microsserviços

## 1. Documento de Arquitetura V12

### 1.1 Visão Geral
A V12 do DIYAPP evolui para uma arquitetura de microsserviços baseada em eventos, com comunicação assíncrona via gRPC para operações síncronas críticas. O sistema é projetado para 100% de autonomia operacional com tolerância a falhas e recuperação automática.

### 1.2 Princípios Arquiteturais
1. **Autonomia por Domínio**: Cada microsserviço possui seu próprio banco de dados
2. **Comunicação Assíncrona**: Eventos como padrão, gRPC apenas quando necessário
3. **Resiliência**: Circuit breakers, retry com backoff exponencial
4. **Observabilidade**: Logs estruturados, métricas e tracing distribuído
5. **Auto-cura**: Health checks e reinicialização automática

### 1.4 Padrões de Comunicação

#### 1.4.1 gRPC (Operações Síncronas)
```protobuf
// protobuf/auth_service.proto
syntax = "proto3";

package diyapp.v12;

service AuthService {
  rpc Authenticate(AuthRequest) returns (AuthResponse);
  rpc ValidateToken(TokenRequest) returns (TokenResponse);
  rpc RefreshToken(RefreshRequest) returns (RefreshResponse);
}

message AuthRequest {
  string username = 1;
  string password = 2;
  string client_id = 3;
}

message AuthResponse {
  string access_token = 1;
  string refresh_token = 2;
  int64 expires_in = 3;
  string user_id = 4;
}
```

### 1.5 Diagramas de Sequência

#### 1.5.1 Fluxo: Criação de Tarefa com IA
```
Participante Client
Participant API Gateway
Participant Auth Service
Participant Task Service
Participant AI Service
Participant Kafka
Participant Notification Service

Client->API Gateway: POST /api/v12/tasks
API Gateway->Auth Service: gRPC ValidateToken
Auth Service-->API Gateway: Token válido
API Gateway->Task Service: gRPC CreateTask
Task Service->Task Service: Persiste no PostgreSQL
Task Service->Kafka: Publica "task.created"
Kafka-->AI Service: Consome "task.created"
AI Service->AI Service: Processa com LLM
AI Service->Task Service: gRPC UpdateTaskWithAI
Task Service->Kafka: Publica "task.ai_processed"
Kafka-->Notification Service: Consome evento
Notification Service->Client: WebSocket/Push notification
```

#### 1.5.2 Fluxo: Autenticação com Circuit Breaker
```
Participant Client
Participant API Gateway
Participant Auth Service
Participant Redis
Participant Circuit Breaker

Client->API Gateway: POST /api/v12/auth/login
API Gateway->Circuit Breaker: Chama Auth Service
alt Serviço disponível
  Circuit Breaker->Auth Service: gRPC Authenticate
  Auth Service->Redis: Cache session
  Auth Service-->Circuit Breaker: Success
  Circuit Breaker-->API Gateway: Auth Response
  API Gateway-->Client: Tokens
else Serviço indisponível
  Circuit Breaker->Circuit Breaker: Trip circuit
  Circuit Breaker-->API Gateway: Fallback response
  API Gateway-->Client: 503 Service Unavailable
end
```

#### 2.3.2 CD Pipeline
```yaml
# .github/workflows/cd.yml
name: CD Pipeline

on:
  workflow_run:
    workflows: ["CI Pipeline"]
    branches: [main]
    types:
      - completed

jobs:
  deploy-staging:
    if: ${{ github.event.workflow_run.conclusion == 'success' }}
    runs-on: ubuntu-latest
    environment: staging
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Configure kubectl
      uses: azure/setup-kubectl@v3
      with:
        version: 'latest'
    
    - name: Deploy to staging
      run: |
        kubectl config set-cluster k8s \
          --server=${{ secrets.K8S_STAGING_SERVER }} \
          --certificate-authority=${{ secrets.K8S_STAGING_CA }}
        kubectl config set-credentials ci-user \
          --token=${{ secrets.K8S_STAGING_TOKEN }}
        kubectl config set-context staging \
          --cluster=k8s \
          --user=ci-user \
          --namespace=staging
        kubectl config use-context staging
        
        # Deploy using Helm
        helm upgrade --install ${{ github.event.repository.name }} \
          ./deployments/helm \
          --namespace staging \
          --set image.tag=${{ github.sha }} \
          --wait
    
    - name: Run smoke tests
      run: |
        npm run test:smoke \
          --base-url=${{ secrets.STAGING_API_URL }}

  deploy-production:
    needs: deploy-staging
    runs-on: ubuntu-latest
    environment: production
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Wait for approval
      uses: trstringer/manual-approval@v1
      with:
        secret: ${{ github.token }}
        approvers: ${{ vars.PRODUCTION_APPROVERS }}
        minimum-approvals: 2
    
    - name: Deploy to production
      run: |
        kubectl config set-cluster k8s-prod \
          --server=${{ secrets.K8S_PROD_SERVER }} \
          --certificate-authority=${{ secrets.K8S_PROD_CA }}
        kubectl config set-credentials ci-user-prod \
          --token=${{ secrets.K8S_PROD_TOKEN }}
        kubectl config set-context production \
          --cluster=k8s-prod \
          --user=ci-user-prod \
          --namespace=production
        kubectl config use-context production
        
        # Blue-green deployment
        helm upgrade --install ${{ github.event.repository.name }}-green \
          ./deployments/helm \
          --namespace production \
          --set image.tag=${{ github.sha }} \
          --set service.type=ClusterIP \
          --set service.port=3001 \
          --wait
        
        # Switch traffic
        kubectl patch svc ${{ github.event.repository.name }} \
          -n production \
          -p '{"spec":{"selector":{"version":"green"}}}'
        
        # Cleanup old deployment
        kubectl delete deployment ${{ github.event.repository.name }}-blue \
          -n production --ignore-not-found=true
```

    ];

    patterns.forEach(pattern => {
      masked = masked.replace(pattern.regex, pattern.replacement);
    });

const dbLogger = new Logger();

const pool = new Pool({
  host: config.database.host,
  port: config.database.port,

  user: config.database.user,
  password: config.database.password,
  max: config.database.poolMax,

  connectionTimeoutMillis: 5000, // 5 seconds connection timeout
});

// Pool event listeners
pool.on('connect', () => {
  dbLogger.debug('New database connection established');
});

pool.on('error', (err) => {
  dbLogger.error('Unexpected database pool error', { error: err.message });
});

pool.on('remove', () => {
  dbLogger.debug('Database connection removed from pool');
});

// Query wrapper with logging and timeout
export const query = async <T = any>(
  text: string,
  params?: any[],
  timeoutMs: number = 10000 // Default 10 second timeout
): Promise<T[]> => {
  const startTime = Date.now();
  const client = await pool.connect();
  
  try {
    // Set statement timeout for this query
    await client.query(`SET statement_timeout = ${timeoutMs}`);
    
    const result = await client.query(text, params);
    const durationMs = Date.now() - startTime;
    
    dbLogger.databaseQuery(text, durationMs, {
      rowCount: result.rowCount,
      params: params ? this.maskSensitiveParams(params) : undefined
    });
    
    return result.rows;
  } catch (error) {
    const durationMs = Date.now() - startTime;
    dbLogger.error('Database query failed', {
      query: text,
      durationMs,
      error: error instanceof Error ? error.message : 'Unknown error',
      params: params ? this.maskSensitiveParams(params) : undefined
    });
    
    // Re-throw with additional context
    throw new Error(`Database query failed: ${error instanceof Error ? error.message : 'Unknown error'}`);
  } finally {
    client.release();
  }
};

// Mask sensitive parameters in logs
const maskSensitiveParams = (params: any[]): any[] => {
  return params.map(param => {
    if (typeof param === 'string') {
      // Check if it looks like email, password, token, etc.
      if (param.includes('@') && param.includes('.')) {
        const [name, domain] = param.split('@');
        return `${name.charAt(0)}***@${domain}`;
      }
      if (param.length > 8 && /[A-Za-z0-9]{20,}/.test(param)) {
        return '***'; // Likely token or hash
      }
    }
    return param;
  });
};

-- Função para atualizar updated_at automaticamente
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ language 'plpgsql';

-- Aplica trigger a todas as tabelas com updated_at
DO $$ 
DECLARE
    t text;
BEGIN
    FOR t IN 
        SELECT table_name 
        FROM information_schema.columns 
        WHERE column_name = 'updated_at' 
        AND table_schema = 'diyapp_v12'
    LOOP
        EXECUTE format('
            DROP TRIGGER IF EXISTS update_%s_updated_at ON %s;
            CREATE TRIGGER update_%s_updated_at
            BEFORE UPDATE ON %s
            FOR EACH ROW
            EXECUTE FUNCTION update_updated_at_column();
        ', t, t, t, t);
    END LOOP;
END;
$$;

-- Função para gerar slug único
CREATE OR REPLACE FUNCTION generate_unique_slug(title VARCHAR, table_name VARCHAR, id_column VARCHAR)
RETURNS VARCHAR AS $$
DECLARE
    base_slug VARCHAR;
    final_slug VARCHAR;
    counter INTEGER := 1;
BEGIN
    -- Remove acentos, converte para minúsculo, substitui espaços
    base_slug := lower(unaccent(title));
    base_slug := regexp_replace(base_slug, '[^a-z0-9]+', '-', 'gi');
    base_slug := trim(both '-' from base_slug);
    
    final_slug := base_slug;
    
    -- Verifica se slug já existe
    WHILE EXISTS (
        EXECUTE format('SELECT 1 FROM %I WHERE slug = $1 AND %I != $2', 
                      table_name, id_column) 
        USING final_slug, NULL
    ) LOOP
        final_slug := base_slug || '-' || counter;
        counter := counter + 1;
    END LOOP;
    
    RETURN final_slug;
END;
$$ LANGUAGE plpgsql;

    WHERE event_type IN ('project_view', 'project_save', 'project_clone', 'project_create');

CREATE INDEX idx_projects_for_recommendation ON projects 
    (category, difficulty_level, created_at DESC) 
    WHERE visibility = 'public' AND status = 'published';

    CHECK (email ~* '^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$');

-- Validação de URL
CREATE OR REPLACE FUNCTION is_valid_url(url TEXT)
RETURNS BOOLEAN AS $$
BEGIN
    RETURN url ~* '^https?://[^\s/$.?#].[^\s]*$';
END;
$$ LANGUAGE plpgsql IMMUTABLE;

-- Validação de datas lógicas
ALTER TABLE projects ADD CONSTRAINT logical_dates_check 
    CHECK (
        (completed_at IS NULL OR completed_at >= created_at) AND
        (published_at IS NULL OR published_at >= created_at) AND
        (completed_at IS NULL OR published_at IS NULL OR completed_at >= published_at)
    );

-- View para dashboard de usuário
CREATE VIEW user_dashboard_stats AS
SELECT 
    u.user_id,
    u.email,
    u.subscription_tier,
    u.created_at as user_created_at,
    
    -- Estatísticas de projetos
    COUNT(DISTINCT p.project_id) as total_projects,
    COUNT(DISTINCT CASE WHEN p.status = 'published' THEN p.project_id END) as published_projects,
    COUNT(DISTINCT CASE WHEN p.status = 'completed' THEN p.project_id END) as completed_projects,
    
    -- Engajamento
    COALESCE(SUM(pm.views_count), 0) as total_views,
    COALESCE(SUM(pm.saves_count), 0) as total_saves,
    COALESCE(SUM(pm.clones_count), 0) as total_clones,
    
    -- Atividade recente
    MAX(p.updated_at) as last_project_update,
    MAX(pm.metric_date) as last_metric_date
    
FROM users u
LEFT JOIN projects p ON u.user_id = p.user_id
LEFT JOIN project_metrics_daily pm ON p.project_id = pm.project_id
WHERE u.deleted_at IS NULL
GROUP BY u.user_id, u.email, u.subscription_tier, u.created_at;
