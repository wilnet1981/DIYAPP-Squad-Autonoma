### **2. Proposta de Arquitetura para V8 Core**
#### **ADR-001: Modularização com Domain-Driven Design (DDD)**
**Contexto**:
A V7 Core é um monolito com baixa coesão. A V8 Core deve ser dividida em **domínios autônomos** (ex: `auth`, `workflow`, `storage`), cada um com:
- **API pública clara** (contratos definidos).
- **Dependências injetadas** (para facilitar testes e mocks).
- **Isolamento de falhas** (um módulo não deve derrubar outro).

**Decisão**:
Adotar **DDD** com a seguinte estrutura:
```
/src
  /domains
    /auth          # Autenticação e RBAC
    /workflow      # Gerenciamento de tarefas
    /storage       # Persistência de dados
    /logging       # Logging centralizado
    /cache         # Cache distribuído
  /shared          # Código compartilhado (ex: utils, interfaces)
  /infra           # Infraestrutura (DB, APIs externas)
```

**Opções Consideradas**:
- **Opção A**: Microsserviços → Rejeitada (complexidade desnecessária para o escopo).
- **Opção B**: Modularização monolítica (DDD) → Escolhida (equilíbrio entre autonomia e simplicidade).

**Consequências**:
- **Positivas**: Manutenção mais fácil, testes isolados, escalabilidade horizontal.
- **Negativas**: Overhead inicial de refatoração.
- **Riscos**: Acoplamento residual entre domínios → Mitigação: Revisões de PR rigorosas.

---

#### **ADR-002: Otimização de Banco de Dados**
**Contexto**:
A V7 Core usa consultas SQL brutas e não otimizadas. A V8 Core precisa:
- **ORM/ODM** para abstração e segurança.
- **Pooling de conexões** para performance.
- **Índices e queries otimizadas**.

**Decisão**:
- **PostgreSQL** (relacional) + **Prisma ORM** (para queries seguras e tipadas).
- **Redis** para caching de queries frequentes (ex: listas de tarefas).
- **Estratégia de pooling**: `pg-pool` com configuração padrão (10 conexões).

**Exemplo de Código (Prisma Schema)**:
```prisma
// src/infra/prisma/schema.prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String
  role      Role     @default(USER)
  createdAt DateTime @default(now())
}

enum Role {
  USER
  ADMIN
}
```

**Consequências**:
- **Positivas**: Segurança (SQL injection mitigado), performance (pooling), manutenção (schema versionado).
- **Negativas**: Curva de aprendizado do Prisma.
- **Riscos**: Lock-in do ORM → Mitigação: Usar queries brutas apenas quando necessário.

#### **ADR-003: Caching com Redis**
**Contexto**:
A V7 Core não usa caching, causando lentidão em queries repetitivas (ex: listas de tarefas).

**Decisão**:
- **Redis** para caching de:
  - Dados estáticos (ex: configurações do sistema).
  - Queries frequentes (ex: `GET /tasks`).
- **Estratégia de invalidação**: TTL (Time-To-Live) + invalidação manual em writes.

**Exemplo de Código**:
```javascript
// src/domains/cache/redisClient.js
import { createClient } from 'redis';

const redisClient = createClient({
  url: process.env.REDIS_URL,
});

redisClient.on('error', (err) => console.error('Redis error:', err));

export const getCached = async (key) => {
  await redisClient.connect();
  const value = await redisClient.get(key);
  await redisClient.disconnect();
  return value ? JSON.parse(value) : null;
};

export const setCached = async (key, value, ttl = 3600) => {
  await redisClient.connect();
  await redisClient.set(key, JSON.stringify(value), { EX: ttl });
  await redisClient.disconnect();
};
```

**Consequências**:
- **Positivas**: Redução de 70%+ em queries repetitivas.
- **Negativas**: Complexidade adicional (gerenciamento de cache).
- **Riscos**: Cache desatualizado → Mitigação: TTL curto + invalidação em writes.

---

#### **ADR-004: Logging Centralizado com Winston**
**Contexto**:
A V7 Core usa `console.log` e logs dispersos. A V8 Core precisa:
- **Logs estruturados** (JSON).
- **Níveis de severidade** (debug, info, warn, error).
- **Persistência** (arquivos + envio para serviço externo opcional).

**Decisão**:
- **Winston** para logging centralizado.
- **Transports**:
  - Console (desenvolvimento).
  - Arquivo (produção).
  - Serviço externo (ex: Sentry) opcional.

**Exemplo de Código**:
```javascript
// src/domains/logging/logger.js
import winston from 'winston';

const logger = winston.createLogger({
  level: 'info',
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  ),
  transports: [
    new winston.transports.Console(),
    new winston.transports.File({ filename: 'logs/error.log', level: 'error' }),
    new winston.transports.File({ filename: 'logs/combined.log' }),
  ],
});

export default logger;
```

**Consequências**:
- **Positivas**: Debugging mais fácil, auditoria, integração com ferramentas de observabilidade.
- **Negativas**: Overhead de armazenamento → Mitigação: Rotação de logs (ex: `winston-daily-rotate-file`).

### **6. Código de Exemplo: Módulo `auth` na V8 Core**
```javascript
// src/domains/auth/authService.js
import { PrismaClient } from '@prisma/client';
import { setCached, getCached } from '../cache/redisClient.js';
import logger from '../logging/logger.js';

const prisma = new PrismaClient();

export const login = async (email, password) => {
  try {
    // Cache: Verifica se o usuário está cacheado
    const cachedUser = await getCached(`user:${email}`);
    if (cachedUser) return cachedUser;

    // DB: Busca no PostgreSQL
    const user = await prisma.user.findUnique({ where: { email } });
    if (!user) throw new Error('User not found');

    // Validação de senha (exemplo simplificado)
    if (user.password !== password) throw new Error('Invalid password');

    // Cache: Armazena usuário por 1h
    await setCached(`user:${email}`, user, 3600);

    logger.info(`User ${email} logged in successfully`);
    return user;
  } catch (error) {
    logger.error(`Login failed for ${email}: ${error.message}`);
    throw error;
  }
};
```

```javascript
function cacheGet(key) {
  return new Promise((resolve, reject) => {

      resolve(result);
```

```javascript
const crypto = require('crypto');

// Criptografar dados
function encryptData(data) {
    const cipher = crypto.createCipher('aes-256-cbc', process.env.CRYPTO_KEY);
    let encrypted = cipher.update(data, 'utf8', 'hex');
    encrypted += cipher.final('hex');
    return encrypted;
}

function decryptData(encrypted) {
    const decipher = crypto.createDecipher('aes-256-cbc', process.env.CRYPTO_KEY);
    let decrypted = decipher.update(encrypted, 'hex', 'utf8');
    decrypted += decipher.final('utf8');
    return decrypted;
}
```

```python
import numpy as np
from sklearn.ensemble import IsolationForest

class AnomalyDetector:
    def __init__(self):
        self.model = IsolationForest(n_estimators=100, contamination=0.1, random_state=42)
    
    def fit(self, data):
        self.model.fit(data)
        
    def predict(self, data):
        return self.model.predict(data)

def preprocess_log_data(log_lines):
    # Convert logs into a numerical format suitable for the model
    return np.array([[len(line), line.count('ERROR'), line.count('WARN')] for line in log_lines])

```

```python
import psutil

class AutoScaler:
    def __init__(self, threshold_cpu=70, threshold_memory=80):
        self.threshold_cpu = threshold_cpu
        self.threshold_memory = threshold_memory

    def check_system_load(self):
        cpu_load = psutil.cpu_percent(interval=1)
        memory_load = psutil.virtual_memory().percent
        return cpu_load, memory_load

    def scale_resources(self):
        cpu_load, memory_load = self.check_system_load()
        if cpu_load > self.threshold_cpu or memory_load > self.threshold_memory:
            self.trigger_scaling()
        
    def trigger_scaling(self):
        print("Scaling up resources...")

```

#### `log_monitor.py`

```python
import os
from src.anomaly_detector import AnomalyDetector, preprocess_log_data
from src.utils import read_logs

class LogMonitor:
    def __init__(self, log_dir='data/logs/'):
        self.log_dir = log_dir
        self.detector = AnomalyDetector()
        
    def load_and_process_logs(self):
        log_lines = read_logs(self.log_dir)
        data = preprocess_log_data(log_lines)
        self.detector.fit(data)
        
    def detect_anomalies(self):
        log_lines = read_logs(self.log_dir)
        data = preprocess_log_data(log_lines)
        predictions = self.detector.predict(data)
        anomalies = [i for i, val in enumerate(predictions) if val == -1]
        return anomalies
```

#### `utils.py`

```python
import os

def read_logs(log_dir):
    log_files = [os.path.join(log_dir, f) for f in os.listdir(log_dir) if f.endswith('.log')]
    logs = []
    for log_file in log_files:
        with open(log_file, 'r') as file:
            logs.extend(file.readlines())
    return logs
```
