# DIYAPP Evolution - V12 Core - Arquitetura de Microsserviços

## ADR-001: Arquitetura de Microsserviços para V12

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V12, Especialista Infra, Especialista Segurança

### CONTEXTO:
A V11 do DIYAPP atingiu limites de escalabilidade com arquitetura monolítica. O crescimento de features de IA, processamento de documentos em tempo real e integrações com múltiplos provedores externos exige:
1. Deploy independente de componentes
2. Escala horizontal seletiva
3. Stack tecnológica heterogênea (Python para ML, Node.js para APIs, Go para workers)
4. Tolerância a falhas entre componentes
5. Observabilidade granular por serviço

### DECISÃO:
Adotar arquitetura de microsserviços com os seguintes princípios:
1. **Serviços por domínio de negócio** (não por tecnologia)
2. **Comunicação assíncrona via eventos** como padrão primário
3. **API Gateway único** para clientes externos
4. **Service Mesh** para comunicação interna entre serviços
5. **Banco de dados por serviço** (Database-per-service)
6. **Deploy via containers** com orquestração Kubernetes

### OPÇÕES CONSIDERADAS:

**Opção A: Microsserviços puros (escolhida)**
- Prós: 
  - Escalabilidade independente por serviço
  - Stack tecnológica adequada por domínio
  - Falhas isoladas não derrubam sistema inteiro
  - Times podem desenvolver/deployar independentemente
- Contras:
  - Complexidade operacional aumentada
  - Necessidade de Service Mesh e observabilidade robusta
  - Sincronização de dados entre serviços
  - Testes end-to-end mais complexos

**Opção B: Arquitetura em camadas com módulos**
- Prós:
  - Mais simples de operar
  - Transações ACID fáceis
  - Debug mais simples
- Contras:
  - Acoplamento alto entre componentes
  - Escala apenas vertical
  - Deploy de todo sistema para qualquer mudança
  - Stack tecnológica única

**Opção C: Monólito com workers assíncronos**
- Prós:
  - Balanceamento entre simplicidade e escalabilidade
  - Reuso de código fácil
- Contras:
  - Não resolve problema de stack heterogênea
  - Falhas em componentes críticos afetam sistema inteiro

### CONSEQUÊNCIAS:

**Positivas:**
- Cada serviço pode escalar independentemente baseado em carga
- Falhas em serviços não críticos não derrubam aplicação
- Times podem escolher stack tecnológica ideal para cada domínio
- Deploy contínuo sem downtime para todo sistema
- Maior resiliência através de circuit breakers e retries

**Negativas:**
- Aumento de 40% em custos de infraestrutura inicial
- Curva de aprendizado para time em sistemas distribuídos
- Necessidade de equipe SRE dedicada
- Latência adicional em comunicações entre serviços

**Riscos:**
1. **Consistência eventual:** Mitigar com padrões SAGA e compensação
2. **Debug distribuído:** Mitigar com correlation IDs e tracing centralizado
3. **Cascata de falhas:** Mitigar com circuit breakers e timeouts configurados
4. **Complexidade de deploy:** Mitigar com GitOps e pipelines automatizadas

**REVISÃO:** 2024-04-15 (3 meses após implementação)

```yaml
# .github/workflows/ci-service.yml
name: Service CI Pipeline

on:
  push:
    branches: [ main, develop ]
    paths:
      - 'services/**'
      - 'libs/**'
      - '.github/workflows/ci-service.yml'
  pull_request:
    branches: [ main, develop ]

jobs:
  validate:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        service: [auth, document, ai-orchestrator, notification, user-profile, billing]
    
    steps:
    - name: Checkout
      uses: actions/checkout@v3
      
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
        
    - name: Install dependencies
      run: npm ci
      working-directory: services/${{ matrix.service }}
      
    - name: Lint
      run: npm run lint
      working-directory: services/${{ matrix.service }}
      
    - name: Type check
      run: npm run type-check
      working-directory: services/${{ matrix.service }}
      
    - name: Unit tests
      run: npm test -- --coverage
      working-directory: services/${{ matrix.service }}
      
    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        file: ./services/${{ matrix.service }}/coverage/lcov.info
        
    - name: Security scan
      uses: snyk/actions/node@master
      env:
        SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
      with:
        args: --severity-threshold=high
      working-directory: services/${{ matrix.service }}
      
    - name: Build container
      run: |
        docker build \
          --tag diyapp/${{ matrix.service }}:${GITHUB_SHA::8} \
          --file services/${{ matrix.service }}/Dockerfile \
          services/${{ matrix.service }}
          
    - name: Container scan
      uses: aquasecurity/trivy-action@master
      with:
        image-ref: 'diyapp/${{ matrix.service }}:${GITHUB_SHA::8}'
        format: 'sarif'
        output: 'trivy-results.sarif'
        
    - name: Upload container scan results
      uses: github/codeql-action/upload-sarif@v2
      if: always()
      with:
        sarif_file: 'trivy-results.sarif'
```

```yaml
# .github/workflows/cd-service.yml
name: Service CD Pipeline

on:
  push:
    tags:
      - 'services/*-v*'

jobs:
  deploy-staging:
    runs-on: ubuntu-latest
    environment: staging
    steps:
    - name: Extract service name
      id: extract
      run: |
        SERVICE=$(echo ${{ github.ref_name }} | cut -d'-' -f1)
        echo "service=${SERVICE}" >> $GITHUB_OUTPUT
        
    - name: Checkout
      uses: actions/checkout@v3
      
    - name: Configure AWS credentials
      uses: aws-actions/configure-aws-credentials@v2
      with:
        aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
        aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
        aws-region: us-east-1
        
    - name: Login to ECR
      id: login-ecr
      uses: aws-actions/amazon-ecr-login@v1
      
    - name: Build and push
      env:
        ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
        ECR_REPOSITORY: diyapp/${{ steps.extract.outputs.service }}
        IMAGE_TAG: ${{ github.sha }}
      run: |
        docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG \
          -f services/${{ steps.extract.outputs.service }}/Dockerfile \
          services/${{ steps.extract.outputs.service }}
        docker push $ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG
        
    - name: Deploy to Kubernetes
      uses: steebchen/kubectl@v2
      with:
        config: ${{ secrets.KUBE_CONFIG_STAGING }}
        command: |
          set -x
          kubectl set image deployment/${{ steps.extract.outputs.service }} \
            ${{ steps.extract.outputs.service }}=$ECR_REGISTRY/$ECR_REPOSITORY:$IMAGE_TAG \
            -n diyapp-staging
          kubectl rollout status deployment/${{ steps.extract.outputs.service }} \
            -n diyapp-staging --timeout=300s
            
    - name: Run smoke tests
      run: |
        npm run test:smoke
      working-directory: services/${{ steps.extract.outputs.service }}
      
  deploy-production:
    needs: deploy-staging
    runs-on: ubuntu-latest
    environment: production
    if: github.ref == 'refs/tags/services/*-v*'
    
    steps:
    - name: Canary deployment
      uses: steebchen/kubectl@v2
      with:
        config: ${{ secrets.KUBE_CONFIG_PROD }}
        command: |
          # Apply canary deployment
          kubectl apply -f infrastructure/k8s/canary/${{ steps.extract.outputs.service }}.yaml
          
          # Monitor canary metrics for 5 minutes
          sleep 300
          
          # Check error rate and latency
          ERROR_RATE=$(curl -s $PROMETHEUS_URL/api/v1/query?query=...)
          if [ "$ERROR_RATE" -lt "0.01" ]; then
            # Promote canary to full deployment
            kubectl apply -f infrastructure/k8s/full/${{ steps.extract.outputs.service }}.yaml
          else
            # Rollback canary
            kubectl rollout undo deployment/${{ steps.extract.outputs.service }}-canary
            exit 1
          fi
```

## Service Template Structure

```typescript
// services/template-service/src/index.ts
import express from 'express';
import helmet from 'helmet';
import cors from 'cors';
import { createLogger, metricsMiddleware, errorHandler } from '@diyapp/observability';
import { connectDatabase } from './database';
import { setupEventConsumers } from './events';
import { registerRoutes } from './routes';
import { healthCheck } from './health';

const app = express();
const logger = createLogger('template-service');
const PORT = process.env.PORT || 3000;

// Security middleware
app.use(helmet());
app.use(cors());
app.use(express.json({ limit: '10mb' }));

// Observability middleware
app.use(metricsMiddleware);
app.use((req, res, next) => {
  req.id = req.headers['x-correlation-id'] || crypto.randomUUID();
  next();
});

// Request logging
app.use((req, res, next) => {
  const start = Date.now();
  res.on('finish', () => {
    const duration = Date.now() - start;
    logger.info({
      method: req.method,
      url: req.url,
      status: res.statusCode,
      duration,
      correlationId: req.id,
      userAgent: req.get('user-agent')
    });
  });
  next();
});

// Health check
app.get('/health', healthCheck);
app.get('/ready', async (req, res) => {
  const isReady = await checkReadiness();
  res.status(isReady ? 200 : 503).json({ ready: isReady });
});

// API Routes
registerRoutes(app);

// Error handling
app.use(errorHandler);

// Startup
async function start() {
  try {
    // Connect to database
    await connectDatabase();
    
    // Start event consumers
    await setupEventConsumers();
    
    // Start server
    app.listen(PORT, () => {
      logger.info(`Service started on port ${PORT}`);
      logger.info(`Environment: ${process.env.NODE_ENV}`);
      logger.info(`Version: ${process.env.APP_VERSION}`);
    });
    
    // Graceful shutdown
    process.on('SIGTERM', gracefulShutdown);
    process.on('SIGINT', gracefulShutdown);
    
  } catch (error) {
    logger.error('Failed to start service', { error });
    process.exit(1);
  }
}

async function gracefulShutdown() {
  logger.info('Shutdown signal received');
  
  // Give connections 30 seconds to close
  setTimeout(() => {
    logger.error('Forceful shutdown');
    process.exit(1);
  }, 30000);
  
  // Close server
  server.close(async () => {
    // Close database connections
    await closeDatabase();
    
    // Close event consumers
    await closeEventConsumers();
    
    logger.info('Graceful shutdown complete');
    process.exit(0);
  });
}

start();
```

BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;

    SELECT
        user_id::STRING AS user_id,
        email,
        username,
        created_at AS user_created_at,
        last_login_at,

        metadata,
        -- Extrair localização do metadata

        -- Calcular dias desde criação
        DATE_DIFF(CURRENT_DATE(), DATE(created_at), DAY) AS days_since_creation
    FROM {{ source('oltp', 'users') }}
    WHERE created_at >= DATE_SUB(CURRENT_DATE(), INTERVAL {{ var('lookback_days') }} DAY)
),

enriched AS (
    SELECT
        *,
        -- Segmentação de usuário
        CASE
            WHEN days_since_creation < 30 THEN 'new'
            WHEN days_since_creation BETWEEN 30 AND 180 THEN 'active'
            WHEN days_since_creation > 180 AND last_login_at >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY) THEN 'retained'
            ELSE 'churned'
        END AS user_segment,
        -- Tier baseado em atividade
        CASE
            WHEN last_login_at >= DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY) THEN 'highly_active'
            WHEN last_login_at >= DATE_SUB(CURRENT_DATE(), INTERVAL 30 DAY) THEN 'active'
            ELSE 'inactive'
        END AS activity_tier
    FROM source
)

// src/core/utils/idempotency.ts
import { Redis } from 'ioredis';
import { v4 as uuidv4 } from 'uuid';
import { logger } from './logger';

export class IdempotencyService {
  private redis: Redis;
  private readonly TTL = 24 * 60 * 60; // 24 hours

  constructor(redis: Redis) {
    this.redis = redis;
  }

  async generateKey(operation: string, payload: any): Promise<string> {
    const sortedPayload = JSON.stringify(payload, Object.keys(payload).sort());
    return `${operation}:${sortedPayload}`;
  }

  async executeWithIdempotency<T>(
    operation: string,
    payload: any,
    executor: () => Promise<T>,
    clientId?: string
  ): Promise<{ result: T; isCached: boolean }> {
    const idempotencyKey = await this.generateKey(operation, payload);
    const cacheKey = `idempotency:${idempotencyKey}`;
    
    // Check if operation was already executed
    const cachedResult = await this.redis.get(cacheKey);
    if (cachedResult) {
      logger.info('Idempotent operation hit', { 
        operation, 
        clientId,
        idempotencyKey: idempotencyKey.substring(0, 50) 
      });
      return { result: JSON.parse(cachedResult), isCached: true };
    }

    // Generate request ID for tracking
    const requestId = uuidv4();
    
    // Use Redis SET with NX to prevent concurrent execution
    const lockKey = `lock:${idempotencyKey}`;
    const lockAcquired = await this.redis.set(lockKey, requestId, 'PX', 10000, 'NX');
    
    if (!lockAcquired) {
      // Wait and retry for concurrent request
      await new Promise(resolve => setTimeout(resolve, 100));
      return this.executeWithIdempotency(operation, payload, executor, clientId);
    }

    try {
      // Execute the operation
      const result = await executor();
      
      // Cache the result
      await this.redis.setex(cacheKey, this.TTL, JSON.stringify(result));
      
      logger.info('Idempotent operation executed', { 
        operation, 
        clientId,
        requestId,
        idempotencyKey: idempotencyKey.substring(0, 50) 
      });
      
      return { result, isCached: false };
    } finally {
      // Release the lock (only if it's still our lock)
      const script = `
        if redis.call("get", KEYS[1]) == ARGV[1] then
          return redis.call("del", KEYS[1])
        else
          return 0
        end
      `;
      await this.redis.eval(script, 1, lockKey, requestId);
    }
  }

  async clearIdempotencyKey(operation: string, payload: any): Promise<void> {
    const idempotencyKey = await this.generateKey(operation, payload);
    const cacheKey = `idempotency:${idempotencyKey}`;
    await this.redis.del(cacheKey);
  }
}

// src/core/utils/resilience.ts
import CircuitBreaker from 'circuit-breaker-js';
import { logger } from './logger';

export interface CircuitBreakerOptions {
  timeout?: number;
  maxFailures?: number;
  resetTimeout?: number;
}

export class ResilientService {
  private circuitBreaker: CircuitBreaker;
  private serviceName: string;
  private metrics: {
    totalCalls: number;
    successfulCalls: number;
    failedCalls: number;
    circuitOpened: number;
  };

  constructor(serviceName: string, options: CircuitBreakerOptions = {}) {
    this.serviceName = serviceName;
    this.metrics = {
      totalCalls: 0,
      successfulCalls: 0,
      failedCalls: 0,
      circuitOpened: 0,
    };

    this.circuitBreaker = new CircuitBreaker({
      timeoutDuration: options.timeout || 5000,
      volumeThreshold: 10,
      errorThreshold: 50,
      onCircuitOpen: () => {
        this.metrics.circuitOpened++;
        logger.warn('Circuit breaker opened', { 
          service: this.serviceName,
          metrics: this.metrics 
        });
      },
      onCircuitClose: () => {
        logger.info('Circuit breaker closed', { service: this.serviceName });
      },
    });
  }

  async execute<T>(
    operation: () => Promise<T>,
    fallback?: () => Promise<T>
  ): Promise<T> {
    this.metrics.totalCalls++;

    return new Promise((resolve, reject) => {
      this.circuitBreaker.run(
        async (success: (result: T) => void, failure: (error: Error) => void) => {
          try {
            const result = await operation();
            this.metrics.successfulCalls++;
            success(result);
          } catch (error) {
            this.metrics.failedCalls++;
            failure(error as Error);
          }
        },
        async (result: T) => {
          resolve(result);
        },
        async (error: Error) => {
          logger.error('Service call failed', { 
            service: this.serviceName, 
            error: error.message,
            metrics: this.metrics 
          });

          if (fallback) {
            try {
              const fallbackResult = await fallback();
              logger.info('Fallback executed successfully', { 
                service: this.serviceName 
              });
              resolve(fallbackResult);
            } catch (fallbackError) {
              reject(fallbackError);
            }
          } else {
            reject(error);
          }
        }
      );
    });
  }

  getMetrics() {
    return { ...this.metrics };
  }

  getState(): 'closed' | 'open' | 'half-open' {
    return this.circuitBreaker.isOpen() ? 'open' : 'closed';
  }
}

import Redis from 'ioredis';
import { config } from '../../config';
import { logger } from '../utils/logger';
import { ResilientService } from '../utils/resilience';

export class RedisClient {
  private client: Redis;
  private resilientService: ResilientService;
  private isConnected = false;

  constructor() {
    this.client = new Redis({
      host: config.redis.host,
      port: config.redis.port,
      password: config.redis.password,
      retryStrategy: (times) => {
        const delay = Math.min(times * 50, 2000);
        return delay;
      },
      maxRetriesPerRequest: 3,
    });

    this.resilientService = new ResilientService('redis', {
      timeout: 1000,
      maxFailures: 5,
      resetTimeout: 30000,
    });

    this.setupEventListeners();
  }

  private setupEventListeners(): void {
    this.client.on('connect', () => {
      this.isConnected = true;
      logger.info('Redis connected successfully');
    });

    this.client.on('error', (error) => {
      logger.error('Redis error', { error: error.message });
    });

    this.client.on('close', () => {
      this.isConnected = false;
      logger.warn('Redis connection closed');
    });

    this.client.on('reconnecting', () => {
      logger.info('Redis reconnecting...');
    });
  }

  async get(key: string): Promise<string | null> {
    return this.resilientService.execute(
      async () => {
        if (!this.isConnected) {
          throw new Error('Redis not connected');
        }
        return await this.client.get(key);
      },
      async () => {
        logger.warn('Redis get fallback - returning null', { key });
        return null;
      }
    );
  }

  async set(
    key: string, 
    value: string, 
    ttl?: number
  ): Promise<void> {
    await this.resilientService.execute(
      async () => {
        if (!this.isConnected) {
          throw new Error('Redis not connected');
        }
        if (ttl) {
          await this.client.setex(key, ttl, value);
        } else {
          await this.client.set(key, value);
        }
      },
      async () => {
        logger.warn('Redis set fallback - operation skipped', { key });
      }
    );
  }

  async del(key: string): Promise<void> {
    await this.resilientService.execute(
      async () => {
        if (!this.isConnected) {
          throw new Error('Redis not connected');
        }
        await this.client.del(key);
      },
      async () => {
        logger.warn('Redis del fallback - operation skipped', { key });
      }
    );
  }

  async healthCheck(): Promise<boolean> {
    try {
      await this.client.ping();
      return true;
    } catch (error) {
      return false;
    }
  }

  async disconnect(): Promise<void> {
    await this.client.quit();
  }
}

import { tokens } from './tokens'

export const theme = {
  // Utility functions for accessing tokens
  color: (colorPath: string) => {
    const [category, shade] = colorPath.split('.')
    return tokens.colors[category as keyof typeof tokens.colors][shade as any]
  },

  fontSize: (size: keyof typeof tokens.typography.fontSize) =>

    tokens.typography.fontWeight[weight],
  borderRadius: (radius: keyof typeof tokens.borderRadius) =>
    tokens.borderRadius[radius],
  shadow: (shadow: keyof typeof tokens.shadows) => tokens.shadows[shadow],
