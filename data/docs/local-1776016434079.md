## ADR-001: Arquitetura de Microsserviços

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead

### CONTEXTO:
Precisamos evoluir a arquitetura monolítica atual para uma arquitetura de microsserviços que permita:
1. Escalabilidade independente de componentes
2. Deploy contínuo sem downtime
3. Isolamento de falhas
4. Autonomia das squads no desenvolvimento

### DECISÃO:
Adotar arquitetura de microsserviços com os seguintes princípios:
- Cada serviço é responsável por um domínio específico
- Comunicação via API REST para operações síncronas
- Comunicação via eventos (RabbitMQ) para operações assíncronas
- API Gateway como ponto único de entrada
- Banco de dados por serviço (Database per Service)

### OPÇÕES CONSIDERADAS:
- **Opção A:** Arquitetura monolítica com módulos
  - Prós: Simplicidade, deploy único, transações ACID fáceis
  - Contras: Escalabilidade limitada, acoplamento alto, dificuldade de evolução independente
- **Opção B:** Microsserviços com mensageria
  - Prós: Escalabilidade independente, resiliência, deploy contínuo
  - Contras: Complexidade operacional, consistência eventual, debugging mais difícil
- **Opção escolhida:** B - Justificativa: Alinha com objetivos de autonomia da squad e permite evolução independente dos componentes

### CONSEQUÊNCIAS:
**Positivas:**
- Cada squad pode desenvolver e deployar independentemente
- Falhas isoladas não derrubam o sistema inteiro
- Escalabilidade granular por serviço

**Negativas:**
- Maior complexidade operacional
- Necessidade de orquestração (Kubernetes/Docker)
- Consistência eventual em alguns cenários

**Riscos:**
- Latência em chamadas entre serviços
- Gerenciamento de transações distribuídas
- Mitigação: Circuit breakers, retry policies, sagas pattern

## Engineering Standards

```markdown
# DIYAPP V11 - Engineering Standards

## 1. Padrões de Código

### 1.1 JavaScript/TypeScript
- TypeScript estrito (strict: true)
- ESLint com Airbnb Style Guide
- Prettier para formatação
- Nomenclatura:
  - Interfaces: PascalCase com prefixo I (IUser)
  - Classes: PascalCase
  - Funções: camelCase
  - Constantes: UPPER_SNAKE_CASE
  - Arquivos: kebab-case

### 1.2 Testes
- Jest para unit tests
- Supertest para API tests
- Cypress para E2E
- Cobertura mínima: 80%
- Testes devem ser independentes e idempotentes

### 1.3 Git
- Conventional Commits
- Branch strategy: Git Flow
- PRs devem ter:
  - Descrição clara do problema/solução
  - Link para issue
  - Screenshots quando aplicável
  - Todos os testes passando

## 2. Padrões de API

### 2.1 REST APIs
- Versionamento: /api/v1/resource
- Métodos HTTP semânticos
- JSON como formato padrão
- Status codes apropriados
- Paginação para listagens
- Rate limiting

### 2.2 Contratos
- OpenAPI 3.0 para documentação
- Validação de schemas com Zod
- Versionamento semântico (SemVer)

## 3. Segurança
- Autenticação JWT
- Autorização baseada em roles
- Input validation em todas as APIs
- Sanitização de dados
- Logs sem dados sensíveis
```

## Configuração do CI/CD Pipeline

### .github/workflows/ci.yml
```yaml
name: CI Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main, develop ]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    
    strategy:
      matrix:
        service: ['auth', 'user', 'project', 'gateway']
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
      working-directory: ./src/services/${{ matrix.service }}-service
    
    - name: Lint
      run: npm run lint
      working-directory: ./src/services/${{ matrix.service }}-service
    
    - name: Run tests
      run: npm test -- --coverage
      working-directory: ./src/services/${{ matrix.service }}-service
    
    - name: Upload coverage
      uses: codecov/codecov-action@v3
      with:
        directory: ./src/services/${{ matrix.service }}-service/coverage
  
  build-and-push:
    needs: lint-and-test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v2
    
    - name: Login to DockerHub
      uses: docker/login-action@v2
      with:
        username: ${{ secrets.DOCKER_USERNAME }}
        password: ${{ secrets.DOCKER_PASSWORD }}
    
    - name: Build and push
      run: |
        docker build -t diyapp/${{ matrix.service }}:latest -f docker/Dockerfile.${{ matrix.service }} .
        docker push diyapp/${{ matrix.service }}:latest
```

info:
  title: User Service API
  version: 1.0.0
  description: API para gerenciamento de usuários

  - url: http://localhost:3001/api/v1
    description: Development server

paths:
  /users:
    get:
      summary: Lista usuários
      parameters:
        - name: page
          in: query
          schema:
            type: integer
            minimum: 1
            default: 1
        - name: limit
          in: query
          schema:
            type: integer
            minimum: 1
            maximum: 100
            default: 20
      responses:
        '200':
          description: Lista de usuários
          content:
            application/json:
              schema:
                type: object
                properties:
                  data:
                    type: array
                    items:
                      $ref: '#/components/schemas/User'
                  pagination:
                    $ref: '#/components/schemas/Pagination'
    
    post:
      summary: Cria novo usuário
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CreateUserRequest'
      responses:
        '201':
          description: Usuário criado
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/User'

  /users/{id}:
    get:
      summary: Obtém usuário por ID
      parameters:
        - name: id
          in: path
          required: true
          schema:
            type: string
      responses:
        '200':
          description: Usuário encontrado
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/User'
        '404':
          description: Usuário não encontrado

components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: string
          format: uuid
        email:
          type: string
          format: email
        name:
          type: string
        role:
          type: string
          enum: [user, admin, moderator]
        createdAt:
          type: string
          format: date-time
        updatedAt:
          type: string
          format: date-time
    
    CreateUserRequest:
      type: object
      required:
        - email
        - name
        - password
      properties:
        email:
          type: string
          format: email
        name:
          type: string
        password:
          type: string
          minLength: 8
    
    Pagination:
      type: object
      properties:
        page:
          type: integer
        limit:
          type: integer
        total:
          type: integer
        pages:
          type: integer

        allow_credentials=True,

    @app.middleware("http")
    async def log_requests(request: Request, call_next):
        correlation_id = str(uuid.uuid4())
        request.state.correlation_id = correlation_id
        
        start_time = time.time()
        
        # Log request
        logger.info(
            "request_started",
            correlation_id=correlation_id,
            method=request.method,
            url=str(request.url),
            client_ip=request.client.host if request.client else None,
            user_agent=request.headers.get("user-agent")
        )
        
        try:
            response = await call_next(request)
            process_time = time.time() - start_time
            
            # Log response
            logger.info(
                "request_completed",
                correlation_id=correlation_id,
                method=request.method,
                url=str(request.url),
                status_code=response.status_code,
                duration_ms=round(process_time * 1000, 2)
            )
            
            # Record metrics
            endpoint = request.url.path
            REQUEST_COUNT.labels(
                method=request.method,
                endpoint=endpoint,
                status=response.status_code
            ).inc()
            
            REQUEST_LATENCY.labels(
                method=request.method,
                endpoint=endpoint
            ).observe(process_time)
            
            return response
            
        except Exception as e:
            process_time = time.time() - start_time
            logger.error(
                "request_failed",
                correlation_id=correlation_id,
                method=request.method,
                url=str(request.url),
                error=str(e),
                error_type=type(e).__name__,
                duration_ms=round(process_time * 1000, 2)
            )
            raise
    
    # Rate limiting
    app.state.limiter = limiter
    app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)
    
    # Metrics endpoint
    @app.get("/metrics")
    async def metrics():
        return Response(generate_latest(), media_type="text/plain")

pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")


def verify_password(plain_password: str, hashed_password: str) -> bool:
    """Verify a password against its hash."""
    try:
        return pwd_context.verify(plain_password, hashed_password)
    except Exception as e:
        logger.error("password_verification_failed", error=str(e))
        return False


def get_password_hash(password: str) -> str:
    """Generate password hash."""
    return pwd_context.hash(password)


def create_access_token(data: dict, expires_delta: Optional[timedelta] = None) -> str:
    """Create JWT access token."""
    to_encode = data.copy()
    if expires_delta:
        expire = datetime.utcnow() + expires_delta
    else:
        expire = datetime.utcnow() + timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    
    to_encode.update({"exp": expire, "type": "access"})
    encoded_jwt = jwt.encode(to_encode, settings.SECRET_KEY, algorithm=settings.ALGORITHM)
    return encoded_jwt


def create_refresh_token(data: dict) -> str:
    """Create JWT refresh token."""
    to_encode = data.copy()
    expire = datetime.utcnow() + timedelta(days=settings.REFRESH_TOKEN_EXPIRE_DAYS)
    to_encode.update({"exp": expire, "type": "refresh"})
    encoded_jwt = jwt.encode(to_encode, settings.SECRET_KEY, algorithm=settings.ALGORITHM)
    return encoded_jwt


def verify_token(token: str, token_type: str = "access") -> dict:
    """Verify JWT token and return payload."""
    try:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=[settings.ALGORITHM])
        
        if payload.get("type") != token_type:
            raise UnauthorizedException("Invalid token type")
        
        return payload
    except JWTError as e:
        logger.warning("token_verification_failed", error=str(e))
        raise UnauthorizedException("Could not validate credentials")

// src/components/CodeEditor/CodeEditor.tsx
import React, { useRef, useEffect, useState } from 'react'
import Editor, { Monaco, OnMount } from '@monaco-editor/react'
import { Box, Paper, Typography, IconButton, Tooltip, CircularProgress } from '@mui/material'
import PlayArrowIcon from '@mui/icons-material/PlayArrow'
import SaveIcon from '@mui/icons-material/Save'
import DownloadIcon from '@mui/icons-material/Download'
import FormatIcon from '@mui/icons-material/FormatListBulleted'
import { useTheme } from '@mui/material/styles'

interface CodeEditorProps {
  initialCode?: string
  language?: string
  onCodeChange?: (code: string) => void
  onRun?: (code: string) => Promise<void>
  onSave?: (code: string) => Promise<void>
  readOnly?: boolean
  height?: string
}

export const CodeEditor: React.FC<CodeEditorProps> = ({
  initialCode = '// Write your code here\nconsole.log("Hello DIYAPP!");',
  language = 'javascript',
  onCodeChange,
  onRun,
  onSave,
  readOnly = false,
  height = '500px'
}) => {
  const theme = useTheme()
  const [code, setCode] = useState(initialCode)
  const [isRunning, setIsRunning] = useState(false)
  const [isSaving, setIsSaving] = useState(false)
  const editorRef = useRef<any>(null)
  const monacoRef = useRef<Monaco | null>(null)

  const handleEditorDidMount: OnMount = (editor, monaco) => {
    editorRef.current = editor
    monacoRef.current = monaco
    
    // Configure Monaco theme
    monaco.editor.defineTheme('diyapp-theme', {
      base: 'vs-dark',
      inherit: true,
      rules: [],
      colors: {
        'editor.background': theme.palette.background.paper,
        'editor.foreground': theme.palette.text.primary,
        'editor.lineHighlightBackground': theme.palette.grey[100],
        'editorCursor.foreground': theme.palette.primary.main,
        'editor.selectionBackground': `${theme.palette.primary.light}40`,
        'editor.inactiveSelectionBackground': `${theme.palette.primary.light}20`,
      }
    })
    
    monaco.editor.setTheme('diyapp-theme')
    
    // Add custom formatting
    editor.addAction({
      id: 'format-code',
      label: 'Format Code',
      keybindings: [monaco.KeyMod.CtrlCmd | monaco.KeyMod.Shift | monaco.KeyCode.KeyF],
      contextMenuGroupId: 'navigation',
      contextMenuOrder: 1.5,
      run: async () => {
        const model = editor.getModel()
        if (model) {
          const formatted = await formatCode(model.getValue(), language)
          model.setValue(formatted)
        }
      }
    })
  }

  const formatCode = async (code: string, lang: string): Promise<string> => {
    // Mock formatting - in production, use a proper formatter
    if (lang === 'javascript' || lang === 'typescript') {
      try {
        // Simple formatting logic
        return code
          .replace(/\s+/g, ' ')
          .replace(/\s*{\s*/g, ' { ')
          .replace(/\s*}\s*/g, ' } ')
          .replace(/\s*\(\s*/g, ' (')
          .replace(/\s*\)\s*/g, ') ')
          .trim()
      } catch {
        return code
      }
    }
    return code
  }

  const handleCodeChange = (value: string | undefined) => {
    const newCode = value || ''
    setCode(newCode)
    onCodeChange?.(newCode)
  }

  const handleRun = async () => {
    if (!onRun) return
    
    setIsRunning(true)
    try {
      await onRun(code)
    } catch (error) {
      console.error('Error running code:', error)
    } finally {
      setIsRunning(false)
    }
  }

  const handleSave = async () => {
    if (!onSave) return
    
    setIsSaving(true)
    try {
      await onSave(code)
    } catch (error) {
      console.error('Error saving code:', error)
    } finally {
      setIsSaving(false)
    }
  }

  const handleDownload = () => {
    const blob = new Blob([code], { type: 'text/plain' })
    const url = URL.createObjectURL(blob)
    const a = document.createElement('a')
    a.href = url
    a.download = `code.${language === 'typescript' ? 'ts' : 'js'}`
    document.body.appendChild(a)
    a.click()
    document.body.removeChild(a)
    URL.revokeObjectURL(url)
  }

  const handleFormat = async () => {
    if (editorRef.current) {
      const model = editorRef.current.getModel()
      if (model) {
        const formatted = await formatCode(model.getValue(), language)
        model.setValue(formatted)
      }
    }
  }

  return (
    <Paper 
      elevation={1} 
      sx={{ 
        display: 'flex', 
        flexDirection: 'column',
        height,
        overflow: 'hidden'
      }}
    >
      <Box 
        sx={{ 
          display: 'flex', 
          alignItems: 'center', 
          justifyContent: 'space-between',
          p: 2,
          borderBottom: 1,
          borderColor: 'divider',
          bgcolor: 'background.default'
        }}
      >
        <Typography variant="h6" component="div">
          Code Editor
        </Typography>
        <Box sx={{ display: 'flex', gap: 1 }}>
          <Tooltip title="Format Code">
            <IconButton onClick={handleFormat} size="small">
              <FormatIcon />
            </IconButton>
          </Tooltip>
          {onSave && (
            <Tooltip title="Save">
              <IconButton onClick={handleSave} disabled={isSaving} size="small">
                {isSaving ? <CircularProgress size={20} /> : <SaveIcon />}
              </IconButton>
            </Tooltip>
          )}
          <Tooltip title="Download">
            <IconButton onClick={handleDownload} size="small">
              <DownloadIcon />
            </IconButton>
          </Tooltip>
          {onRun && (
            <Tooltip title="Run Code">
              <IconButton 
                onClick={handleRun} 
                disabled={isRunning}
                color="primary"
                size="small"
              >
                {isRunning ? <CircularProgress size={20} /> : <PlayArrowIcon />}
              </IconButton>
            </Tooltip>
          )}
        </Box>
      </Box>
      
      <Box sx={{ flex: 1, overflow: 'hidden' }}>
        <Editor
          height="100%"
          language={language}
          value={code}
          onChange={handleCodeChange}
          onMount={handleEditorDidMount}
          options={{
            minimap: { enabled: true },
            scrollBeyondLastLine: false,
            fontSize: 14,
            wordWrap: 'on',
            automaticLayout: true,
            readOnly,
            tabSize: 2,
            suggestOnTriggerCharacters: true,
            acceptSuggestionOnEnter: 'on',
            snippetSuggestions: 'inline',
            padding: { top: 16 },
            scrollbar: {
              vertical: 'visible',
              horizontal: 'visible'
            }
          }}
        />
      </Box>
    </Paper>
  )
}

    def generate_threat_report(self) -> Dict:
        """Gera relatório completo de ameaças"""
        report = {
            "version": "V11.0.0",
            "generated_at": self.last_updated.isoformat(),
            "threat_summary": {
                "total": len(self.threats),
                "critical": len([t for t in self.threats if t.severity == ThreatSeverity.CRITICAL]),
                "high": len([t for t in self.threats if t.severity == ThreatSeverity.HIGH]),
                "medium": len([t for t in self.threats if t.severity == ThreatSeverity.MEDIUM]),
                "low": len([t for t in self.threats if t.severity == ThreatSeverity.LOW])
            },
            "threats_by_category": {},
            "detailed_threats": []
        }
        
        # Agrupar por categoria
        for category in ThreatCategory:
            category_threats = [t for t in self.threats if t.category == category]
            report["threats_by_category"][category.value] = {
                "count": len(category_threats),
                "threats": [t.id for t in category_threats]
            }
        
        # Detalhes completos
        for threat in self.threats:
            report["detailed_threats"].append({
                "id": threat.id,
                "category": threat.category.value,
                "asset": threat.asset,
                "severity": threat.severity.value,
                "description": threat.description,
                "mitigation": threat.mitigation,
                "controls_required": threat.controls_required,
                "risk_accepted": threat.risk_accepted
            })
        
        return report

# Arquitetura DIYAPP V11 Core

## Visão Geral
DIYAPP V11 é uma plataforma autônoma de desenvolvimento de software com múltiplos agentes especializados trabalhando em modo Hive. O sistema é projetado para estabilidade 100% autônoma com capacidade de auto-refatoração.

## Princípios Arquiteturais

### 1. Autonomia Total
- Cada agente opera com mandatos específicos e protocolos de decisão claros
- Sistema continua funcionando mesmo com falha de componentes individuais
- Decisões distribuídas com consenso automático

### 2. Modo Hive
- Refatoração paralela sem downtime
- Agentes especializados colaboram em tempo real
- Estado compartilhado através de eventos

### 3. Estabilidade Garantida
- Health checks contínuos
- Rollback automático em falhas
- Monitoramento proativo

## Componentes Principais

### Agentes Especializados
1. **Squad Leader** - Orquestração e priorização
2. **Product Owner** - Definição de valor e backlog
3. **Tech Lead** - Decisões técnicas e ADRs
4. **Dev Backend** - APIs e lógica de negócio
5. **Dev Frontend** - Interface e experiência
6. **SRE** - Infraestrutura e confiabilidade
7. **QA Engineer** - Qualidade e testes
8. **Tech Writer** - Documentação e conhecimento
9. **Especialista em Inovação** - Experimentação

  constructor(db) {

  /**
   * Avalia se uma feature requer DPIA
   * Critérios: dados sensíveis, monitoramento sistemático, processamento em larga escala
   */
  async requiresDPIA(featureData) {
    const riskFactors = {
      sensitiveData: this.hasSensitiveData(featureData.dataCategories),
      systematicMonitoring: featureData.systematicMonitoring || false,
      largeScaleProcessing: featureData.expectedUsers > 10000,
      automatedDecisionMaking: featureData.automatedDecisions || false,
      dataMatchingCombining: featureData.dataCombination || false,
      vulnerableSubjects: featureData.vulnerableSubjects || false,
      innovativeUse: featureData.innovativeUse || false,
      preventsExerciseOfRights: featureData.preventsRights || false
    };

    const riskScore = Object.values(riskFactors).filter(Boolean).length;
    const requiresDPIA = riskScore >= 2; // Dois ou mais fatores de risco

    await complianceLogger.logDPIAAssessment({
      featureId: featureData.featureId,
      riskFactors,
      riskScore,
      requiresDPIA,
      timestamp: new Date()
    });

    return {
      requiresDPIA,
      riskFactors,
      riskScore,
      recommendation: requiresDPIA
