# DIYAPP Evolution - V12 Core - Arquitetura Técnica

## 1. DECISÃO ARQUITETURAL PRINCIPAL - ADR-001: Arquitetura Híbrida Microsserviços/Monolito

**Data:** 2024-01-15
**Status:** Aceita
**Autores:** Tech Lead V12

### CONTEXTO:
O DIYAPP evoluiu para uma plataforma complexa com múltiplos módulos (WhatsApp Automation, IA Agents, Dashboard, API Gateway). O monolito atual enfrenta:
- Acoplamento excessivo entre módulos
- Dificuldade de deploy independente
- Escalabilidade desigual (alguns módulos são heavy, outros light)
- Time-to-market lento para novas features

### DECISÃO:
Adotar arquitetura híbrida:
- **Core Services** (microsserviços): WhatsApp Engine, AI Orchestrator, Task Scheduler
- **Shared Monolith:** Dashboard, User Management, Billing
- **API Gateway** unificada com rate limiting e auth
- **Message Bus** para comunicação assíncrona

### OPÇÕES CONSIDERADAS:
1. **Monolito puro** - Manter status quo
   - Prós: Simplicidade, deploy único
   - Contras: Acoplamento, dificuldade de escala
2. **Microsserviços completos** - Tudo separado
   - Prós: Escalabilidade máxima, deploy independente
   - Contras: Complexidade operacional, overhead de rede
3. **Híbrida (escolhida)** - Separação estratégica
   - Prós: Balanceamento ideal, evolução gradual
   - Contras: Gerenciamento de duas arquiteturas

### CONSEQUÊNCIAS:
**Positivas:**
- Deploy independente dos serviços críticos
- Escala seletiva baseada em carga
- Evolução gradual sem big bang

**Negativas:**
- Complexidade aumentada
- Necessidade de orquestração (Kubernetes)
- Monitoramento distribuído

**Riscos:**
- Latência entre serviços → Mitigação: Cache Redis, conexões persistentes
- Consistência de dados → Mitigação: Event Sourcing, Sagas

import { NestFactory } from '@nestjs/core';
import { ValidationPipe, Logger } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';
import { WhatsAppEngineModule } from './whatsapp-engine.module';
import { setupSwagger } from '@diyapp/common/swagger';
import { setupMetrics } from '@diyapp/common/metrics';
import { setupTracing } from '@diyapp/common/tracing';

async function bootstrap() {
  const logger = new Logger('WhatsAppEngine');
  
  // Configuração
  const app = await NestFactory.create(WhatsAppEngineModule, {
    logger: ['error', 'warn', 'log', 'debug'],
    bufferLogs: true,
  });

  const configService = app.get(ConfigService);
  
  // Middlewares
  app.useGlobalPipes(new ValidationPipe({ transform: true, whitelist: true }));
  
  // Observability
  setupTracing(app, 'whatsapp-engine');
  setupMetrics(app);
  
  // Documentação
  if (configService.get('NODE_ENV') !== 'production') {
    setupSwagger(app, 'WhatsApp Engine API', 'v1');
  }
  
  // Health checks
  app.enableShutdownHooks();
  
  const port = configService.get('PORT', 3001);
  await app.listen(port);
  
  logger.log(`🚀 WhatsApp Engine running on port ${port}`);
  logger.log(`📊 Metrics: http://localhost:${port}/metrics`);
  logger.log(`📚 API Docs: http://localhost:${port}/api/docs`);
}

bootstrap().catch((error) => {
  console.error('Failed to bootstrap:', error);
  process.exit(1);
});

```typescript
// services/whatsapp-engine/src/services/whatsapp.service.ts
import { Injectable, Logger, Inject } from '@nestjs/common';
import { ClientProxy } from '@nestjs/microservices';
import { RedisService } from '@diyapp/common/redis';
import { BaileysProvider } from '../providers/baileys.provider';
import { 
  SendMessageDto, 
  MessageStatus, 
  WhatsAppSession 
} from '@diyapp/types/whatsapp';

@Injectable()
export class WhatsAppService {
  private readonly logger = new Logger(WhatsAppService.name);
  private sessions: Map<string, WhatsAppSession> = new Map();

  constructor(
    private readonly baileys: BaileysProvider,
    private readonly redis: RedisService,
    @Inject('AI_SERVICE') private readonly aiClient: ClientProxy,
  ) {}

  async sendMessage(dto: SendMessageDto): Promise<MessageStatus> {
    try {
      // Validação de rate limiting
      const key = `rate_limit:${dto.from}:${dto.to}`;
      const count = await this.redis.incr(key);
      
      if (count === 1) {
        await this.redis.expire(key, 60); // 1 minuto
      }
      
      if (count > 10) {
        throw new Error('Rate limit exceeded');
      }

      // Processamento com IA (opcional)
      let processedContent = dto.content;
      if (dto.useAI) {
        processedContent = await this.aiClient
          .send('process_message', { content: dto.content })
          .toPromise();
      }

      // Envio via Baileys
      const result = await this.baileys.sendMessage({
        to: dto.to,
        content: processedContent,
        options: dto.options,
      });

      // Log e métricas
      this.logger.log(`Message sent to ${dto.to}`);
      await this.redis.publish('whatsapp:messages:sent', JSON.stringify({
        ...dto,
        timestamp: new Date().toISOString(),
        messageId: result.messageId,
      }));

      return {
        success: true,
        messageId: result.messageId,
        timestamp: new Date(),
        status: 'delivered',
      };
    } catch (error) {
      this.logger.error(`Failed to send message: ${error.message}`, error.stack);
      
      await this.redis.publish('whatsapp:messages:failed', JSON.stringify({
        ...dto,
        error: error.message,
        timestamp: new Date().toISOString(),
      }));

      throw error;
    }
  }

  async getSessionStatus(sessionId: string): Promise<WhatsAppSession> {
    const cached = await this.redis.get(`session:${sessionId}`);
    if (cached) {
      return JSON.parse(cached);
    }

    const session = this.sessions.get(sessionId) || {
      id: sessionId,
      status: 'disconnected',
      lastActive: null,
      qrCode: null,
    };

    await this.redis.setex(
      `session:${sessionId}`,
      300, // 5 minutos TTL
      JSON.stringify(session)
    );

    return session;
  }
}

class AuthenticationSystem {
    constructor() {
        this.config = securityConfig;
        this.failedAttempts = new Map();
        this.lockedAccounts = new Map();
    }

    /**
     * Valida política de senha
     */
    validatePasswordPolicy(password) {
        const policy = this.config.owaspControls.authentication.passwordPolicy;
        
        if (password.length < policy.minLength) {
            throw new Error(`Senha deve ter no mínimo ${policy.minLength} caracteres`);
        }
        
        if (policy.requireUppercase && !/[A-Z]/.test(password)) {
            throw new Error('Senha deve conter pelo menos uma letra maiúscula');
        }
        
        if (policy.requireLowercase && !/[a-z]/.test(password)) {
            throw new Error('Senha deve conter pelo menos uma letra minúscula');
        }
        
        if (policy.requireNumbers && !/\d/.test(password)) {
            throw new Error('Senha deve conter pelo menos um número');
        }
        
        if (policy.requireSpecialChars && !/[!@#$%^&*(),.?":{}|<>]/.test(password)) {
            throw new Error('Senha deve conter pelo menos um caractere especial');
        }
        
        // Verificar senhas comuns/vazadas (simplificado)
        const commonPasswords = ['123456', 'password', '123456789', '12345678', '12345'];
        if (commonPasswords.includes(password.toLowerCase())) {
            throw new Error('Senha muito comum. Escolha uma senha mais segura');
        }
        
        return true;
    }

    /**
     * Hash de senha com bcrypt
     */
    async hashPassword(password) {
        const saltRounds = this.config.owaspControls.cryptography.saltRounds;
        return await bcrypt.hash(password, saltRounds);
    }

    /**
     * Verifica senha
     */
    async verifyPassword(password, hash) {
        return await bcrypt.compare(password, hash);
    }

    /**
     * Gera token JWT
     */
    generateToken(userId, roles = [], additionalClaims = {}) {
        const payload = {
            sub: userId,
            roles: roles,
            iss: this.config.auth.jwt.issuer,
            aud: this.config.auth.jwt.audience,
            iat: Math.floor(Date.now() / 1000),
            ...additionalClaims
        };

        const token = jwt.sign(
            payload,
            process.env.JWT_SECRET || 'default-secret-change-in-production',
            { expiresIn: this.config.auth.jwt.accessTokenExpiry }
        );

        const refreshToken = jwt.sign(
            { sub: userId, type: 'refresh' },
            process.env.JWT_REFRESH_SECRET || 'default-refresh-secret-change',
            { expiresIn: this.config.auth.jwt.refreshTokenExpiry }
        );

        return { token, refreshToken, expiresIn: this.config.auth.jwt.accessTokenExpiry };
    }

    /**
     * Verifica token JWT
     */
    verifyToken(token, isRefresh = false) {
        try {
            const secret = isRefresh 
                ? process.env.JWT_REFRESH_SECRET 
                : process.env.JWT_SECRET;
            
            return jwt.verify(token, secret || 'default-secret-change-in-production', {
                issuer: this.config.auth.jwt.issuer,
                audience: this.config.auth.jwt.audience
            });
        } catch (error) {
            throw new Error(`Token inválido: ${error.message}`);
        }
    }

    /**
     * Gerencia tentativas falhas de login
     */
    trackFailedLogin(username) {
        const now = Date.now();
        const attempts = this.failedAttempts.get(username) || [];
        
        // Remove tentativas antigas (últimas 15 minutos)
        const recentAttempts = attempts.filter(time => now - time < 15 * 60 * 1000);
        recentAttempts.push(now);
        
        this.failedAttempts.set(username, recentAttempts);
        
        // Bloqueia conta se exceder limite
        if (recentAttempts.length >= 5) {
            this.lockedAccounts.set(username, now + 15 * 60 * 1000); // Bloqueio por 15 minutos
            throw new Error('Conta temporariamente bloqueada devido a múltiplas tentativas falhas');
        }
        
        return 5 - recentAttempts.length; // Tentativas restantes
    }

    /**
     * Limpa tentativas falhas após login bem-sucedido
     */
    clearFailedAttempts(username) {
        this.failedAttempts.delete(username);
        this.lockedAccounts.delete(username);
    }

    /**
     * Verifica se conta está bloqueada
     */
    isAccountLocked(username) {
        const lockUntil = this.lockedAccounts.get(username);
        if (lockUntil && Date.now() < lockUntil) {
            return true;
        }
        
        if (lockUntil && Date.now() >= lockUntil) {
            this.lockedAccounts.delete(username);
            this.failedAttempts.delete(username);
        }
        
        return false;
    }

    /**
     * Gera código MFA
     */
    generateMFACode() {
        return crypto.randomInt(100000, 999999).toString();
    }

    /**
     * Valida código MFA com TOTP
     */
    validateTOTPCode(secret, code) {
        // Implementação simplificada de TOTP
        // Em produção, usar biblioteca como 'speakeasy' ou 'otplib'
        const crypto = require('crypto');
        const epoch = Math.floor(Date.now() / 1000);
        const timeWindow = Math.floor(epoch / 30);
        
        const hmac = crypto.createHmac('sha1', secret);
        hmac.update(Buffer.from(timeWindow.toString(16).padStart(16, '0'), 'hex'));
        const hmacResult = hmac.digest();
        
        const offset = hmacResult[hmacResult.length - 1] & 0xf;
        const binary = (
            (hmacResult[offset] & 0x7f) << 24 |
            (hmacResult[offset + 1] & 0xff) << 16 |
            (hmacResult[offset + 2] & 0xff) << 8 |
            (hmacResult[offset + 3] & 0xff)
        );
        
        const totp = binary % 1000000;
        return totp.toString().padStart(6, '0') === code;
    }

function generateCSSVariables() {
  let css = ':root {\n';
  
  // Color variables
  Object.entries(DesignTokens.colors).forEach(([category, values]) => {
    if (typeof values === 'object') {
      Object.entries(values).forEach(([key, value]) => {
        if (typeof value === 'string' && value.startsWith('var(')) {
          css += `  --color-${category}-${key}: ${value};\n`;
        } else if (typeof value === 'object') {
          Object.entries(value).forEach(([subKey, subValue]) => {
            css += `  --color-${category}-${key}-${subKey}: ${subValue};\n`;
          });
        } else {
          css += `  --color-${category}-${key}: ${value};\n`;
        }
      });
    }
  });
  
  // Typography variables
  Object.entries(DesignTokens.typography).forEach(([category, values]) => {
    if (typeof values === 'object') {
      Object.entries(values).forEach(([key, value]) => {
        css += `  --font-${category}-${key}: ${value};\n`;
      });
    }
  });
  
  // Spacing variables
  Object.entries(DesignTokens.spacing).forEach(([key, value]) => {
    css += `  --spacing-${key}: ${value};\n`;
  });
  
  // Border radius variables
  Object.entries(DesignTokens.borderRadius).forEach(([key, value]) => {
    css += `  --radius-${key}: ${value};\n`;
  });
  
  // Elevation variables
  Object.entries(DesignTokens.elevation).forEach(([key, value]) => {
    css += `  --elevation-${key}: ${value};\n`;
  });
  
  // Animation variables
  Object.entries(DesignTokens.animation.duration).forEach(([key, value]) => {
    css += `  --duration-${key}: ${value};\n`;
  });
  
  Object.entries(DesignTokens.animation.easing).forEach(([key, value]) => {
    css += `  --easing-${key}: ${value};\n`;
  });
  
  css += '}\n';
  return css;
}

export const useMediaQuery = (query: string): boolean => {
  const [matches, setMatches] = useState(false);

  useEffect(() => {
    const media = window.matchMedia(query);
    if (media.matches !== matches) {
      setMatches(media.matches);
    }
    
    const listener = () => setMatches(media.matches);
    window.addEventListener('resize', listener);
    
    return () => window.removeEventListener('resize', listener);
  }, [matches, query]);

  return matches;
};

  return useMediaQuery(`(min-width: ${breakpoints[breakpoint]})`);

        """Verify password against hash"""
        return pwd_context.verify(plain_password, hashed_password)
    
    @staticmethod
    def get_password_hash(password: str) -> str:
        """Generate password hash"""
        return pwd_context.hash(password)
    
    @staticmethod
    def create_access_token(data: dict, expires_delta: Optional[timedelta] = None) -> str:
        """Create JWT access token"""
        to_encode = data.copy()
        if expires_delta:
            expire = datetime.utcnow() + expires_delta
        else:
            expire = datetime.utcnow() + timedelta(minutes=Config.ACCESS_TOKEN_EXPIRE_MINUTES)
        
        to_encode.update({"exp": expire, "type": "access"})
        encoded_jwt = jwt.encode(to_encode, Config.SECRET_KEY, algorithm=Config.ALGORITHM)
        return encoded_jwt
    
    @staticmethod
    def create_refresh_token(data: dict) -> str:
        """Create JWT refresh token"""
        to_encode = data.copy()
        expire = datetime.utcnow() + timedelta(days=Config.REFRESH_TOKEN_EXPIRE_DAYS)
        to_encode.update({"exp": expire, "type": "refresh"})
        encoded_jwt = jwt.encode(to_encode, Config.SECRET_KEY, algorithm=Config.ALGORITHM)
        return encoded_jwt
    
    @staticmethod
    def decode_token(token: str) -> dict:
        """Decode and validate JWT token"""
        try:
            payload = jwt.decode(token, Config.SECRET_KEY, algorithms=[Config.ALGORITHM])
            return payload
        except jwt.ExpiredSignatureError:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="Token has expired"
            )
        except jwt.InvalidTokenError:
            raise HTTPException(
                status_code=status.HTTP_401_UNAUTHORIZED,
                detail="Invalid token"
            )

-- Query 1: User activity dashboard (PM metrics)
CREATE OR REPLACE FUNCTION get_user_activity_metrics(
    start_date DATE, 
    end_date DATE
) RETURNS TABLE (
    date DATE,
    active_users INTEGER,
    new_users INTEGER,
    projects_created INTEGER,
    deployments_count INTEGER,
    avg_session_minutes DECIMAL(5,2)
) AS $$
BEGIN
    RETURN QUERY
    WITH daily_stats AS (
        SELECT 
            DATE(ae.created_at) as stat_date,
            COUNT(DISTINCT ae.user_id) as active_users,
            COUNT(DISTINCT CASE WHEN u.created_at::DATE = DATE(ae.created_at) THEN u.user_id END) as new_users,
            COUNT(DISTINCT CASE WHEN ae.event_type = 'project_created' THEN ae.project_id END) as projects_created,
            COUNT(CASE WHEN ae.event_type = 'deployment_completed' THEN 1 END) as deployments_count
        FROM analytics_events ae
        LEFT JOIN users u ON ae.user_id = u.user_id
        WHERE DATE(ae.created_at) BETWEEN start_date AND end_date
        GROUP BY DATE(ae.created_at)
    )
    SELECT 
        ds.stat_date,
        ds.active_users,
        ds.new_users,
        ds.projects_created,
        ds.deployments_count,
        COALESCE((
            SELECT AVG(EXTRACT(EPOCH FROM (MAX(session_end) - MIN(session_start)))/60)
            FROM (
                SELECT 
                    session_id,
                    MIN(created_at) as session_start,
                    MAX(created_at) as session_end
                FROM analytics_events
                WHERE DATE(created_at) = ds.stat_date
                GROUP BY session_id
            ) sessions
        ), 0) as avg_session_minutes
    FROM daily_stats ds
    ORDER BY ds.stat_date;
END;
$$ LANGUAGE plpgsql;

-- Query 2: LLM cost by feature (AI Ops metrics)
CREATE OR REPLACE FUNCTION get_llm_cost_by_feature(
    start_date TIMESTAMP, 
    end_date TIMESTAMP
) RETURNS TABLE (
    feature VARCHAR(50),
    provider VARCHAR(50),
    model VARCHAR(100),
    total_requests BIGINT,
    total_tokens BIGINT,
    total_cost DECIMAL(12,6),
    avg_latency_ms DECIMAL(8,2)
) AS $$
BEGIN
    RETURN QUERY
    SELECT 
        lr.operation_type as feature,
        lr.provider,
        lr.model,
        COUNT(*) as total_requests,
        SUM(lr.total_tokens) as total_tokens,
        SUM(lr.cost_usd) as total_cost,
        AVG(lr.latency_ms) as avg_latency_ms
    FROM llm_requests lr
    WHERE lr.created_at BETWEEN start_date AND end_date
    GROUP BY lr.operation_type, lr.provider, lr.model
    ORDER BY total_cost DESC;
END;
$$ LANGUAGE plpgsql;

-- Query 3: Project health metrics (SRE dashboard)
CREATE OR REPLACE FUNCTION get_project_health_metrics(
    project_uuid UUID
) RETURNS TABLE (
    metric_name VARCHAR(50),
    metric_value DECIMAL(12,4),
    metric_trend VARCHAR(10),
    last_updated TIMESTAMP
) AS $$
BEGIN
    RETURN QUERY
    WITH deployment_stats AS (
        SELECT 
            'deployment_success_rate' as metric_name,
            AVG(CASE WHEN status = 'success' THEN 1.0 ELSE 0.0 END) * 100 as metric_value,
            CASE 
                WHEN AVG(CASE WHEN status = 'success' THEN 1.0 ELSE 0.0 END) > 
                     LAG(AVG(CASE WHEN status = 'success' THEN 1.0 ELSE 0.0 END), 7) 
                     OVER (ORDER BY DATE(created_at)) THEN 'improving'
                ELSE 'declining'
            END as metric_trend,
            MAX(created_at) as last_updated
        FROM deployments
        WHERE project_id = project_uuid
        AND created_at > NOW() - INTERVAL '30 days'
        GROUP BY DATE(created_at)
        ORDER BY DATE(created_at) DESC
        LIMIT 1
    ),
    performance_stats AS (
        SELECT 
            'avg_build_time' as metric_name,
            AVG(duration_ms) as metric_value,
            CASE 
                WHEN AVG(duration_ms) < 
                     LAG(AVG(duration_ms), 7) OVER (ORDER BY DATE(created_at)) THEN 'improving'
                ELSE 'declining'
            END as metric_trend,
            MAX(created_at) as last_updated
        FROM deployments
        WHERE project_id = project_uuid
        AND status = 'success'
        AND created_at > NOW() - INTERVAL '30 days'
        GROUP BY DATE(created_at)
        ORDER BY DATE(created_at) DESC
        LIMIT 1
    )
    SELECT * FROM deployment_stats
    UNION ALL
    SELECT * FROM performance_stats;
END;
$$ LANGUAGE plpgsql;

# DIYAPP V12 - Visão Geral do Sistema

**Data de criação:** 2024-01-15  
**Última atualização:** 2024-01-15  
**Owner:** Tech Lead  
**Status:** Ativo

## 1. Objetivo do Sistema
DIYAPP V12 é uma plataforma de automação de workflows empresariais com suporte a múltiplos canais de comunicação (WhatsApp, Email, SMS) e integração com sistemas legados.

## 2. Princípios Arquiteturais

### 2.1 Autonomia da Squad
- Cada módulo opera independentemente
- Comunicação via APIs bem definidas
- Fallback automático em caso de falha

### 2.2 Estabilidade 100% Autônoma
- Health checks automatizados
- Circuit breakers em todas as integrações externas
- Auto-recovery de serviços

### 2.3 Modo Hive
- Refatoração paralela sem downtime
- Versionamento de APIs com backward compatibility
- Feature flags para rollouts controlados

## 3. Componentes Principais

### 3.1 API Gateway
- **Tecnologia:** Express.js com rate limiting
- **Responsabilidade:** Roteamento, autenticação, logging
- **Porta:** 3000

### 3.2 Core Service
- **Tecnologia:** Node.js + TypeScript
- **Responsabilidade:** Lógica de negócio principal
- **Banco de dados:** PostgreSQL (transações) + Redis (cache)

### 3.3 WhatsApp Integration
- **Tecnologia:** Baileys (WhatsApp Web API)
- **Responsabilidade:** Comunicação com WhatsApp
- **Recursos:** Envio/recebimento de mensagens, grupos, mídia

### 3.4 Worker Service
- **Tecnologia:** Bull (Redis queue)
- **Responsabilidade:** Processamento assíncrono
- **Jobs:** Notificações, relatórios, limpeza

### 3.5 Monitoring Service
- **Tecnologia:** Winston + Prometheus
- **Responsabilidade:** Métricas, logs, alertas
- **Dashboard:** Grafana em `:3001/metrics`

## 4. Fluxo de Dados

```
Cliente → API Gateway → Autenticação → Core Service → (WhatsApp/Email/SMS) → Worker → Database
```

## 5. Decisões de Design (ADRs)

### ADR-001: Comunicação Assíncrona
**Contexto:** Necessidade de processar mensagens sem bloquear a API
**Decisão:** Usar Redis + Bull para filas de jobs
**Consequências:** Maior resiliência, complexidade adicional

### ADR-002: Multi-tenant Isolation
**Contexto:** Suporte a múltiplos clientes no mesmo sistema
**Decisão:** Schema separation no PostgreSQL
**Consequências:** Segurança melhorada, overhead de migração

## 6. Dependências Externas

# Guia de Autenticação - DIYAPP V12 API

**Data de criação:** 2024-01-15  
**Última atualização:** 2024-01-15  
**Owner:** Tech Writer

## Visão Geral
A API DIYAPP V12 usa autenticação baseada em JWT (JSON Web Tokens). Todos os endpoints (exceto `/auth/login` e `/health`) requerem um token válido.

## Fluxo de Autenticação

### 1. Obter Token
```bash
curl -X POST https://api.diyapp.example.com/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "seu@email.com",
    "password": "sua-senha"
  }'
```

**Resposta de Sucesso:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 3600,
  "user": {
    "id": "uuid",
    "email": "seu@email.com",
    "name": "Seu Nome",
    "role": "user"
  }
}
```

### 2. Usar Token
Inclua o token no header `Authorization`:

```bash
curl -X GET https://api.diyapp.example.com/v1/workflows \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## Tipos de Token

### Access Token
- **Validade:** 1 hora
- **Uso:** Acesso à API
- **Renovação:** Via refresh token ou novo login

### Refresh Token (Em breve)
- **Validade:** 7 dias
- **Uso:** Obter novo access token
- **Armazenamento:** HTTP-only cookie

## Roles e Permissões

| Role | Permissões |
|------|------------|
| admin | Todas operações (CRUD completo) |
| user | Criar/ler/atualizar próprios recursos |
| viewer | Apenas leitura |

## Rate Limiting

- **Limite padrão:** 100 requests por minuto por token
- **Headers de resposta:**
  ```
  X-RateLimit-Limit: 100
  X-RateLimit-Remaining: 99
  X-RateLimit-Reset: 1617032400
  ```

## Erros Comuns

### 401 Unauthorized
```json
{
  "error": "InvalidToken",
  "message": "Token is invalid or expired"
}
```

**Solução:**
1. Verifique se o token está correto
2. Renove o token se expirado
3. Faça login novamente

### 403 Forbidden
```json
{
  "error": "InsufficientPermissions",
  "message": "User does not have required permissions"
}
```

**Solução:** Contate um administrador para ajustar suas permissões.
